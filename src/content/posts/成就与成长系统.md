## 1. 系统概述

成就与成长系统是提升玩家长期游戏动力的核心机制，在本项目中主要体现在以下几个维度：

| 成长维度 | 实现方式 | 数据来源 |
|---|---|---|
| 玩家等级（Level） | 经验值积累升级 | `PlayerComponent.Level` |
| 角色好感度（Favor） | 剧本互动积累 | `IPCharacterData.FavorLevel` |
| 养成成就（Cultivation） | 剧本日程/赛事结果 | `CultivationComponentSystem` |
| 回忆卡收集（MemoryCard） | 剧本内解锁截图留念 | `MemoryCardComponentSystem.cs` |
| 角色解锁序列（IPSequence） | 星图解锁顺序 | `CharacterComponent.IPSequence` |
| 历史记录（History） | 所有日程决策留档 | `CultivationHistoryData` |

## 2. 架构设计

### 2.1 成长体系总览

```
成长层次
├── 账号级成长（全局持久化）
│     ├── 玩家等级（Level）
│     ├── 称号/头衔
│     └── 解锁角色数量
│
├── 角色级成长（per-IP 持久化）
│     ├── 好感度等级（FavorLevel）
│     └── 好感度数值（FavorValue）
│
└── 剧本级成长（per-Script 持久化）
      ├── 日程安排历史
      ├── 赛事成绩历史
      ├── 疗愈历史
      └── 回忆卡解锁状态
```

### 2.2 回忆卡系统（MemoryCard）

回忆卡是游戏的特色成就收集系统，玩家在剧本中的重要时刻可以解锁专属的"照片卡"，形成独一无二的养成记忆：

```
MemoryCardComponent（回忆卡组件）
├── TempMemoryCardName: string    临时回忆卡文件名（待确认）
└── （通过网络接口拉取完整数据）

MemoryCardBriefData（简要数据，用于列表展示）
├── ScriptID: int         所属剧本
├── CardID:   int         卡片 ID
├── IsUnlocked: bool      是否已解锁
└── UnlockTime: long      解锁时间戳

MemoryCardDetailData（详细数据，用于大图展示）
├── ScriptID:  int
├── CardID:    int
├── ImagePath: string     图片资源路径
├── Title:     string     卡片标题
├── Description: string   剧情描述
└── ...
```

### 2.3 成就历史记录系统

养成历史（`CultivationHistoryData`）记录了玩家整个剧本的每一次决策，形成完整的成长档案：

```
CultivationHistoryData
├── ScheduleHistories: List<CultivationScheduleHistoryData>  每日日程历史
│     └── CultivationScheduleHistoryData
│           ├── ScheduleID:  int
│           ├── ScheduleDay: int
│           ├── ScheduleWatchingCharacter: IPCharacterEnum
│           └── CharacterScheduleHistories: List<...>
│                 └── CultivationCharacterScheduleHistoryData
│                       ├── CharacterIP: IPCharacterEnum
│                       ├── ArrangeHistories: List<CultivationCharacterArrangeHistoryData>
│                       └── IsLiveHost: bool
│
└── HealingHistories: List<...>   夜晚疗愈历史
```

### 2.4 好感度成长流程

```
玩家完成日程（日程结算）
         │
         ▼
服务端计算好感度变化
         │
         ▼
剧本结算通知（CellBoxCharacterUpdateInfoMsg）
         │
         ▼
CharacterComponentSystem.SyncCharacterFavorFromCultivateScriptComplete()
         │
         ▼
IPCharacterData.FavorLevel = CultivationUtil.GetLevelByFavorValue(value)
         │
         ▼
Evt_CharacterFavorUpdated → UI 展示好感度升级动效
```

/// <summary>
/// 请求回忆卡简要数据集合（成就列表）
/// </summary>
public static async ETTask<List<MemoryCardBriefData>> RequestMemoryCardList(
    this MemoryCardComponent self)
{
    Log.LogInfo("[MemoryCard][Network] 协议 ZoneCsCultivateMemoryCardBriefInfo 请求开始");

    var req = new ZoneCultivatesMemoryCardBriefInfoReq();
    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneCultivatesMemoryCardBriefInfoResp>(
            (uint)ZoneClientCmd.ZoneCsCultivateMemoryCardBriefInfo, req);

    if (nr.IsCompleteSuccess)
    {
        Log.LogInfo("[MemoryCard][Network] 接收成功");
        return ServerDataConverter.ToList(
            nr.Data?.MemoryCardIndexList,
            ServerDataConverter.GetMemoryCardBriefData);
    }

    Log.LogError("[MemoryCard][Network] 接收失败");
    return null;
}

/// <summary>
/// 请求指定回忆卡的详细数据（成就详情）
/// </summary>
public static async ETTask<MemoryCardDetailData> RequestMemoryCardDetail(
    this MemoryCardComponent self, int scriptID, int cardID)
{
    Log.LogInfo("[MemoryCard][Network] 协议 ZoneCsCultivateMemoryCardInfo 请求开始");

    var req = new ZoneCultivatesMemoryCardInfoReq
    {
        ScriptId    = scriptID,
        CardIdList  = { cardID },
    };
    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneCultivatesMemoryCardInfoResp>(
            (uint)ZoneClientCmd.ZoneCsCultivateMemoryCardInfo, req);

    if (nr.IsCompleteSuccess)
    {
        var cardList = nr.Data?.MemoryCardList;
        if (cardList != null && cardList.Count > 0)
        {
            Log.LogInfo("[MemoryCard][Network] 接收成功");
            return ServerDataConverter.GetMemoryCardDetailData(cardList[0]);
        }
    }

    Log.LogError("[MemoryCard][Network] 接收失败");
    return null;
}

### 3.3 历史记录（成长档案写入）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Cultivation\
//   CultivationComponentSystem.ScriptHistory.cs

/// <summary>
/// 记录角色安排历史数据（成长档案：每日决策留档）
/// </summary>
public static void RecordCharacterScheduleHistory(this CultivationComponent self)
{
    if (CultivationUtil.IsMatchScheduleType(self.GetScheduleType()))
        return; // 比赛日不记录安排历史

    var (_, scheduleHistory) = self.GetOrAddScheduleHistory();
    if (scheduleHistory == null) return;

    var characterScheduleHistories = new List<CultivationCharacterScheduleHistoryData>();
    foreach (var character in self.GetCharacters())
    {
        var arrangeHistories = new List<CultivationCharacterArrangeHistoryData>();

        // 记录三种安排（直播/训练/休息）
        RecordSchedule(self.GetLiveSchedule(character.LiveID));
        RecordSchedule(self.GetTrainingSchedule(character.TrainingID));
        RecordSchedule(self.GetRestSchedule(character.RestID));

        characterScheduleHistories.Add(new CultivationCharacterScheduleHistoryData
        {
            CharacterIP          = character.IP,
            StoryCharacterID     = character.StoryCharacterID,
            ArrangeHistories     = arrangeHistories,
            IsLiveHost           = character.IsLiveHost,
        });

        void RecordSchedule(IScheduleDetailData scheduleData)
        {
            if (scheduleData == null) return;
            arrangeHistories.Add(new CultivationCharacterArrangeHistoryData
            {
                ScheduleType    = scheduleData.ScheduleType,
                ScheduleSubType = scheduleData.ScheduleSubType,
                ScheduleID      = scheduleData.ScheduleID,
            });
        }
    }

    scheduleHistory.CharacterScheduleHistories = characterScheduleHistories;
    scheduleHistory.ScheduleWatchingCharacter = self.GetScheduleWatchingCharacterIP();
}
```

### 3.4 角色好感度成长同步

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Character\
//   CharacterComponentSystem.cs

/// <summary>
/// 从剧本结算同步角色好感度（成长：亲密度升级）
/// </summary>
public static void SyncCharacterFavorFromCultivateScriptComplete(
    this CharacterComponent self,
    RepeatedField<CellBoxCharacterUpdateInfoMsg> characterInfos)
{
    if (characterInfos == null) return;

    foreach (var characterInfo in characterInfos)
    {
        var character = self.GetUnlockedIPCharacter((IPCharacterEnum)characterInfo.IpId);
        if (character == null) continue;

        // 将好感度数值转换为好感度等级（配置表驱动换算）
        character.FavorLevel = CultivationUtil.GetLevelByFavorValue(characterInfo.Favorability);
    }
}
```

## 4. 设计亮点

### 4.1 回忆卡的命名规则设计
回忆卡文件名包含 ScriptID、CardID、PlayerGID 和时间戳，形成全局唯一标识。这种设计：
- 支持玩家在不同设备查看同一张回忆卡；
- 防止文件名冲突；
- 便于服务端验证图片归属。

### 4.2 好感度数值与等级分离
服务端存储和传输的是**好感度数值**（精确），客户端展示用的是**好感度等级**（玩家友好）。两者通过 `CultivationUtil.GetLevelByFavorValue` 换算，升级阈值由配置表定义，策划可以自由调整升级曲线而无需改代码。

### 4.3 历史记录的层次化存储
历史记录按"日程→角色→安排类型"的三级层次存储，既保留了完整的决策记录（支持"回忆"功能回放），又保持了数据结构的规整性，便于后期数据分析和成就判断。

### 4.4 BitArray 高效记录成就完成状态
与新手引导同理，大量成就的完成标记使用 `BitArray` 存储，8 个成就标记只占 1 字节，序列化效率远优于 `bool[]` 或 `List<int>`。

## 5. 常见问题与最佳实践

### Q1: 如何在 UI 上展示好感度升级动效？
```csharp
// 监听好感度更新事件
[Event(SceneType.Client)]
public class CharacterFavorUpdatedEvent : AEvent<Evt_CharacterFavorUpdated>
{
    protected override void Run(Scene scene, Evt_CharacterFavorUpdated a)
    {
        // 检查等级是否真的提升了
        if (a.OldFavorLevel < a.NewFavorLevel)
        {
            // 触发升级动效、音效、提示文字
            UIManager.ShowFavorLevelUpEffect(a.CharacterIP, a.NewFavorLevel);
        }
    }
}
```

### Q2: 回忆卡解锁是服务端触发还是客户端触发？
由服务端判断解锁条件（特定剧情完成、特定选项选择等）并下发解锁通知，客户端只负责展示解锁动效和保存本地截图。这保证了成就的公平性，防止客户端自行解锁。

### Q3: 历史记录数据量会不会过大？
每个日程的历史记录大约包含：3-5名角色 × 3种安排 = 约 9-15 条记录。一个剧本通常有 20-50 个日程，总记录量在 200-700 条之间，数据量可控。服务端可以按剧本 ID 分表存储，历史数据不影响热点读写。

### Q5: 玩家等级成长如何驱动功能解锁？
玩家等级（`PlayerComponent.Level`）变化时，服务端重新计算所有 `SystemEntry` 的解锁状态并推送。客户端在收到 `Evt_PlayerLevelUp` 事件后，可以主动查询 `SystemEntries` 检查是否有新解锁的功能，并展示"新功能解锁"弹窗引导。
