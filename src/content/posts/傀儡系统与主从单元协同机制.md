# 傀儡系统与主从单元协同机制

在许多动作游戏中，玩家角色并不是孤独作战。他可能带着一个伴生精灵，或者战斗时从宠物中召唤出一个辅助单位，这些"从属单位"需要与主角精确协同——跟随主角的位置、响应主角的技能指令、随主角的状态显示或隐藏。

这套机制就是"傀儡系统"（Puppet System）。本文深入解析这套系统如何管理复杂的主从关系。

---

## 什么是傀儡（Puppet）？

```csharp
public enum EEntityType
{
    Player,     // 玩家角色（主控）
    Monster,    // 怪物
    Puppet,     // 傀儡（从属于某个 Player 或 Monster）
    BattleItem, // 战斗道具
    Director,   // 导演
}
```

傀儡是一种特殊的战斗单元，特点是：
- **有宿主**：通过 `RootUnit` 指向创建它的主单元
- **跟随主人**：默认情况下跟随宿主的位置偏移运动
- **与主人共命运**：主人隐藏时，傀儡也隐藏；主人消亡，傀儡也消亡

典型应用：
- 召唤类角色带着一个"鬼魂"跟随
- 某些角色有"分身"机制
- 特殊技能召唤出一个辅助攻击单位

## 傀儡的创建

```csharp
// UnitFactory.Create 中的傀儡创建逻辑
var puppetConf = unit.GetConfig<CCharacter>()?.PuppetId_Ref;
if (puppetConf != null)
{
    var puppetInfo = new UnitInfo();
    puppetInfo.ConfigId = puppetConf.Id;
    puppetInfo.Type = (int)EEntityType.Puppet;
    puppetInfo.Group = unit.Group;  // 和主人同队伍
    
    // 计算初始位置偏移
    var followInfo = CfgManager.tables.TbPuppetFollowInfo.Get(puppetConf.Id, unit.ConfigId);
    if (followInfo == null)
    {
        puppetInfo.Pos = unit.Position;
    }
    else
    {
        // 相对于主人的旋转方向偏移
        puppetInfo.Pos = unit.Position + unit.Rotation * new TSVector(
            followInfo.OffsetX, followInfo.OffsetY, followInfo.OffsetZ);
    }
    puppetInfo.Forward = unit.Forward;
    puppetInfo.RootUnit = unit;  // 关键：记录宿主
    
    var puppet = UnitFactory.Create(currentScene, puppetInfo);
    puppet.FollowUnit = unit;    // 关键：跟随目标
    
    // 在宿主的 PuppetOwnerComponent 中注册
    var puppetComp = unit.GetComponent<PuppetOwnerComponent>();
    puppetComp.RegisterPuppet(PuppetOwnerComponent.KeyDefault, puppet);
}
```

两个关键引用：
- `RootUnit`：永不改变，指向创建者，代表"我属于谁"
- `FollowUnit`：可以改变，指向当前跟随目标，代表"我现在跟着谁移动"

这种设计允许傀儡"转移"——可以让一个傀儡从原来跟随宿主，改为跟随另一个活跃单位（如队伍切换到新角色时，某些傀儡跟随新主角）。

## PuppetOwnerComponent：傀儡管理器

```csharp
public class PuppetOwnerComponent : Entity
{
    public Dictionary<string, PuppetInfo> PuppetDic;  // 傀儡信息字典
    public List<string> PuppetKeyList;  // 有序的键列表（保证遍历顺序）
    public bool bShow = true;  // 宿主是否显示（影响傀儡显示）
    public bool bHasEvent = false;  // 是否有待处理的事件
}

public class PuppetInfo
{
    public Unit puppet;                     // 傀儡实例
    public bool bAlive;                     // 是否存活
    public bool bShow;                      // 当前显示状态
    public EPuppetFollowMode followMode;    // 跟随模式
    public Dictionary<EStateTag, int> stateList = new(); // 状态引用计数
    public List<string> eventList = new();  // 已触发的事件
}
```

### 注册傀儡

```csharp
public static void RegisterPuppet(this PuppetOwnerComponent self, string key, Unit puppet)
{
    if (!self.PuppetDic.ContainsKey(key))
    {
        self.PuppetKeyList.Add(key);
        var newInfo = new PuppetInfo();
        newInfo.puppet = puppet;
        newInfo.bAlive = true;
        self.PuppetDic.Add(key, newInfo);
    }
}
```

用字符串 `key` 而不是索引来标识傀儡，方便通过名称引用特定傀儡（如"召唤的幻影"、"主要宠物"）。

## 跟随移动实现

```csharp
// SimpleMoveComponentSystem 中的傀儡移动
private static void PuppetMove(this SimpleMoveComponent self, FP fixedDeltaTime, bool immediate)
{
    var unit = self.GetParent<Unit>();
    if (unit.Type != EEntityType.Puppet) return;  // 只对傀儡生效

    var conf = unit.GetConfig<CPuppet>();
    if (conf == null) return;

    var followUnit = unit.FollowUnit;  // 跟随目标
    var followInfo = CfgManager.tables.TbPuppetFollowInfo.Get(unit.ConfigId, followUnit.ConfigId);
    if (followInfo == null) followInfo = CfgManager.tables.TbPuppetFollowInfo.DataList[0];
    
    // 目标跟随位置 = 主人位置 + 旋转偏移
    var followPos = followUnit.Position + followUnit.Rotation * new TSVector(
        followInfo.OffsetX, followInfo.OffsetY, followInfo.OffsetZ);
    
    FP followAreaX = followInfo.AreaX;
    FP followAreaY = followInfo.AreaY;
    FP followAreaZ = followInfo.AreaZ;
    
    // ... 平滑插值到目标位置
}
```

`followInfo` 包含了配置化的跟随参数：
- `OffsetX/Y/Z`：相对于主人的偏移位置（如"站在主人右侧1米"）
- `AreaX/Y/Z`：允许的误差区域（在此范围内不需要移动，避免颤抖）

跟随时使用插值而不是直接设置位置，产生流畅的跟随效果。

## 跟随模式（EPuppetFollowMode）

```csharp
public enum EPuppetFollowMode
{
    Onwer,       // 跟随宿主（默认）
    MainMember,  // 跟随队伍当前上场主角
}
```

两种模式的切换场景：

**Owner 模式**：正常情况，傀儡跟随创建它的宿主。宿主隐藏 → 傀儡隐藏。

**MainMember 模式**：当队伍切换主角时，傀儡不再跟随原宿主（可能已经换到后排），而是跟随当前上场的主角。这用于"队伍共享的傀儡"场景（如某个阵容特性召唤的傀儡，任何主角都可以使用它）。

```csharp
// 刷新跟随目标和显示状态
private static void FreshFollowUnitAndShowState(this PuppetOwnerComponent self, string key, bool bIgnoreEff)
{
    PuppetInfo puppetInfo;
    if (self.PuppetDic.TryGetValue(key, out puppetInfo))
    {
        bool bShow = false;
        switch (puppetInfo.followMode)
        {
            case EPuppetFollowMode.Onwer:
                puppetInfo.puppet.FollowUnit = puppetInfo.puppet.RootUnit;
                bShow = self.bShow;  // 显示状态跟随宿主
                break;
            case EPuppetFollowMode.MainMember:
                TeamComponent teamSys = self.CurrentScene().GetComponent<TeamComponent>();
                var team = teamSys.GetTeamByTeamID(puppetInfo.puppet.Group);
                puppetInfo.puppet.FollowUnit = team.GetCurMainMember();  // 跟随当前主角
                bShow = true;  // 始终显示（不随宿主隐藏）
                break;
        }
        self.SetPuppetShowState(key, bShow, bIgnoreEff);
    }
}
```

## 显隐状态管理

傀儡的显示/隐藏需要双重确认：

```csharp
private static void SetPuppetShowState(this PuppetOwnerComponent self, string key, bool newState, bool bIgnoreEff)
{
    var unit = self.PuppetDic[key].puppet;
    // 最终显示状态 = 傀儡存活 AND 新状态
    bool bShow = self.PuppetDic[key].bAlive & newState;
    
    if (self.PuppetDic[key].bShow != bShow)  // 状态有变化才处理
    {
        self.PuppetDic[key].bShow = bShow;
        
        // 发布显示状态变化事件（渲染层响应）
        EventSystem.Instance.Publish(self.DomainScene(), new Evt_SetGoShowState() { 
            owner = unit, 
            bShow = bShow, 
            IgnoreTS = team.bInForceChange,  // 强制切换时忽略时间缩放
            bIgnoreEff = bIgnoreEff          // 是否跳过特效
        });
        
        // 对应地暂停/恢复战斗逻辑
        unit.SetBattlePause(!bShow, EPauseFlag.puppetShow);
    }
}
```

注意：隐藏的傀儡不只是视觉上消失，还会进入 `BattlePause` 状态（通过 `EPauseFlag.puppetShow` 标记），这意味着它的逻辑也暂停——不参与碰撞检测，不执行状态机，不浪费性能。

显示状态由两个因素共同决定：
1. `bAlive`：傀儡是否"存活"（逻辑上是否应该存在）
2. `newState`：外部状态（宿主是否显示、跟随模式）

只有两个都为 true 时，傀儡才显示。

## 虚拟输入转发

主人可以向傀儡发送虚拟按键，让傀儡执行特定动作：

```csharp
public static void InputKey(this PuppetOwnerComponent self, Unit followUnit, string key, EInputKey vkey)
{
    if (self.PuppetDic.ContainsKey(key))
    {
        var info = self.PuppetDic[key];
        var unit = info.puppet;
        
        // 只对当前跟随特定单位且存活可见的傀儡生效
        if (info.bAlive && unit.FollowUnit == followUnit && !unit.BattlePause)
        {
            var inputComponent = unit.GetComponent<VirtualInputComponent>();
            inputComponent.OnTap((int)vkey);  // 触发虚拟按键
            
            var fsmComp = unit.GetComponent<FSMComponent>();
            fsmComp.CheckAnyState();  // 立刻检查状态转换
        }
    }
}
```

通过虚拟输入，主人技能图中的节点可以控制傀儡执行特定攻击，实现"主人大招，傀儡同步发动配合技"的效果。

## 傀儡事件系统

傀儡可以记录执行过的事件，主人可以查询这些事件来做决策：

```csharp
// 傀儡记录事件
public static void RecordPuppetEvent(this PuppetOwnerComponent self, Unit puppet, string eventStr)
{
    string key = self.GetPuppetKey(puppet);
    if (key != null)
    {
        self.PuppetDic[key].eventList.Add(eventStr);
        self.bHasEvent = true;
    }
}

// 主人触发傀儡事件（每帧统一触发）
public static void FirePuppetEvent(this PuppetOwnerComponent self)
{
    if (self.bHasEvent)
    {
        var unit = self.GetParent<Unit>();
        unit.GetComponent<EventDispatcherComponent>().FireEvent(new Evt_OnPupetEvent());
        self.bHasEvent = false;
    }
}

// 主人查询傀儡是否触发了特定事件
public static bool CheckPuppetEvent(this PuppetOwnerComponent self, bool bAny, string key, string eventStr)
{
    if (bAny)
    {
        // 任意傀儡触发了该事件
        foreach (var k in self.PuppetKeyList)
        {
            if (self.PuppetDic[k].eventList.Contains(eventStr))
                return true;
        }
    }
    else
    {
        // 指定 key 的傀儡触发了该事件
        if (self.PuppetDic.ContainsKey(key))
        {
            if (self.PuppetDic[key].eventList.Contains(eventStr))
                return true;
        }
    }
    return false;
}
```

这套事件系统让主人的技能逻辑可以感知傀儡的行为，实现复杂的主从协同逻辑，而不需要傀儡直接调用主人的函数（保持了解耦）。

## 更新顺序保证

傀儡的更新必须晚于主人：

```csharp
// WorldComponent.EnterFrame
private static void GetUnits(this WorldComponent self, List<Unit> allUnits, 
    List<Unit> units1, List<Unit> units2)
{
    foreach (var unit in allUnits)
    {
        if (unit.Type == EEntityType.Puppet)
            units2.Add(unit);  // 傀儡放第二批
        else
            units1.Add(unit);  // 普通单位放第一批
    }
}

// 先更新普通单位
self.Tick(activeUnits1, deltaTime);
// 再更新傀儡
self.Tick(activeUnits2, deltaTime);
```

这确保了傀儡在每帧中读取的主人位置，是主人本帧更新后的最新位置，而不是上帧的旧位置（滞后一帧的问题）。

## 设计总结

傀儡系统的核心设计哲学是：**最大化复用战斗单元系统，最小化特殊处理**。

傀儡本质上是一个普通的 `Unit`，只是多了：
- `RootUnit` 和 `FollowUnit` 两个引用
- 由 `PuppetOwnerComponent` 统一管理显隐和跟随
- 更新顺序保证在主人之后

这种设计让傀儡可以拥有完整的战斗能力（技能、碰撞、攻击），同时保持与主人的协同关系。

| 特性 | 实现方式 |
|------|---------|
| 跟随移动 | SimpleMoveComponent 中的 PuppetMove |
| 显隐同步 | bAlive × bShow 双重条件 + BattlePause |
| 逻辑暂停 | EPauseFlag.puppetShow 标记 |
| 输入转发 | VirtualInputComponent.OnTap |
| 事件通信 | eventList 记录 + CheckPuppetEvent 查询 |
| 更新顺序 | WorldComponent 分批更新（普通先，傀儡后）|

理解傀儡系统，不只是理解一个具体功能，更是理解如何基于同一套基础架构，用最小的成本扩展出多样的战斗单元类型。
