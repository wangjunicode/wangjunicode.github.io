# 根运动系统与动画驱动位移

在动作游戏中，"打一刀向前冲刺三步"这种效果需要动画和物理的精确配合。如果用代码硬编码"冲刺3步"的移动量，和动画脚步完全对不上，玩家会感觉"脚踩着地面但身体在滑行"。

根运动（Root Motion）技术解决了这个问题：**让动画本身包含位移数据，物理引擎直接读取这些位移数据来移动角色**。本文深入解析这套系统的技术实现。

---

## 第一性原理：什么是根运动？

在 3D 动画中，角色的骨骼通常有一个根骨骼（Root Bone）。根运动是指：根骨骼在动画播放过程中的位置变化。

**没有根运动**：角色在原地跑步，用代码手动计算移动量

**有根运动**：动画本身就包含了每帧根骨骼的位移量，代码读取并应用这个位移

根运动的核心优势：
1. **动画与移动完全吻合**：脚步声、位移、动画姿态完美同步
2. **策划可调整**：修改动画就修改了移动轨迹，无需改代码
3. **支持复杂运动**：旋转、跳跃弧线、特殊攀爬动作均可由动画驱动

## RootMotionData：根运动数据结构

```csharp
[System.Serializable]
[MemoryPackable]
public partial class RootMotionData
{
    // 每帧的根骨骼位移（定点数，保证帧同步一致性）
    public List<TSVector> rootMotionPositions;
    
    // 每帧的根骨骼旋转
    public List<TSQuaternion> rootMotionRotations;
}
```

这是**预计算（Baked）**的根运动数据：在动画导入时，系统会采样动画中每帧的根骨骼位置和旋转，存储为这个列表。运行时直接读取预计算数据，不需要实时计算骨骼变换。

**为什么要预计算？**

1. **帧同步需要定点数**：Unity 的动画系统使用 `float`，在不同平台可能有微小误差。预计算时将 `float` 精确转换为定点数 `TSVector`，保证所有平台结果一致。

2. **性能优化**：运行时只需要按帧索引查找数组，O(1) 复杂度，比实时骨骼计算快得多。

3. **序列化友好**：预计算数据可以序列化为文件，作为技能包的一部分加载。

## 按帧采样与时间插值

```csharp
// 按帧索引采样（整帧）
public void GetSampleData(int frameIdx, ref TSVector displacement, ref TSQuaternion rotation)
{
    if (frameIdx >= 0 && frameIdx < rootMotionPositions.Count)
    {
        displacement = rootMotionPositions[frameIdx];
        rotation = rootMotionRotations[frameIdx];
    }
    else
    {
        // 超出范围：使用最后一帧（动画停在末尾）
        displacement = rootMotionPositions[^1];
        rotation = rootMotionRotations[^1];
    }
}

// 按时间采样（带帧间插值）
public void GetSampleData(FP time, ref TSVector displacement, ref TSQuaternion rotation)
{
    // 1. 计算当前时间对应的帧范围
    var frameIdx = (time / EngineDefine.fixedDeltaTime_Orignal).AsInt();
    var nextFrameIdx = frameIdx + 1;
    
    // 2. 获取相邻两帧的数据
    TSVector displacement1 = TSVector.zero;
    TSQuaternion rotation1 = TSQuaternion.identity;
    TSVector displacement2 = TSVector.zero;
    TSQuaternion rotation2 = TSQuaternion.identity;
    GetSampleData(frameIdx, ref displacement1, ref rotation1);
    GetSampleData(nextFrameIdx, ref displacement2, ref rotation2);
    
    // 3. 计算在两帧之间的比例
    var modTime = time - frameIdx * EngineDefine.fixedDeltaTime_Orignal;
    var t = modTime / EngineDefine.fixedDeltaTime_Orignal;  // [0, 1)
    
    // 4. 线性插值位移，球面插值旋转
    displacement = TSVector.Lerp(displacement1, displacement2, t);
    rotation = TSQuaternion.Slerp(rotation1, rotation2, t);
}
```

两种采样模式：

**按帧索引采样**（逻辑帧专用）：每个逻辑帧固定采样一次，结果确定性最强。

**按时间采样**（支持插值）：渲染层可以在逻辑帧之间进行时间插值，让根运动在渲染帧之间也是平滑的，而不是每逻辑帧"跳一次"。

`TSQuaternion.Slerp`（球面线性插值）而不是 `TSVector.Lerp`：旋转的插值必须用 Slerp，才能保证旋转路径是角度空间中的最短弧，而不是欧拉角的简单线性混合（后者可能产生奇怪的旋转轨迹）。

## RootMotionComponent：全局根运动管理

```csharp
public class RootMotionComponent : Singleton<RootMotionComponent>
{
    // 线程安全的资源缓存（多线程异步加载时使用）
    public ConcurrentDictionary<string, object> ResourceMappingDataDict = new();

    // 懒加载：获取数据，不存在则加载
    [LogTrackIgnore]  // 帧轨迹调试时忽略此函数（太频繁）
    public T TryGetData<T>(string path)
    {
        KHDebug.IgnoreTrack();
        if (ResourceMappingDataDict.TryGetValue(path, out var data))
        {
            return (T)data;  // 缓存命中，直接返回
        }
        data = DeserializeData<T>(path);  // 缓存未命中，反序列化加载
        return (T)data;
    }
}
```

`RootMotionComponent` 是一个单例，管理所有根运动数据的加载和缓存：

- **线程安全**：`ConcurrentDictionary` 支持异步加载场景下的并发访问
- **懒加载**：只有在需要时才加载数据
- **[LogTrackIgnore]**：帧轨迹调试工具会记录每个函数调用，但 `TryGetData` 每帧都会被调用多次，加了 `IgnoreTrack` 避免日志爆炸

## 根运动在战斗中的应用

### 技能攻击时的位移

一个挥刀技能需要向前冲刺：

```csharp
// 技能图节点：播放根运动
public class GS_ApplyRootMotion : FlowNodeBase
{
    public override void OnExecute(Flow flow)
    {
        var unit = flow.GetVar<Unit>("self");
        var skillComp = unit.GetComponent<SkillComponent>();
        
        // 获取当前技能的根运动数据
        var currentSkill = skillComp.GetCurrentSkill();
        var rootMotionData = currentSkill.GetRootMotionData();
        
        if (rootMotionData != null)
        {
            // 告诉物理系统：接下来由根运动数据驱动位移
            var physicsComp = unit.GetComponent<PhysicsComponent>();
            physicsComp.SetRootMotionEnabled(true, rootMotionData);
        }
        
        output.Call(flow);
    }
}
```

### 物理系统中的根运动应用

```csharp
// PhysicsComponentSystem.UpdatePhase2 中
private static void ApplyRootMotion(this PhysicsComponent self, FP deltaTime)
{
    if (!self.useRootMotion || self.rootMotionData == null) return;
    
    var currentFrame = self.rootMotionCurrentFrame;
    
    // 获取当前帧的根运动位移（相对于角色本地空间）
    TSVector localDisplacement = TSVector.zero;
    TSQuaternion localRotation = TSQuaternion.identity;
    self.rootMotionData.GetSampleData(currentFrame, ref localDisplacement, ref localRotation);
    
    // 转换到世界空间（考虑角色朝向）
    var worldDisplacement = self.TransientRotation * localDisplacement;
    
    // 应用到位置（叠加到目标速度）
    self.TransientPosition += worldDisplacement;
    
    self.rootMotionCurrentFrame++;
    if (self.rootMotionCurrentFrame >= self.rootMotionData.rootMotionPositions.Count)
    {
        self.SetRootMotionEnabled(false);  // 根运动数据结束，关闭
    }
}
```

根运动的位移是在**角色本地空间**中定义的（通常是向前 Z 轴方向的位移），转换到世界空间时需要乘以角色的旋转矩阵。

## 运动扭曲（Motion Warping）

根运动的问题：如果攻击目标不在正确的位置怎么办？角色会移动到"空气"里。

运动扭曲（Motion Warping）技术解决这个问题：在根运动的基础上，实时调整位移量，让角色在动画结束时恰好到达目标位置。

```csharp
public class MotionWarpingData
{
    public List<MotionWarpingPoint> Points;  // 扭曲关键帧
    
    public struct MotionWarpingPoint
    {
        public int Frame;      // 在第几帧应用扭曲
        public FP WarpRatio;   // 扭曲比例（0=不扭曲，1=完全扭曲到目标）
    }
}
```

扭曲的计算：

```csharp
// 计算经过扭曲后的帧位移
private TSVector GetWarpedDisplacement(int frame, TSVector originalDisplacement)
{
    var warpRatio = warpingData.GetWarpRatio(frame);
    if (warpRatio <= 0) return originalDisplacement;
    
    var targetPos = GetTargetPosition();  // 目标位置
    var currentPos = unit.Position;
    var idealDisplacement = (targetPos - currentPos) / remainingFrames;  // 理想每帧位移
    
    // 在原始根运动位移和理想位移之间插值
    return TSVector.Lerp(originalDisplacement, idealDisplacement, warpRatio);
}
```

运动扭曲让技能打击"吸附"到目标上，视觉上感觉是精确命中目标。

## 编辑器预览

```csharp
#if UNITY_EDITOR
public AnimationCurve GetPreviewCurve(int startCut, int length)
{
    if (_previewRange.x != startCut || _previewRange.y != length) _previewCurve = null;
    if (_previewCurve != null) return _previewCurve;
    
    _previewRange = new Vector2Int(startCut, length);
    _previewCurve = new AnimationCurve();
    
    var len = Mathf.Min(rootMotionPositions.Count, length);
    for (int i = startCut; i < len; i++)
    {
        // 每帧的移动量大小（向量长度）
        _previewCurve.AddKey(1f * i / len, rootMotionPositions[i].magnitude.AsFloat());
    }
    return _previewCurve;
}
#endif
```

这个编辑器专用函数将根运动数据可视化为 `AnimationCurve`，可以在 Unity Editor 的 Inspector 中查看每帧的移动量曲线。

程序可以直观地看到：
- 技能的哪个阶段位移最大（爆发力）
- 位移是否平滑（非平滑可能导致穿墙）
- 每段动画的实际移动轨迹

`#if UNITY_EDITOR` 确保这段代码只在编辑器中编译，不影响打包后的性能。

## 设计总结

| 特性 | 实现方式 |
|------|---------|
| 预计算 | 动画导入时采样，存储为定点数列表 |
| 帧同步安全 | TSVector/TSQuaternion 定点数存储 |
| 时间插值 | Lerp + Slerp 平滑帧间过渡 |
| 懒加载缓存 | ConcurrentDictionary 线程安全缓存 |
| 运动扭曲 | 实时调整位移量，吸附到目标 |
| 编辑器工具 | AnimationCurve 可视化根运动 |

根运动是动作游戏"打击感"的重要组成部分。它让每个技能的位移都与动画精确对齐，玩家会自然感觉到攻击的"真实物理重量"，而不是动画和移动各走各的路。

对初学者来说，理解根运动的核心思路：**动画决定移动，而不是代码决定移动**。这个思维转变，是从技术美术过渡到动作游戏程序员的关键一步。
