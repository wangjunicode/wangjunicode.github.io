## 前言

一套成熟的游戏框架，除了 ECS、事件系统这类"大件"，同样离不开一批小而精的工具类。这些工具类往往只有几十行代码，却频繁出现在框架各处，是整个系统的"毛细血管"。

本文剖析游戏框架 Core 层中五个工具类的实现：
- **BitwiseOpHelper**：位运算帮助类
- **RandomGenerator**：线程安全随机数生成器
- **ByteHelper**：字节数组操作扩展
- **StringHelper**：字符串与字节数组互转
- **LinqExtensions**：IEnumerable 扩展方法集

## 一、BitwiseOpHelper：位运算的系统性封装

### 1.1 为什么要封装位运算？

位运算是游戏开发中的高频操作，但原始位运算代码可读性很差：

```csharp
// 原始写法：很难看出在做什么
flags = flags | (1 << 3);
flags = flags & ~(1 << 3);
bool hasFlag = (flags & (1 << 3)) != 0;

// 封装后
flags = BitwiseOpHelper.SetBit(flags, 3);
flags = BitwiseOpHelper.ClearBit(flags, 3);
bool hasFlag = BitwiseOpHelper.IsBitSet(flags, 3);
```

封装后的代码语义清晰，降低出错概率（如忘记 `~` 符号）。

### 1.2 基础位操作

```csharp
// 将第 position 位设为 1
public static int SetBit(int value, int position)
    => value | (1 << position);

// 将第 position 位清为 0
public static int ClearBit(int value, int position)
    => value & ~(1 << position);

// 翻转第 position 位
public static int ToggleBit(int value, int position)
    => value ^ (1 << position);

// 检查第 position 位是否为 1
public static bool IsBitSet(int value, int position)
    => (value & (1 << position)) != 0;
```

这四个是"原子操作"，覆盖了99%的单位操作场景。

### 1.3 多位段操作（掩码范围）

游戏中经常需要在一个整数里打包多个小数值，如技能标志位、状态压缩等：

```csharp
// 创建从 startPos 开始、长度为 length 的掩码
public static int CreateMaskRange(int startPos, int length)
    => ((1 << length) - 1) << startPos;

// 提取某段位的值
public static int GetBits(int value, int startPos, int length)
{
    int mask = CreateMaskRange(startPos, length);
    return (value & mask) >> startPos;  // 与运算 + 右移对齐到 0
}

// 将某段位设为新值
public static int SetBitsValue(int value, int startPos, int length, int newValue)
{
    int mask = CreateMaskRange(startPos, length);
    value = value & ~mask;              // 清除目标段
    return value | ((newValue << startPos) & mask);  // 写入新值
}
```

**实战示例：技能状态压缩**

```csharp
// 用一个 int 存储技能的多个状态标志
// bit 0-2: 等级（0-7）
// bit 3-7: 冷却阶段（0-31）
// bit 8: 是否激活

int skillState = 0;
skillState = BitwiseOpHelper.SetBitsValue(skillState, 0, 3, 5);  // 设等级为 5
skillState = BitwiseOpHelper.SetBitsValue(skillState, 3, 5, 12); // 设冷却阶段为 12
skillState = BitwiseOpHelper.SetBit(skillState, 8);              // 激活

int level = BitwiseOpHelper.GetBits(skillState, 0, 3);           // 读取等级 = 5
bool isActive = BitwiseOpHelper.IsBitSet(skillState, 8);         // 是否激活 = true
```

### 1.4 高级位运算技巧

**Brian Kernighan 算法统计 1 的个数：**

```csharp
public static int CountSetBits(int value)
{
    int count = 0;
    while (value != 0)
    {
        value &= value - 1;  // 每次消除最低位的 1
        count++;
    }
    return count;
}
```

`value &= value - 1` 是经典技巧：`value - 1` 会把最低位的 1 变为 0，同时低位全变 1，与原值做与运算就消除了最低位的 1。

**判断是否为 2 的幂：**

```csharp
public static bool IsPowerOfTwo(int value)
    => value > 0 && (value & (value - 1)) == 0;
```

2 的幂的二进制只有一个 1（如 8 = 1000），减 1 后恰好全是 0（0111），与运算结果为 0。

### 1.5 游戏中的经典位运算场景

| 场景 | 数据结构 | 操作 |
|------|----------|------|
| 技能标志位 | `int flags` | SetBit / ClearBit / IsBitSet |
| 状态压缩存储 | `int packed` | SetBitsValue / GetBits |
| 图层遮罩（LayerMask） | `int mask` | IsAllBitsSet / IsAnyBitSet |
| 权限系统 | `long permissions` | SetBit(long) / IsBitSet(long) |

## 二、RandomGenerator：线程安全的随机数工具

### 2.1 直接用 `new Random()` 的问题

```csharp
// 危险！
private static Random _random = new Random();
// 多线程并发调用 _random.Next() 会导致内部状态损坏，返回 0 或重复值
```

`System.Random` 不是线程安全的，多线程场景下需要加锁或使用线程本地存储。

### 2.2 线程本地随机数方案

```csharp
public static class RandomGenerator
{
    [StaticField]
    [ThreadStatic]
    private static Random random;  // 每个线程独立实例

    private static Random GetRandom()
    {
        return random ??= new Random(Guid.NewGuid().GetHashCode());
        // 首次访问时用 GUID 哈希值做种子，每个线程种子不同
    }
}
```

关键设计：
- `[ThreadStatic]`：每个线程持有独立的 `Random` 实例，彻底避免竞态
- **GUID 种子**：`Guid.NewGuid().GetHashCode()` 保证不同线程的种子不同，不会产生相同的随机序列
- `[StaticField]`：这是框架的自定义特性，标记需要在域重置时清理的静态字段

### 2.3 功能覆盖

```csharp
// 整数范围随机（常用）
public static int RandomNumber(int lower, int upper)
    => GetRandom().Next(lower, upper);  // 含 lower，不含 upper

// 布尔随机
public static bool RandomBool()
    => GetRandom().Next(2) == 0;

// 从数组/List 随机取一个元素
public static T RandomArray<T>(T[] array)
    => array[RandomNumber(0, array.Length)];

// Fisher-Yates 洗牌算法
public static void BreakRank<T>(List<T> arr)
{
    for (int i = 0; i < arr.Count; i++)
    {
        int index = GetRandom().Next(0, arr.Count);
        (arr[index], arr[i]) = (arr[i], arr[index]);
    }
}

// 0~1 浮点数（精度到百万分之一）
public static float RandFloat01()
    => RandomNumber(0, 1000000) / 1000000f;
```

### 2.4 大整数随机数的拼接方式

```csharp
public static long RandInt64()
{
    uint r1 = RandUInt32();
    uint r2 = RandUInt32();
    return (long)(((ulong)r1 << 32) | r2);
}
```

`Random.Next()` 最大只能生成 31 位的正整数（避免符号位问题）。要生成 64 位随机数，需要生成两个 32 位值拼接，这是标准做法。

### 2.5 洗牌算法分析

```csharp
public static void BreakRank<T>(List<T> arr)
{
    for (int i = 0; i < arr.Count; i++)
    {
        int index = GetRandom().Next(0, arr.Count);
        (arr[index], arr[i]) = (arr[i], arr[index]);
    }
}
```

注意：这里用的是"每次从全范围随机取位置"，而不是标准的 Fisher-Yates（标准版是从 `i` 到末尾随机取）。两种方案结果都能打乱，但标准 Fisher-Yates 在概率分布上更均匀。游戏场景下这个差异通常可以接受。

## 三、ByteHelper 与 StringHelper：字节操作的扩展方法

### 3.1 ByteHelper：字节数组的调试与序列化

```csharp
public static class ByteHelper
{
    // 单字节转十六进制字符串
    public static string ToHex(this byte b) => b.ToString("X2");

    // 字节数组转完整十六进制字符串
    public static string ToHex(this byte[] bytes)
    {
        var sb = new StringBuilder();
        foreach (byte b in bytes)
            sb.Append(b.ToString("X2"));
        return sb.ToString();
    }

    // 带偏移量的部分转换
    public static string ToHex(this byte[] bytes, int offset, int count) { ... }

    // 字节数组转字符串（系统默认编码）
    public static string ToStr(this byte[] bytes) => Encoding.Default.GetString(bytes);
    public static string Utf8ToStr(this byte[] bytes) => Encoding.UTF8.GetString(bytes);
}
```

`ToHex` 系列方法在网络协议调试时极为有用，可以快速打印数据包内容。

**低级字节写入（小端序）：**

```csharp
public static void WriteTo(this byte[] bytes, int offset, uint num)
{
    bytes[offset]     = (byte)(num & 0xff);
    bytes[offset + 1] = (byte)((num & 0xff00) >> 8);
    bytes[offset + 2] = (byte)((num & 0xff0000) >> 16);
    bytes[offset + 3] = (byte)((num & 0xff000000) >> 24);
}
```

这是小端序（Little-Endian）写入，低位字节在低地址。网络协议通常用大端序（Big-Endian），但游戏内部存储用小端序与 x86 CPU 原生格式一致，效率更高。

unsafe 版本的 long 写入：

```csharp
public static unsafe void WriteTo(this byte[] bytes, int offset, long num)
{
    byte* bPoint = (byte*)&num;
    for (int i = 0; i < sizeof(long); ++i)
        bytes[offset + i] = bPoint[i];
}
```

直接操作指针，避免多次移位运算，性能更高（但需要 `unsafe` 上下文）。

### 3.2 StringHelper：字符串与字节数组的桥接

```csharp
public static class StringHelper
{
    // 字符串 → 字节数组
    public static byte[] ToByteArray(this string str)
        => Encoding.Default.GetBytes(str);

    public static byte[] ToUtf8(this string str)
        => Encoding.UTF8.GetBytes(str);

    // 十六进制字符串 → 字节数组（解析）
    public static byte[] HexToBytes(this string hexString)
    {
        if (hexString.Length % 2 != 0)
            throw new ArgumentException("...");
        var hexAsBytes = new byte[hexString.Length / 2];
        for (int index = 0; index < hexAsBytes.Length; index++)
        {
            string byteValue = hexString.Substring(index * 2, 2);
            hexAsBytes[index] = byte.Parse(byteValue, NumberStyles.HexNumber);
        }
        return hexAsBytes;
    }

    // 格式化字符串（语法糖）
    public static string Fmt(this string text, params object[] args)
        => string.Format(text, args);
}
```

`Fmt` 扩展方法让格式化字符串更流畅：

```csharp
// 传统写法
string msg = string.Format("玩家 {0} 击杀了 {1}", playerName, targetName);

// 扩展方法写法
string msg = "玩家 {0} 击杀了 {1}".Fmt(playerName, targetName);
```

## 四、LinqExtensions：强化版的 LINQ 扩展

### 4.1 调试神器：Examine

```csharp
public static IEnumerable<T> Examine<T>(this IEnumerable<T> source, Action<T> action)
{
    foreach (T obj in source)
    {
        action(obj);   // 先执行 action（如打日志）
        yield return obj;  // 再 yield
    }
}
```

在 LINQ 管道中间插入调试代码，不打断链式调用：

```csharp
var result = entities
    .Where(e => e.IsAlive)
    .Examine(e => Debug.Log($"Processing: {e.Id}"))  // 链式调试
    .Select(e => e.GetComponent<HealthComponent>())
    .ToList();
```

### 4.2 ForEach：让 LINQ 支持副作用

标准 LINQ 没有 `ForEach`（因为 LINQ 设计上是无副作用的函数式风格），但游戏开发中经常需要：

```csharp
// 返回 source 自身，支持链式
public static IEnumerable<T> ForEach<T>(this IEnumerable<T> source, Action<T> action)
{
    foreach (T obj in source)
        action(obj);
    return source;
}

// 带索引版本
public static IEnumerable<T> ForEach<T>(this IEnumerable<T> source, Action<T, int> action)
{
    int num = 0;
    foreach (T obj in source)
        action(obj, num++);
    return source;
}
```

使用示例：

```csharp
// 给所有敌人施加减速效果，并记录日志
enemies
    .Where(e => e.IsInRange(player.Position, 10f))
    .ForEach((e, i) => {
        e.ApplyDebuff(DebuffType.Slow);
        Debug.Log($"Enemy {i}: {e.Id} slowed");
    });
```

### 4.3 PrependWith / AppendWith：动态序列拼接

```csharp
// 在序列开头插入元素
public static IEnumerable<T> PrependWith<T>(this IEnumerable<T> source, T prepend)
{
    yield return prepend;
    foreach (T obj in source)
        yield return obj;
}

// 条件性插入
public static IEnumerable<T> PrependIf<T>(
    this IEnumerable<T> source, bool condition, T prepend)
{
    if (condition) yield return prepend;
    foreach (T obj in source) yield return obj;
}
```

游戏中常见的"默认选项排第一"场景：

```csharp
var options = availableSkills
    .PrependIf(hasDefaultSkill, defaultSkill)  // 条件性加在最前
    .AppendWith(cancelOption);                  // 取消选项加在最后
```

### 4.4 IsNullOrEmpty 与 Populate

```csharp
// 判断 IList 为空（比 source == null || source.Count == 0 简洁）
public static bool IsNullOrEmpty<T>(this IList<T> list)
    => list == null || list.Count == 0;

// 批量填充
public static void Populate<T>(this IList<T> list, T item)
{
    int count = list.Count;
    for (int index = 0; index < count; ++index)
        list[index] = item;
}
```

`Populate` 可以快速重置列表（比 `Clear` + `AddRange` 少一次内存重新分配）：

```csharp
// 每帧重置状态数组
_damageBuffer.Populate(0f);  // 比 new float[size] 开销更小
```

### 4.5 FilterCast：安全类型过滤

```csharp
public static IEnumerable<T> FilterCast<T>(this IEnumerable source)
{
    foreach (object obj1 in source)
    {
        if (obj1 is T obj2)
            yield return obj2;
    }
}
```

比 `OfType<T>()` 语义更明确，在处理混合类型容器时很实用（如游戏对象列表中筛选特定组件类型）。

## 五、工具类的设计原则总结

通过分析以上五个工具类，可以提炼出一套工具库的设计原则：

### 原则一：扩展方法优先
所有工具都以扩展方法（`this T obj`）形式提供，让调用代码更自然：
```csharp
// 比 StringHelper.ToUtf8(str) 更自然
byte[] data = "hello".ToUtf8();
```

### 原则二：null 安全
`RandomGenerator.BreakRank` 开头检查 `arr == null || arr.Count < 2`，`BitwiseOpHelper` 不处理 null（因为 int/long 不能为 null），所有涉及引用类型的工具都优先检查 null。

### 原则三：线程安全明确标注
`RandomGenerator` 通过 `[ThreadStatic]` 解决并发问题，并且在类注释中写明"支持多线程"，让使用者有明确的信息。

### 原则四：性能意识
- `ByteHelper.WriteTo` 用 unsafe 指针操作提升性能
- `LinqExtensions.AddRange` 对 `List<T>` 做类型判断，走更快的原生实现
- `FixedLengthQueue` 固定容量，避免动态扩容

### 原则五：单一职责
每个类只做一件事：ByteHelper 只处理字节数组，StringHelper 只处理字符串，BitwiseOpHelper 只做位运算。不把所有工具塞进一个"Utils"类。

## 六、总结

这五个工具类虽然代码量不大，但各自解决了一个具体的痛点：

| 工具类 | 核心价值 |
|--------|---------|
| BitwiseOpHelper | 位操作语义化，降低状态压缩的出错率 |
| RandomGenerator | 线程安全随机数，避免多线程状态损坏 |
| ByteHelper | 字节数组调试与序列化的基础设施 |
| StringHelper | 字符串↔字节数组转换的统一入口 |
| LinqExtensions | 扩展 LINQ 的副作用能力和调试能力 |

工具库往往是团队技术积累的缩影——它们不显眼，但质量直接影响整个代码库的健壮性。写好工具类，是成为合格框架工程师的必修课。
