# 游戏内存优化：对象池与内存复用

内存管理是游戏性能优化中最关键的环节之一。频繁的内存分配和释放会触发 GC（垃圾回收），导致帧率抖动，严重影响玩家体验。本文深入讲解对象池模式与内存复用策略。

## 一、为什么需要对象池

### 1.1 GC 触发的根本原因

```csharp
// 反面教材：频繁创建销毁子弹
void FireBullet()
{
    // 每次开枪都 new 一个新对象 → 堆内存分配 → 触发 GC
    GameObject bullet = new GameObject("Bullet");
    bullet.AddComponent<BulletController>();
    Destroy(bullet, 3f);
}
```

每次 `new` 操作：
1. 在托管堆上申请内存
2. 初始化对象头
3. 触发 GC 根扫描（当堆达到阈值时）

在高频操作（子弹、特效、UI动画）下，GC 停顿可达 **10~50ms**，直接表现为卡顿。

### 1.2 内存碎片化问题

```
分配前：[空闲][已用][空闲][已用][空闲]
释放后：[空闲][空闲][空闲][已用][空闲]  ← 碎片化
再分配：找不到连续大块 → 触发压缩 GC（更耗时）
```

## 二、基础对象池实现

### 2.1 通用泛型对象池

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

public class ObjectPool<T> where T : class
{
    private readonly Func<T> _createFunc;
    private readonly Action<T> _onGet;
    private readonly Action<T> _onRelease;
    private readonly Action<T> _onDestroy;
    private readonly int _maxSize;
    
    private readonly Stack<T> _pool;
    private int _activeCount;

    public int CountInactive => _pool.Count;
    public int CountActive => _activeCount;
    public int CountAll => CountInactive + CountActive;

    public ObjectPool(
        Func<T> createFunc,
        Action<T> onGet = null,
        Action<T> onRelease = null,
        Action<T> onDestroy = null,
        bool collectionCheck = true,
        int defaultCapacity = 10,
        int maxSize = 1000)
    {
        _createFunc = createFunc ?? throw new ArgumentNullException(nameof(createFunc));
        _onGet = onGet;
        _onRelease = onRelease;
        _onDestroy = onDestroy;
        _maxSize = maxSize;
        _pool = new Stack<T>(defaultCapacity);
    }

    public T Get()
    {
        T element;
        if (_pool.Count == 0)
        {
            element = _createFunc();
        }
        else
        {
            element = _pool.Pop();
        }
        
        _activeCount++;
        _onGet?.Invoke(element);
        return element;
    }

    public void Release(T element)
    {
        _onRelease?.Invoke(element);
        _activeCount--;
        
        if (_pool.Count < _maxSize)
        {
            _pool.Push(element);
        }
        else
        {
            _onDestroy?.Invoke(element);
        }
    }

    public void Clear()
    {
        while (_pool.Count > 0)
        {
            _onDestroy?.Invoke(_pool.Pop());
        }
    }
    
    // 预热：提前创建指定数量对象
    public void Prewarm(int count)
    {
        for (int i = 0; i < count; i++)
        {
            var obj = _createFunc();
            _onRelease?.Invoke(obj);
            _pool.Push(obj);
        }
    }
}
```

### 2.2 GameObject 专用池

```csharp
public class GameObjectPool : MonoBehaviour
{
    [SerializeField] private GameObject prefab;
    [SerializeField] private int initialSize = 10;
    [SerializeField] private int maxSize = 100;
    
    private Queue<GameObject> _pool;
    private Transform _poolParent;

    private void Awake()
    {
        _poolParent = new GameObject($"[Pool] {prefab.name}").transform;
        _pool = new Queue<GameObject>(initialSize);
        
        // 预填充
        for (int i = 0; i < initialSize; i++)
        {
            CreateNewInstance();
        }
    }

    private GameObject CreateNewInstance()
    {
        var go = Instantiate(prefab, _poolParent);
        go.SetActive(false);
        _pool.Enqueue(go);
        return go;
    }

    public GameObject Spawn(Vector3 position, Quaternion rotation)
    {
        GameObject go;
        
        if (_pool.Count > 0)
        {
            go = _pool.Dequeue();
        }
        else if (GetTotalCount() < maxSize)
        {
            go = Instantiate(prefab);
        }
        else
        {
            Debug.LogWarning($"Pool {prefab.name} 已达最大容量 {maxSize}");
            return null;
        }
        
        go.transform.SetPositionAndRotation(position, rotation);
        go.SetActive(true);
        
        // 通知对象被激活
        go.GetComponent<IPoolable>()?.OnSpawn();
        
        return go;
    }

    public void Despawn(GameObject go)
    {
        go.GetComponent<IPoolable>()?.OnDespawn();
        go.SetActive(false);
        go.transform.SetParent(_poolParent);
        _pool.Enqueue(go);
    }

    private int GetTotalCount()
    {
        // 池中 + 场景中激活的
        return _pool.Count; // 简化版
    }
}

// 对象池接口
public interface IPoolable
{
    void OnSpawn();
    void OnDespawn();
}
```

## 三、高级内存复用技术

### 3.1 数组池（避免临时数组分配）

```csharp
using System.Buffers;

// 反面教材：每帧分配临时数组
void Update()
{
    // 每帧 new 数组 → GC 压力
    RaycastHit[] hits = new RaycastHit[20];
    int count = Physics.RaycastNonAlloc(ray, hits, 100f);
    ProcessHits(hits, count);
}

// 正确做法：使用 ArrayPool
void UpdateOptimized()
{
    // 从池中租借数组，无堆分配
    RaycastHit[] hits = ArrayPool<RaycastHit>.Shared.Rent(20);
    try
    {
        int count = Physics.RaycastNonAlloc(ray, hits, 100f);
        ProcessHits(hits, count);
    }
    finally
    {
        // 归还到池，不触发 GC
        ArrayPool<RaycastHit>.Shared.Return(hits);
    }
}
```

### 3.2 自定义内存分配器

```csharp
// 线性分配器：适合帧内临时数据
public unsafe class LinearAllocator : IDisposable
{
    private byte* _buffer;
    private int _offset;
    private readonly int _capacity;

    public LinearAllocator(int capacity)
    {
        _capacity = capacity;
        _buffer = (byte*)System.Runtime.InteropServices.Marshal.AllocHGlobal(capacity);
        _offset = 0;
    }

    public T* Allocate<T>() where T : unmanaged
    {
        int size = sizeof(T);
        int aligned = (_offset + size - 1) & ~(size - 1); // 内存对齐
        
        if (aligned + size > _capacity)
            throw new OutOfMemoryException("线性分配器已满");
        
        T* ptr = (T*)(_buffer + aligned);
        _offset = aligned + size;
        return ptr;
    }

    // 帧末重置，O(1) 复杂度
    public void Reset() => _offset = 0;

    public void Dispose()
    {
        if (_buffer != null)
        {
            System.Runtime.InteropServices.Marshal.FreeHGlobal((IntPtr)_buffer);
            _buffer = null;
        }
    }
}
```

### 3.3 字符串池（避免字符串拼接 GC）

```csharp
public static class StringPool
{
    private static readonly Dictionary<string, string> _pool 
        = new Dictionary<string, string>(StringComparer.Ordinal);

    // 对于频繁使用的固定字符串进行池化
    public static string Intern(string str)
    {
        if (_pool.TryGetValue(str, out var cached))
            return cached;
        
        _pool[str] = str;
        return str;
    }
    
    public static void Clear() => _pool.Clear();
}

// 使用 StringBuilder 缓存
public class CachedStringBuilder
{
    private static readonly StringBuilder _sb = new StringBuilder(256);
    
    public static string Format(string format, int value)
    {
        _sb.Clear();
        _sb.Append(format);
        _sb.Append(value);
        return _sb.ToString();
    }
}
```

## 四、Unity 原生池支持（Unity 2021+）

```csharp
using UnityEngine.Pool;

public class BulletManager : MonoBehaviour
{
    [SerializeField] private Bullet bulletPrefab;
    
    // Unity 内置 ObjectPool（线程安全版本）
    private ObjectPool<Bullet> _bulletPool;

    private void Awake()
    {
        _bulletPool = new ObjectPool<Bullet>(
            createFunc: () => Instantiate(bulletPrefab),
            actionOnGet: bullet => bullet.gameObject.SetActive(true),
            actionOnRelease: bullet => bullet.gameObject.SetActive(false),
            actionOnDestroy: bullet => Destroy(bullet.gameObject),
            collectionCheck: false, // 生产环境关闭，提升性能
            defaultCapacity: 50,
            maxSize: 200
        );
    }

    public void Fire(Vector3 pos, Vector3 dir)
    {
        var bullet = _bulletPool.Get();
        bullet.Initialize(pos, dir, () => _bulletPool.Release(bullet));
    }
}
```

## 五、内存分析与监控

### 5.1 Profiler 内存分析

```csharp
// 标记 GC 分配热点
using Unity.Profiling;

public class MemoryProfiler : MonoBehaviour
{
    private ProfilerRecorder _gcAllocRecorder;
    private ProfilerRecorder _totalMemRecorder;

    void OnEnable()
    {
        _gcAllocRecorder = ProfilerRecorder.StartNew(
            ProfilerCategory.Memory, "GC Allocated In Frame");
        _totalMemRecorder = ProfilerRecorder.StartNew(
            ProfilerCategory.Memory, "Total Used Memory");
    }

    void OnDisable()
    {
        _gcAllocRecorder.Dispose();
        _totalMemRecorder.Dispose();
    }

    void Update()
    {
        long gcAlloc = _gcAllocRecorder.LastValue;
        if (gcAlloc > 1024) // 超过 1KB 报警
        {
            Debug.LogWarning($"本帧 GC 分配: {gcAlloc / 1024f:F2} KB");
        }
    }
}
```

### 5.2 内存泄漏检测

```csharp
public class PoolLeakDetector
{
    private readonly Dictionary<int, string> _activeObjects 
        = new Dictionary<int, string>();

    public void TrackGet(object obj, string location)
    {
#if UNITY_EDITOR || DEVELOPMENT_BUILD
        _activeObjects[obj.GetHashCode()] = 
            $"{location} @ {System.DateTime.Now:HH:mm:ss}";
#endif
    }

    public void TrackRelease(object obj)
    {
#if UNITY_EDITOR || DEVELOPMENT_BUILD
        _activeObjects.Remove(obj.GetHashCode());
#endif
    }

    public void ReportLeaks()
    {
        if (_activeObjects.Count > 0)
        {
            Debug.LogWarning($"检测到 {_activeObjects.Count} 个对象未归还到池:");
            foreach (var kvp in _activeObjects)
            {
                Debug.LogWarning($"  HashCode={kvp.Key}, 获取位置: {kvp.Value}");
            }
        }
    }
}
```

## 六、最佳实践总结

| 场景 | 推荐方案 | 预期收益 |
|------|----------|----------|
| 子弹/特效 | GameObject 对象池 | 消除 Instantiate/Destroy GC |
| 临时数组 | ArrayPool\<T\> | 消除堆分配 |
| 碰撞检测结果 | 预分配+NonAlloc API | 零 GC 检测 |
| UI 文本拼接 | StringBuilder 缓存 | 减少 string GC |
| 帧内临时数据 | 线性分配器 | 零 GC 临时分配 |
| 网络包数据 | MemoryPool\<byte\> | 复用网络缓冲区 |

### 核心原则

1. **预分配优于运行时分配** — 在加载阶段预热对象池
2. **复用优于创建** — 宁可多占内存，也要减少 GC 压力
3. **NonAlloc API 优先** — Physics、Audio 等系统的 NonAlloc 版本
4. **结构体优于类** — 值类型栈上分配，无 GC 压力
5. **监控驱动优化** — 先用 Profiler 定位热点，再针对性优化

> 💡 **黄金法则**：移动端游戏每帧 GC 分配应控制在 **0KB**，桌面端控制在 **1KB 以内**。
