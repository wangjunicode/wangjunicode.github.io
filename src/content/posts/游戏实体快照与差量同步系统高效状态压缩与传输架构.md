# 游戏实体快照与差量同步系统：高效状态压缩与传输架构

## 一、为什么需要快照差量同步？

在多人网络游戏中，服务器需要持续将世界状态同步给所有客户端。如果每帧广播所有实体的完整状态，即使只有 100 个实体、每个实体 50 字节，20tick 的服务器每秒就需要向每个客户端发送：

```
100 实体 × 50 字节 × 20 tick = 100KB/s（单客户端）
100KB/s × 64 客户端 = 6.4MB/s（服务器出带宽）
```

这在移动端和大规模同屏场景下完全不可接受。**快照差量同步**（Snapshot Delta Sync）正是解决这一问题的核心技术。

### 1.1 核心思路

```
完整快照（Full Snapshot）
  ↓ 包含所有实体所有字段的状态
  
差量快照（Delta Snapshot）
  ↓ 只包含相对于上一帧有变化的字段
  
进一步优化：
  ↓ 基线快照 + 多帧差量序列
  ↓ 字段级别位域标记
  ↓ 量化压缩浮点数
```

### 1.2 工业案例参考

- **Quake 3**：先驱，完整快照 + 差量编码，基于 Baseline 快照压缩
- **Overwatch**：每 tick（1/64s）发送差量，使用确认 ACK 追踪各客户端基线
- **Warzone/COD**：多优先级实体分层发送，视距外实体低频同步

```csharp
[System.Serializable]
public struct EntitySnapshot
{
    // 标识
    public uint EntityId;
    public byte EntityType;    // 0=Player, 1=Enemy, 2=Projectile
    
    // 变换
    public short PosX;        // 量化后的坐标（×100 存储 cm 精度）
    public short PosY;
    public short PosZ;
    public short RotY;        // 量化后的 Y 旋转（/180 × 32767）

    public ushort MaxHp;
    public byte   State;      // 状态机枚举
    public byte   AnimState;  // 动画状态

    public short VelX;
    public short VelZ;
    
    // 变更标记（哪些字段有变化）
    public uint DirtyFlags;  // 位域，最多 32 个字段
}

// 字段对应的 DirtyFlag 位
public static class EntityDirtyBits
{
    public const uint Position  = 1 << 0;
    public const uint Rotation  = 1 << 1;
    public const uint Hp        = 1 << 2;
    public const uint State     = 1 << 3;
    public const uint AnimState = 1 << 4;
    public const uint Velocity  = 1 << 5;
    public const uint All       = uint.MaxValue;
}
```

### 2.2 快照存储：环形缓冲区

```csharp
// 服务端维护最近 N 帧快照，用于差量编码
public class SnapshotRingBuffer
{
    private const int BUFFER_SIZE = 64; // 保留最近 64 帧（~3.2 秒 @ 20tick）
    
    private readonly EntitySnapshot[][] m_Frames; // [frame_index][entity_index]
    private readonly uint[]             m_Ticks;  // 每帧对应的 tick 号
    private          int                m_Head;   // 最新帧的环形下标
    
    public SnapshotRingBuffer(int maxEntities)
    {
        m_Frames = new EntitySnapshot[BUFFER_SIZE][];
        m_Ticks  = new uint[BUFFER_SIZE];
        
        for (int i = 0; i < BUFFER_SIZE; i++)
            m_Frames[i] = new EntitySnapshot[maxEntities];
    }
    
    // 写入新帧快照
    public void PushFrame(uint tick, EntitySnapshot[] snapshots)
    {
        m_Head = (m_Head + 1) % BUFFER_SIZE;
        m_Ticks[m_Head] = tick;
        Array.Copy(snapshots, m_Frames[m_Head], snapshots.Length);
    }
    
    // 按 tick 查找帧（用于差量编码的基线）
    public EntitySnapshot[] GetFrame(uint tick)
    {
        for (int i = 0; i < BUFFER_SIZE; i++)
        {
            if (m_Ticks[i] == tick)
                return m_Frames[i];
        }
        return null; // 已超出缓冲范围
    }
    
    // 获取最新帧
    public EntitySnapshot[] GetLatest() => m_Frames[m_Head];
    public uint LatestTick => m_Ticks[m_Head];
}
```

## 三、差量编码器

### 3.1 核心差量计算

```csharp
public static class SnapshotDelta
{
    /// <summary>
    /// 计算从 baseline 到 current 的差量，填充 DirtyFlags
    /// </summary>
    public static EntitySnapshot ComputeDelta(
        ref EntitySnapshot baseline,
        ref EntitySnapshot current)
    {
        var delta = current; // 复制当前状态
        delta.DirtyFlags = 0;
        
        // 逐字段比较，设置变更位
        if (current.PosX  != baseline.PosX ||
            current.PosY  != baseline.PosY ||
            current.PosZ  != baseline.PosZ)
        {
            delta.DirtyFlags |= EntityDirtyBits.Position;
        }
        
        if (current.RotY != baseline.RotY)
            delta.DirtyFlags |= EntityDirtyBits.Rotation;
        
        if (current.Hp != baseline.Hp)
            delta.DirtyFlags |= EntityDirtyBits.Hp;
        
        if (current.State != baseline.State)
            delta.DirtyFlags |= EntityDirtyBits.State;
        
        if (current.AnimState != baseline.AnimState)
            delta.DirtyFlags |= EntityDirtyBits.AnimState;
        
        if (current.VelX != baseline.VelX || current.VelZ != baseline.VelZ)
            delta.DirtyFlags |= EntityDirtyBits.Velocity;
        
        return delta;
    }
    
    /// <summary>
    /// 将差量快照序列化到 BitWriter（仅写有变化的字段）
    /// </summary>
    public static void Serialize(ref EntitySnapshot delta, BitWriter writer)
    {
        writer.WriteUInt(delta.EntityId, 16);
        writer.WriteUInt(delta.DirtyFlags, 6); // 最多 6 个字段标记
        
        if ((delta.DirtyFlags & EntityDirtyBits.Position) != 0)
        {
            writer.WriteShort(delta.PosX);
            writer.WriteShort(delta.PosY);
            writer.WriteShort(delta.PosZ);
        }
        
        if ((delta.DirtyFlags & EntityDirtyBits.Rotation) != 0)
            writer.WriteShort(delta.RotY);
        
        if ((delta.DirtyFlags & EntityDirtyBits.Hp) != 0)
            writer.WriteUShort(delta.Hp);
        
        if ((delta.DirtyFlags & EntityDirtyBits.State) != 0)
            writer.WriteByte(delta.State);
        
        if ((delta.DirtyFlags & EntityDirtyBits.AnimState) != 0)
            writer.WriteByte(delta.AnimState);
        
        if ((delta.DirtyFlags & EntityDirtyBits.Velocity) != 0)
        {
            writer.WriteShort(delta.VelX);
            writer.WriteShort(delta.VelZ);
        }
    }
    
    /// <summary>
    /// 从 BitReader 反序列化差量，叠加到 baseline
    /// </summary>
    public static EntitySnapshot Deserialize(ref EntitySnapshot baseline, BitReader reader)
    {
        var result = baseline; // 从基线开始
        
        result.EntityId  = reader.ReadUInt(16);
        uint dirtyFlags  = reader.ReadUInt(6);
        
        if ((dirtyFlags & EntityDirtyBits.Position) != 0)
        {
            result.PosX = reader.ReadShort();
            result.PosY = reader.ReadShort();
            result.PosZ = reader.ReadShort();
        }
        
        if ((dirtyFlags & EntityDirtyBits.Rotation) != 0)
            result.RotY = reader.ReadShort();
        
        if ((dirtyFlags & EntityDirtyBits.Hp) != 0)
            result.Hp = reader.ReadUShort();
        
        if ((dirtyFlags & EntityDirtyBits.State) != 0)
            result.State = reader.ReadByte();
        
        if ((dirtyFlags & EntityDirtyBits.AnimState) != 0)
            result.AnimState = reader.ReadByte();
        
        if ((dirtyFlags & EntityDirtyBits.Velocity) != 0)
        {
            result.VelX = reader.ReadShort();
            result.VelZ = reader.ReadShort();
        }
        
        result.DirtyFlags = dirtyFlags;
        return result;
    }
}
```

### 3.2 高性能 BitWriter/BitReader

```csharp
using System;

/// <summary>位级别写入器，压缩字段到最小位数</summary>
public unsafe class BitWriter
{
    private byte[] m_Buffer;
    private int    m_BitPos; // 当前写入的位位置
    
    public BitWriter(int capacityBytes = 1024)
    {
        m_Buffer = new byte[capacityBytes];
        m_BitPos = 0;
    }
    
    public void WriteUInt(uint value, int bits)
    {
        // 断言 value 在 bits 范围内
        for (int i = bits - 1; i >= 0; i--)
        {
            int  byteIndex = m_BitPos / 8;
            int  bitIndex  = 7 - (m_BitPos % 8);
            bool bit       = ((value >> i) & 1) == 1;
            
            if (bit) m_Buffer[byteIndex] |=  (byte)(1 << bitIndex);
            else     m_Buffer[byteIndex] &= (byte)~(1 << bitIndex);
            
            m_BitPos++;
        }
    }
    
    public void WriteShort(short value)  => WriteUInt((uint)(ushort)value, 16);
    public void WriteUShort(ushort value) => WriteUInt(value, 16);
    public void WriteByte(byte value)    => WriteUInt(value, 8);
    
    // 获取已写入的字节（向上取整）
    public ArraySegment<byte> GetBuffer() 
        => new ArraySegment<byte>(m_Buffer, 0, (m_BitPos + 7) / 8);
    
    public void Reset() => m_BitPos = 0;
}

public class BitReader
{
    private readonly byte[] m_Buffer;
    private          int    m_BitPos;
    
    public BitReader(byte[] buffer, int offset = 0)
    {
        m_Buffer = buffer;
        m_BitPos = offset * 8;
    }
    
    public uint ReadUInt(int bits)
    {
        uint value = 0;
        for (int i = bits - 1; i >= 0; i--)
        {
            int  byteIndex = m_BitPos / 8;
            int  bitIndex  = 7 - (m_BitPos % 8);
            uint bit       = (uint)((m_Buffer[byteIndex] >> bitIndex) & 1);
            value |= bit << i;
            m_BitPos++;
        }
        return value;
    }
    
    public short  ReadShort()  => (short)(ushort)ReadUInt(16);
    public ushort ReadUShort() => (ushort)ReadUInt(16);
    public byte   ReadByte()   => (byte)ReadUInt(8);
}
```

## 四、坐标量化（Quantization）

浮点坐标直接传输需要 4 字节/轴，量化后可压缩到 2 字节：

```csharp
public static class Quantize
{
    // 坐标精度：0.01m（1cm），范围 ±327.67m
    private const float POS_SCALE  = 100f;
    private const float POS_MAX    = 32767f;
    
    public static short EncodePosition(float worldPos)
    {
        float scaled = worldPos * POS_SCALE;
        scaled = Mathf.Clamp(scaled, -POS_MAX, POS_MAX);
        return (short)Mathf.RoundToInt(scaled);
    }
    
    public static float DecodePosition(short encoded)
        => encoded / POS_SCALE;
    
    // 旋转量化：精度 360/65535 ≈ 0.0055°
    public static ushort EncodeRotation(float angle)
    {
        angle = ((angle % 360f) + 360f) % 360f; // 归一化到 [0, 360)
        return (ushort)(angle / 360f * 65535f);
    }
    
    public static float DecodeRotation(ushort encoded)
        => encoded / 65535f * 360f;
    
    // 速度量化：精度 0.01m/s，范围 ±327.67m/s
    public static short EncodeVelocity(float vel) => EncodePosition(vel);
    public static float DecodeVelocity(short encoded) => DecodePosition(encoded);
    
    // 将 Vector3 转为量化快照坐标
    public static (short x, short y, short z) EncodeVector3(Vector3 pos)
        => (EncodePosition(pos.x), EncodePosition(pos.y), EncodePosition(pos.z));
    
    public static Vector3 DecodeVector3(short x, short y, short z)
        => new Vector3(DecodePosition(x), DecodePosition(y), DecodePosition(z));
}
```

## 五、服务端：每帧差量包生成

```csharp
public class SnapshotServer
{
    private readonly SnapshotRingBuffer m_SnapshotBuffer;
    private          uint               m_CurrentTick;
    
    // 每个客户端的 ACK 状态（知道客户端确认到哪一帧）
    private readonly Dictionary<int, uint> m_ClientAckedTick = new Dictionary<int, uint>();
    
    public SnapshotServer(int maxEntities)
    {
        m_SnapshotBuffer = new SnapshotRingBuffer(maxEntities);
    }
    
    // 每 Tick 调用：收集当前状态，生成差量包
    public void Tick(EntitySnapshot[] currentState)
    {
        m_CurrentTick++;
        m_SnapshotBuffer.PushFrame(m_CurrentTick, currentState);
        
        // 向每个客户端发送差量包
        foreach (var (clientId, ackedTick) in m_ClientAckedTick)
        {
            byte[] packet = BuildDeltaPacket(clientId, ackedTick, currentState);
            SendToClient(clientId, packet);
        }
    }
    
    private byte[] BuildDeltaPacket(int clientId, uint ackedTick, EntitySnapshot[] current)
    {
        var writer = new BitWriter(4096);
        
        // 包头
        writer.WriteUInt(m_CurrentTick, 32);
        writer.WriteUInt(ackedTick, 32);      // 告知客户端基线 tick
        
        // 获取基线快照
        EntitySnapshot[] baseline = m_SnapshotBuffer.GetFrame(ackedTick);
        bool isFullUpdate = (baseline == null); // 基线过期，发全量
        
        writer.WriteUInt(isFullUpdate ? 1u : 0u, 1);
        
        int changedCount = 0;
        int countPos     = writer.BitPosition; // 记录 count 写入位置（后续回填）
        writer.WriteUInt(0, 16); // 占位：变化实体数量
        
        for (int i = 0; i < current.Length; i++)
        {
            EntitySnapshot delta;
            
            if (isFullUpdate)
            {
                delta = current[i];
                delta.DirtyFlags = EntityDirtyBits.All;
            }
            else
            {
                delta = SnapshotDelta.ComputeDelta(ref baseline[i], ref current[i]);
            }
            
            if (delta.DirtyFlags == 0 && !isFullUpdate) continue;
            
            SnapshotDelta.Serialize(ref delta, writer);
            changedCount++;
        }
        
        // 回填实体数量（需要 BitWriter 支持随机写入，此处简化）
        // 实际可在头部预留 2 字节再回填
        
        return writer.GetBuffer().ToArray();
    }
    
    // 客户端 ACK 处理
    public void OnClientAck(int clientId, uint ackedTick)
    {
        m_ClientAckedTick[clientId] = ackedTick;
    }
    
    private void SendToClient(int clientId, byte[] data)
    {
        // 调用底层网络发送（UDP/KCP等）
    }
    
    public int BitPosition => 0; // 占位，实际在 BitWriter 中实现
}
```

## 六、客户端：快照插值与应用

### 6.1 快照缓冲区（Jitter Buffer）

```csharp
public class ClientSnapshotBuffer
{
    // 保存已收到但尚未应用的快照（按 tick 排序）
    private readonly SortedDictionary<uint, EntitySnapshot[]> m_Buffer
        = new SortedDictionary<uint, EntitySnapshot[]>();
    
    private const int BUFFER_DELAY_TICKS = 3; // 缓冲 3 帧用于抗抖动
    
    private uint m_LastAppliedTick = 0;
    private uint m_LastReceivedTick = 0;
    
    public void OnReceiveSnapshot(uint tick, EntitySnapshot[] snapshots)
    {
        m_Buffer[tick] = snapshots;
        m_LastReceivedTick = Math.Max(m_LastReceivedTick, tick);
    }
    
    // 返回应当应用的当前帧快照（若有）
    public bool TryGetApplyFrame(uint currentTick, out EntitySnapshot[] snapshots)
    {
        // 应用 currentTick - BUFFER_DELAY_TICKS 的快照
        uint applyTick = currentTick > BUFFER_DELAY_TICKS
            ? currentTick - BUFFER_DELAY_TICKS
            : 0;
        
        if (m_Buffer.TryGetValue(applyTick, out snapshots))
        {
            m_Buffer.Remove(applyTick);
            m_LastAppliedTick = applyTick;
            return true;
        }
        return false;
    }
    
    // 获取插值用的相邻两帧
    public bool TryGetInterpolateFrames(uint fromTick, uint toTick,
        out EntitySnapshot[] from, out EntitySnapshot[] to)
    {
        return m_Buffer.TryGetValue(fromTick, out from)
            && m_Buffer.TryGetValue(toTick,   out to);
    }
}
```

## 七、优先级系统：按重要性分配带宽

当实体数量很多时，需要对实体进行优先级排序：

```csharp
public class SnapshotPriorityScheduler
{
    // 实体优先级累积桶
    private readonly Dictionary<uint, float> m_PriorityBucket = new Dictionary<uint, float>();
    
    // 计算实体对某客户端的优先级
    public float CalculatePriority(EntitySnapshot entity, Vector3 clientPos, float deltaTime)
    {
        Vector3 entityPos = Quantize.DecodeVector3(entity.PosX, entity.PosY, entity.PosZ);
        float   dist      = Vector3.Distance(entityPos, clientPos);
        
        // 距离衰减：距离越远优先级越低
        float distFactor = 1f / (1f + dist * 0.1f);
        
        // 移动中的实体优先级更高
        bool isMoving   = entity.VelX != 0 || entity.VelZ != 0;
        float moveFactor = isMoving ? 2f : 1f;
        
        // 战斗状态（HP 变化）优先级最高
        float combatFactor = (entity.DirtyFlags & EntityDirtyBits.Hp) != 0 ? 3f : 1f;
        
        float priority = distFactor * moveFactor * combatFactor;
        
        // 累积（每帧未发送时积分，避免饥饿）
        uint id = entity.EntityId;
        if (!m_PriorityBucket.ContainsKey(id)) m_PriorityBucket[id] = 0f;
        m_PriorityBucket[id] += priority * deltaTime;
        
        return m_PriorityBucket[id];
    }
    
    // 选取本帧发送的实体（按优先级排序，取 Top-N）
    public List<uint> SelectEntitiesForUpdate(
        EntitySnapshot[] all, Vector3 clientPos, int maxEntities, float deltaTime)
    {
        var scored = new List<(uint id, float score)>();
        
        foreach (var snap in all)
        {
            float score = CalculatePriority(snap, clientPos, deltaTime);
            scored.Add((snap.EntityId, score));
        }
        
        scored.Sort((a, b) => b.score.CompareTo(a.score));
        
        var result = new List<uint>();
        for (int i = 0; i < Math.Min(maxEntities, scored.Count); i++)
        {
            result.Add(scored[i].id);
            m_PriorityBucket[scored[i].id] = 0f; // 发送后清空积分
        }
        return result;
    }
}
```

```csharp
            {
                bool posChanged = UnityEngine.Random.value < 0.30f;
                bool hpChanged  = UnityEngine.Random.value < 0.05f;
                
                // 全量包：每个实体固定 16 字节
                // EntityId(2) + PosXYZ(6) + RotY(2) + HP(2) + State(1) + Anim(1) + Vel(2) = 16B
                fullBytesTotal += 16;
                
                // 差量包：只写有变化的字段
                uint dirtyFlags = 0;
                if (posChanged) dirtyFlags |= EntityDirtyBits.Position; // +6B
                if (hpChanged)  dirtyFlags |= EntityDirtyBits.Hp;       // +2B
                
                // EntityId(2) + DirtyFlags(1) + 可选字段
                float deltaSize = 3f;
                if (posChanged) deltaSize += 6f;
                if (hpChanged)  deltaSize += 2f;
                
                // 未变化的实体可完全跳过（DirtyFlags = 0 不发送）
                if (dirtyFlags != 0) deltaBytesTotal += deltaSize;
            }
        }
        
        float fullBPS  = fullBytesTotal  / (TICKS / 20f); // 每秒字节数
        float deltaBPS = deltaBytesTotal / (TICKS / 20f);
        
        UnityEngine.Debug.Log($"全量同步: {fullBPS:F0} B/s ({fullBPS/1024:F1} KB/s)");
        UnityEngine.Debug.Log($"差量同步: {deltaBPS:F0} B/s ({deltaBPS/1024:F1} KB/s)");
        UnityEngine.Debug.Log($"压缩率: {(1f - deltaBPS/fullBPS)*100:F1}%");
        
        // 典型结果：全量 ~32KB/s，差量 ~3.5KB/s，压缩率 ~89%
    }
}
```

## 十、最佳实践总结

### 10.1 架构设计原则

| 场景 | 推荐策略 |
|------|---------|
| 低频状态变化（HP、状态机） | 差量 + 事件触发，不随 tick 发送 |
| 高频状态变化（位置、速度） | 每 tick 差量，配合客户端插值 |
| 重要实体（玩家自身、Boss） | 高优先级，强制每 tick 更新 |
| 远距离实体 | 降频（每 5 tick 一次）或只发事件 |
| 断线重连 | 强制全量快照（最近一帧） |

### 10.2 常见优化手段

1. **位域差量**：只传有变化字段，减少 70-90% 带宽
2. **坐标量化**：4字节 float → 2字节 short，精度 1cm
3. **死区过滤**：位置变化小于 1cm 不标记为 dirty
4. **优先级调度**：近处/战斗实体高频，远处低频
5. **ACK 滑动窗口**：参考 QUAKE3 的多基线方案，丢包时仍有候选基线

### 10.3 调试工具建议

```csharp
// 带宽监控
public class SnapshotBandwidthMonitor : MonoBehaviour
{
    private long m_BytesSentThisSecond;
    private float m_ResetTimer;
    
    void Update()
    {
        m_ResetTimer += Time.deltaTime;
        if (m_ResetTimer >= 1f)
        {
            UnityEngine.Debug.Log($"[Snapshot] 出带宽: {m_BytesSentThisSecond / 1024f:F1} KB/s");
            m_BytesSentThisSecond = 0;
            m_ResetTimer -= 1f;
        }
    }
    
    public void RecordSent(int bytes) => m_BytesSentThisSecond += bytes;
}
```

> **参考目标**：移动端游戏，100 个可见实体的快照差量带宽控制在 **10KB/s 以内**（客户端下行），服务端对 64 玩家的总出带宽控制在 **1MB/s 以内**，这是工业级多人游戏的常见目标值。
