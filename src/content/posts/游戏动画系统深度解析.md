## 动画系统的层次

```
Unity 动画系统的层次结构：

AnimationClip（动画片段）
  最基础的单元：一段关键帧动画数据

Animator Controller（动画控制器）
  状态机：管理多个 AnimationClip 的切换逻辑
  Blend Tree：多个动画的混合（如移动速度混合走/跑）

Animation Rigging（程序化动画）
  基于物理/逻辑的实时骨骼控制
  例：手持物品时手指自动调整、IK（逆运动学）

Timeline（时间轴）
  用于过场动画、复杂序列动画
```

## 一、Animator Controller 设计

### 1.1 状态机设计原则

```
常见错误：把所有状态都放在一层
  → Animator 里有 50+ 个状态，过渡线条密密麻麻
  → 难以维护，Bug 难以排查
  
正确做法：分层（Layers）+ Sub-State Machine（子状态机）

Layer 设计：
  Base Layer（0）：主要动画（Idle/Walk/Run/Fall/Jump）
  Upper Body（1）：上半身叠加动画（攻击/射击）权重 1，Additive 模式
  Face（2）：面部动画（仅头部骨骼）
  
  好处：上半身可以独立于下半身播放（跑步同时攻击）
  
Sub-State Machine 用于状态分组：
  Combat 子状态机：Attack1/Attack2/Attack3/ComboFinisher
  Movement 子状态机：Walk/Run/Crouch/CrouchWalk
```

### 1.2 参数管理

```csharp
/// <summary>
/// Animator 参数管理
/// 使用 Hash 替代字符串（性能优化）
/// </summary>
public class CharacterAnimator : MonoBehaviour
{
    private Animator _animator;
    
    // ✅ 预先缓存参数 Hash，避免每帧字符串查找
    private static readonly int SpeedHash = Animator.StringToHash("Speed");
    private static readonly int IsGroundedHash = Animator.StringToHash("IsGrounded");
    private static readonly int AttackHash = Animator.StringToHash("Attack");
    private static readonly int JumpHash = Animator.StringToHash("Jump");
    private static readonly int DeathHash = Animator.StringToHash("Death");
    
    void Awake() => _animator = GetComponent<Animator>();
    
    // ❌ 低效：每帧字符串 Hash 计算
    void Update_Wrong(float speed, bool isGrounded)
    {
        _animator.SetFloat("Speed", speed);     // 每帧 Hash("Speed")
        _animator.SetBool("IsGrounded", isGrounded); // 每帧 Hash
    }
    
    // ✅ 高效：使用预缓存的 Hash
    public void SetMovement(float speed, bool isGrounded)
    {
        _animator.SetFloat(SpeedHash, speed);
        _animator.SetBool(IsGroundedHash, isGrounded);
    }
    
    public void TriggerAttack() => _animator.SetTrigger(AttackHash);
    public void TriggerJump() => _animator.SetTrigger(JumpHash);
    public void TriggerDeath() => _animator.SetTrigger(DeathHash);
    
    // 动画事件（在 AnimationClip 中设置时间点触发）
    // 方法名必须匹配 AnimationClip 中配置的事件函数名
    void OnAttackHit()
    {
        // 攻击帧命中时触发伤害
        _attackComponent?.ApplyDamage();
    }
    
    void OnFootstep()
    {
        // 脚步动画帧触发音效
        _audioComponent?.PlayFootstep();
    }
    
    private AttackComponent _attackComponent;
    private AudioComponent _audioComponent;
}
```

## 二、Blend Tree 进阶

### 2.1 运动混合

```
Blend Tree 最常见用途：移动速度混合

简单 1D Blend Tree（移动速度）：
  参数：Speed（0~1）
  0.0：Idle（站立）
  0.5：Walk（走路）
  1.0：Run（跑步）
  
  效果：随着 Speed 平滑过渡，不会突然切换动画

2D Blend Tree（移动方向）：
  参数：SpeedX（横向速度）, SpeedZ（纵向速度）
  中心（0,0）：Idle
  上（0,1）：Forward_Run
  下（0,-1）：Backward_Walk
  左（-1,0）：Strafe_Left
  右（1,0）：Strafe_Right
  
  效果：可以平滑表现斜向移动
  
  设置方法：
  Blend Type → 2D Freeform Cartesian
  参数 X → SpeedX
  参数 Y → SpeedZ
```

## 三、程序化动画（Animation Rigging）

### 3.1 IK（逆运动学）

```csharp
/// <summary>
/// 脚步 IK：让角色在不平地面上双脚落地准确
/// </summary>
[RequireComponent(typeof(Animator))]
public class FootIK : MonoBehaviour
{
    private Animator _animator;
    
    [Header("IK 参数")]
    [SerializeField] private LayerMask _groundLayer;
    [SerializeField] private float _distanceToGround = 0.1f;
    [SerializeField] [Range(0, 1)] private float _ikWeight = 1f;
    
    void Awake() => _animator = GetComponent<Animator>();
    
    // OnAnimatorIK 在动画 IK pass 中调用
    void OnAnimatorIK(int layerIndex)
    {
        if (!_animator) return;
        
        // 设置两只脚的 IK 目标
        AdjustFootIK(AvatarIKGoal.LeftFoot);
        AdjustFootIK(AvatarIKGoal.RightFoot);
    }
    
    private void AdjustFootIK(AvatarIKGoal foot)
    {
        // 设置 IK 权重
        _animator.SetIKPositionWeight(foot, _ikWeight);
        _animator.SetIKRotationWeight(foot, _ikWeight);
        
        // 从动画中获取脚的当前位置
        Vector3 footPos = _animator.GetIKPosition(foot);
        
        // 向下射线检测地面
        if (Physics.Raycast(footPos + Vector3.up, Vector3.down, out RaycastHit hit, 
                            1.5f, _groundLayer))
        {
            // 将脚放到地面上
            _animator.SetIKPosition(foot, hit.point + Vector3.up * _distanceToGround);
            
            // 让脚的旋转与地面法线对齐
            Quaternion footRotation = Quaternion.LookRotation(
                Vector3.ProjectOnPlane(transform.forward, hit.normal), hit.normal);
            _animator.SetIKRotation(foot, footRotation);
        }
    }
}
```

### 3.2 Look-At IK

```csharp
/// <summary>
/// 头部朝向 IK：角色自然看向目标
/// </summary>
public class LookAtIK : MonoBehaviour
{
    private Animator _animator;
    
    [SerializeField] private Transform _lookTarget;
    [SerializeField] [Range(0, 1)] private float _bodyWeight = 0.1f;
    [SerializeField] [Range(0, 1)] private float _headWeight = 0.9f;
    [SerializeField] private float _turnSpeed = 5f;
    
    private Vector3 _currentLookPos;
    
    void Awake() => _animator = GetComponent<Animator>();
    
    void OnAnimatorIK(int layerIndex)
    {
        if (_lookTarget == null) return;
        
        // 平滑插值（避免头部突然转向）
        _currentLookPos = Vector3.Lerp(
            _currentLookPos, _lookTarget.position, 
            Time.deltaTime * _turnSpeed);
        
        _animator.SetLookAtWeight(1f, _bodyWeight, _headWeight);
        _animator.SetLookAtPosition(_currentLookPos);
    }
}
```

## 四、大量角色的动画优化

### 4.1 GPU Skinning

```
CPU Skinning vs GPU Skinning：

CPU Skinning（默认）：
  每帧在 CPU 上计算所有骨骼变换和顶点蒙皮
  50 个角色 → 50 倍 CPU 开销
  
GPU Skinning（Compute Shader）：
  蒙皮计算在 GPU 上并行进行
  适合大量相同骨架的角色（如 MMO 中大量怪物）
  
设置：
  Skinned Mesh Renderer → Skinned Motion Vectors
  Renderer → GPU Skinning（需要项目设置开启）
```

### 4.2 LOD 动画系统

```csharp
/// <summary>
/// 动画 LOD：根据与摄像机距离降低动画更新频率
/// 对于大型 MMO，可以大幅减少 CPU 动画开销
/// </summary>
public class AnimationLOD : MonoBehaviour
{
    private Animator _animator;
    private Transform _camera;
    
    // LOD 层级配置
    private readonly float[] _lodDistances = { 20f, 40f, 60f };
    private readonly float[] _updateIntervals = { 0f, 0.1f, 0.2f, 0.5f }; // 0=每帧, 0.1=10fps, ...
    
    private float _nextUpdateTime;
    private int _currentLOD = 0;
    
    void Start()
    {
        _animator = GetComponent<Animator>();
        _camera = Camera.main.transform;
    }
    
    void Update()
    {
        // 计算到摄像机的距离，确定 LOD 级别
        float dist = Vector3.Distance(transform.position, _camera.position);
        
        _currentLOD = 0;
        for (int i = 0; i < _lodDistances.Length; i++)
        {
            if (dist > _lodDistances[i]) _currentLOD = i + 1;
            else break;
        }
        
        // 根据 LOD 更新频率决定是否更新动画
        if (Time.time < _nextUpdateTime) return;
        
        _animator.enabled = true;
        _nextUpdateTime = Time.time + _updateIntervals[_currentLOD];
        
        // LOD 最高时，完全停止动画更新
        if (_currentLOD >= 3)
            _animator.enabled = false;
    }
}
```

## 总结

动画系统优化核心：

| 策略 | 适用场景 | 效果 |
|------|---------|------|
| 参数 Hash 缓存 | 所有有 Animator 的角色 | 减少每帧字符串开销 |
| 动画 LOD | MMO/大量 NPC 的游戏 | 大幅减少 CPU 动画开销 |
| GPU Skinning | 大量同类角色 | 把蒙皮计算转移到 GPU |
| 合理分 Layer | 所有角色 | 支持上下半身独立动画 |
| IK 系统 | 角色与环境互动 | 提升自然感，无需手制作动画 |
