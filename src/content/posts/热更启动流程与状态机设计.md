# 07_热更启动流程与状态机设计

> 面向刚入行的毕业生 · 建议搭配 Index.cs、ClientBridge.cs、PatchManager.cs 阅读

---

## 1. 系统概述

游戏的启动流程是整个热更新系统的"主干"，所有热更相关的操作都嵌入在这个流程中。本文档详细拆解从 App 启动到玩家进入游戏大厅的完整时序，帮助读者理解各个组件之间的协作关系。

### 1.1 启动流程的阶段划分

```
阶段 0：App 进程启动
阶段 1：Unity 场景初始化（Index Awake/Start）
阶段 2：闪屏序列（SplashUI）
阶段 3：热更检测（PatchManager + Dolphin）
阶段 4：资源 Catalog 更新（Addressables）
阶段 5：热更 DLL 加载（ClientBridge + HybridCLR）
阶段 6：游戏主逻辑初始化（ClientEntry.DllPostLoad）
阶段 7：游戏正式运行
```

### 2.2 关键状态变量

```csharp
// PatchManager 中的状态变量（理解这些是理解流程的关键）
private bool hadInit;               // PatchManager 是否已初始化
private bool hadCheckUpdateBarOver; // 进度条是否已满
private bool isUpdating;            // Dolphin 是否正在更新
private bool exitUpdate;            // 是否要退出更新（完成或失败）
private bool appReadyBeginRes;      // 程序包检查完成，可以开始资源检查
private bool updateingAA;           // Addressables 是否正在更新
```

这些布尔变量构成了一个隐式状态机，在 `Update()` 每帧检查驱动流程推进。

## 3. 核心代码展示

### 3.1 SplashUI 启动序列

```csharp
// Index.cs
private void StartSplashSequence()
{
    // 同步加载 SplashUI 预制体（启动阶段可以接受同步加载）
    var handle = Addressables.LoadAssetAsync<GameObject>("Patch/SplashUI.prefab");
    var prefab = handle.WaitForCompletion();
    
    if (prefab == null)
    {
        // SplashUI 加载失败，跳过直接进入热更流程
        Log.LogWarning("[Index] SplashUI.prefab load failed, skip splash");
        OnSplashComplete();
        return;
    }
    
    var splashGo = Instantiate(prefab);
    splashGo.name = "SplashUI";
    
    var splashUI = splashGo.GetComponent<SplashUI>();
    if (splashUI != null)
    {
        splashUI.onComplete = OnSplashComplete; // 注册完成回调
        splashUI.StartSequence();               // 开始播放闪屏
    }
    else
    {
        Destroy(splashGo);
        OnSplashComplete(); // 没有 SplashUI 组件，直接继续
    }
}
```

### 3.2 SplashUI 序列（视频+图片）

```csharp
// SplashUI.cs - 使用协程实现顺序播放
private IEnumerator PlaySequence()
{
    // 初始化：隐藏所有内容
    if (displayUGUI != null) displayUGUI.gameObject.SetActive(false);
    if (splashImage != null) splashImage.gameObject.SetActive(false);
    if (splashWarning != null) splashWarning.gameObject.SetActive(false);
    
    // 1. 播放启动视频（腾讯 Logo 视频）
    if (!string.IsNullOrEmpty(videoPath))
        yield return PlayVideo();
    
    // 2. 显示图片1（公司 Logo）
    if (splashImage != null)
        yield return ShowImage(splashImage, splashImageDisplayTime);
    
    // 3. 显示图片2（游戏警告信息）
    if (splashWarning != null)
        yield return ShowImage(splashWarning, splashWarningDisplayTime);
    
    // 完成！触发回调，继续热更流程
    onComplete?.Invoke();
    
    Cleanup();
    Destroy(gameObject);
}

// 图片显示：淡入 → 停留 → 淡出
private IEnumerator ShowImage(Image image, float displayTime)
{
    var imageCanvasGroup = image.GetComponent<CanvasGroup>() 
        ?? image.gameObject.AddComponent<CanvasGroup>();
    
    imageCanvasGroup.alpha = 0;
    image.gameObject.SetActive(true);
    
    yield return imageCanvasGroup.DOFade(1, fadeInDuration).SetUpdate(true).WaitForCompletion();
    yield return new WaitForSecondsRealtime(displayTime);
    yield return imageCanvasGroup.DOFade(0, fadeOutDuration).SetUpdate(true).WaitForCompletion();
    
    image.gameObject.SetActive(false);
}
```

### 3.3 闪屏完成后的回调链

### 3.4 PatchManager Update 驱动状态机

```csharp
// PatchManager.cs - 每帧驱动状态推进
void Update()
{
    if (isSkipPatch) return;
    
    if (hadInit)
    {
        // [状态1] 驱动 Dolphin 回调
        if (isUpdating && mgr != null)
        {
            mgr.DriveUpdateService();
        }

        // [状态2] 退出更新（完成或用户取消）
        if (exitUpdate)
        {
            if (isUpdating && mgr != null)
            {
                mgr.StopUpdateService();
                ResetParam();
            }
        }

        // [状态3] 程序包检查完毕，开始资源更新
        if (appReadyBeginRes)
        {
            SourceUpdate(); // 启动第二阶段 Dolphin 更新
            appReadyBeginRes = false;
        }

        // [状态4] 进度条动画
        if (!isUpdating || !hadCheckUpdateBarOver)
        {
            if (preLoadABProgress < 0.99f)
            {
                if (!updateingAA)
                    preLoadABProgress += 0.01f;  // 模拟进度（每帧+1%）
                onUpdateBar(preLoadABProgress, true);
            }
            else if (preLoadABProgress >= 0.99f)
            {
                onUpdateBar(1, true);
                hadCheckUpdateBarOver = true;
            }
        }
    }
}
```

**进度条设计细节**：当没有实际下载进度时（如只是初始化 SDK），进度条会每帧自增 1%，给用户"在做事"的视觉反馈，防止白屏。实际下载时，进度条由 `OnUpdateProgressInfo` 回调精确控制。

## 4. 设计亮点

### 4.1 协程驱动的顺序流程

整个启动流程大量使用 Unity 协程（`IEnumerator` + `yield return`）来实现**看起来是同步的、实际是异步的**操作：

```csharp
// 等待 Addressables 异步加载完成
var handle = Addressables.LoadContentCatalogAsync(catalogPath);
yield return handle; // 等待加载，期间 Unity 继续渲染其他内容

// 等待下一帧再继续（防止某些异步操作在同帧未完全就绪）
yield return null;
```

协程的优势：代码逻辑线性、易读，同时不阻塞主线程。

### 4.2 闪屏时间利用

SplashUI 播放视频和图片的同时，后台可以开始 GCloud SDK 初始化、网络检查等操作。合理利用启动等待时间，减少总体加载耗时。

### 4.3 优雅降级（每个步骤都有 Fallback）

```
SplashUI 加载失败？→ 跳过，直接进热更
Catalog 加载失败？→ 跳过，使用安装包内资源
DLL 加载失败？   → 打印错误，跳过该程序集（不崩溃）
```

每个步骤都有 Fallback 路径，防止一个环节失败导致整个游戏无法启动。

## 6. 常见问题与最佳实践

### Q1：启动黑屏太久怎么解决？
**A**：检查以下几点：
1. SplashUI 的视频是否过大（建议 MP4，1080p，时长 < 5 秒）
2. Addressables.LoadAssetAsync 是否被同步调用（`WaitForCompletion()`）且资源过大
3. GCloud SDK 初始化是否耗时过长（网络请求超时）
4. 考虑将 GCloud 初始化移到 SplashUI 播放期间并行执行

### Q2：`appReadyBeginRes = true` 后为什么要等一帧才触发 SourceUpdate？
**A**：`appReadyBeginRes` 是在 `OnNoticeNewVersionInfo` 回调中设置的，而回调本身是在 `DriveUpdateService()` 调用栈内触发的。如果立即调用 `SourceUpdate()`，会在 `DriveUpdateService()` 的调用栈中初始化新的 Dolphin 管理器，可能造成状态混乱。延迟到下一帧 `Update()` 中处理，确保了状态的干净切换。

### Q3：游戏更新中途断网怎么处理？
**A**：Dolphin 内置了断点续传能力。当网络恢复后，玩家只需重新启动游戏，Dolphin 会从上次中断处继续下载。`OnUpdateMessageBoxInfo` 中的"重试"按钮调用 `mgr.Continue()` 也可以触发重试。

### Q4：热更流程能否在后台完成（不阻塞玩家进入游戏）？
**A**：当前设计是**阻塞式**的，玩家必须等待热更完成才能进入游戏。这是为了保证代码版本一致性（避免新旧代码混用）。如果需要后台下载（玩家可以先进游戏，后台继续下载），需要将 Dolphin 的资源更新改为可选模式，并在下载完成后提示玩家重启。对于代码热更（HybridCLR DLL），由于需要在启动时加载，必须阻塞等待。

### Q5：如何在启动流程中添加自定义步骤？
**A**：在 `ClientBridge.RunAfterPatchFlow()` 中扩展（当前有被注释掉的 `ResDownloadWsClient.TryCreateAfterPatchAsync()`）：
```csharp
private async ETTask RunAfterPatchFlow(string token)
{
    // 在这里添加热更后、DLL 加载前的自定义步骤
    await MyCustomStep(); // 如：验证用户登录态、预下载核心资源等
    StartGame("");
}
```
