## 设计原则

设计模式的根基是面向对象的 SOLID 原则，遵循这些原则可以让代码更稳定、易维护、具有一致性：

| 原则 | 说明 |
|------|------|
| **单一职责原则 (SRP)** | 一个类只负责一件事 |
| **开闭原则 (OCP)** | 对扩展开放，对修改关闭 |
| **里氏替换原则 (LSP)** | 子类可以替换父类而不影响程序正确性 |
| **依赖倒置原则 (DIP)** | 高层模块不依赖低层模块，都依赖于抽象 |
| **接口隔离原则 (ISP)** | 不强迫使用者依赖不需要的接口 |
| **迪米特法则 (LoD)** | 一个对象应当对其他对象了解最少 |
| **合成复用原则** | 少用继承，多用组合 |

## 游戏开发常用设计模式

### 1. 单例模式（Singleton）

最常用也最容易被滥用的模式，适合全局唯一的管理器类：

```csharp
public class GameManager : MonoBehaviour
{
    public static GameManager Instance { get; private set; }
    
    void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);
    }
}
```

### 2. 观察者模式（Observer）

事件系统的基础，解耦事件发送者和接收者：

```csharp
// 使用 C# 事件
public class EventManager
{
    public static event Action<int> OnPlayerDamaged;
    public static event Action OnPlayerDied;
    
    public static void TriggerPlayerDamaged(int damage)
    {
        OnPlayerDamaged?.Invoke(damage);
    }
}

// 监听者
void OnEnable()  => EventManager.OnPlayerDamaged += HandleDamage;
void OnDisable() => EventManager.OnPlayerDamaged -= HandleDamage;

void HandleDamage(int damage) { /* 处理伤害 */ }
```

### 3. 状态机模式（State Machine）

非常适合角色、游戏流程的状态管理：

```csharp
public interface IState
{
    void Enter();
    void Update();
    void Exit();
}

public class StateMachine
{
    private IState _currentState;
    
    public void ChangeState(IState newState)
    {
        _currentState?.Exit();
        _currentState = newState;
        _currentState?.Enter();
    }
    
    public void Update() => _currentState?.Update();
}
```

### 4. 命令模式（Command）

将操作封装为对象，支持撤销/重做，适合技能系统和编辑器工具：

```csharp
public interface ICommand
{
    void Execute();
    void Undo();
}

public class AttackCommand : ICommand
{
    private readonly Player _player;
    private readonly float _damage;
    
    public AttackCommand(Player player, float damage)
    {
        _player = player;
        _damage = damage;
    }
    
    public void Execute() => _player.TakeDamage(_damage);
    public void Undo()    => _player.HealDamage(_damage);
}
```

### 5. 对象池模式（Object Pool）

避免频繁实例化/销毁对象，减少 GC 压力：

```csharp
public class BulletPool : MonoBehaviour
{
    [SerializeField] private GameObject bulletPrefab;
    private readonly Queue<GameObject> _pool = new();
    
    public GameObject Get()
    {
        if (_pool.Count > 0)
        {
            var obj = _pool.Dequeue();
            obj.SetActive(true);
            return obj;
        }
        return Instantiate(bulletPrefab);
    }
    
    public void Return(GameObject bullet)
    {
        bullet.SetActive(false);
        _pool.Enqueue(bullet);
    }
}
```

### 6. 策略模式（Strategy）

运行时切换算法，适合 AI 行为、移动策略等：

```csharp
public interface IAttackStrategy
{
    void Attack(Transform target);
}

public class MeleeAttack : IAttackStrategy
{
    public void Attack(Transform target) { /* 近战攻击 */ }
}

public class RangedAttack : IAttackStrategy
{
    public void Attack(Transform target) { /* 远程攻击 */ }
}

public class Enemy : MonoBehaviour
{
    private IAttackStrategy _strategy;
    
    void Start()
    {
        // 根据距离切换攻击策略
        _strategy = IsNearPlayer() ? new MeleeAttack() : new RangedAttack();
    }
}
```

## 设计模式的使用原则

> 不要为了使用设计模式而使用设计模式，模式是解决特定问题的工具，要根据实际需求选择合适的模式。
