## 这个博客的起点

这是我游戏开发技术博客的第一篇文章，记录于 2017 年。

从一个游戏开发入门者，到逐渐理解技术框架的设计哲学，这个博客是我技术成长的轨迹记录。

## 核心方法论

### 任何框架都遵循同一条路径

```
原理 → 设计 → 实现 → 优化
```

- **原理**：理解技术解决的本质问题是什么
- **设计**：理解如何通过抽象和架构来解决这个问题
- **实现**：掌握具体的代码实现细节
- **优化**：在实际工程中发现瓶颈并改进

以 Unity 的 Addressables 资源管理系统为例：
- **原理**：游戏需要按需加载资源，避免内存溢出
- **设计**：通过地址（Address）解耦资源路径，引入异步加载 + 引用计数
- **实现**：`Addressables.LoadAssetAsync<T>()` + `AsyncOperationHandle`
- **优化**：分析加载时序，预加载热点资源，合理分组减少 Bundle 数量

### 抓住关键问题，避免迷失在细节中

学习技术时，容易陷入两个极端：

1. **只学皮毛**：会用 API 但不理解原理，遇到问题无从下手
2. **过度深究**：在非关键的实现细节上花费大量时间，忘记了学习的目标

正确的方式是：**先抓主线，再补细节**。

先理解 "这个技术解决什么问题" 和 "它的整体工作流程"，再根据实际需要深入特定细节。

### 做难而正确的事

> 你要创造价值。

游戏开发的价值体现在：
- 做出让玩家享受的游戏体验
- 构建可维护、可扩展的技术架构
- 在性能约束下实现最佳效果
