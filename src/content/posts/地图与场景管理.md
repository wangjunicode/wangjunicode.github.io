## 1. 系统概述

地图与场景管理系统负责游戏中所有场景的**加载、切换、资源管理和生命周期控制**。在本项目中，场景切换与系统域（SystemDomain）切换深度集成，形成了一套完整的"大场景 + 功能系统"双层切换机制。

## 2. 架构设计

### 2.1 双层切换模型

本项目将"切换"分为两个层次：

```
┌─────────────────────────────────────────────┐
│              场景切换（SceneSwitch）          │
│  触发：跨场景跳转（如 主城 → 战斗场景）       │
│  成本：高（需要 LoadScene，销毁/创建对象）    │
└──────────────────┬──────────────────────────┘
                   │ 场景内
┌──────────────────▼──────────────────────────┐
│           系统域切换（SystemDomainSwitch）    │
│  触发：同场景内功能大区域切换                 │
│        （如 大厅 → 养成剧本 → PVP 竞技场）   │
│  成本：中（无需重加载 Scene，只换 UI 和逻辑） │
└─────────────────────────────────────────────┘
```

### 2.2 场景类型

通过 `GameSceneType` 枚举区分不同场景类型：
- `Boot`（启动场景）：游戏初始化、加载配置表
- `Login`（登录场景）：登录界面
- `Main`（主城场景）：游戏主场景（包含大多数功能系统）
- `Battle`（战斗场景）：PVE/PVP 战斗专用场景

### 2.3 Loading 层级（ELoadingLevel）

切换时显示的 Loading 界面分为两个层级：
```
ELoadingLevel
├── Scene  (场景级)   -- 场景切换时的全屏 Loading
└── System (系统级)   -- 系统域切换时的局部或全屏 Loading
```

系统级 Loading 可以由 `SystemDomainHandler.GetLoadingName()` 覆写场景级 Loading，实现差异化 Loading 界面（如养成剧本进入时播放专属动画）。

### 2.4 资源预加载机制

场景切换过程中，资源预加载分为多轮（`LoadDependentResources1~4 + LoadExtraResources`），每轮对应不同类型的资源：

```
资源加载轮次
├── Round 1: 基础 UI 资源（面板、图集）
├── Round 2: 角色模型、动作
├── Round 3: 特效、音效
├── Round 4: 场景相关配置数据
└── Extra:   由下一个 SystemDomain 指定的额外资源
```

这种多轮设计允许各轮次并发加载，同时保证依赖关系（如 Round2 需要 Round1 的图集已加载）。

### 2.5 场景切换完整生命周期

```
Evt_SwitchScene 发布
       │
       ▼
  Init()              初始化上下文
       │
       ▼
  Prepare()           准备（校验、收集下一个 Domain 信息）
       │
       ▼
  ExitPrevDomain      前置退出（清理当前系统状态）
       │
       ▼
  ShowLoading()       显示 Loading 界面
       │
       ▼
  ExitPrevDomain      后置退出（关闭 UI 面板）
       │
       ▼
  LoadAndEnterScene() 加载并进入新 Unity Scene
       │
       ▼
  LoadDependentResources × 4  多轮资源预加载
       │
       ▼
  LoadExtraResources  加载 SystemDomain 额外资源
       │
       ▼
  SwitchSystemDomain  切换到新系统域
       │
       ▼
  EnterNextDomain     前置进入（准备新系统）
       │
       ▼
  HideLoading()       隐藏 Loading 界面
       │
       ▼
  EnterNextDomain     后置进入（打开 UI、播放动效）
       │
       ▼
  Run()               执行额外收尾逻辑
```

## 3. 核心代码展示

### 3.1 场景切换事件处理器

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Scene\SceneSwitchEvent.cs

[Event(SceneType.Client)]
public class SceneSwitchEvent : EventWithRes2<Evt_SwitchScene>
{
    // 只在需要时缓存，避免异步切换时的竞态条件
    private SystemDomainSwitchContext _systemSwitchContext;

    public override async ETTask Handle(Scene scene, Evt_SwitchScene a)
    {
        try
        {
            Init(scene, ref a);

            var (success, context) = await Prepare(scene, a);
            if (!success)
            {
                // 准备失败 → 回退到启动场景（安全兜底）
                SceneChangeHelper.BackToBootScene(scene).Coroutine();
                return;
            }
            _systemSwitchContext = context;

            // 严格有序的异步管道（每步都是独立的 await）
            await ExitPrevSystemDomainPreprocess(context);
            var loadingOptions = await ShowLoading(scene, a);
            context.LoadingPanelName = loadingOptions.UIPath;
            await ExitPrevSystemDomainPostprocess(context);
            await LoadAndEnterScene(scene, a, loadingOptions);
            await LoadDependentResources(scene, a);
            await LoadDependentResources2(scene, a);
            await LoadDependentResources3(scene, a);
            await LoadDependentResources4(scene, a);
            await LoadExtraResources(scene, a);
            await SwitchSystemDomain(context);
            await EnterNextSystemDomainPreprocess(context);
            await HideLoading(scene, a, loadingOptions);
            await EnterNextSystemDomainPostprocess(context);
            await Run(scene, a);
        }
        catch (Exception e)
        {
            Log.Error(e);
        }
    }

    /// <summary>
    /// 收集下一个系统所需的资源（由 SystemDomainHandler 定义）
    /// </summary>
    protected override void GetDependentRes(Scene scene, Evt_SwitchScene evt,
        List<string> lstRes, List<string> lstOther)
    {
        var systemDomainComp = scene.GetComponent<SystemDomainComponent>();
        if (systemDomainComp.TryGetDomainHandler(
            _systemSwitchContext.NextDomainType, out var nextHandler))
        {
            nextHandler.GatherResources(lstRes, lstOther);
        }
    }

    /// <summary>
    /// 预加载额外资源（异步，与场景加载并行）
    /// </summary>
    private async ETTask LoadExtraResources(Scene scene, Evt_SwitchScene evt)
    {
        var systemDomainComp = scene.GetComponent<SystemDomainComponent>();
        if (systemDomainComp.TryGetDomainHandler(
            _systemSwitchContext.NextDomainType, out var nextHandler))
        {
            await nextHandler.PreloadExtraResources();
        }
    }
}
```

### 3.2 Loading 界面选择逻辑

```csharp
/// <summary>
/// 决定使用哪个 Loading 界面（优先级：系统级 > 场景级 > 通用兜底）
/// </summary>
protected override LoadingOptions GetLoadingOptions(Scene scene, Evt_SwitchScene evt)
{
    // 1. 外部强制指定 Loading（最高优先级）
    var loadingPanelName = evt.LoadingPanelName;
    if (!string.IsNullOrEmpty(loadingPanelName))
        return new LoadingOptions
        {
            UIPath          = loadingPanelName,
            Level           = (int)ELoadingLevel.Scene,
            GameSceneType   = evt.SceneType,
        };

    // 2. 当前场景指定的 Loading
    var sceneComp = scene.GetComponent<SceneComponent>();
    loadingPanelName = sceneComp.GetLoadingPanelName();

    // 3. 下一个 SystemDomain 是否能覆写场景级 Loading
    var systemDomainComp = scene.GetComponent<SystemDomainComponent>();
    if (systemDomainComp.TryGetDomainHandler(
        _systemSwitchContext.NextDomainType, out var nextHandler))
    {
        var loadingConfig = nextHandler.GetLoadingName(_systemSwitchContext);
        if (loadingConfig.OverrideSceneLevel)
            loadingPanelName = loadingConfig.UIPath;  // 系统级覆写
    }

    // 4. 最终兜底：使用通用 Loading 面板
    if (string.IsNullOrEmpty(loadingPanelName))
        loadingPanelName = PanelNameDefine.CommonLoadingPanel;

    return new LoadingOptions
    {
        UIPath          = loadingPanelName,
        Level           = (int)ELoadingLevel.Scene,
        GameSceneType   = evt.SceneType,
    };
}
```

[Event(SceneType.Client)]
public class SwitchSystemDomainEvent : EventWithRes<Evt_SwitchSystemDomain>
{
    public override async ETTask Handle(Scene scene, Evt_SwitchSystemDomain a)
    {
        try
        {
            Init(scene, ref a);

            var (success, context) = await Prepare(scene, a);
            if (!success)
                return; // 系统域切换失败直接中断（不回退场景）

            _systemSwitchContext = context;

            await ExitPrevSystemDomainPreprocess(context);
            var loadingOptions = await ShowLoading(scene, a);
            context.LoadingPanelName = loadingOptions.UIPath;
            await LoadDependentResources(scene, a);
            await LoadDependentResources2(scene, a);
            await LoadDependentResources3(scene, a);
            await LoadDependentResources4(scene, a);
            await LoadExtraResources(scene, a);
            await ExitPrevSystemDomainPostprocess(context);
            await SwitchSystemDomain(context);
            await EnterNextSystemDomainPreprocess(context);
            await HideLoading(scene, a, loadingOptions);
            await EnterNextSystemDomainPostprocess(context);
            await Run(scene, a);
        }
        catch (Exception e)
        {
            Log.Error(e);
        }
    }

    /// <summary>
    /// 校验：同一域不能重复进入
    /// </summary>
    private async ETTask<(bool, SystemDomainSwitchContext)> Prepare(
        Scene scene, Evt_SwitchSystemDomain evt)
    {
        var systemDomainComp = scene.SystemDomain();
        var prevDomain = systemDomainComp.GetBaseDomain();
        var nextDomain = evt.DomainType;

        if (prevDomain == nextDomain)
        {
            Log.LogError("[System][Domain] 系统作用域 {0} 无法进入，已进入", nextDomain);
            return (false, default);
        }

        // ...构建切换上下文
        return (true, context);
    }
}

// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\SystemDomain\SystemDomainHandler.cs

public abstract class SystemDomainHandler : ISystemDomainHandler
{
    public ESystemDomainType Domain { get; set; }
    public SystemDomainComponent Owner { get; set; }

    /// <summary>
    /// 声明该系统域所需的预加载资源列表
    /// </summary>
    public virtual void GatherResources(List<string> lstRes, List<string> lstOther) { }

    /// <summary>
    /// 异步预加载额外资源（如大型 AssetBundle）
    /// </summary>
    public virtual async ETTask PreloadExtraResources()
    {
        await ETTask.CompletedTask;
    }

    /// <summary>
    /// 获取进入此系统域时的 Loading 配置
    /// </summary>
    public virtual LoadingConfig GetLoadingName(SystemDomainSwitchContext context)
    {
        return LoadingUtil.GetSystemLoadingName(
            context.PrevDomainType, context.NextDomainType);
    }

    /// <summary>
    /// 退出后置处理（默认关闭所有 UI，仅保留 Loading 面板）
    /// </summary>
    public virtual async ETTask ExitPostprocess(SystemDomainSwitchContext context)
    {
        await ETTask.CompletedTask;
        EventSystem.Instance.Publish(Owner.ClientScene(),
            new Evt_CloseAllUIPanelExceptOne(context.LoadingPanelName));
    }
}

// 公共域（默认兜底）

public class CommonDomainHandler : SystemDomainHandler { }

// 未知域（异常兜底）
[SystemDomainBinding(ESystemDomainType.Unknown)]
public class UnknownDomainHandler : SystemDomainHandler { }

## 4. 设计亮点

### 4.1 Pipeline 模式保障切换原子性
场景切换的每一步都是独立的 `await`，步骤之间互不干扰，任何一步抛出异常都会被外层 `try-catch` 捕获并记录，不会导致游戏卡死。切换失败时有明确的兜底策略（回到启动场景）。

### 4.2 分层 Loading 系统
通过场景级和系统级两层 Loading 的优先级设计，做到了：
- 普通场景切换：自动使用通用 Loading 界面
- 特殊玩法进入：自定义专属 Loading 动画（如养成剧本的"开场动画"）
- 调用方可强制指定 Loading（用于特殊 AB 测试或运营活动）

### 4.3 多轮资源预加载
将资源预加载拆分为 4 轮 + Extra，便于：
- 不同类型资源按优先级加载（UI 先于模型先于特效）
- 各轮次内部可以并发加载（Addressables 批量请求）
- Loading 进度条可以按轮次划分百分比，展示更流畅

### 4.4 域切换防重入机制
系统域切换会检查 `prevDomain == nextDomain`，直接阻断无意义的重复切换，避免了多次点击按钮导致的状态混乱。

## 5. 常见问题与最佳实践

### Q1: 如何添加一个新游戏场景（如新玩法场景）？
1. 在 Unity Editor 中创建新的 Scene 文件，加入 Build Settings；
2. 在 `GameSceneType` 枚举中添加对应值；
3. 如果该场景有对应的系统域，创建继承 `SystemDomainHandler` 的处理类；
4. 在新场景对应的 `SceneComponent` 中设置 `LoadingPanelName`；
5. 在需要的地方发布 `Evt_SwitchScene { SceneType = YourScene }` 触发切换。

### Q2: 场景切换时为什么要分 Preprocess 和 Postprocess？
- `Preprocess`（前置）：在 Loading 显示**之前**执行，可用于保存当前状态、发送服务端通知等（玩家还能看到当前界面）；
- `Postprocess`（后置）：在 Loading 显示**之后**执行，可以安全地销毁资源、清空 UI（玩家看不到这些操作，Loading 遮挡了）。

### Q3: 为什么场景切换失败时要回退到 Boot 场景？
Boot 场景是游戏最基础的安全状态，其 Component 最少，出错风险最低。回退到 Boot 场景后，可以重新走完整的登录流程，恢复到已知的正确状态。这是游戏客户端常见的"安全回退"策略。

### Q4: 如何在不切场景的情况下刷新当前场景的所有资源？
发布 `Evt_SwitchSystemDomain` 时将 `NextDomainType` 设为当前域可能不行（被重入检查拦截）。正确做法是先切换到一个中间域（如 Common），再切换回目标域，整个过程会触发完整的退出→进入生命周期，从而刷新所有资源。

### Q5: 场景加载进度如何显示在 Loading 界面上？
在 `LoadAndEnterScene` 阶段，通过 Addressables 的异步加载句柄（`AsyncOperationHandle`）获取加载进度（0~1），将进度值通过事件推送给 Loading UI：
```csharp
// 每帧更新进度（使用 UniTask 或协程）
while (!sceneHandle.IsDone)
{
    float progress = sceneHandle.PercentComplete;
    EventSystem.Instance.Publish(scene, new Evt_LoadingProgress { Progress = progress });
    await ETTask.Yield();
}
```
