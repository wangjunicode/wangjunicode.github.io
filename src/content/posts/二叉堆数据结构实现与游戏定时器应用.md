# 二叉堆数据结构实现与游戏定时器应用

## 为什么游戏需要二叉堆

定时器是游戏中使用频率极高的系统——技能 CD、状态持续时间、延迟触发事件，本质上都是"在 T 毫秒后执行某个回调"。最朴素的实现是每帧遍历所有定时器，时间复杂度 O(n)，当定时器数量膨胀时性能急剧下降。

二叉堆提供了另一条路：**堆顶永远是最近要触发的定时器**，每帧只需比较堆顶是否到期，Pop 操作 O(log n)。这是定时器系统的经典基础设施。

---

## 接口设计：IBinaryHeap\<T\>

框架将堆的行为抽象为接口，核心方法如下：

```csharp
public interface IBinaryHeap<T> : IEnumerable<T>
{
    int    Count   { get; }
    bool   IsEmpty { get; }
    IComparer<T> Comparer { get; }

    T    Peek();              // 查看堆顶，不移除
    bool TryPeek(out T item); // 安全版本

    T    Pop();               // 移除并返回堆顶
    bool TryPop(out T item);  // 安全版本

    void Push(T item);        // 插入元素
    void PushRange(IEnumerable<T> collection); // 批量插入

    T PushPop(T item);  // 先插入再弹出（可优化为一次堆化）
    T PopPush(T item);  // 先弹出再插入（可优化为一次堆化）

    void Rebuild();     // 重建堆（Remove 后调用）
    bool Remove(T item);
    int  RemoveAll(Func<T, bool> predicate);

    bool TryGet(Func<T, bool> predicate, out T element);
    bool TryGetAll(Func<T, bool> predicate, out IList<T> elements);
}
```

`PushPop` 和 `PopPush` 是两个容易混淆的操作：

| 操作 | 顺序 | 适用场景 |
|------|------|----------|
| `PushPop` | 先插入，再弹出堆顶 | 插入新元素后立即取最小/最大值 |
| `PopPush` | 先弹出，再插入新元素 | 替换堆顶，维持堆大小不变 |

两者都可以在一次 `SiftDown` 内完成，避免两次 O(log n) 操作。

## 最小堆实现：MinBinaryHeap\<T\>

### 数据结构

```csharp
public class MinBinaryHeap<T> : IBinaryHeap<T>
{
    private readonly List<T> _items;
    private readonly IComparer<T> _comparer;
    
    // 下标关系
    // parent(i) = (i - 1) / 2
    // leftChild(i)  = i * 2 + 1
    // rightChild(i) = i * 2 + 2
}
```

用 `List<T>` 作为底层存储，通过下标公式模拟树结构，避免了节点对象的额外内存分配。

### 两种堆化方向

**SiftUp（自底向上）** —— Push 时使用：

```csharp
private void SiftUp(int start)
{
    var child  = start;
    var parent = (child - 1) / 2;
    while (child > 0)
    {
        // 最小堆：子节点 >= 父节点时停止
        if (_comparer.Compare(_items[child], _items[parent]) >= 0)
            break;
        Swap(_items, parent, child);
        child  = parent;
        parent = (child - 1) / 2;
    }
}
```

每次 Push 只需最多 O(log n) 次交换。

**SiftDown（自顶向下）** —— Pop / Heapify 时使用：

```csharp
private void SiftDown(int start)
{
    var parent = start;
    var lChild = parent * 2 + 1;
    var rChild = parent * 2 + 2;
    while (lChild < _items.Count)
    {
        var temp = parent;
        if (lChild < _items.Count && _comparer.Compare(_items[lChild], _items[temp]) < 0)
            temp = lChild;
        if (rChild < _items.Count && _comparer.Compare(_items[rChild], _items[temp]) < 0)
            temp = rChild;
        if (temp == parent) break;
        Swap(_items, parent, temp);
        parent = temp;
        lChild = parent * 2 + 1;
        rChild = parent * 2 + 2;
    }
}
```

### Pop 的核心技巧

```csharp
private T RemoveRootNode()
{
    var head = 0;
    var tail = _items.Count - 1;
    Swap(_items, head, tail);      // 1. 堆顶与末尾交换
    var item = _items[tail];
    _items.RemoveAt(tail);         // 2. 删除末尾（原堆顶）
    SiftDown(head);                // 3. 新堆顶下沉修复堆序
    return item;
}
```

不直接删除堆顶，而是交换到末尾再删除，避免了数组移位的 O(n) 开销。

### 批量建堆的时间复杂度优化

```csharp
private void Heapify()
{
    if (_items.Count <= 1) return;
    
    // Floyd 算法：从最后一个父节点向上逐个 SiftDown
    var lastParentWithChildren = (_items.Count - 1) / 2;
    for (var i = lastParentWithChildren; i >= 0; --i)
        SiftDown(i);
}
```

逐个 SiftUp 的时间复杂度是 O(n log n)，而 Floyd 建堆算法使用 SiftDown 从底层开始，**整体复杂度降至 O(n)**。

这是因为底层节点数量多但下沉距离短，顶层节点数量少但下沉距离长，综合求和后为线性时间。

## 最大堆：MaxBinaryHeap\<T\>

最大堆与最小堆实现完全对称，只有比较方向不同：

```csharp
// SiftUp：最大堆中子节点 <= 父节点时停止
if (_comparer.Compare(_items[child], _items[parent]) <= 0)
    break;

// SiftDown：寻找最大的子节点
if (lChild < _items.Count && _comparer.Compare(_items[lChild], _items[temp]) > 0)
    temp = lChild;
if (rChild < _items.Count && _comparer.Compare(_items[rChild], _items[temp]) > 0)
    temp = rChild;
```

通过 `IComparer<T>` 注入比较逻辑，两种堆的代码结构完全一致，只是">" 和 "<" 互换，这是比较器模式的典型应用。

## 迭代器的正确实现

```csharp
public IEnumerator<T> GetEnumerator()
{
    // 遍历副本，避免修改源集合
    var clone = new MinBinaryHeap<T>(_items, _comparer);
    while (clone._items.Count > 0)
        yield return clone.RemoveRootNode();
}
```

迭代顺序是**有序弹出序列**（升序/降序），而不是内部数组的存储顺序。直接暴露 `_items` 枚举会得到无序结果，这里通过克隆副本按序弹出，保证语义正确性。

## 在游戏定时器中的应用

游戏框架的 `TimerComponent` 将最小堆作为定时任务的调度核心：

```csharp
// 概念结构（伪代码）
public class TimerComponent
{
    // 按触发时间排序的最小堆
    private MinBinaryHeap<TimerTask> _heap = new(TimerTask.Comparer);

    public void Update(long nowMs)
    {
        // 只要堆顶到期就弹出触发
        while (!_heap.IsEmpty)
        {
            var top = _heap.Peek();
            if (top.TriggerTime > nowMs) break;
            
            _heap.Pop();
            top.Callback?.Invoke();
        }
    }

    public long AddTimer(long delayMs, Action callback)
    {
        var task = new TimerTask
        {
            TriggerTime = TimeHelper.ClientNow() + delayMs,
            Callback    = callback,
        };
        _heap.Push(task);
        return task.Id;
    }
}
```

每帧 `Update` 中最多执行 O(k log n) 次 Pop（k 为本帧到期数量），相比遍历方案效率大幅提升。

## 各操作时间复杂度总结

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| `Push` | O(log n) | SiftUp |
| `Pop` | O(log n) | SiftDown |
| `Peek` | O(1) | 访问索引 0 |
| `PushPop` / `PopPush` | O(log n) | 一次 SiftDown |
| `Remove(item)` | O(n) | 需要线性搜索 + Heapify |
| `Heapify` (批量建堆) | O(n) | Floyd 算法 |
| `Contains` | O(n) | 线性扫描 |

`Remove(item)` 的 O(n) 是堆的固有局限——堆不支持随机访问定位。如果需要高效删除任意元素，可以在外部维护一个 `Dictionary<id, index>` 映射，结合位置标记实现 O(log n) 删除（懒删除方案）。

## 快速开项目清单

1. **定时器调度**：最小堆 + 时间戳比较器，堆顶即最近到期任务
2. **优先级队列**：直接用 `MinBinaryHeap<T>` 替代 `SortedSet`（当不需要随机删除时）
3. **Top-K 问题**：维护大小为 K 的最小堆，扫描时保留最大的 K 个元素
4. **批量初始化**：用 `new MinBinaryHeap<T>(collection)` 触发 Floyd 建堆，比逐个 Push 快
5. **自定义排序**：传入 `IComparer<T>` 实现任意比较逻辑，无需继承 IComparable
