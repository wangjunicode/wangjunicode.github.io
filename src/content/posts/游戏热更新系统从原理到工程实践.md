# 游戏热更新系统：从原理到工程实践

热更新能力是商业手游的标配需求。一套完善的热更系统可以让你在不发版（不经过应用商店审核）的情况下修复 Bug、更新内容甚至更新逻辑代码。

## 一、热更新的两个维度

```
热更新系统
├── 资源热更（Asset Hot Update）
│   ├── 更新内容：图片、音频、模型、预制体、配置表...
│   ├── 技术方案：AssetBundle / Addressables
│   └── iOS/Android 均允许
│
└── 代码热更（Code Hot Update）
    ├── 更新内容：游戏逻辑代码
    ├── 技术方案：Lua(XLua/ToLua) / HybridCLR(C#热更)
    └── iOS 限制脚本执行（需热更方案绕过）
```

## 二、资源热更系统

### 2.1 AssetBundle 基础

```csharp
// 打包资源到 AssetBundle
// 通过 BuildPipeline（通常在 Editor 脚本中执行）

// 运行时加载
public class AssetBundleLoader
{
    private Dictionary<string, AssetBundle> _loadedBundles = new();
    private const string CDN_URL = "https://cdn.yourgame.com/ab/";
    
    // 异步加载 Bundle
    public async Task<T> LoadAssetAsync<T>(string bundleName, string assetName) 
        where T : UnityEngine.Object
    {
        // 1. 检查是否已加载
        if (!_loadedBundles.TryGetValue(bundleName, out var bundle))
        {
            bundle = await LoadBundleAsync(bundleName);
            _loadedBundles[bundleName] = bundle;
        }
        
        // 2. 从 Bundle 中加载资产
        var request = bundle.LoadAssetAsync<T>(assetName);
        await request;
        return request.asset as T;
    }
    
    private async Task<AssetBundle> LoadBundleAsync(string bundleName)
    {
        string localPath = Path.Combine(Application.persistentDataPath, bundleName);
        
        // 优先加载本地缓存
        if (File.Exists(localPath))
        {
            return await LoadFromLocal(localPath);
        }
        
        // 从 CDN 下载
        return await DownloadAndCacheBundle(CDN_URL + bundleName, localPath);
    }
    
    private async Task<AssetBundle> DownloadAndCacheBundle(string url, string cachePath)
    {
        using var uwr = UnityWebRequestAssetBundle.GetAssetBundle(url);
        await uwr.SendWebRequest();
        
        if (uwr.result != UnityWebRequest.Result.Success)
            throw new Exception($"下载失败: {uwr.error}");
        
        var bundle = DownloadHandlerAssetBundle.GetContent(uwr);
        
        // 写入本地缓存
        File.WriteAllBytes(cachePath, uwr.downloadHandler.data);
        
        return bundle;
    }
}
```

### 2.2 版本对比与差量更新

```csharp
[Serializable]
public class AssetBundleManifest
{
    public string Version;
    public List<BundleInfo> Bundles;
}

[Serializable]
public class BundleInfo
{
    public string Name;
    public string MD5;
    public long Size;
    public string URL;
}

public class HotUpdateChecker
{
    // 检查需要更新的资源列表
    public async Task<List<BundleInfo>> CheckForUpdates()
    {
        // 1. 下载服务器最新 Manifest
        var serverManifest = await FetchServerManifest();
        
        // 2. 读取本地 Manifest
        var localManifest = LoadLocalManifest();
        
        var toUpdate = new List<BundleInfo>();
        
        foreach (var serverBundle in serverManifest.Bundles)
        {
            var localBundle = localManifest?.Bundles
                .FirstOrDefault(b => b.Name == serverBundle.Name);
            
            // MD5 不一致 → 需要更新
            if (localBundle == null || localBundle.MD5 != serverBundle.MD5)
            {
                toUpdate.Add(serverBundle);
            }
        }
        
        return toUpdate;
    }
    
    // 执行下载更新（带进度回调）
    public async Task DownloadUpdates(
        List<BundleInfo> bundles, 
        Action<float, string> onProgress)
    {
        long totalSize = bundles.Sum(b => b.Size);
        long downloaded = 0;
        
        foreach (var bundle in bundles)
        {
            await DownloadBundle(bundle);
            downloaded += bundle.Size;
            onProgress?.Invoke((float)downloaded / totalSize, bundle.Name);
        }
        
        // 保存新 Manifest
        SaveLocalManifest(await FetchServerManifest());
    }
}
```

### 2.3 Addressables 热更（推荐）

```csharp
// Unity Addressables 简化了热更流程
public class AddressablesUpdater
{
    public async Task<bool> CheckAndUpdate(Action<float> onProgress)
    {
        // 1. 检查是否有更新
        var handle = Addressables.CheckForCatalogUpdates(false);
        await handle.Task;
        
        var catalogsToUpdate = handle.Result;
        Addressables.Release(handle);
        
        if (catalogsToUpdate.Count == 0)
        {
            Debug.Log("无需更新");
            return false;
        }
        
        // 2. 更新目录
        var updateHandle = Addressables.UpdateCatalogs(catalogsToUpdate, false);
        await updateHandle.Task;
        Addressables.Release(updateHandle);
        
        // 3. 获取下载大小
        var sizeHandle = Addressables.GetDownloadSizeAsync("default");
        await sizeHandle.Task;
        long downloadSize = sizeHandle.Result;
        Addressables.Release(sizeHandle);
        
        if (downloadSize > 0)
        {
            // 4. 执行下载
            var downloadHandle = Addressables.DownloadDependenciesAsync("default", false);
            while (!downloadHandle.IsDone)
            {
                onProgress?.Invoke(downloadHandle.PercentComplete);
                await Task.Yield();
            }
            Addressables.Release(downloadHandle);
        }
        
        return true;
    }
}
```

## 三、代码热更：HybridCLR（推荐C#热更方案）

HybridCLR 是腾讯 ILRuntime 的后继者，让 Unity 支持 **完整 C# 热更新**，无需学习 Lua。

### 3.1 工作原理

```
传统 Unity（AOT编译）：
  C# 代码 → IL2CPP → 本机代码（打入安装包，无法修改）

HybridCLR：
  热更程序集 → IL字节码 → 随 AssetBundle 下载
                             ↓
                  运行时解释执行（JIT+AOT混合）
```

### 3.2 项目结构设计

```
Assets/
├── Scripts/
│   ├── Core/           ← AOT程序集（随包发布，稳定代码）
│   │   ├── Framework/
│   │   ├── Network/
│   │   └── Bootstrap.cs  ← 启动入口，负责加载热更程序集
│   │
│   └── HotUpdate/      ← 热更程序集（独立DLL，可热更）
│       ├── Game/
│       ├── UI/
│       ├── Battle/
│       └── GameEntry.cs  ← 热更代码入口
```

### 3.3 启动流程实现

```csharp
// Bootstrap.cs（AOT，随包发布）
public class Bootstrap : MonoBehaviour
{
    async void Start()
    {
        // 1. 初始化 HybridCLR
        LoadMetadataForAOTAssemblies();
        
        // 2. 检查更新
        var updater = new HotUpdateManager();
        bool hasUpdate = await updater.CheckAndDownloadHotfix();
        
        // 3. 加载热更程序集
        Assembly hotUpdateAssembly = LoadHotUpdateAssembly();
        
        // 4. 反射调用热更入口
        Type gameEntryType = hotUpdateAssembly.GetType("GameEntry");
        MethodInfo startMethod = gameEntryType.GetMethod("Start");
        startMethod.Invoke(null, null);
    }
    
    // 加载 AOT 元数据（泛型支持必需）
    private void LoadMetadataForAOTAssemblies()
    {
        // 将 AOT 程序集的 metadata 加载到 HybridCLR 运行时
        // 这些文件在打包时由 HybridCLR 生成
        string[] aotAssemblies = { "mscorlib.dll", "System.dll", "UnityEngine.dll" };
        
        foreach (var aot in aotAssemblies)
        {
            byte[] data = LoadAssemblyBytes($"HotUpdateDlls/{aot}");
            var err = HybridCLR.RuntimeApi.LoadMetadataForAOTAssembly(data, 
                HybridCLR.HomologousImageMode.SuperSet);
            Debug.Log($"加载AOT元数据 {aot}: {err}");
        }
    }
    
    private Assembly LoadHotUpdateAssembly()
    {
        // 从 AssetBundle 或 persistentDataPath 加载热更 DLL
        byte[] dllBytes = LoadAssemblyBytes("HotUpdateDlls/HotUpdate.dll");
        byte[] pdbBytes = LoadAssemblyBytes("HotUpdateDlls/HotUpdate.pdb"); // 调试符号
        
        return Assembly.Load(dllBytes, pdbBytes);
    }
    
    private byte[] LoadAssemblyBytes(string path)
    {
        string fullPath = Path.Combine(Application.persistentDataPath, path);
        if (File.Exists(fullPath))
            return File.ReadAllBytes(fullPath);
        
        // Fallback: 从 StreamingAssets 加载（首包内置）
        return LoadFromStreamingAssets(path);
    }
}
```

## 五、热更新工程化体系

### 5.1 完整发布流程

```
开发阶段
  ↓ 提交代码到 Git
CI/CD 流水线
  ├── 编译打包
  ├── 生成热更资源（AssetBundle + DLL）
  ├── 计算 MD5，生成 Manifest
  ├── 上传到 CDN
  └── 更新版本配置表

客户端启动
  ├── 检查版本配置表
  ├── 对比本地/服务器 Manifest
  ├── 下载差量资源
  └── 加载热更代码，进入游戏
```

### 5.2 版本管理策略

```
版本号规范：A.B.C.D
A = 大版本（重大改版，必须重新安装）
B = 中版本（功能版本，可能需要重新安装）
C = 小版本（内容更新，热更即可）
D = 热修版本（Bug修复，热更即可）

示例：
1.5.0.0 → 正式发布版本
1.5.1.0 → 活动内容热更
1.5.1.1 → 紧急Bug修复热更
1.6.0.0 → 新功能（可能需要安装包更新）
```

### 5.3 灰度发布

```csharp
// 灰度发布：先推给 1% 用户，验证无问题后全量
public class GrayReleaseManager
{
    public string GetHotfixVersion(string userId)
    {
        // 基于用户ID哈希决定灰度分组
        int hash = userId.GetHashCode() & 0x7FFFFFFF;
        int bucket = hash % 100; // 0-99
        
        // 读取灰度策略配置
        var policy = FetchGrayPolicy();
        
        foreach (var rule in policy.Rules)
        {
            // 白名单（内部测试）
            if (rule.Whitelist?.Contains(userId) == true)
                return rule.Version;
            
            // 百分比灰度
            if (bucket < rule.Percentage)
                return rule.Version;
        }
        
        return policy.DefaultVersion;
    }
}
```

## 六、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| iOS 审核被拒 | 热更包含可执行代码 | 使用 Lua/HybridCLR（解释执行）；资源热更不受限 |
| 热更后闪退 | 新旧代码接口不兼容 | 版本号管理 + 接口向后兼容规范 |
| 下载中断 | 网络不稳定 | 断点续传 + MD5 校验 |
| 包体越来越大 | AB 无法删除旧资源 | 定期清理本地缓存（保留N个版本） |
| 泛型方法报错 | HybridCLR AOT 元数据缺失 | 补充 AOT 程序集元数据列表 |

> 💡 **技术负责人建议**：热更新是双刃剑。滥用热更会导致版本混乱、问题难以追踪。建议建立明确的热更纪律：Bug修复走热更，新功能不建议热更，重要逻辑强制走安装包发版接受审计。
