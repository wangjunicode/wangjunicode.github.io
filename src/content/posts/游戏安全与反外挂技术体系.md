# 游戏安全与反外挂技术体系

游戏安全是多人游戏的生死线。一个充斥外挂的游戏会迅速流失正常玩家，毁掉游戏生态。本文系统讲解从客户端防护到服务端验证的完整安全体系。

## 一、外挂类型与威胁模型

```
外挂类型分类：

内存修改类（工具：CE / 内存搜索器）
  └── 防护：数值加密存储

速度类（工具：Speed Hack / 变速齿轮）
  └── 防护：服务端时间验证

透视/自瞄类（工具：内存读写 + DX Hook）
  └── 防护：服务端 FOG OF WAR

数据包篡改类（工具：WireShark / 封包编辑器）
  └── 防护：数据包签名 + HTTPS + 服务端验证

脚本/自动化类（工具：Auto Hotkey / Macro）
  └── 防护：行为分析 + 验证码 + 封号

注入类（工具：DLL Injector）
  └── 防护：进程完整性检测
```

## 二、客户端数值保护

### 2.1 数值加密存储

```csharp
// 保护内存中的数值，防止 CE 等工具扫描修改
public struct EncryptedInt
{
    private int _encryptedValue;
    private int _key;
    private int _checksum;

    public EncryptedInt(int value)
    {
        _key = UnityEngine.Random.Range(int.MinValue, int.MaxValue);
        _encryptedValue = value ^ _key;
        _checksum = value + _key;
    }

    public int Value
    {
        get
        {
            int decrypted = _encryptedValue ^ _key;

            if (decrypted + _key != _checksum)
            {
                Debug.LogWarning("检测到内存被篡改！");
                AntiCheatSystem.ReportMemoryTampering();
                return 0;
            }

            return decrypted;
        }
        set
        {
            _key = UnityEngine.Random.Range(int.MinValue, int.MaxValue);
            _encryptedValue = value ^ _key;
            _checksum = value + _key;
        }
    }

    public static implicit operator int(EncryptedInt e) => e.Value;
    public static implicit operator EncryptedInt(int v) => new EncryptedInt(v);
}

// 使用：与普通 int 完全相同的体验
public class PlayerStats
{
    private EncryptedInt _hp = 100;
    private EncryptedInt _gold = 0;

    public void TakeDamage(int damage) => _hp -= damage;
}
```

### 2.2 速度外挂检测

```csharp
public class SpeedHackDetector : MonoBehaviour
{
    private float _lastRealTime;
    private float _lastGameTime;
    private const float TOLERANCE = 0.15f;
    private int _suspicionCount = 0;

    private void Start()
    {
        _lastRealTime = Time.realtimeSinceStartup;
        _lastGameTime = Time.time;
        InvokeRepeating(nameof(DetectSpeedHack), 30f, 30f);
    }

    private void DetectSpeedHack()
    {
        float realElapsed = Time.realtimeSinceStartup - _lastRealTime;
        float gameElapsed = Time.time - _lastGameTime;
        float ratio = gameElapsed / realElapsed;

        if (ratio > 1f + TOLERANCE || ratio < 1f - TOLERANCE)
        {
            _suspicionCount++;
            Debug.LogWarning($"速度异常: 时间比率 = {ratio:F2} (第{_suspicionCount}次)");

            if (_suspicionCount >= 5)
                ReportAndPunish();
        }
        else
        {
            _suspicionCount = Mathf.Max(0, _suspicionCount - 1);
        }

        _lastRealTime = Time.realtimeSinceStartup;
        _lastGameTime = Time.time;
    }

    private void ReportAndPunish()
    {
        AntiCheatSystem.Report(new CheatReport
        {
            Type = CheatType.SpeedHack,
            Evidence = $"时间比率持续异常，共检测{_suspicionCount}次"
        });
    }
}
```

## 三、网络数据安全

### 3.1 数据包签名

```csharp
public class NetworkSecurity
{
    private static readonly string APP_SECRET = GetSecretFromSecureSource();

    public static string GenerateSignature(Dictionary<string, string> parameters, long timestamp)
    {
        var sortedParams = parameters.OrderBy(p => p.Key)
                                      .Select(p => $"{p.Key}={p.Value}")
                                      .ToList();
        sortedParams.Add($"timestamp={timestamp}");
        string rawString = string.Join("&", sortedParams) + APP_SECRET;

        using var sha256 = SHA256.Create();
        byte[] hash = sha256.ComputeHash(Encoding.UTF8.GetBytes(rawString));
        return Convert.ToHexString(hash).ToLower();
    }

    public static async Task<string> SendSecureRequest(string url, Dictionary<string, string> data)
    {
        long timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        string nonce = Guid.NewGuid().ToString("N");
        data["timestamp"] = timestamp.ToString();
        data["nonce"] = nonce;
        data["sign"] = GenerateSignature(data, timestamp);

        var form = new WWWForm();
        foreach (var (key, value) in data)
            form.AddField(key, value);

        using var request = UnityWebRequest.Post(url, form);
        await request.SendWebRequest();
        return request.downloadHandler.text;
    }
}
```

## 四、服务端权威验证

```
反外挂铁律：
  一切重要数值，服务端是唯一权威。
  客户端只负责"展示"，不负责"裁决"。

不应信任客户端的数据：
  ❌ 玩家当前 HP（客户端可以修改）
  ❌ 玩家位置（速度/穿墙外挂）
  ❌ 伤害数值（伤害外挂）
  ❌ 金币数量（绕过消耗检查）

可以（适度）信任的数据：
  ✅ 玩家操作指令（移动方向、技能使用）
  ✅ 客户端时间戳（用于校验，不作为权威）
```

## 五、行为分析反机器人

```csharp
public class BotDetectionSystem
{
    private List<float> _actionIntervals = new();
    private float _lastActionTime = 0;
    private float _suspicionScore = 0;

    public void RecordAction()
    {
        float currentTime = Time.time;

        if (_lastActionTime > 0)
        {
            float interval = currentTime - _lastActionTime;
            _actionIntervals.Add(interval);

            if (_actionIntervals.Count > 50)
                _actionIntervals.RemoveAt(0);

            if (_actionIntervals.Count >= 20)
            {
                float stdDev = CalculateStdDev(_actionIntervals);

                if (stdDev < 0.01f) // 间隔波动小于10ms，极度可疑（机器人）
                {
                    _suspicionScore += 10;
                    Debug.LogWarning($"操作间隔异常规律: stdDev={stdDev:F4}s");
                }
            }
        }

        _lastActionTime = currentTime;

        if (_suspicionScore > 50)
        {
            AntiCheatSystem.Report(new CheatReport
            {
                Type = CheatType.BotScript,
                Evidence = $"行为分析得分: {_suspicionScore}"
            });
        }
    }

    private float CalculateStdDev(List<float> data)
    {
        float mean = data.Average();
        float variance = data.Average(x => Mathf.Pow(x - mean, 2));
        return Mathf.Sqrt(variance);
    }
}
```

## 六、安全体系建设原则

| 原则 | 说明 |
|------|------|
| 深度防御 | 多层防护，不依赖单一手段 |
| 服务端权威 | 关键逻辑服务端验证，不信任客户端 |
| 惩罚梯度 | 警告 → 封禁功能 → 封号，降低误伤率 |
| 数据留证 | 保留证据再封号，防止误封投诉 |
| 持续对抗 | 外挂方会持续更新，安全需要持续投入 |
| 成本博弈 | 提高作弊成本 > 试图完全杜绝作弊 |

> ⚠️ **重要原则**：安全是一场持续的攻防战，不存在"一劳永逸"的解决方案。服务端权威验证是最根本的防线，客户端防护只是增加攻击难度。发现-封禁-迭代的运营闭环，比任何单一技术手段都更重要。
