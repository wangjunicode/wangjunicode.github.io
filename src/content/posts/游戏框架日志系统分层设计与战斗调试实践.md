# 游戏框架日志系统分层设计与战斗调试实践

## 日志系统的核心矛盾

游戏日志面临两个相互对立的需求：

- **开发期**：需要详尽的调试信息，包括战斗状态快照、帧计数、调用堆栈
- **运行期**：每帧调用数百次日志，任何分配和字符串拼接都会触发 GC

框架通过三层架构解决这个矛盾：**ILog 接口层 → Log 静态调用层 → LogManager 工厂初始化层**，底层引入 ZLogger 实现零分配日志写入。

---

## 第一层：ILog 接口

```csharp
namespace ET
{
    public interface ILog
    {
        void Trace(string message);
        void Warning(string message);
        void Info(string message);
        void Debug(string message);
        void Error(string message);
    }
}
```

极简接口，只定义五个级别的无参数日志方法。`UnityLogger` 是其在客户端的实现：

```csharp
public class UnityLogger : ILog
{
    public void Trace(string msg)   => UnityEngine.Debug.Log(msg);
    public void Debug(string msg)   => UnityEngine.Debug.Log(msg);
    public void Info(string msg)    => UnityEngine.Debug.Log(msg);
    public void Warning(string msg) => UnityEngine.Debug.LogWarning(msg);
    public void Error(string msg)   => UnityEngine.Debug.LogError(msg);
    // Error(Exception e) 额外重载 → Debug.LogException
}
```

这一层的职责是**平台适配**，将统一接口映射到具体平台的日志输出。服务端可替换为文件日志，测试时可替换为 Mock 实现。

## 第二层：LogManager 工厂

`LogManager` 是日志系统的初始化入口，基于 `Microsoft.Extensions.Logging` 生态，核心是 ZLogger：

```csharp
public static class LogManager
{
    static ILoggerFactory loggerFactory;
    static ILogger globalLogger;
    static string[] levelShort = { "T", "D", "I", "W", "E", "C" };

    static LogManager()
    {
#if UNITY_EDITOR
        CreateLogger(LogLevel.Information); // 编辑器下显示所有 Info 以上
#else
        CreateLogger(LogLevel.Warning);     // 运行时只显示 Warning 以上
#endif
    }

    private static void CreateLogger(LogLevel level)
    {
#if ONLY_CLIENT
        loggerFactory = UnityLoggerFactory.Create(builder =>
        {
            builder
                .SetMinimumLevel(level)
                .AddZLoggerUnityDebug()          // 输出到 Unity Console
                .AddZLoggerRollingFile(          // 同时写入滚动文件
                    (dt, x) => $"{LogPathRoot}/game_{dt:yyyy-MM-dd_HH-mm-ss}_{x:000}.log",
                    x => x.ToLocalTime().Date,   // 按天滚动
                    4096,                        // 最大文件数
                    x =>
                    {
                        x.PrefixFormatter = (writer, info) =>
                            ZString.Utf8Format(writer, "[{0}][{1}]",
                                levelShort[(int)info.LogLevel],
                                info.Timestamp.ToLocalTime().DateTime);
                    });
        });
#endif
        globalLogger = loggerFactory.CreateLogger("Global");
    }
}
```

### 跨平台路径处理

```csharp
private static string LogPathRoot => cachedLogPathRoot ??= 
#if UNITY_EDITOR
    Application.dataPath + "/../Logs"   // 项目根目录下的 Logs/
#elif ONLY_CLIENT
    Application.persistentDataPath       // 移动端持久化路径
#else
    "./Logs"                             // 服务端当前目录
#endif
;
```

一处定义，自动适配编辑器 / 移动端 / 服务端三种路径策略。

### 运行时动态调整日志级别

```csharp
public static void ChangeLogLevel(LogLevel level)
{
    var loggersArray = (Array)globalLogger
        .GetType()
        .GetProperty("MessageLoggers", BindingFlags.Public | BindingFlags.Instance)
        ?.GetValue(globalLogger);

    if (loggersArray != null)
    {
        for (int i = 0; i < loggersArray.Length; i++)
        {
            var info = loggersArray.GetValue(i);
            var piMinLevel = info?.GetType().GetProperty("MinLevel");
            var fiMinLevel = piMinLevel.GetBackingField(); // 反射获取 backing field
            fiMinLevel.SetValue(info, level);
            loggersArray.SetValue(info, i);
        }
    }
}
```

这段代码通过反射绕过 `Microsoft.Extensions.Logging` 不支持运行时修改级别的限制。`GetBackingField` 是自定义扩展方法，通过命名规则 `<PropertyName>k__BackingField` 定位自动属性的底层字段。

> ⚠️ 生产环境慎用此方案，依赖编译器命名约定，可能因版本变化失效。

## 第三层：Log 静态调用层

`Log` 是业务代码的统一入口，通过 ZLogger 实现**零分配的格式化日志**：

```csharp
public static partial class Log
{
    static ILogger logger = LogManager.Logger;

    // Warning/Error 级别自动附加堆栈
    private static void ZLog(LogLevel logLevel, EventId eventId, Exception exception, string message)
    {
        logger.Log(logLevel, eventId, new MessageLogState<object>(null,
                (int)logLevel >= (int)LogLevel.Warning
                    ? ZString.Concat(message, '\n', new StackTrace(2, true).ToString())
                    : message),
            exception,
            (state, ex) => state.Message);
    }

    // 泛型格式化版本，避免 params object[] 装箱
    private static void ZLog<T1>(LogLevel logLevel, EventId eventId, Exception exception,
        string format, T1 arg1)
    {
        logger.Log(logLevel, eventId,
            new FormatLogState<object, T1>(null, format, arg1),
            exception,
            (state, ex) => ZString.Format(state.Format, state.Arg1));
    }
}
```

### ZLogger 的零分配原理

传统日志：
```csharp
logger.LogWarning($"Player {playerId} took {damage} damage"); // 字符串插值 → 堆分配
```

ZLogger 方案：
```csharp
Log.LogWarning("Player {0} took {1} damage", playerId, damage);
// 内部用 Utf8ValueStringBuilder 直接写入，不产生中间字符串
```

`FormatLogState<TPayload, T1>` 是一个值类型（struct），以泛型参数携带格式化参数，延迟到实际写入时才格式化，整个路径**零堆分配**。

## 战斗调试系统：BattleLog

`LogEx.cs` 展示了如何在通用日志之上叠加战斗专用的调试能力。

### 标签体系

```csharp
public enum BattleLogTag
{
    Default, Combo, Perform, Input, ComboDetail,
    SelectPerform, RegisterPerform, Turn, ChangeDef,
    Skill, Node, Damage, Debug, Error, // ...
}

[Flags]
public enum LogFilterType
{
    Default    = 0,
    Turn       = 1 << 0,
    Input      = 1 << 1,
    Token      = 1 << 3,
    BattlePoint= 1 << 4,
    Perform    = 1 << 5,
    Combo      = 1 << 6,
    Skill      = 1 << 7,
    // ...
}
```

`BattleLogTag` 是细粒度标签（枚举值），`LogFilterType` 是位掩码过滤器。一个过滤器可以匹配多个标签：

```csharp
public static readonly Dictionary<LogFilterType, List<string>> LogFilter = new()
{
    { LogFilterType.Perform, new List<string>
        {
            LogPrefix + TryUsePerform,
            LogPrefix + RegisterPerform,
            LogPrefix + UnRegisterPerform,
            LogPrefix + Perform,
            LogPrefix + PerformDetail,
        }
    },
    // ...
};

// 过滤使用
public static bool FilterResult(this LogFilterType type, string str)
{
    return LogFilter
        .Where(x => ((int)x.Key & (int)type) == (int)x.Key)
        .Any(filter => filter.Value.Any(str.StartsWith));
}
```

实际使用：只需启用 `LogFilterType.Perform` 就能看到所有技能释放相关日志，无需逐条开关。

### 状态快照机制

```csharp
private static Dictionary<string, string> LogStateCache = new();

public static void LogState<T1>(LogKey key, bool append, T1 t1)
{
    if (EnableStateLog) LogState(key, t1.ToString(), append);
}

private static void LogState(LogKey key, string message, bool append)
{
    var str = key.ToString();
    if (LogStateCache.TryGetValue(str, out var old))
    {
        LogStateCache[str] = append
            ? ZString.Concat(old, message)  // 追加模式
            : message;                       // 覆盖模式
    }
    else
    {
        LogStateCache[str] = message;
    }
}
```

`LogKey` 是一个值类型：

```csharp
public partial struct LogKey
{
    private string name;
    
    public static LogKey Make(long id, string name)
    {
        if (Log.EnableStateLog)
            return new LogKey { name = ZString.Concat(id, " ", name) };
        return default; // 未启用时返回默认值，所有操作空操作
    }
}
```

**关键设计**：`EnableStateLog = false` 时，`LogKey.Make` 直接返回 `default`，所有 `LogState` 调用都会因为 `if (!EnableStateLog) return;` 提前返回，**生产构建零开销**。

### 战斗帧计数注入

```csharp
public static Func<int> BattleFrameGetter;

private static void LogBattleInfo(BattleLogTag tag, string message)
{
    if (!EnableStateLog) return;
    LogInfo(ZString.Concat(
        LogPrefix,
        LogTagTable[tag], " ",
        BattleFrameGetter?.Invoke() ?? Game.FixedFrames, " ",
        message));
}
```

`BattleFrameGetter` 是可注入的帧计数委托，战斗系统初始化时注入，日志输出中自动附加当前逻辑帧号，便于回放分析时对齐时间线。

## 架构总结

```
业务代码
    │
    ▼
Log.Info() / Log.Warning() / Log.LogBattleInfo()   ← 静态入口层
    │
    ▼ ZLogger（零分配格式化）
Microsoft.Extensions.Logging.ILogger                ← 标准日志抽象
    │
    ├─ ZLoggerUnityDebug  → UnityEngine.Debug.Log
    └─ ZLoggerRollingFile → game_yyyy-MM-dd_xxx.log  ← LogManager 初始化层
```

## 快速开项目清单

1. **引入 ZLogger**：NuGet 包 `ZLogger`，替换所有 `$"..."` 字符串拼接
2. **环境分级**：编辑器 `Information`，客户端 `Warning`，服务端 `Warning`
3. **战斗调试**：用 `BattleLogTag` 标记日志分类，用 `LogFilterType` 控制展示范围
4. **状态快照**：`LogState(key, append:false, value)` 覆盖，`LogStateAppend` 追加，查询用 `GetLogState(key)`
5. **帧追踪**：注入 `BattleFrameGetter = () => battleSystem.Frame`，所有战斗日志自动附帧号
6. **零开销 Guard**：所有调试日志路径以 `if (!EnableStateLog) return;` 开头，发布构建无性能损失
