## 1. 系统概述

剧情与对话系统是本项目的**核心差异化玩法**，它驱动了整个养成剧本的叙事体验，包括：
- **主线剧情播放**（CutScene 类型剧情）
- **气泡对话（Bubble Dialogue）**：角色在场景中的随机轻量对话
- **NPC 对话（Npc Talk）**：与场景 NPC 的互动
- **条件触发剧情**：根据玩家决策、日程安排、角色状态等条件动态触发

系统基于 **UniScript 可视化节点图（Node Canvas）** 实现，策划可以在编辑器中直接设计剧情流程，无需程序介入即可创作复杂的分支故事。

## 2. 架构设计

### 2.1 核心概念图

```
StoryComponent（剧情管理组件）
  ├── 全局模式（bGlobalMode=true）：挂载在 Scene 上，管理全局剧情
  └── 单元模式（bGlobalMode=false）：挂载在 Unit 上，管理单角色剧情
       │
       ▼
  StoryList: List<Story>     当前活跃的剧情列表
       │
       └── Story（单个剧情实体）
             ├── StoryId: int               剧情 ID
             ├── StoryInfo: GraphData       节点图数据（策划创作的流程图）
             │     ├── localBlackboard       本地黑板（剧情变量存储）
             │     └── triggerGroups         触发组列表
             │
             └── TriggerGroups: List<TriggerGroup>  已激活的触发组
                   └── TriggerGroup
                         ├── GroupInfo       触发组配置
                         └── TriggerGraphs: List<Graph>  触发图列表（具体逻辑）
```

### 2.2 UniScript 节点图系统

剧情内容由策划在 Unity Editor 中通过**可视化节点图**（类似 Blueprint/Flow Graph）编辑：

```
节点类型（部分）：
├── StartEvent        开始节点（每个触发图的入口）
├── CommonSkipEvent   可跳过节点（允许玩家快速跳过）
├── IStoryServersCmdNode  服务端命令节点（需要 C/S 同步）
├── IStoryCutsceneNode    过场动画节点
├── StoryCmdNodeBase      基础命令节点基类
└── IStoryStateTipsNode   状态提示节点（显示角色状态变化）
```

节点图以 JSON 格式序列化存储（使用 `FullSerializer`），运行时反序列化并执行。

### 2.3 Blackboard 变量系统

每个剧情有一个 **Blackboard（黑板）**，类似于一块白板，记录剧情运行过程中的所有变量：

```
Blackboard（黑板）
├── self: Unit           当前主角 Unit 引用
├── DELAY_SHOW_SCHEDULE_WATCH_PANEL: bool  特殊延迟显示标记
├── 自定义变量...         由策划在节点图中定义的任意变量
└── 全局变量（GV）        跨剧情共享的全局状态
```

### 2.4 触发机制

```
剧情触发器类型（Triggers 目录）：
├── TG_EventHandler          游戏事件触发（如背包更新、日程变化）
├── TG_BattlePointEventHandler  战斗中特定时间点触发
├── TG_DelayHandler          延迟 N 秒后触发
├── TG_TimerHandler          周期性定时触发
├── TG_EnterAreaHandler      进入指定区域触发
├── TG_BoardBuildEvtHandler  棋盘构建触发
├── TG_ChessBuildEvtHandler  棋子构建触发
├── TG_PSkillInitEvtHandler  技能初始化触发
└── TG_PSkillUnInitEvtHandler 技能销毁触发
```

### 2.5 气泡对话子系统

气泡对话（Bubble Dialogue）是轻量级的角色随机对话，独立于主剧情流程：

```
BubbleDialogueSystem（气泡对话系统）
├── 在 StoryComponent.Awake 时初始化
├── 由定时器/事件触发
├── 根据当前日程、角色状态、关系值选择对话内容
└── 通过 UI 气泡组件展示（不进入剧情流程）
```

### 3.2 剧情加载与启动

```csharp
/// <summary>
/// 加载并启动剧情（核心入口）
/// </summary>
private static async ETTask<Story> LoadStory(
    this StoryComponent self, int storyId, bool force = false)
{
    // 已加载则直接返回（避免重复加载）
    if (self.ContainsStory(storyId))
        return self.GetStory(storyId);

    Log.Info(ZString.Format("<color=#ff0>LoadAndStartStory</color>:{0}", storyId));

    var story = self.AddChild<Story>();
    story.AddComponent<BlackboardComponent>();

    // 加载节点图资源
    string path = PathUtil.GetStoryPath(storyId, self.bGlobalMode);
    story.StoryInfo = self.LoadStoryGraph(path);

    if (story.StoryInfo == null)
    {
        Log.LogError(ZString.Format("Story not found! id: {0}", storyId));
        return null;
    }

    // 单元模式：将 Unit 注入到黑板，供节点图访问
    if (!self.bGlobalMode)
    {
        var me = self.GetParent<Unit>();
        var selfVar = story.StoryInfo.localBlackboard.GetVariable<Unit>("self");
        if (selfVar == null)
            story.StoryInfo.localBlackboard.AddVariable("self", me);
        else
            story.StoryInfo.localBlackboard.SetVariableValue("self", me);
    }

    // 初始化所有触发组的黑板引用
    foreach (var triggerGroup in story.StoryInfo.triggerGroups)
    {
        if (triggerGroup.graphs != null)
        {
            foreach (var triggerGraph in triggerGroup.graphs)
            {
                triggerGraph.localBlackboard = story.StoryInfo.localBlackboard;
            }
        }
    }

    return story;
}
```

### 3.3 剧情实体的触发组控制

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Story\Runtime\StorySystem.cs

/// <summary>
/// 启动剧情（运行默认触发组）
/// </summary>
public static void Start(this Story self)
{
    if (!self.IsInit)
    {
        self.RunDefaultGroup();  // 运行所有已启用的触发组
        self.IsInit = true;
    }
    Log.Info(ZString.Format("[Story]Story start:{0}", self.StoryId));
}

/// <summary>
/// 运行所有默认启用的触发组
/// </summary>
public static void RunDefaultGroup(this Story self)
{
    for (var i = 0; i < self.StoryInfo.triggerGroups.Count; i++)
    {
        var group = self.StoryInfo.triggerGroups[i];
        if (group.enabled)
        {
            var triggerGroup = self.CreateTriggerGroup(group.id);
            if (triggerGroup != null)
                triggerGroup.Run();
        }
    }
}

/// <summary>
/// 精确启动指定触发组（剧情分支切换）
/// </summary>
public static void StartTriggerGroup(this Story self, int triggerGroupId)
{
    var group = self.CreateTriggerGroup(triggerGroupId);
    if (group == null) return;
    self.IsInit = true;
    group.Run();
}

/// <summary>
/// 精确停止指定触发组（结束分支）
/// </summary>
public static void StopTriggerGroup(this Story self, int triggerGroupId)
{
    var group = self.GetTriggerGroup(triggerGroupId);
    if (group == null) return;
    foreach (var triggerGraph in group.TriggerGraphs)
        triggerGraph.Stop();
}
```

/// <summary>
/// 统一的剧情条件评估入口
/// </summary>
public static bool Evaluate(StoryComponent comp, condition conditionName, string conditionParam)
{
    switch (conditionName)
    {
        case condition.None:
            return true;  // 无条件，直接通过

        case condition.SL_GV:
            return SL_GV(comp, conditionParam);    // 全局变量条件

        case condition.SL_BATTLE:
            return SL_BATTLE(comp, conditionParam); // 战斗结果条件

        case condition.SL_SCHEDULE_CHARACTER:
            return SL_SCHEDULE_CHARACTER(comp, conditionParam); // 日程角色

        case condition.SL_IN_TEAM:
            return SL_IN_TEAM(comp, conditionParam); // 队伍成员

        case condition.ScheduleNum:
            return ScheduleNum(comp, conditionParam); // 日程天数

        case condition.SL_PRESSURE_CHARACTER:
            return SL_PRESSURE_CHARACTER(comp, conditionParam); // 压力值

        case condition.SL_RELATION_CHARACTER:
            return SL_RELATION_CHARACTER(comp, conditionParam); // 角色关系

        case condition.SL_STATE_CHARACTER:
            return SL_STATE_CHARACTER(comp, conditionParam); // 角色状态

        case condition.SL_CURRENT_CHARACTOR:
            return SL_CURRENT_CHARACTER(comp, conditionParam); // 当前角色

        case condition.SL_STORY_CHARACTOR:
            return SL_STORY_CHARACTER(comp, conditionParam); // 剧情角色

        default:
            Log.LogError("条件 = {0} 未开发", conditionName.ToString());
            return false;
    }
}

### 3.5 服务端命令节点计数（性能优化）

```csharp
// StorySystem.cs

/// <summary>
/// 递归统计剧情图中的服务端命令节点数量（DFS）
/// 用于在剧情开始前预判网络交互次数，优化 Loading 进度估算
/// </summary>
public static void RecordSpecialNodeCounts(this Story self, Node node = null,
    HashSetComponent<Node> searchedNodes = null)
{
    bool needDispose = searchedNodes == null;
    searchedNodes ??= HashSetComponent<Node>.Create();

    if (node == null)
    {
        // 从所有 StartEvent / CommonSkipEvent 开始遍历
        foreach (var _node in self.StoryInfo.triggerGroups[0].graphs[0].allNodes)
        {
            if (_node is StartEvent or CommonSkipEvent)
                self.RecordSpecialNodeCounts(_node, searchedNodes);
        }
        if (needDispose) searchedNodes.Dispose();
        return;
    }

    if (searchedNodes.Contains(node))
    {
        // 遇到回环（循环分支）：停止递归，避免死循环
        Log.Info(ZString.Format("[Story]RecordSpecialNodeCounts: 检测到循环节点 {0}", node.name));
        return;
    }

    searchedNodes.Add(node);  // 进入 DFS

    // 递归计算子节点
    foreach (var connection in node.outConnections)
    {
        self.RecordSpecialNodeCounts(connection.targetNode, searchedNodes);
        node.CmdNodeCountCache += connection.targetNode.CmdNodeCountCache;
    }

    // 如果自身是服务端命令节点，计数 +1
    if (node is IStoryServersCmdNode)
        node.CmdNodeCountCache++;

    searchedNodes.Remove(node);  // 退出 DFS
    if (needDispose) searchedNodes.Dispose();
}
```

/// <summary>

    string key, T value)
{
    if (self.GlobalVariables == null)
        self.GlobalVariables = new Dictionary<string, object>();

}

/// 获取剧情全局变量
/// </summary>
public static T GetGlobalVariable<T>(this StoryVariableComponent self, string key)
{
    if (self.GlobalVariables == null) return default;
    return self.GlobalVariables.TryGetValue(key, out var val) ? (T)val : default;
}

## 4. 设计亮点

### 4.1 可视化节点图驱动，策划自主创作
整个剧情流程由策划在 Unity Editor 的节点图编辑器中设计，节点之间用连线表示流程方向，无需编写代码即可创作复杂的分支故事。程序只需维护**节点类型的实现**，内容创作完全由策划独立完成，极大地提升了内容生产效率。

### 4.2 Blackboard 解耦数据传递
Blackboard（黑板）是节点图中所有节点共享数据的统一容器，节点之间不直接引用，而是通过 Blackboard 读写变量通信。这种设计：
- **解耦节点**：任意节点都可以安全地独立测试
- **方便调试**：所有运行时变量都在 Blackboard 中可视
- **支持热重载**：Blackboard 数据在 Scene 生命周期内持久，不受节点重启影响

### 4.3 DFS 预计算服务端命令节点数量
在剧情开始前递归统计 `CmdNodeCountCache`，使得 Loading 进度可以精确估算"还有多少次网络交互"，避免了 Loading 进度停滞或突然跳跃的体验问题。同时通过 `HashSetComponent<Node>` 检测回环，安全处理了循环分支的场景。

### 4.4 触发器模式的剧情驱动
通过各类 `TriggerHandler`（事件触发、延迟触发、计时触发、区域触发等），剧情的启动时机与游戏事件系统完全解耦。策划只需在节点图的触发器节点中配置条件，无需程序为每种触发条件写专用代码。

### 4.5 气泡对话与主剧情分离
气泡对话系统独立于主剧情流程，不占用剧情队列，可以在任何时候打断并显示，实现了"轻量对话"和"重量剧情"并行的设计。这种分层设计既保证了叙事的连贯性，又丰富了角色的日常表现。

## 5. 常见问题与最佳实践

### Q1: 如何新增一种剧情触发条件？
1. 在 `condition` 枚举中添加新值（配置表同步更新）；
2. 在 `StoryLogicCondition.Evaluate` 的 switch 中添加 case；
3. 实现对应的静态判断方法；
4. 策划在节点图的条件节点中选择新条件类型并填写参数即可。

### Q2: 剧情播放时如何暂停/恢复？
```csharp
// 暂停指定剧情
var story = storyComp.GetStory(storyId);
story?.StopTriggerGroup(triggerGroupId);

// 恢复指定剧情
story?.StartTriggerGroup(triggerGroupId);
```
注意：停止触发组会中断当前节点的执行。如果需要"暂停"而非"停止"，需要节点本身支持暂停状态（通过 Blackboard 变量标记）。

### Q3: 剧情中的玩家选项（分支对话）是如何实现的？
选项节点（`StoryCmdNodeBase` 的子类）在执行时会向 `Story.OptionsLs` 添加选项 ID，UI 层监听到这个列表后展示选项按钮。玩家点击后触发对应的选项事件，驱动节点图从对应分支继续执行。

### Q4: 如何调试正在播放的剧情？
1. 在 `StoryComponent.StoryList` 中找到活跃的 `Story` 对象；
2. 查看 `Story.StoryInfo.localBlackboard` 中的所有变量值；
3. 在 `StoryLogicCondition.Evaluate` 中打断点，查看条件判断结果；
4. 通过 `Log.LogInfo("[Story]...")` 风格的日志追踪剧情推进步骤。

### Q5: 剧情资源（节点图 JSON）如何热更新？
节点图文件通过 Addressables 加载（`PathUtil.GetStoryPath`），支持热更新。策划修改节点图后，更新 Addressables 资源包，客户端下载新资源包后，下次进入剧情时自动加载新版本，无需重启游戏。注意：正在播放中的剧情不会中途切换，需要在剧情结束后才会使用新版本。

### Q6: 如何确保剧情内容的多语言支持？
剧情文本（对话文字）不直接写在节点图中，而是存储在配置表（`TbStory` 或 `TbDialogue`）中，节点图中只引用文本 ID。切换语言时，客户端重新加载对应语言的配置表，剧情文本自动切换，无需修改节点图内容。
