# 游戏事件溯源与命令模式撤销重做系统完全指南：从编辑器到战斗系统

## 一、什么是事件溯源？为什么游戏需要它？

**事件溯源（Event Sourcing）** 的核心思想是：**不保存当前状态，而是保存产生该状态的事件序列**。任何时刻的状态都可以通过从头回放事件序列来重建。

### 游戏中的典型应用场景

| 应用场景 | 传统方案的痛点 | 事件溯源的优势 |
|---------|--------------|-------------|
| 编辑器撤销/重做 | 深拷贝状态开销大 | 只存操作记录，轻量 |
| 战斗回放 | 状态快照文件大 | 事件流压缩率高 |
| 反作弊验证 | 难以复现客户端计算 | 在服务端重放事件验证结果 |
| 调试时间回溯 | 无法"倒退"到某时刻 | 从初始状态重放到目标时刻 |
| 多人协作编辑 | 冲突难以合并 | 事件序列可合并/排序 |

public interface ICommand
{
    /// <summary>执行命令</summary>
    void Execute();
    
    /// <summary>撤销命令（逆操作）</summary>
    void Undo();
    
    /// <summary>重做（默认重新执行）</summary>
    void Redo() => Execute();
    
    /// <summary>命令描述（用于日志/UI显示）</summary>
    string Description { get; }
    
    /// <summary>是否可与相邻同类命令合并（如连续移动）</summary>
    bool CanMergeWith(ICommand other) => false;
    
    /// <summary>合并命令（返回合并后的新命令）</summary>
    ICommand MergeWith(ICommand other) => throw new NotSupportedException();
}

// IReversibleCommand.cs - 可完整记录执行前/后状态的命令（用于差量存储）
public interface IReversibleCommand : ICommand
{
    /// <summary>执行前的状态快照（用于精确撤销）</summary>
    void CaptureBeforeState();
    
    /// <summary>执行后的状态快照</summary>
    void CaptureAfterState();
}

### 2.2 撤销重做栈管理器

```csharp
// CommandHistory.cs
using System;
using System.Collections.Generic;
using UnityEngine;

public class CommandHistory
{
    private readonly Stack<ICommand> _undoStack = new();
    private readonly Stack<ICommand> _redoStack = new();
    private readonly int _maxHistory;
    
    // 历史记录变更事件
    public event Action OnHistoryChanged;
    
    // 当前操作描述（用于UI显示"撤销 移动物体"）
    public string UndoDescription => _undoStack.Count > 0 
        ? $"撤销 {_undoStack.Peek().Description}" 
        : "无法撤销";
    
    public string RedoDescription => _redoStack.Count > 0 
        ? $"重做 {_redoStack.Peek().Description}" 
        : "无法重做";

    public bool CanUndo => _undoStack.Count > 0;
    public bool CanRedo => _redoStack.Count > 0;

    public CommandHistory(int maxHistory = 200)
    {
        _maxHistory = maxHistory;
    }

    /// <summary>执行并记录命令</summary>
    public void Execute(ICommand command)
    {
        // 检查是否可以与栈顶命令合并（减少历史记录条目）
        if (_undoStack.Count > 0 && _undoStack.Peek().CanMergeWith(command))
        {
            var merged = _undoStack.Pop().MergeWith(command);
            merged.Execute();
            _undoStack.Push(merged);
        }
        else
        {
            command.Execute();
            _undoStack.Push(command);
            
            // 限制历史记录深度
            if (_undoStack.Count > _maxHistory)
                TrimHistory();
        }
        
        // 执行新命令后清空重做栈
        _redoStack.Clear();
        OnHistoryChanged?.Invoke();
    }

    /// <summary>撤销最近一次操作</summary>
    public bool Undo()
    {
        if (!CanUndo) return false;
        
        var command = _undoStack.Pop();
        command.Undo();
        _redoStack.Push(command);
        
        OnHistoryChanged?.Invoke();
        return true;
    }

    /// <summary>重做被撤销的操作</summary>
    public bool Redo()
    {
        if (!CanRedo) return false;
        
        var command = _redoStack.Pop();
        command.Redo();
        _undoStack.Push(command);
        
        OnHistoryChanged?.Invoke();
        return true;
    }

    /// <summary>批量执行（作为一个原子操作记录）</summary>
    public void ExecuteBatch(string description, Action action)
    {
        var batchCommand = new BatchCommand(description);
        
        // 临时拦截子命令
        var tempHistory = new CommandHistory(1000);
        tempHistory.OnHistoryChanged += () => { }; // 静默
        
        // 这里需要更复杂的实现，简化展示思路
        action();
        
        _undoStack.Push(batchCommand);
        _redoStack.Clear();
        OnHistoryChanged?.Invoke();
    }

    public void Clear()
    {
        _undoStack.Clear();
        _redoStack.Clear();
        OnHistoryChanged?.Invoke();
    }

    private void TrimHistory()
    {
        // 将 Stack 转为 List 去掉最旧的记录
        var items = _undoStack.ToArray();
        _undoStack.Clear();
        for (int i = items.Length - 1; i >= items.Length - _maxHistory; i--)
            _undoStack.Push(items[i]);
    }
}
```

// MoveObjectCommand.cs - 移动物体命令
public class MoveObjectCommand : ICommand
{
    private readonly Transform _target;
    private readonly Vector3 _fromPosition;
    private Vector3 _toPosition; // 可更新（用于合并连续移动）
    
    public string Description => $"移动 {_target.name}";

    public MoveObjectCommand(Transform target, Vector3 toPosition)
    {
        _target = target;
        _fromPosition = target.position; // 记录执行前位置
        _toPosition = toPosition;
    }

    public void Execute() => _target.position = _toPosition;
    
    public void Undo() => _target.position = _fromPosition;

    // 连续移动可合并（鼠标拖拽期间不产生大量历史记录）
    public bool CanMergeWith(ICommand other)
    {
        return other is MoveObjectCommand otherMove 
            && otherMove._target == _target
            && Time.realtimeSinceStartup - _mergeTimeStamp < 0.3f; // 0.3秒内的移动合并
    }

    public ICommand MergeWith(ICommand other)
    {
        var otherMove = (MoveObjectCommand)other;
        return new MoveObjectCommand(_target, otherMove._toPosition)
        {
            // 保留原始起始位置用于撤销到最初位置
        };
    }
    
    private float _mergeTimeStamp = Time.realtimeSinceStartup;
}

// PlaceObjectCommand.cs - 放置新物体
public class PlaceObjectCommand : ICommand
{
    private readonly GameObject _prefab;
    private readonly Vector3 _position;
    private readonly Quaternion _rotation;
    private readonly Transform _parent;
    private GameObject _instantiated;
    
    public string Description => $"放置 {_prefab.name}";

    public PlaceObjectCommand(GameObject prefab, Vector3 pos, Quaternion rot, Transform parent = null)
    {
        _prefab = prefab;
        _position = pos;
        _rotation = rot;
        _parent = parent;
    }

    public void Execute()
    {
        _instantiated = Object.Instantiate(_prefab, _position, _rotation, _parent);
        _instantiated.name = _prefab.name; // 去掉 "(Clone)" 后缀
    }

    public void Undo()
    {
        if (_instantiated != null)
            Object.Destroy(_instantiated);
    }

    public void Redo()
    {
        // 重建同一个实例（而非新实例），确保引用一致性
        Execute();
    }
}

// DeleteObjectCommand.cs - 删除物体（需要保存完整状态以恢复）
public class DeleteObjectCommand : ICommand
{
    private readonly GameObject _target;
    private readonly Vector3 _savedPosition;
    private readonly Quaternion _savedRotation;
    private readonly Vector3 _savedScale;
    private readonly Transform _savedParent;
    private readonly int _savedSiblingIndex;
    private GameObject _deleted; // 禁用而非销毁，以便撤销

    public string Description => $"删除 {_target?.name}";

    public DeleteObjectCommand(GameObject target)
    {
        _target = target;
        _savedPosition = target.transform.position;
        _savedRotation = target.transform.rotation;
        _savedScale = target.transform.localScale;
        _savedParent = target.transform.parent;
        _savedSiblingIndex = target.transform.GetSiblingIndex();
    }

    public void Execute()
    {
        _deleted = _target;
        _deleted.SetActive(false); // 禁用而非销毁
    }

    public void Undo()
    {
        _deleted.SetActive(true);
        _deleted.transform.SetParent(_savedParent);
        _deleted.transform.position = _savedPosition;
        _deleted.transform.rotation = _savedRotation;
        _deleted.transform.localScale = _savedScale;
        _deleted.transform.SetSiblingIndex(_savedSiblingIndex);
    }
}

// BatchCommand.cs - 批量操作（多选删除、批量属性修改等）
public class BatchCommand : ICommand
{
    private readonly List<ICommand> _commands = new();
    public string Description { get; }

    public BatchCommand(string description) => Description = description;

    public void Add(ICommand cmd) => _commands.Add(cmd);

    public void Execute()
    {
        foreach (var cmd in _commands)
            cmd.Execute();
    }

    public void Undo()
    {
        // 逆序撤销
        for (int i = _commands.Count - 1; i >= 0; i--)
            _commands[i].Undo();
    }
}

// BattleEvent.cs - 战斗事件基类
[Serializable]
public abstract class BattleEvent
{
    public long Timestamp;       // 帧序号（确定性）
    public int ActorEntityId;    // 发起者实体ID
    
    public abstract void Apply(BattleWorld world);   // 正向应用
    public abstract void Rollback(BattleWorld world); // 回滚（可选）
}

// 具体战斗事件
[Serializable]
public class DamageEvent : BattleEvent
{
    public int TargetEntityId;
    public float Amount;
    public DamageType Type;
    public int SkillId;
    
    // 记录执行前状态（用于回滚）
    public float TargetHpBefore;

    public override void Apply(BattleWorld world)
    {
        var target = world.GetEntity(TargetEntityId);
        TargetHpBefore = target.Hp; // 捕获前置状态
        target.Hp = Mathf.Max(0, target.Hp - Amount);
        
        if (target.Hp <= 0)
            world.ScheduleEvent(new DeathEvent { ActorEntityId = TargetEntityId, Timestamp = Timestamp + 1 });
    }

    public override void Rollback(BattleWorld world)
    {
        var target = world.GetEntity(TargetEntityId);
        target.Hp = TargetHpBefore; // 精确恢复
    }
}

[Serializable]
public class MoveEvent : BattleEvent
{
    public Vector2 FromPosition;
    public Vector2 ToPosition;
    
    public override void Apply(BattleWorld world)
    {
        var entity = world.GetEntity(ActorEntityId);
        FromPosition = entity.Position;
        entity.Position = ToPosition;
    }

    public override void Rollback(BattleWorld world)
    {
        world.GetEntity(ActorEntityId).Position = FromPosition;
    }
}

[Serializable]
public class SkillCastEvent : BattleEvent
{
    public int SkillId;
    public Vector2 TargetPosition;
    public int[] TargetEntityIds;
    
    public override void Apply(BattleWorld world)
    {
        // 触发技能效果，产生子事件（伤害、状态等）
        var skill = world.SkillDatabase.Get(SkillId);
        skill.Execute(world, ActorEntityId, TargetPosition, TargetEntityIds);
    }

    public override void Rollback(BattleWorld world)
    {
        // 技能效果由子事件逆序回滚，此处不需要单独处理
    }
}

### 4.2 事件存储与回放引擎

```csharp
// BattleEventStore.cs - 战斗事件存储与回放
public class BattleEventStore
{
    private readonly List<BattleEvent> _events = new(1024);
    private readonly Dictionary<long, int> _frameToEventIndex = new(); // 帧号 -> 事件起始索引
    
    // 快照列表（每 N 帧存一次完整快照，加速跳转）
    private readonly List<(long frame, BattleWorldSnapshot snapshot)> _snapshots = new();
    private const int SNAPSHOT_INTERVAL = 300; // 每300帧存一次快照

    /// <summary>记录事件</summary>
    public void Record(BattleEvent evt)
    {
        if (!_frameToEventIndex.ContainsKey(evt.Timestamp))
            _frameToEventIndex[evt.Timestamp] = _events.Count;
        
        _events.Add(evt);
    }

    /// <summary>从头回放到指定帧（用于观战、回放验证）</summary>
    public BattleWorld ReplayToFrame(long targetFrame, BattleWorldFactory factory)
    {
        // 找到最近的快照（避免从头回放）
        var (startFrame, snapshot) = FindNearestSnapshot(targetFrame);
        
        var world = factory.CreateFromSnapshot(snapshot);
        
        // 从快照位置回放到目标帧
        for (long frame = startFrame + 1; frame <= targetFrame; frame++)
        {
            if (_frameToEventIndex.TryGetValue(frame, out int startIdx))
            {
                int endIdx = GetEndIndex(frame);
                for (int i = startIdx; i < endIdx; i++)
                    _events[i].Apply(world);
            }
            
            // 检查是否需要生成新快照
            if (frame % SNAPSHOT_INTERVAL == 0)
                _snapshots.Add((frame, world.TakeSnapshot()));
        }
        
        return world;
    }

    /// <summary>验证客户端结果（反作弊）</summary>
    public bool ValidateClientResult(BattleResultReport clientReport)
    {
        var replayedWorld = ReplayToFrame(clientReport.FinalFrame, new BattleWorldFactory());
        
        // 逐一比对关键结果
        foreach (var entityResult in clientReport.EntityResults)
        {
            var entity = replayedWorld.GetEntity(entityResult.EntityId);
            if (Mathf.Abs(entity.Hp - entityResult.FinalHp) > 0.01f)
            {
                Debug.LogWarning($"反作弊检测：实体 {entityResult.EntityId} HP 不一致！" +
                    $"服务端={entity.Hp}, 客户端={entityResult.FinalHp}");
                return false;
            }
        }
        return true;
    }

    /// <summary>序列化事件流（用于持久化/网络传输）</summary>
    public byte[] Serialize()
    {
        using var ms = new System.IO.MemoryStream();
        using var writer = new System.IO.BinaryWriter(ms);
        
        writer.Write(_events.Count);
        foreach (var evt in _events)
        {
            // 写入事件类型ID和数据
            writer.Write(EventTypeRegistry.GetId(evt.GetType()));
            // 使用 MessagePack 或 MemoryPack 序列化具体事件
            var data = MessagePackSerializer.Serialize(evt.GetType(), evt);
            writer.Write(data.Length);
            writer.Write(data);
        }
        
        return ms.ToArray();
    }

    private (long frame, BattleWorldSnapshot snapshot) FindNearestSnapshot(long targetFrame)
    {
        for (int i = _snapshots.Count - 1; i >= 0; i--)
        {
            if (_snapshots[i].frame <= targetFrame)
                return _snapshots[i];
        }
        return (0, BattleWorldSnapshot.Initial);
    }

    private int GetEndIndex(long frame)
    {
        // 找到下一帧的起始索引
        long nextFrame = frame + 1;
        return _frameToEventIndex.TryGetValue(nextFrame, out int idx) ? idx : _events.Count;
    }
}

// BattleTimeline.cs - 支持任意时刻回溯的战斗调试工具
public class BattleTimeline : MonoBehaviour
{
    [Header("调试控制")]
    [SerializeField] [Range(0, 1)] private float _playbackPosition = 1f; // 0=最初, 1=当前
    [SerializeField] private bool _isPaused = false;
    [SerializeField] private float _playbackSpeed = 1f;
    
    private BattleEventStore _store;
    private BattleWorld _currentWorld;
    private long _maxFrame;
    private long _displayFrame;
    
    // 时间轴 UI 信息
    public long DisplayFrame => _displayFrame;
    public long MaxFrame => _maxFrame;
    public int TotalEventCount => _store?.EventCount ?? 0;
    
    void Update()
    {
        if (!_isPaused)
        {
            // 正常推进
            _playbackPosition = Mathf.Min(1f, _playbackPosition + Time.deltaTime * _playbackSpeed / GetTotalDurationSeconds());
        }
        
        // 根据滑块位置计算目标帧
        long targetFrame = (long)(_playbackPosition * _maxFrame);
        
        // 若目标帧变化，触发重放
        if (targetFrame != _displayFrame)
        {
            SeekToFrame(targetFrame);
        }
    }

    public void SeekToFrame(long frame)
    {
        _displayFrame = frame;
        _currentWorld = _store.ReplayToFrame(frame, new BattleWorldFactory());
        OnWorldStateUpdated?.Invoke(_currentWorld);
    }

    public void StepForward()
    {
        if (_displayFrame < _maxFrame)
            SeekToFrame(_displayFrame + 1);
    }

    public void StepBackward()
    {
        if (_displayFrame > 0)
            SeekToFrame(_displayFrame - 1);
    }

    public event Action<BattleWorld> OnWorldStateUpdated;

    float GetTotalDurationSeconds() => _maxFrame / 30f; // 假设 30 帧/秒
}
```

## 六、事件流压缩与差量快照

### 6.1 差量编码

```csharp
// DeltaEncoder.cs - 对连续移动事件进行差量压缩
public static class BattleEventCompressor
{
    /// <summary>压缩移动事件序列（差量编码）</summary>
    public static byte[] CompressMoveEvents(List<MoveEvent> events)
    {
        if (events.Count == 0) return Array.Empty<byte>();
        
        using var ms = new System.IO.MemoryStream();
        using var writer = new System.IO.BinaryWriter(ms);
        
        writer.Write(events.Count);
        
        // 第一帧完整编码
        var first = events[0];
        writer.Write(first.Timestamp);
        writer.Write(first.ActorEntityId);
        WriteVector2(writer, first.ToPosition);
        
        // 后续帧差量编码（只存变化量）
        for (int i = 1; i < events.Count; i++)
        {
            var prev = events[i - 1];
            var curr = events[i];
            
            long timeDelta = curr.Timestamp - prev.Timestamp;
            float dx = curr.ToPosition.x - prev.ToPosition.x;
            float dy = curr.ToPosition.y - prev.ToPosition.y;
            
            // 使用变长整数编码时间戳差值（小数字占用字节少）
            WriteVarInt(writer, timeDelta);
            WriteFixedPoint(writer, dx); // 定点数，精度0.001
            WriteFixedPoint(writer, dy);
        }
        
        return ms.ToArray();
    }

    private static void WriteVector2(System.IO.BinaryWriter w, Vector2 v)
    {
        w.Write((short)(v.x * 100)); // 定点数：精度 0.01 单位
        w.Write((short)(v.y * 100));
    }

    private static void WriteFixedPoint(System.IO.BinaryWriter w, float v)
    {
        w.Write((short)(v * 100)); // 差量一般很小，short 足够
    }

    private static void WriteVarInt(System.IO.BinaryWriter w, long value)
    {
        // ZigZag 编码 + LEB128 变长编码
        ulong encoded = (ulong)((value << 1) ^ (value >> 63));
        do
        {
            byte b = (byte)(encoded & 0x7F);
            encoded >>= 7;
            if (encoded != 0) b |= 0x80;
            w.Write(b);
        } while (encoded != 0);
    }
}
```

## 七、多人协作编辑的事件合并

### 7.1 操作转换（Operational Transformation，OT）

```csharp
// CollaborativeEditor.cs - 基于 OT 的多人协作关卡编辑
public class OperationTransformer
{
    // 转换两个并发操作，使其可以按任意顺序应用
    public static (ICommand, ICommand) Transform(ICommand cmdA, ICommand cmdB)
    {
        // 移动 + 删除 的冲突处理
        if (cmdA is MoveObjectCommand moveA && cmdB is DeleteObjectCommand deleteB)
        {
            if (moveA._target == deleteB._target)
            {
                // 删除优先：移动操作变为空操作
                return (NoOpCommand.Instance, deleteB);
            }
        }
        
        // 两个移动同一物体的冲突（后到者优先）
        if (cmdA is MoveObjectCommand moveA2 && cmdB is MoveObjectCommand moveB2)
        {
            if (moveA2._target == moveB2._target)
            {
                // 采用 "最后写者胜" 策略，cmdB 的目标位置生效
                var transformedA = new MoveObjectCommand(moveA2._target, moveB2._toPosition);
                return (transformedA, moveB2);
            }
        }
        
        // 无冲突：原样返回
        return (cmdA, cmdB);
    }
}
```

## 八、最佳实践总结

### ✅ 命令模式推荐做法

1. **命令不应引用 UI 对象**：命令只操作数据模型，与视图解耦，方便服务端验证和无头重放
2. **合并短命令**：鼠标拖拽、属性滑块等高频操作，通过 `CanMergeWith` 合并，避免历史记录爆炸
3. **禁用而非销毁**：删除物体时用 `SetActive(false)` 保留实例，撤销只需 `SetActive(true)`
4. **批量操作原子化**：多选删除等复合操作包装为 `BatchCommand`，一次 Undo 即可完整撤销

### ✅ 事件溯源推荐做法

1. **事件不可变**：一旦记录的事件不应被修改，修正错误通过追加补偿事件实现
2. **事件足够细粒度**：伤害、移动、状态变化各自独立事件，方便精确回滚和分析
3. **快照加速回放**：每 N 帧存一次快照，从最近快照重放，避免每次从头回放
4. **压缩传输**：差量编码 + ZigZag + LEB128，事件流体积可压缩至原始的 15-20%

### ⚠️ 注意事项

1. 命令中捕获的状态（`_fromPosition`）必须在 `Execute` 调用**之前**记录
2. 事件溯源的"当前帧"概念需与游戏逻辑帧严格对齐，不能使用 `Time.time`（浮点不确定）
3. 多人协作时需要向量时钟（Vector Clock）或中央化序列号来全序排列事件
4. 战斗回放验证中，浮点计算差异问题需用定点数或允许一定容差阈值

---

## 总结

**命令模式 + 事件溯源** 是游戏开发中最有价值的架构组合之一：

- 在**编辑器**层面，提供流畅的撤销重做体验，让设计师自由探索创意而无需担心误操作
- 在**战斗**层面，事件流既是完整的战斗记录，又是反作弊验证的铁证，还支持任意时刻的调试回溯
- 在**多人协作**层面，通过操作转换（OT）解决并发冲突，让多人同时编辑同一场景成为可能

掌握这两种模式，能够显著提升游戏工程的可维护性和调试效率。
