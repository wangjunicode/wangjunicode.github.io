## 前言

在游戏框架开发中，有两类看似简单却极具实用价值的工具：**容量受控的队列**与**灵活的字符串模板替换系统**。前者用于滑动窗口、历史记录、帧数据缓冲等场景；后者用于本地化、动态提示、任务描述拼接等需要灵活文本替换的场景。

本文将深入剖析 `VGame.Framework` 命名空间下的 `FixedLengthQueue<T>` 和 `StringReplacer` 两个核心工具类，分析其设计动机、实现细节，并给出典型应用案例。

## 一、固定长度队列：FixedLengthQueue\<T\>

### 1.1 设计动机

标准的 `Queue<T>` 是无容量上限的，元素可以无限入队。但在游戏开发中，很多场景需要**"只保留最近 N 条记录"**的滑动窗口语义：

- **帧输入缓冲**：仅保留最近 5 帧的操作指令，用于连招判断
- **伤害数字历史**：UI 层展示最近 8 条飘字记录，超出自动丢弃最旧的
- **调试日志滚动**：Console 面板只保留最近 100 条日志，超出自动清除
- **网络延迟采样**：每秒记录 RTT 值，只保留最近 60 个样本计算平均值

如果用 `Queue<T>` 手动实现，每次入队都要检查 `Count >= Capacity` 再手动 `Dequeue`，代码散落各处，很容易遗漏。**封装成 `FixedLengthQueue<T>` 后，所有容量控制逻辑集中在一处，调用方只需正常 `Enqueue` 即可。**

### 1.2 完整源码分析

```csharp
public class FixedLengthQueue<T> : Queue<T>
{
    private readonly int _length = 1;

    public FixedLengthQueue(int length) : base(length)
    {
        this._length = length;
    }

    public virtual void Enqueue(T item)
    {
        if (base.Count >= _length)
        {
            var t = base.Dequeue();
            OnDrop(t);
        }
        base.Enqueue(item);
    }

    protected virtual void OnDrop(T item)
    {
        // 子类重写，处理被丢弃的元素
    }

    public bool IsEmpty => base.Count == 0;
}
```

**关键设计要点：**

#### （1）继承而非组合

类继承自 `Queue<T>` 而非包装它，优点是可以直接使用 `Queue<T>` 的所有遍历、`Peek`、`Dequeue` 等方法，无需额外委托。缺点是需要用 `base.Count`、`base.Enqueue` 来避免递归调用自身重写的 `Enqueue`。

```csharp
// 注意：这里必须用 base.Count，不能用 this.Count
// 因为 this.Count 最终也指向 base 的实现（Queue<T>.Count 是非虚的）
if (base.Count >= _length)
{
    var t = base.Dequeue(); // 弹出最旧的元素
    OnDrop(t);              // 通知子类处理被丢弃的元素
}
base.Enqueue(item);         // 入队新元素
```

#### （2）OnDrop 钩子方法

`OnDrop` 是一个 `protected virtual` 方法，空实现。这是**模板方法模式**的典型应用：

- 基类定义算法骨架（超出容量 → 丢弃最旧 → 入队新元素）
- 子类通过重写 `OnDrop` 定制"丢弃时做什么"的具体行为

这样的设计让 `FixedLengthQueue<T>` 既可以作为"无感知丢弃"的简单容器使用，也可以扩展出"丢弃时回收到对象池"、"丢弃时触发事件"等复杂语义。

#### （3）IsEmpty 属性

```csharp
public bool IsEmpty => base.Count == 0;
```

这是一个便利属性，语义比 `Count == 0` 更清晰，符合"代码即文档"的原则。

### 1.3 构造时传 length 给 base(length)

```csharp
public FixedLengthQueue(int length) : base(length)
{
    this._length = length;
}
```

`base(length)` 调用的是 `Queue<T>(int capacity)` 构造函数，它会预分配内部数组容量，避免后续频繁扩容导致 GC 压力。这在游戏中是细节优化：**对于已知固定容量的容器，构造时直接传容量值**。

### 1.4 应用案例：帧输入缓冲

```csharp
// 连招系统：保留最近 6 帧的输入
public class ComboInputBuffer
{
    private readonly FixedLengthQueue<InputKey> _buffer = new FixedLengthQueue<InputKey>(6);

    public void RecordInput(InputKey key)
    {
        _buffer.Enqueue(key);
    }

    public bool MatchCombo(InputKey[] pattern)
    {
        if (_buffer.Count < pattern.Length) return false;
        var recent = _buffer.ToArray();
        int offset = recent.Length - pattern.Length;
        for (int i = 0; i < pattern.Length; i++)
        {
            if (recent[offset + i] != pattern[i]) return false;
        }
        return true;
    }
}
```

### 1.5 应用案例：带回收的对象池缓冲

```csharp
// 子类重写 OnDrop，将被丢弃的对象归还到对象池
public class PooledFixedQueue<T> : FixedLengthQueue<T> where T : class, new()
{
    private readonly ObjectPool<T> _pool;

    public PooledFixedQueue(int length, ObjectPool<T> pool) : base(length)
    {
        _pool = pool;
    }

    protected override void OnDrop(T item)
    {
        _pool.Return(item); // 归还到对象池，避免 GC
    }
}
```

### 1.6 注意事项

1. **`Enqueue` 是 `virtual` 的**：在 C# 中 `Queue<T>.Enqueue` 并非虚方法，`FixedLengthQueue` 通过 `new` 关键字隐藏（shadow）了基类的 `Enqueue`。这意味着如果通过 `Queue<T>` 引用调用 `Enqueue`，会绕过容量检查。因此，**持有 `FixedLengthQueue<T>` 时永远用具体类型变量，不要向上转型为 `Queue<T>`**。

2. **线程安全**：`Queue<T>` 不是线程安全的，`FixedLengthQueue<T>` 同样如此。多线程场景需要外部加锁或使用 `ConcurrentQueue<T>` 方案。

## 二、字符串占位符替换系统：StringReplacer

### 2.1 设计动机

游戏中大量字符串需要动态填充，例如：

```
"玩家 {player:name} 在 {scene:name} 击杀了 {enemy:name}，获得 {reward:gold} 金币"
"冷却时间：{skill:cd} 秒，当前等级：{skill:level}"
"欢迎回来，{player:name}！你已离线 {player:offlinetime}"
```

朴素方案是用 `string.Replace("{player:name}", playerName)`，问题是：

1. 每次新增占位符都要在代码里加一行 `Replace`，散乱且难维护
2. 占位符逻辑与字符串模板耦合，无法复用
3. 带参数的占位符（如 `{item:12}` 表示第 12 号道具名称）无法用固定替换搞定

`StringReplacer` 的解决方案是：**用 `[Placeholder]` Attribute 标注方法，通过反射自动注册替换规则**，实现零代码维护的占位符扩展。

### 2.2 PlaceholderAttribute 定义

```csharp
[AttributeUsage(AttributeTargets.Method, AllowMultiple = true)]
public class PlaceholderAttribute : Attribute
{
    public string Placeholder { get; }

    public PlaceholderAttribute(string placeholder)
    {
        Placeholder = placeholder;
    }
}
```

- `AttributeTargets.Method`：只能标注在方法上
- `AllowMultiple = true`：同一个方法可以绑定多个占位符（如同一个方法既处理 `{player:name}` 也处理 `{player:nickname}`）

### 2.3 StringReplacer 核心实现

```csharp
public class StringReplacer
{
    // 固定占位符规则：{player:name} → Func<string>（无参，直接返回值）
    private readonly Dictionary<string, Func<string>> _fixedRules;
    
    // 通配符规则：{item:*} → Func<string, string>（接收完整占位符，返回替换值）
    private readonly Dictionary<string, Func<string, string>> _wildcardRules;
    
    public object ReplacementMethodsInstance = null;

    public StringReplacer(object replacementMethodsInstance)
    {
        ReplacementMethodsInstance = replacementMethodsInstance;
        // 反射扫描所有带 [Placeholder] 的 public 实例方法
        var methods = replacementMethodsInstance.GetType()
            .GetMethods(BindingFlags.Public | BindingFlags.Instance)
            .Where(m => m.GetCustomAttributes(typeof(PlaceholderAttribute), false).Any());

        foreach (var method in methods)
        {
            var attribute = (PlaceholderAttribute)method
                .GetCustomAttributes(typeof(PlaceholderAttribute), false).First();
            string placeholder = attribute.Placeholder.ToLower();

            if (placeholder.EndsWith("*")) // 通配符规则
            {
                string prefix = placeholder.Substring(0, placeholder.IndexOf(":"));
                _wildcardRules[prefix] = (p) => 
                    (string)method.Invoke(replacementMethodsInstance, new object[] { p });
            }
            else // 固定占位符规则
            {
                _fixedRules[placeholder] = () => 
                    (string)method.Invoke(replacementMethodsInstance, null);
            }
        }
    }
}
```

### 2.4 替换执行逻辑

```csharp
public string ReplacePlaceholders(string input)
{
    if (string.IsNullOrEmpty(input)) return input;

    // 正则：匹配 {word} 或 {word:param} 形式
    Regex regex = new Regex(@"\{(\w+)(?::([^}]+))?\}");

    return regex.Replace(input, match =>
    {
        string prefix = match.Groups[1].Value.ToLower();
        string fullPlaceholder = match.Value.Substring(1, match.Value.Length - 2).ToLower();

        // 1. 优先查找固定规则（完整匹配）
        if (_fixedRules.TryGetValue(fullPlaceholder, out Func<string> fixedRule))
            return fixedRule();

        // 2. 查找通配符规则（前缀匹配）
        if (match.Groups[2].Success) // 有参数
        {
            if (_wildcardRules.TryGetValue(prefix, out Func<string, string> wildcardRule))
                return wildcardRule(fullPlaceholder);
        }

        // 3. 未找到规则，原样保留
        return match.Value;
    });
}
```

**正则表达式解析：**

```
\{(\w+)(?::([^}]+))?\}
  ^^^    ^^^^^^^^^^^^
  前缀     可选参数部分
```

- `\{` / `\}` — 匹配大括号
- `(\w+)` — 捕获组1：前缀，如 `player`、`item`
- `(?::([^}]+))?` — 可选的 `:参数` 部分，捕获组2：参数值，如 `name`、`12`

### 2.5 使用示例

**步骤一：定义替换规则类**

```csharp
public class GameTextReplacements
{
    private readonly PlayerModel _player;
    private readonly ItemDatabase _itemDB;

    public GameTextReplacements(PlayerModel player, ItemDatabase itemDB)
    {
        _player = player;
        _itemDB = itemDB;
    }

    // 固定占位符：{player:name}
    [Placeholder("player:name")]
    public string GetPlayerName() => _player.DisplayName;

    // 固定占位符：{player:level}
    [Placeholder("player:level")]
    public string GetPlayerLevel() => _player.Level.ToString();

    // 通配符：{item:*} 接收完整占位符如 "item:12"，解析道具ID
    [Placeholder("item:*")]
    public string GetItemName(string fullPlaceholder)
    {
        // fullPlaceholder = "item:12"
        int itemId = int.Parse(fullPlaceholder.Split(':')[1]);
        return _itemDB.GetName(itemId);
    }
}
```

**步骤二：创建 StringReplacer 实例**

```csharp
var replacements = new GameTextReplacements(currentPlayer, itemDB);
var replacer = new StringReplacer(replacements);
```

**步骤三：执行替换**

```csharp
string template = "勇士 {player:name}（Lv.{player:level}）获得了 {item:1024}！";
string result = replacer.ReplacePlaceholders(template);
// 输出："勇士 kyriewang（Lv.42）获得了 传说之剑！"
```

### 2.6 性能考量与改进建议

| 问题 | 原因 | 改进方案 |
|---|---|---|
| 每次替换都执行正则 | `new Regex(...)` 在方法内，每次调用都编译正则 | 将 `Regex` 缓存为类成员 `static readonly` |
| 反射 Invoke 较慢 | 每次替换都通过反射调用方法 | 构造时将方法编译为 `Func<string>` 委托（已做），避免每次反射查找 |
| 大量长字符串替换 | `Regex.Replace` 逐字符处理 | 对于极热路径，可预解析模板为段列表，运行时只填充变量段 |

**当前实现的优化点：**构造函数中已经将方法包装为 `Func<string>` 和 `Func<string, string>` 委托，运行时调用委托而非每次反射查找方法，这是正确的做法。唯一需要改进的是正则的缓存问题。

```csharp
// 推荐改法：缓存编译好的 Regex
private static readonly Regex PlaceholderRegex = 
    new Regex(@"\{(\w+)(?::([^}]+))?\}", RegexOptions.Compiled);
```

## 三、两个工具类的设计共性

| 特征 | FixedLengthQueue | StringReplacer |
|---|---|---|
| **核心思想** | 继承扩展 + 模板方法 | 反射注册 + 委托替换 |
| **扩展方式** | 继承重写 OnDrop | 新增 [Placeholder] 方法 |
| **初始化时机** | 构造时预分配容量 | 构造时反射扫描注册 |
| **运行时性能** | O(1) 入队/出队 | 委托调用，接近直接调用 |
| **适用场景** | 固定窗口数据缓存 | 动态文本模板渲染 |

两个类都遵循同一个设计原则：**将变化封装到可扩展点（OnDrop / [Placeholder] 方法），让调用方写最少的代码**。这正是游戏框架工具类设计的精髓所在。

## 四、总结

- `FixedLengthQueue<T>` 通过继承 `Queue<T>` 并重写 `Enqueue`，实现了**自动丢弃最旧元素**的容量受控队列，结合 `OnDrop` 钩子可灵活扩展丢弃逻辑
- `StringReplacer` 通过 `[PlaceholderAttribute]` 标注方法，在构造时反射扫描并注册委托，运行时用正则表达式驱动占位符替换，支持固定占位符和通配符两种匹配模式
- 两者都是**小而美**的工具类，代码量不多但设计扎实，是学习"框架级工具类设计"的好素材

掌握这类工具类的设计思路，能让你在面对新的"定制容器"或"动态文本"需求时，快速构建出可维护、可扩展的解决方案。
