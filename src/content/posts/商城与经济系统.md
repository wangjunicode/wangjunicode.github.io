# 07 商城与经济系统

> 面向对象：刚入行的游戏客户端开发工程师
> 文档版本：v1.0
> 更新日期：2026-03-19

---

## 1. 系统概述

商城与经济系统是游戏商业化的核心支柱，负责管理玩家的**货币流转**、**商品购买**、**充值入口**和**经济数值平衡**。

在本项目中，经济系统通过以下几个组件协同实现：

| 模块 | 功能 | 代码位置 |
|---|---|---|
| 虚拟货币（Currency） | 金币/钻石/体力的存储与查询 | `BackpackComponentSystem.cs` |
| 体力系统（Physical） | 体力上限、当前值、恢复时间 | `PlayerComponentSystem.cs` |
| 奖励发放（AwardNotify） | 购买/活动奖励的统一下发 | `AwardNotifyExtensions.cs` |
| 道具使用（ItemUse） | 消耗道具换取游戏资源 | `BackpackComponentSystem.cs` |
| 系统开关（SystemEntry） | 商城功能的开放控制 | `PlayerComponentSystem.cs` |

> 注：独立的商城 UI 和充值接入层代码通常位于 `Biz/` 或 `UI/` 目录，当前可见的相关逻辑主要是**经济数据的底层支撑**。本文将结合实际代码进行深度讲解，并补充标准商城系统设计方案。

## 2. 架构设计

### 2.1 货币体系

```
货币层次
├── 充值货币（钻石/月卡币）
│     ├── 来源：充值（MSDK 支付接入）
│     ├── 存储：VirtualItems["钻石ID"]
│     └── 消耗：购买商城道具、抽卡、跳过等待等
│
├── 免费货币（金币/声望点）
│     ├── 来源：关卡奖励、任务奖励、活动奖励
│     ├── 存储：VirtualItems["金币ID"]
│     └── 消耗：购买免费商城道具、强化材料等
│
└── 行动力（体力/AP）
      ├── 来源：自然回复（时间）+ 道具恢复
      ├── 存储：PlayerComponent.PlayerPhysicalInfo
      └── 消耗：进入关卡、执行活动等
```

### 2.2 体力（Physical）数据结构

```csharp
class PlayerPhysicalInfo
{
    int Physical;           // 当前体力值
    long LastPhysicalTime;  // 上次更新时间戳（用于客户端本地推算）
    int MaxPhysical;        // 上限体力值
}
```

体力的恢复是**时间驱动**的：每 X 分钟恢复 1 点，客户端根据 `LastPhysicalTime` 和当前时间差在本地推算当前实际体力，不需要每秒都向服务端轮询。

### 2.3 商城系统架构（标准范式）

```
ShopComponent（商城组件）
├── ShopTabs: List<ShopTabData>        商城标签页列表
│     └── ShopTabData
│           ├── TabID:  int
│           ├── Name:   string
│           └── Items:  List<ShopItemData>
│
└── PurchaseHistory: Dict<int, int>    已购记录（key=商品ID, value=购买次数）

ShopItemData（商品数据）
├── ItemID:         int         道具 ID
├── Price:          int         价格
├── CurrencyType:   ECurrencyType  货币类型
├── StockLimit:     int         库存上限（-1=无限）
├── DailyLimit:     int         每日购买上限
├── IsOnSale:       bool        是否打折
└── DiscountRate:   float       折扣率
```

### 2.4 充值系统接入（MSDK）

```
充值流程（已在 ClientEntryEvent.cs 初始化 MSDK）：
玩家点击"充值"
    │
    ▼
调用 MSDK 支付接口（平台层）
    │  （苹果 App Store / Google Play / 微信支付 / QQ 支付）
    ▼
平台返回支付结果
    │
    ▼
客户端上报订单到游戏服务端
    │
    ▼
服务端验证 + 发放充值奖励
    │
    ▼
ZoneInfoChangeNotify → BackpackComponent 更新货币
```

### 3.2 道具使用（道具换资源）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Backpack\BackpackComponentSystem.cs

/// <summary>
/// 请求使用道具（核心经济交互：消耗道具获取资源）
/// </summary>
public static async ETTask<bool> RequestItemUse(
    this BackpackComponent self, ItemData item, int count)
{
    Log.LogInfo("[Backpack][Network] 协议 ZoneCsItemUse 请求开始");

    // 使用前记录当前体力（用于计算体力恢复道具的收益对比）
    var energyCount = self.Player().GetCurrentEnergyCount();

    var req = new ZoneItemUseReq
    {
        Pos      = item.Pos,    // 服务端槽位
        UseCount = count,       // 使用数量
    };

    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneItemUseResp>((uint)ZoneClientCmd.ZoneCsItemUse, req);

    if (nr.IsCompleteSuccess)
    {
        var useCount = nr.Data?.UseCount ?? 0;

        // 发布道具使用成功事件（含奖励详情，供商城/背包 UI 展示）
        EventSystem.Instance.Publish(self.ClientScene(), new Evt_BackpackItemUsed
        {
            ItemID          = item.ItemID,
            UseCount        = useCount,
            AwardNotify     = nr.Data?.AwardNotify,    // 本次获得的奖励
            PrevEnergyCount = energyCount,              // 使用前体力（对比展示）
        });
        return true;
    }

    Log.LogError("[Backpack][Network] 协议 ZoneCsItemUse 接收失败");
    return false;
}
```

### 3.3 货币数量查询

```csharp
// BackpackComponentSystem.cs 扩展方法（示意）

/// <summary>
/// 获取指定虚拟货币数量（如金币、钻石）
/// </summary>
public static long GetCurrencyCount(this BackpackComponent self, int currencyItemId)
{
    if (self.VirtualItems == null) return 0;
    return self.VirtualItems.TryGetValue(currencyItemId, out var item) ? item.Count : 0;
}

/// <summary>
/// 设置虚拟道具数量（服务端推送时调用）
/// </summary>
public static void SetVirtualItemCount(this BackpackComponent self, int itemId, long count)
{
    if (self.VirtualItems == null)
        self.VirtualItems = new Dictionary<int, ItemData>();

    if (self.VirtualItems.TryGetValue(itemId, out var item))
        item.Count = count;
    else
        self.VirtualItems[itemId] = new ItemData { ItemID = itemId, Count = count };
}
```

### 3.4 奖励汇总（多渠道发放）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Backpack\AwardNotifyExtensions.cs

/// <summary>
/// 获取本次奖励中指定货币的总变更量（+/- 都可能）
/// 用于展示"购买成功：+500金币 -30钻石"
/// </summary>
public static long GetItemChangeCount(this AwardNotify self, int itemID)
{
    var count = 0L;
    // 直发奖励
    count += GetItemChangeCount(self.AwardItem, itemID);
    // 邮件发放（背包满时）
    count += GetItemChangeCount(self.MailItem,  itemID);
    // 仅展示（不实际入库）
    count += GetItemChangeCount(self.ShowItem,  itemID);
    return count;
}

/// <summary>
/// 获取总量（用于展示奖励弹窗的合计数量）
/// </summary>
public static long GetItemTotalCount(this AwardNotify self, int itemID)
{
    var count = 0L;
    count += GetItemTotalCount(self.AwardItem, itemID);
    count += GetItemTotalCount(self.MailItem,  itemID);
    count += GetItemTotalCount(self.ShowItem,  itemID);
    return count;
}
```

### 3.5 标准商城购买请求（设计方案）

```csharp
/// <summary>
/// 请求购买商城商品（标准设计方案）
/// </summary>
public static async ETTask<bool> RequestShopBuy(
    this ShopComponent self, int shopItemId, int count = 1)
{
    Log.LogInfo("[Shop][Network] 请求购买商品 {0} x{1}", shopItemId, count);

    // 客户端预校验（货币是否足够）
    var shopItem = self.GetShopItem(shopItemId);
    if (shopItem == null)
    {
        Log.LogError("[Shop] 商品 {0} 不存在", shopItemId);
        return false;
    }

    var backpack = self.Player().Backpack();
    var currencyCount = backpack.GetCurrencyCount(shopItem.CurrencyItemId);
    if (currencyCount < (long)shopItem.Price * count)
    {
        // 货币不足，引导充值
        EventSystem.Instance.Publish(self.ClientScene(),
            new Evt_ShopCurrencyInsufficient { CurrencyType = shopItem.CurrencyType });
        return false;
    }

    // 发起购买请求
    var req = new ZoneShopBuyReq
    {
        ShopItemId = shopItemId,
        Count      = count,
    };
    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneShopBuyResp>((uint)ZoneClientCmd.ZoneCsShopBuy, req);

    if (nr.IsCompleteSuccess)
    {
        // 服务端扣除货币 + 发放商品，通过 ZoneInfoChangeNotify 推送
        EventSystem.Instance.Publish(self.ClientScene(), new Evt_ShopBuySuccess
        {
            ShopItemId  = shopItemId,
            Count       = count,
            AwardNotify = nr.Data?.AwardNotify,
        });
        return true;
    }

    Log.LogError("[Shop][Network] 购买失败，ShopItemId={0}", shopItemId);
    return false;
}
```

## 4. 设计亮点

### 4.1 服务端权威货币系统
所有货币增减都由**服务端最终确认**，客户端的货币显示来自服务端下发数据，杜绝了客户端篡改货币数值的可能性。客户端的"预校验"只是为了提升体验（避免无谓的网络请求），不作为安全防护手段。

### 4.2 体力的本地推算
体力恢复不需要服务端推送每秒变化，客户端通过存储的 `LastPhysicalTime` 和 `Physical` 在本地推算：
```
当前体力 = Physical + floor((当前时间 - LastPhysicalTime) / 恢复间隔)
```
这种设计大幅减少了服务端推送频率，同时保证了展示的实时性。

### 4.3 AwardNotify 三路汇总降低展示复杂度
商城购买、关卡奖励、活动奖励的结果展示统一使用 `AwardNotify` 结构，通过扩展方法 `GetItemTotalCount / GetItemChangeCount` 汇总三路数据，UI 层只需调用一个方法即可获取完整奖励信息，无需区分来源。

### 4.4 道具使用记录前置体力快照
在道具使用前记录 `PrevEnergyCount`，使用后计算差值，可以精准展示"此次恢复体力 +X 点"，提升玩家的获得感和使用道具的操作正反馈。

## 5. 常见问题与最佳实践

### Q1: 体力恢复时间如何展示倒计时？
```csharp
// 推算下次恢复的剩余时间
public static long GetNextPhysicalRecoverRemainSeconds(PlayerComponent player)
{
    var info = player.PlayerPhysicalInfo;
    if (info.Physical >= info.MaxPhysical)
        return 0; // 已满，无需倒计时

    var recoverIntervalSeconds = CfgManager.tables.TbGameConst.PhysicalRecoverInterval;
    var elapsed = TimeHelper.ServerNow() - info.LastPhysicalTime;
    var remain = recoverIntervalSeconds - (elapsed % recoverIntervalSeconds);
    return remain;
}
```

### Q2: 如何防止重复购买（防重提交）？
1. 在发起请求时立即禁用购买按钮（`button.interactable = false`）；
2. 请求返回（成功/失败）后再恢复按钮状态；
3. 服务端做幂等性校验，同一订单号不重复处理；
4. 客户端维护本地 `_isPurchasing` 标志位作为双重保障。

### Q3: 限时折扣如何控制？
限时折扣的时间由服务端控制，客户端只展示折扣价格和倒计时。商品数据 `ShopItemData` 中包含 `IsOnSale`、`DiscountRate` 和折扣结束时间戳，客户端仅做展示逻辑，价格计算以服务端为准。

### Q4: 购买失败后如何给玩家友好提示？
根据服务端返回的错误码（`nr.ErrCode`）映射到对应的提示文本：
- `ERR_SHOP_ITEM_NOT_FOUND`  → "商品不存在"
- `ERR_SHOP_CURRENCY_LACK`   → "货币不足，是否前往充值？"
- `ERR_SHOP_STOCK_EMPTY`     → "库存已售罄"
- `ERR_SHOP_DAILY_LIMIT`     → "今日购买次数已达上限"

### Q5: 如何在经济系统中避免"无限刷资源"漏洞？
1. **服务端做所有核心校验**，客户端校验只是 UI 优化；
2. 所有资源增减操作都有**服务端日志**，方便事后审计；
3. 为高价值道具设置**每日购买上限**（`DailyLimit`）；
4. 对异常请求（如短时间大量购买）触发**风控告警**（通常由 MSDK 的反外挂模块处理）。
