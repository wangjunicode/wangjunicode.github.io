## 一、向量（Vector）

### 1.1 向量的本质

向量有两层含义：
1. **几何意义**：有方向和长度的箭头
2. **物理意义**：位移、速度、力的方向和大小

```csharp
// Unity中的向量
Vector3 position = new Vector3(1, 2, 3);  // 位置（点）
Vector3 velocity = new Vector3(0, 0, 5);  // 速度（方向+大小）
Vector3 direction = Vector3.forward;       // 纯方向（单位向量）
```

### 1.2 向量运算及游戏应用

**点积（Dot Product）：**

```
a · b = |a| × |b| × cos(θ)

几何意义：两个向量的"相似度"
- 结果 > 0：夹角 < 90°（同方向趋势）
- 结果 = 0：垂直
- 结果 < 0：夹角 > 90°（反方向趋势）
```

```csharp
// 游戏应用1：判断敌人是否在玩家前方
Vector3 playerForward = transform.forward;
Vector3 toEnemy = (enemy.position - player.position).normalized;
float dot = Vector3.Dot(playerForward, toEnemy);
// dot > 0 → 前方   dot < 0 → 后方
// dot > 0.866 → 前方60度范围内（cos60° = 0.5，但这是cos30°= 0.866）

// 游戏应用2：背刺判断
bool IsBackstab(Vector3 attackerPos, Vector3 targetPos, Vector3 targetForward)
{
    Vector3 toAttacker = (attackerPos - targetPos).normalized;
    float dot = Vector3.Dot(targetForward, toAttacker);
    return dot < -0.5f; // 夹角 > 120°，视为背后
}

// 游戏应用3：投影（计算在斜坡上的速度分量）
Vector3 velocity = new Vector3(1, 0, 1);
Vector3 slopeNormal = new Vector3(0, 0.866f, 0.5f); // 30度斜坡
Vector3 projectedVelocity = velocity - Vector3.Dot(velocity, slopeNormal) * slopeNormal;
// projectedVelocity是沿斜坡方向的速度
```

**叉积（Cross Product）：**

```
a × b = 垂直于a和b的向量
方向：右手定则（右手从a转向b，大拇指方向）
大小：|a| × |b| × sin(θ)

几何意义：
- 结果向量垂直于两个输入向量
- 大小等于两向量构成的平行四边形面积
- 用于计算法向量、旋转方向等
```

```csharp
// 游戏应用1：计算面的法线
Vector3 v0 = new Vector3(0, 0, 0);
Vector3 v1 = new Vector3(1, 0, 0);
Vector3 v2 = new Vector3(0, 1, 0);
Vector3 normal = Vector3.Cross(v1 - v0, v2 - v0).normalized; // (0, 0, 1)

// 游戏应用2：判断转向方向（顺时针还是逆时针）
Vector3 forward = transform.forward;
Vector3 toTarget = (target.position - transform.position).normalized;
Vector3 cross = Vector3.Cross(forward, toTarget);
float turnDirection = Mathf.Sign(cross.y); // +1顺时针转，-1逆时针转

// 游戏应用3：构建正交基（TBN矩阵）
Vector3 N = surfaceNormal.normalized;
Vector3 T = Vector3.Cross(N, Vector3.up).normalized; // Tangent
Vector3 B = Vector3.Cross(N, T).normalized;           // Bitangent
```

## 二、矩阵（Matrix）

### 2.1 变换矩阵的意义

**4×4矩阵的作用：**

```
[Xx  Xy  Xz  Tx]   X轴方向    平移X
[Yx  Yy  Yz  Ty]   Y轴方向    平移Y  
[Zx  Zy  Zz  Tz]   Z轴方向    平移Z
[0   0   0   1 ]   齐次坐标
    ↑                  ↑
  旋转/缩放          平移部分
```

**为什么用4×4而不是3×3？**
- 3×3矩阵无法表示平移（平移不是线性变换）
- 引入齐次坐标后，平移也可以用矩阵乘法表示
- 点用w=1，方向向量用w=0（方向向量不应该受平移影响）

### 2.2 变换顺序的重要性

```csharp
// ⚠️ 矩阵乘法不满足交换律！A×B ≠ B×A

// 正确顺序（Unity使用列主序矩阵）：
// 最终变换 = 平移 × 旋转 × 缩放
Matrix4x4 TRS = Matrix4x4.TRS(position, rotation, scale);

// 这等价于：先缩放，再旋转，最后平移
// 对顶点变换：finalPos = T × R × S × localPos

// 实际游戏场景：父子Transform的矩阵关系
// 世界变换矩阵 = 父节点矩阵 × 本地变换矩阵
Matrix4x4 worldMatrix = parentMatrix * localMatrix;

// 逆矩阵应用：将世界坐标转为本地坐标
Matrix4x4 worldToLocal = worldMatrix.inverse;
Vector3 localPos = worldToLocal.MultiplyPoint3x4(worldPosition);
```

### 2.3 MVP矩阵变换（渲染核心）

```
顶点从模型空间到屏幕的变换链：

模型空间（Local Space）
    ↓ Model矩阵（TRS变换）
世界空间（World Space）
    ↓ View矩阵（相机变换的逆）
相机空间（View/Eye Space）
    ↓ Projection矩阵（透视/正交投影）
裁剪空间（Clip Space）
    ↓ 透视除法（/ w）
NDC空间（[-1,1]³）
    ↓ Viewport变换
屏幕空间（像素坐标）
```

## 三、四元数（Quaternion）

### 3.1 为什么需要四元数？

**欧拉角的问题——万向节死锁（Gimbal Lock）：**

```
欧拉角表示旋转：(pitch, yaw, roll) = (绕X, 绕Y, 绕Z)

问题：当pitch = 90°时，绕Z轴旋转和绕X轴旋转变为等价操作
→ 失去一个自由度
→ 某些旋转无法用欧拉角表示

实际案例：飞机俯冲90°时，方向盘（roll）和航向（yaw）的操作混淆
游戏影响：角色仰头90°时，左右转头变成了滚转
```

**四元数的优势：**
- 没有万向节死锁
- 插值平滑（Slerp）
- 组合旋转只需乘法（效率高）
- 数值稳定性好

### 3.2 四元数原理（不需要完全理解数学，但要会用）

```
四元数：q = (w, x, y, z) = w + xi + yj + zk
- w：实部（与旋转角度相关）
- xyz：虚部（与旋转轴相关）

对于绕轴 axis 旋转 angle 角度：
q = (cos(angle/2), sin(angle/2) × axis.x, sin(angle/2) × axis.y, sin(angle/2) × axis.z)

单位四元数（|q| = 1）表示旋转，乘以(-1)不改变旋转（q 和 -q 表示同一旋转）
```

```csharp
// Unity中的四元数使用

// 1. 创建旋转
Quaternion rotation = Quaternion.Euler(0, 90, 0);     // 从欧拉角
Quaternion fromToRot = Quaternion.FromToRotation(Vector3.forward, targetDir); // 从一个方向旋转到另一个
Quaternion lookRot = Quaternion.LookRotation(direction, Vector3.up); // 朝向某方向

// 2. 旋转组合（顺序重要！）
Quaternion combined = parentRotation * localRotation; // 先局部后父级
// 注意：Quaternion乘法顺序与矩阵相反

// 3. 旋转向量
Vector3 rotatedVector = rotation * Vector3.forward; // 将(0,0,1)旋转

// 4. 球面线性插值（Slerp）- 平滑旋转
Quaternion smoothRotation = Quaternion.Slerp(
    transform.rotation,     // 起始旋转
    targetRotation,         // 目标旋转
    Time.deltaTime * speed  // t值（0-1）
);
// Slerp保证角速度均匀，不会出现快慢不一致的问题

// Lerp vs Slerp：
// Lerp：线性插值，快速但不均匀
// Slerp：球面插值，均匀但略慢
// 游戏中大多数相机/角色旋转用Slerp

// 5. 提取欧拉角（用于显示）
Vector3 euler = rotation.eulerAngles;
// ⚠️ 不要用eulerAngles做增量旋转！会有精度问题
// 而是直接操作Quaternion
```

```csharp
    );
}
```

## 四、坐标系变换

### 4.1 游戏中的多个坐标系

```
模型空间（Object/Local Space）：
- 以模型原点为中心
- 设计师建模时使用的坐标系
- transform.localPosition 就是相对父节点的坐标

世界空间（World Space）：
- 游戏世界的全局坐标系
- transform.position 是世界空间位置

相机空间（View/Eye Space）：
- 以相机为原点，相机正对-Z轴方向
- 用于计算光照（光源方向需要在相机空间中一致）

切线空间（Tangent Space）：
- 以顶点为原点，法线为Z轴
- 法线贴图存储的是切线空间中的法线偏移
- 好处：可以在不同朝向的面上复用同一张法线贴图

裁剪空间（Clip Space）：
- 投影变换后的空间
- 可视锥体变成标准正方体

NDC空间（Normalized Device Coordinates）：
- x,y,z 都在 [-1,1] 范围内
- 通过透视除法（/ w）从裁剪空间得到
```

### 4.2 坐标系转换代码

```csharp
// 世界 ↔ 屏幕
Vector3 screenPos = Camera.main.WorldToScreenPoint(worldPos);
Vector3 worldPos = Camera.main.ScreenToWorldPoint(new Vector3(screenX, screenY, distance));
```

## 五、光线投射（Ray Casting）数学

### 5.1 射线与平面求交

```csharp
// 射线：R(t) = origin + t * direction
// 平面：N · P = d（N是法线，d是距离原点的距离）

// 求交：代入射线方程
// N · (origin + t * direction) = d
// t = (d - N · origin) / (N · direction)

bool RayPlaneIntersect(Ray ray, Vector3 planeNormal, float planeDistance, out Vector3 hitPoint)
{
    hitPoint = Vector3.zero;
    float denom = Vector3.Dot(ray.direction, planeNormal);
    
    if (Mathf.Abs(denom) < 0.0001f) return false; // 平行
    
    float t = (planeDistance - Vector3.Dot(ray.origin, planeNormal)) / denom;
    if (t < 0) return false; // 交点在射线后方
    
    hitPoint = ray.origin + ray.direction * t;
    return true;
}

// 实际游戏应用：鼠标点击地面（地面是y=0的平面）
Vector3 GetMouseWorldPositionOnGround()
{
    Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
    if (RayPlaneIntersect(ray, Vector3.up, 0, out var hitPoint))
        return hitPoint;
    return Vector3.zero;
}
```

### 5.2 射线与球体求交（碰撞检测基础）

```csharp
// 射线：R(t) = O + t * D
// 球体：|P - C|² = r²
// 代入：|O + t*D - C|² = r²
// 展开为二次方程求解t

bool RaySphereIntersect(Vector3 origin, Vector3 dir, Vector3 center, float radius, out float t)
{
    t = 0;
    Vector3 oc = origin - center; // 从球心到射线起点的向量
    
    float a = Vector3.Dot(dir, dir);              // 通常是1（方向已归一化）
    float b = 2f * Vector3.Dot(oc, dir);
    float c = Vector3.Dot(oc, oc) - radius * radius;
    
    float discriminant = b * b - 4 * a * c;
    
    if (discriminant < 0) return false; // 不相交
    
    float sqrtDisc = Mathf.Sqrt(discriminant);
    float t1 = (-b - sqrtDisc) / (2 * a); // 较近交点
    float t2 = (-b + sqrtDisc) / (2 * a); // 较远交点
    
    if (t1 > 0) { t = t1; return true; }
    if (t2 > 0) { t = t2; return true; } // 起点在球内
    
    return false;
}
```

## 七、贝塞尔曲线（Bezier Curve）

### 7.1 贝塞尔曲线在游戏中的应用

```csharp
// 二次贝塞尔曲线（3个控制点）
// 应用：技能飞行轨迹、UI动画、相机路径

Vector3 QuadraticBezier(Vector3 p0, Vector3 p1, Vector3 p2, float t)
{
    float u = 1 - t;
    return u * u * p0 + 2 * u * t * p1 + t * t * p2;
    // p0: 起点, p1: 控制点, p2: 终点
}

// 三次贝塞尔曲线（4个控制点，更平滑）
Vector3 CubicBezier(Vector3 p0, Vector3 p1, Vector3 p2, Vector3 p3, float t)
{
    float u = 1 - t;
    return u*u*u*p0 + 3*u*u*t*p1 + 3*u*t*t*p2 + t*t*t*p3;
}

// 技能飞行特效：抛物线轨迹
public class ProjectileMotion : MonoBehaviour
{
    private Vector3 _start, _control, _end;
    private float _t;
    
    public void Launch(Vector3 start, Vector3 end, float height)
    {
        _start = start;
        _end = end;
        // 控制点在中点上方
        _control = (start + end) / 2 + Vector3.up * height;
        _t = 0;
    }
    
    void Update()
    {
        _t += Time.deltaTime / flightTime;
        transform.position = QuadraticBezier(_start, _control, _end, _t);
        
        if (_t >= 1f) OnArrived();
    }
}
```

### Q2：如何判断点在多边形内部？

```csharp
// 射线法：从点向任意方向发射射线，统计与多边形边的交叉次数
// 奇数次 → 内部，偶数次 → 外部
bool IsPointInPolygon(Vector2 point, Vector2[] polygon)
{
    int n = polygon.Length;
    bool inside = false;
    
    for (int i = 0, j = n - 1; i < n; j = i++)
    {
        if (((polygon[i].y > point.y) != (polygon[j].y > point.y)) &&
            (point.x < (polygon[j].x - polygon[i].x) * (point.y - polygon[i].y) 
                       / (polygon[j].y - polygon[i].y) + polygon[i].x))
        {
            inside = !inside;
        }
    }
    
    return inside;
}
```
