# 技能系统的图驱动架构设计

技能系统是战斗引擎中最复杂的部分之一。一个技能可能包含：起手动作、攻击判定、特效播放、Buff 添加、条件分支、连招检测……如果用硬编码实现，每个技能都是一堆 `if-else`，维护噩梦。本文解析一套基于"可视化节点图"的技能架构，带你理解为什么这是大型动作游戏的主流选择。

---

## 第一性原理：技能本质是什么？

剥去技能的华丽外表，其本质是：**一系列按时序执行的操作序列，支持条件分支，可被外部中断**。

这和"程序"的定义完全吻合——程序就是按顺序执行的指令，支持分支和跳转，可以被信号中断。所以，技能系统的理想形态就是：**让策划可以直接"编程"技能逻辑，而不需要程序员介入每一个细节**。

可视化节点图（FlowGraph）正是把"编程"变成了"连线"，让策划直接在编辑器中设计技能行为。

---

## 技能的核心数据结构

```csharp
[GraphType(EGraphType.Skill)]
[MemoryPackable]
public partial class SkillGraph : FlowGraph
{
    [NonSerialized]
    public SkillComponent Parent;   // 所属的技能组件
    
    public CCommonSkill SkillConfig; // 技能配置表数据
    
    public ETTask<bool> tcs;  // 异步完成信号（async/await 支持）

    public override bool UseBinary => true;  // 使用二进制序列化（性能优化）
}
```

`SkillGraph` 继承自 `FlowGraph`，这是整个架构的核心——技能不是硬编码的逻辑，而是一个可以序列化、可以在编辑器可视化的"图"。

**为什么要 `ETTask<bool> tcs`？**

技能的执行是异步的。当技能开始执行，调用者（状态机）可以 `await` 等待技能执行完毕，然后决定下一步行为：

```csharp
// 调用技能并等待结果
bool success = await skillComponent.StartSkill(skillId);
if (success)
{
    // 技能正常完成，进入下一状态
}
else
{
    // 技能被中断，恢复空闲状态
}
```

这种 `async/await` 的使用让复杂的时序逻辑变得极度清晰，而不是传统的回调地狱。

## SkillComponent：技能管理器

每个 Unit 都持有一个 `SkillComponent`，负责管理该 Unit 的所有技能实例：

```csharp
public static async ETTask<bool> StartSkill(this SkillComponent self, int skillId)
{
    var info = self.AddSkillInfo(skillId);  // 获取或创建技能图实例
    return await info.Start();              // 异步执行
}
```

### 技能加载与缓存

```csharp
private static SkillGraph LoadSkillInfo(this SkillComponent self, int skillId)
{
    var path = PathUtil.GetSkillGraphPath(skillId);
    
    // 先查缓存池
    var unitComp = self.Domain.GetComponent<UnitComponent>();
    SkillGraph graph = null;
    if (unitComp.SkillCache.TryGetValue(path, out var skillinfos))
    {
        skillinfos.TryPop(out graph);  // 从缓存池取出（避免重复创建）
    }
    
    // 缓存未命中，从磁盘/Bundle加载
    if (graph == null)
    {
        graph = SerializeHelper.Deserialize<SkillGraph>(path);
    }
    
    // 初始化图的运行环境
    graph.engine = self.Domain.GetComponent<BattleComponent>().Engine;
    
    // 注入 self 变量（技能图中可以直接使用 self.Position 等）
    var v = graph.localBlackboard.GetVariable<Unit>("self");
    if (v == null) v = graph.localBlackboard.AddVariable<Unit>("self");
    v.value = self.GetParent<Unit>();
    v.isReadonly = true;
    
    return graph;
}
```

这里有个重要的设计：**技能图使用对象池缓存**。一个技能图文件可能被多个 Unit 同时使用（比如很多怪物使用同一个技能），通过对象池避免了频繁的反序列化开销。

### 技能变量注入

技能图通过黑板（Blackboard）传递参数，这是策划不需要写代码的关键：

```csharp
public static SkillGraph AddVariable<T>(this SkillGraph skill, string key, T value)
{
    var old = skill.localBlackboard.GetVariable(key);
    if (old != null)
    {
        ((Variable<T>)old).value = value;
        return skill;
    }
    skill.localBlackboard.AddVariable<T>(key, value);
    return skill;
}
```

调用方可以链式设置参数：

```csharp
bool result = await skillComp
    .AddSkillInfo(skillId)
    .AddVariable("IsBackHit", isBackHit)
    .AddVariable("AttackDirection", direction)
    .Start();
```

技能图内部根据这些变量做不同分支，完全由策划在编辑器中配置，不需要修改代码。

## 技能启动的完整流程

```csharp
private static void _realStart(this SkillGraph skill)
{
    skill.tcs = ETTask<bool>.Create();  // 创建完成信号
    
    // 1. 停止当前所有运行中的普通技能（换技能要停掉旧的）
    using var list = ListComponent<SkillGraph>.Create();
    skill.Parent.GetCurrentSkills(list);
    foreach (var skillInfo in list)
    {
        var cfgSkill = CfgManager.tables.TbCommonSkill.Get(skillInfo.graphId);
        if (cfgSkill != null)
        {
            skill.Parent.StopSkill(skillInfo);  // 停止旧技能
        }
    }
    
    // 2. 受击状态下的旋转校正
    var selfUnit = skill.Parent.GetParent<Unit>();
    if (selfUnit.Type != EEntityType.Director)
    {
        var currentStateTag = fsmComp.CurrentRootStateTag();
        if (currentStateTag == EActionType.Hurt || currentStateTag == EActionType.PerformHurt)
        {
            var hurtComp = selfUnit.GetComponent<HurtComponent>();
            // 被击时校正朝向：面对攻击者
            skill.AddVariable("IsBackHit", hurtComp.AttackInfo.IsBackHit);
            if (currentState is NestedFSMState nestedFsmState && ...)
            {
                var rot = TSQuaternion.LookRotation(
                    (hurtComp.AttackInfo.IsBackHit ? 1 : -1) * direction, 
                    TSVector.up);
                selfUnit.GetComponent<PhysicsComponent>().SetRotation(rot);
            }
        }
    }
    
    // 3. 启动图执行
    if (!skill.isRunning)
    {
        skill.StartGraph(
            skill.Parent.Parent as IUniAgent, 
            skill.localBlackboard,
            Graph.UpdateMode.Manual,  // 手动更新模式（由WorldComponent驱动）
            skill.OnSkillEnd);        // 结束回调
    }
}
```

三个关键步骤：
1. **互斥处理**：新技能开始前停止旧技能（保证同时只有一个技能在运行）
2. **状态感知**：技能启动时感知角色当前状态（如受击状态需要特殊处理）
3. **手动更新**：技能图使用 `Manual` 模式，不自动更新，由 `WorldComponent` 的帧循环统一驱动

### 技能帧更新

```csharp
public static void EnterFrame(this SkillComponent self, FP fixedDeltaTime)
{
    VProfiler.BeginDeepSample("SkillComponent.EnterFrame");
    var unit = self.GetParent<Unit>();
    if (unit.bHost)  // 主机模式跳过（远端同步时不执行本地技能逻辑）
    {
        return;
    }
    for (int i = 0; i < self.SkillList.Count; i++)
    {
        var skill = self.SkillList[i];
        if (skill.isRunning && skill.updateMode == Graph.UpdateMode.Manual)
        {
            skill.UpdateGraph(fixedDeltaTime);  // 推进技能图执行
        }
    }
    VProfiler.EndDeepSample();
}
```

每帧调用 `UpdateGraph`，推进技能图中各个节点的执行。节点可能是"播放动画"、"等待N帧"、"检测碰撞"、"施加Buff"等任何操作。

## 防打断机制：AbortInput

技能在执行过程中，需要知道什么输入可以中断当前技能：

```csharp
public static void AddAbortInput(this SkillComponent self, EInputKey key)
{
    if (!self.AbortInputCnt.ContainsKey(key))
    {
        self.AbortInputCnt.Add(key, 1);
    }
    else
    {
        self.AbortInputCnt[key]++;  // 引用计数，支持多次注册
    }
    self.AbortInput.Add(key);
}

public static void RemoveAbortInput(this SkillComponent self, EInputKey key)
{
    if (self.AbortInputCnt.ContainsKey(key))
    {
        self.AbortInputCnt[key]--;
        if (self.AbortInputCnt[key] <= 0)
        {
            self.AbortInput.Remove(key);
        }
    }
}
```

这是一个带引用计数的"允许中断的输入"注册系统。技能节点在执行时，可以声明"我处于可被某个按键中断的时间窗口"，让系统知道什么时候可以响应新的输入来切换技能。

## 破防系统集成

技能系统还集成了破防（BreakDefense）机制：

```csharp
public static void TryBreakDefense(this SkillComponent self)
{
    var unit = self.GetParent<Unit>();
    var TeamSys = self.Domain.GetComponent<TeamComponent>();
    var team = TeamSys.GetTeamByUnit(unit);
    
    if (team.bHoldBreakDefense > 0)
    {
        // 队伍正在持有破防状态，延迟触发
        self.bDelayBreakDefense = true;
    }
    else
    {
        if (!self.GetParent<Unit>().GetComponent<FSMComponent>().DoBreakDefense)
        {
            self.GetParent<Unit>().GetComponent<FSMComponent>().DoBreakDefense = true;
            var hurtComponent = unit.GetComponent<HurtComponent>();
            if (hurtComponent.AttackInfo.Attacker != null)
            {
                self.ClientScene().GetComponent<EventDispatcherComponent>().FireEvent(
                    new Evt_BreakDefense() { 
                        atker = hurtComponent.AttackInfo.Attacker, 
                        target = unit 
                    });   
            }
        }
    }
}
```

破防延迟触发（`bDelayBreakDefense`）是一个精妙设计——当队伍处于"持有破防"状态时（可能正在播放某个重要演出），破防效果先暂存，等状态结束后再触发，保证了表现层和逻辑层的协调。

## 技能系统设计总结

```
技能请求（玩家输入/AI决策）
    ↓
SkillComponent.StartSkill(skillId)
    ↓
AddSkillInfo → 从缓存池获取或反序列化 SkillGraph
    ↓
注入黑板变量（self, 攻击方向, 是否背击等）
    ↓
_realStart → 停止互斥技能 → StartGraph
    ↓
每帧 SkillComponent.EnterFrame → UpdateGraph（推进节点执行）
    ↓
节点执行：播放Timeline → 碰撞检测 → 施加Buff → 等待...
    ↓
技能结束 → OnSkillEnd → tcs.SetResult(bool)
    ↓
await 的调用方收到结果，状态机进行下一步决策
```

核心设计优点：
1. **策划可配**：技能逻辑由节点图描述，策划无需改代码
2. **异步清晰**：`async/await` 让时序逻辑可读性极强
3. **互斥安全**：自动处理技能切换时的清理工作
4. **对象池优化**：技能图复用，避免频繁序列化
5. **状态感知**：技能启动时自动感知并处理角色当前状态

这套架构支撑了数十甚至上百个复杂技能的开发，是大型动作游戏技能系统的工业化解决方案。
