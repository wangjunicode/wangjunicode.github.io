# 受击系统与打击感实现原理

"打击感"是动作游戏的灵魂。玩家抡起武器打下去，如果对方像纸糊的一样没反应，游戏立刻失去吸引力。打击感来自多个维度：受击动画、音效、顿帧、击退力、屏幕震动……本文专注于技术实现，深入解析受击系统如何将这些元素整合，产生真实的打击体验。

---

## 第一性原理：打击感的要素分析

优秀的打击感需要：

1. **视觉反馈**：受击动画、闪白、受击特效
2. **物理反馈**：击退（被打飞一段距离）
3. **时间反馈**：顿帧（被击中瞬间时间停顿）
4. **音频反馈**：沉闷的打击音效
5. **逻辑反馈**：进入受击状态，暂时无法行动

其中，程序最核心的是物理反馈（击退）和时间反馈（顿帧）。它们由 `HurtSystem` 驱动。

## HurtSystem：受击处理核心

```csharp
public static class HurtSystem
{
    public static void HurtMoveAndFreeze(ActionTask task, Unit attacker, Unit defender, 
        int skillID, JumpTextType jumpTextType)
    {
        var hurtComp = defender.GetComponent<HurtComponent>();
        
        if (attacker != null)
        {
            // 1. 广播受击状态（让双方都知道）
            var defenderState = defender.GetComponent<FSMComponent>().CurrentLeafStateTag();
            attacker.GetComponent<EventDispatcherComponent>()
                .FireEvent(new Evt_DefenderHitState() { HitState = defenderState });
            defender.GetComponent<EventDispatcherComponent>()
                .FireEvent(new Evt_DefenderHitState() { HitState = defenderState });
            
            // 2. 执行受击技能（播放受击动画）
            var subskillConfig = hurtComp.AttackInfo.SubSkillConfigClone;
            CastSkill(task, defender, skillID, subskillConfig).Coroutine();
            
            // 3. 立刻触发动画的第0帧预采样（消除动画启动延迟）
            var timeline = defender.GetComponent<TimelineComponent>().ActiveTimeline;
            if (timeline != null && timeline.isWillActive)
            {
                timeline.PreSample(0);
                timeline.Play(0);
            }
            
            // 4. 发布受击特效事件（跳字、特效）
            EventSystem.Instance.Publish(defender.DomainScene(), new Evt_HurtHitEffect()
            {
                Attacker = attacker,
                Defender = defender,
                jumpTextType = jumpTextType
            });
            
            // 5. 击退 + 顿帧
            if (subskillConfig != null)
            {
                defender.GetComponent<PhysicsComponent>().Stop();  // 先停止现有运动
                
                var simpleMoveComp = defender.GetComponent<SimpleMoveComponent>();
                var subskillParam = subskillConfig.GetSubSkillParam(defenderState);
                
                // 计算击退速度：水平方向 + 垂直方向
                var velocity = TSMath.Abs(subskillParam.PowerXz) * hurtComp.AttackInfo.AttackDirection 
                             + subskillParam.PowerY * TSVector.up;
                
                // 摩擦率调整
                if (subskillParam.DefenderFrictionAcceleration > 0)
                {
                    simpleMoveComp.FrictionRate = subskillParam.DefenderFrictionAcceleration 
                                                / simpleMoveComp.FrictionAcceleration;
                }
                
                simpleMoveComp.AddForceVelocity(velocity);  // 施加击退速度
                
                // 顿帧：攻击方和防御方分别冻结不同帧数
                attacker.GetComponent<UnitTimeComponent>()
                    .Freeze(subskillParam.FinalAttackFreezeFrames(defenderState));
                defender.GetComponent<UnitTimeComponent>()
                    .Freeze(subskillParam.FinalHurtFreezeFrames(defenderState));
            }
        }
    }
}
```

这个函数是打击感实现的集中体现，包含了几个关键子系统的协作。

## 顿帧系统详解

顿帧是动作游戏打击感最重要的元素之一。在命中瞬间，时间短暂停顿（通常 2~6 帧），让玩家的大脑有时间"感受"到这次打击。

### 冻结触发

```csharp
// 攻击方冻结（通常比防御方更短，避免攻击方感觉"卡住"）
attacker.GetComponent<UnitTimeComponent>()
    .Freeze(subskillParam.FinalAttackFreezeFrames(defenderState));

// 防御方冻结（硬直时间）
defender.GetComponent<UnitTimeComponent>()
    .Freeze(subskillParam.FinalHurtFreezeFrames(defenderState));
```

`FinalAttackFreezeFrames` 和 `FinalHurtFreezeFrames` 根据防御方当前状态动态计算冻结时长：

```csharp
// 子技能参数（来自策划配置）
public int FinalAttackFreezeFrames(EActionType defenderState)
{
    // 对方处于不同状态时，冻结帧数不同
    // 例如：对方在空中时，冻结时间更短（空中打击感较弱）
    return defenderState switch
    {
        EActionType.Idle => AttackFreezeFrames,
        EActionType.Float => AttackFreezeFrames / 2,
        EActionType.Hurt => (int)(AttackFreezeFrames * 0.8f),
        _ => AttackFreezeFrames
    };
}
```

### 冻结机制实现

```csharp
// UnitTimeComponent 的冻结处理
public static void Freeze(this UnitTimeComponent self, int frames)
{
    if (frames <= 0) return;
    var unit = self.GetParent<Unit>();
    unit.FreezeFrames = TSMath.Max(unit.FreezeFrames, frames);  // 取最大值，不叠加
}
```

**关键细节**：取最大值而不是累加。如果 A 攻击造成 3 帧冻结，0.5 帧后 B 又攻击造成 5 帧冻结，结果是 5 帧而不是 8 帧。这避免了多次同帧命中导致的超长顿帧。

冻结的生效：

```csharp
// WorldComponent.EnterFrame 中
public bool Freeze => this.FreezeFrames > 0;

// 每帧减1
if (unit.FreezeFrames > 0) unit.FreezeFrames--;

// 物理更新跳过冻结单位
if (unit.Freeze) continue;
unit.GetComponent<PhysicsComponent>()?.EnterFrame(deltaTime);
unit.GetComponent<SimpleMoveComponent>()?.EnterFrame(deltaTime);
```

注意：**状态机（FSM）不受冻结影响**，只有物理和移动被冻结。这保证了玩家在顿帧期间仍然可以输入操作（状态机继续响应输入），而不会感觉到操作延迟。

## 击退力系统

```csharp
// 计算击退速度向量
var velocity = TSMath.Abs(subskillParam.PowerXz) * hurtComp.AttackInfo.AttackDirection 
             + subskillParam.PowerY * TSVector.up;

simpleMoveComp.AddForceVelocity(velocity);
```

`AttackDirection` 是攻击的方向向量（从攻击者指向防御者），`PowerXz` 是水平击退力度，`PowerY` 是垂直击退力度（控制"飞起来"的高度）。

```csharp
// AttackInfo 中的方向计算
public TSVector AttackDirection
{
    get
    {
        if (attacker == null || defender == null) return TSVector.forward;
        var dir = (defender.Position - attacker.Position).normalized;
        dir.y = 0;  // 水平方向（不考虑高度差）
        if (dir == TSVector.zero) dir = defender.Forward;  // 重叠时用防御方朝向
        return dir;
    }
}
```

背击处理（被从背后打）：

```csharp
public static bool IsBackHit(this AttackInfo info)
{
    if (info.attacker == null) return false;
    var dot = TSVector.Dot(info.defender.Forward, info.AttackDirection);
    return dot > 0;  // 攻击方向和防御方朝向相同 → 背击
}
```

背击通常有更强的击退效果和特殊的受击动画。

## 摩擦力调整系统

击退速度不是突然停止的，需要逐渐减速到0。这通过摩擦力实现：

```csharp
if (subskillParam.DefenderFrictionAcceleration > 0)
{
    simpleMoveComp.FrictionRate = subskillParam.DefenderFrictionAcceleration 
                                / simpleMoveComp.FrictionAcceleration;
}
```

`FrictionRate`（摩擦率）调整了默认摩擦力的倍数。大于1的值让角色减速更快（重型攻击），小于1的值让角色滑行更远（轻型但有击飞效果的攻击）。

不同类型的攻击可以配置不同的摩擦率：
- 普通攻击：正常摩擦，短距离击退
- 重攻击：低摩擦，滑行距离长
- 上挑攻击：高垂直力，低水平摩擦，打飞效果
- 下砸攻击：强制停止（先调用 `Stop()`），局部爆发击退

## 受击状态感知

受击时，防御方的当前状态影响受击反应：

```csharp
var defenderState = defender.GetComponent<FSMComponent>().CurrentLeafStateTag();

// 广播给双方（用于触发特殊效果）
attacker.GetComponent<EventDispatcherComponent>()
    .FireEvent(new Evt_DefenderHitState() { HitState = defenderState });
defender.GetComponent<EventDispatcherComponent>()
    .FireEvent(new Evt_DefenderHitState() { HitState = defenderState });
```

状态感知的应用场景：
- **防御方处于格挡状态**：触发格挡特效，减少实际伤害，不进入受击动画
- **防御方处于空中**：使用专门的"空中受击"动画和更小的击退力
- **防御方处于倒地**：追打状态，使用"倒地受击"动画
- **防御方处于受击中**：连续受击时，后续攻击可能造成不同效果（如延长硬直）

## 跳字系统（伤害数字显示）

```csharp
EventSystem.Instance.Publish(defender.DomainScene(), new Evt_HurtHitEffect()
{
    Attacker = attacker,
    Defender = defender,
    jumpTextType = jumpTextType
});
```

`JumpTextType` 枚举定义了跳字的显示类型：

```csharp
public enum JumpTextType
{
    Normal,      // 普通白字
    Critical,    // 会心橙字（更大、更显眼）
    Heal,        // 治疗绿字
    Focus,       // 破招
    Parry,       // 格挡
    Miss,        // 闪避
    Immune,      // 免疫
}
```

不同类型触发不同的 UI 表现（颜色、字体大小、飞出动画），让玩家直观感受到攻击效果的差异。

## 动画即时响应：PreSample 优化

```csharp
var timeline = defender.GetComponent<TimelineComponent>().ActiveTimeline;
if (timeline != null && timeline.isWillActive)
{
    timeline.PreSample(0);  // 预采样第0帧
    timeline.Play(0);       // 立刻播放
}
```

通常，动画在 `StartGraph` 后需要等到下一帧才能显示第一帧画面（因为 Update 和逻辑帧的时序关系）。通过 `PreSample(0)` 立刻采样第0帧并提交给渲染，消除了这个一帧延迟，让受击动画响应更加即时，进一步增强打击感。

## HurtComponent 数据模型

`HurtComponent` 存储了受击的完整上下文信息，供受击状态下的技能和 Buff 使用：

```csharp
public class HurtComponent : Entity
{
    public AttackInfo AttackInfo;  // 本次攻击的详细信息
    
    // AttackInfo 包含：
    // - Attacker: 攻击者
    // - Defender: 防御者  
    // - AttackDirection: 攻击方向向量
    // - SubSkillConfigClone: 攻击的子技能配置（决定击退力、冻结帧等）
    // - IsBackHit: 是否背击
    // - JumpTextType: 跳字类型
}
```

## 多段攻击与受击协调

某些技能是多段攻击（如连续刺击3次），每段都需要独立处理受击：

```csharp
// HurtMoveAndFreezeNoCastSkill：不切换技能状态的受击处理
// 用于连击中途的每一段
public static void HurtMoveAndFreezeNoCastSkill(Unit attacker, Unit defender, 
    JumpTextType jumpTextType, CSubSkill subskillConfig = null, 
    bool enableEffect = true, bool enableFreeze = true, 
    bool enableHitPower = true, EHurtResult hurtResult = EHurtResult.None)
{
    // ... 同 HurtMoveAndFreeze 的击退和顿帧逻辑
    // 但不调用 CastSkill（不切换受击动画），保持当前状态继续执行
}
```

`enableEffect`、`enableFreeze`、`enableHitPower` 这三个开关让调用方可以精确控制受击表现：
- 某些轻型命中只需要特效，不需要击退
- 某些表演性攻击只需要效果，不需要实际的顿帧（避免打断演出）

## 打击感设计总结

```
命中判定成功
    ↓
HurtSystem.HurtMoveAndFreeze
    ├─ 状态感知：获取防御方当前状态
    ├─ 受击技能：切换到受击动画（含 PreSample 即时响应）
    ├─ 特效事件：发布跳字 + 受击特效
    └─ 物理效果：
         ├─ Stop() → 清除现有速度
         ├─ AddForceVelocity() → 施加击退速度
         ├─ FrictionRate 调整 → 控制减速速率
         └─ Freeze() → 攻防双方顿帧
```

| 元素 | 技术实现 | 感受作用 |
|------|---------|---------|
| 顿帧 | FreezeFrames 计数器 | 时间停顿，重量感 |
| 击退 | AddForceVelocity | 力量感 |
| 摩擦率 | FrictionRate | 控制滑行距离 |
| 状态感知 | 广播 Evt_DefenderHitState | 不同状态不同表现 |
| 即时动画 | PreSample(0) | 消除一帧延迟 |
| 跳字 | Evt_HurtHitEffect 事件 | 数值可视化 |

理解这套系统，你就掌握了"打击感"的技术密码。好的打击感不是巧合，而是每个细节的精确把控。
