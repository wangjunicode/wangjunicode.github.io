## 基本概念

### 程序、进程、线程

| 概念 | 定义 |
|------|------|
| **程序** | 保存在磁盘上的指令集合，是静态的 |
| **进程** | 程序执行的过程，包含动态创建、调度和消亡，是操作系统资源管理的最小单位 |
| **线程** | 操作系统能够进行运算调度的最小单位，一个进程内可以包含多个线程（最少一个）|
| **协程** | 比线程更轻量级的微线程，一个线程可以有多个协程 |

### 三者关系

```
进程
├── 线程1（主线程）
│   ├── 协程A
│   ├── 协程B
│   └── 协程C
└── 线程2
    └── ...
```

## 线程 vs 协程的核心区别

### 线程（抢占式调度）

线程由**内核调度**，线程切换时需要：
1. 保存用户线程状态到内存
2. 恢复另一个线程状态到寄存器
3. 更新调度器的数据结构

这几步操作涉及**用户态到内核态切换**，开销较大。

### 协程（协作式调度）

协程的调度完全**由用户控制**：
- 协程拥有自己的寄存器上下文和栈
- 切换时保存寄存器上下文和栈到其他地方
- 切回时恢复先前保存的状态
- **直接操作用户空间栈，完全没有内核切换的开销**

## Unity 中的协程

Unity 的协程基于 C# 的 `IEnumerator`（迭代器）实现，本质上是在**主线程**上执行的，并非真正的多线程：

```csharp
// 启动协程
StartCoroutine(MyCoroutine());

IEnumerator MyCoroutine()
{
    Debug.Log("开始");
    yield return new WaitForSeconds(1f);  // 等待1秒（不阻塞主线程）
    Debug.Log("1秒后继续");
    yield return null;  // 等待下一帧
    Debug.Log("下一帧继续");
}
```

### 常用 yield 指令

| yield 指令 | 说明 |
|-----------|------|
| `yield return null` | 等待下一帧 |
| `yield return new WaitForSeconds(t)` | 等待 t 秒 |
| `yield return new WaitForFixedUpdate()` | 等待下次物理更新 |
| `yield return new WaitUntil(() => condition)` | 等待条件为 true |
| `yield return www` | 等待网络请求完成 |

### 协程的注意事项

- 协程在 `MonoBehaviour` 被销毁时自动停止
- 游戏对象被禁用（SetActive(false)）时协程也会停止
- 大量协程会增加 GC 压力，高频场景考虑用 Update 替代

## 在游戏开发中的应用

```csharp
// 异步加载资源（不阻塞主线程）
IEnumerator LoadAssetAsync(string path)
{
    var request = Resources.LoadAsync<GameObject>(path);
    yield return request;
    
    if (request.asset != null)
    {
        Instantiate(request.asset);
    }
}

// 游戏流程控制
IEnumerator GameFlow()
{
    yield return StartCoroutine(ShowTitle());      // 显示标题
    yield return StartCoroutine(LoadGameScene());  // 加载场景
    yield return StartCoroutine(SpawnPlayer());    // 生成玩家
    GameStart();
}
```
