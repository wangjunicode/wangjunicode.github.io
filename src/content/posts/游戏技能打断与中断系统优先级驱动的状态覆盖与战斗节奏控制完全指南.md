# 游戏技能打断与中断系统：优先级驱动的状态覆盖与战斗节奏控制完全指南

## 一、为什么技能打断系统如此重要

### 1.1 技能打断对战斗手感的影响

在动作游戏（ACT）和角色扮演游戏（RPG）中，技能打断系统决定了：

- **操作手感**：玩家能否在合适时机"取消"技能，决定游戏是否流畅
- **战斗策略**：BOSS能否打断玩家技能，决定战斗的挑战性与公平感
- **连招系统**：特定技能能否衔接下一个，是连招设计的核心依赖
- **控制效果**：眩晕、冻结等控制技能如何中断目标当前行为

典型案例对比：

| 游戏类型 | 打断设计特点 |
|---------|------------|
| 魂系游戏 | 绝大多数动作可被强力攻击打断（硬直系统），轻重攻击、翻滚优先级严格分层 |
| MOBA | 施法中被CC控制会中断释放，但部分技能带有"韧性"标签可抵抗 |
| 格斗游戏 | 取消系统（Cancel）：普通攻击可在特定帧窗口取消为特殊技 |
| MMO RPG | 读条技能被移动/受击中断，瞬发技能不可中断 |

### 1.2 打断系统的核心问题

设计一个打断系统需要回答三个核心问题：

1. **什么可以被打断**？（被打断的技能/状态的属性标记）
2. **什么可以打断它**？（触发打断的来源类型和条件）
3. **打断后发生什么**？（技能的清理逻辑、冷却处理、动画过渡）

## 二、打断系统架构设计

### 2.1 核心数据结构

```csharp
using System;
using System.Collections.Generic;

/// <summary>
/// 技能打断标签：定义一个技能"可以被什么类型的事件打断"
/// 使用 [Flags] 位枚举，支持组合多种打断条件
/// </summary>
[Flags]
public enum InterruptFlag : uint
{
    None            = 0,
    
    // ── 来自外部事件的打断 ──
    AnyDamage       = 1 << 0,   // 受到任何伤害
    HeavyDamage     = 1 << 1,   // 受到重型伤害（超过阈值）
    Stun            = 1 << 2,   // 眩晕
    Freeze          = 1 << 3,   // 冰冻
    Knockback       = 1 << 4,   // 击退/击飞
    Grab            = 1 << 5,   // 抓取
    
    // ── 来自玩家输入的打断 ──
    MoveInput       = 1 << 8,   // 移动输入
    DodgeInput      = 1 << 9,   // 翻滚输入
    NormalAttack    = 1 << 10,  // 普通攻击输入
    SpecialSkill    = 1 << 11,  // 特殊技能输入
    UltimateSkill   = 1 << 12,  // 终极技能（通常优先级最高）
    
    // ── 自我中断 ──
    SelfCancel      = 1 << 16,  // 主动取消（玩家按取消键）
    SkillChain      = 1 << 17,  // 技能链衔接（前摇结束可接下一技能）
    
    // ── 常用组合 ──
    AllCC           = Stun | Freeze | Knockback | Grab,          // 所有控制效果
    AllInputs       = MoveInput | DodgeInput | NormalAttack | SpecialSkill | UltimateSkill,
    StandardCast    = AnyDamage | AllCC | DodgeInput,            // 标准读条技能的打断条件
    Unstoppable     = None,                                       // 无敌技能（不可被任何打断）
    Everything      = uint.MaxValue                               // 完全可打断
}

/// <summary>
/// 技能优先级：数值越大优先级越高，高优先级技能可覆盖低优先级
/// </summary>
public enum SkillPriority
{
    Background  = 0,    // 背景动作（待机、走路）
    Normal      = 100,  // 普通攻击
    Skill       = 200,  // 普通技能
    Special     = 300,  // 特殊技能（带特效/大招前摇）
    Ultimate    = 400,  // 终极技能
    Cinematic   = 1000, // 过场动画（不可被任何技能覆盖）
}

/// <summary>
/// 技能阶段：描述技能在生命周期中的当前阶段
/// 不同阶段可以有不同的打断条件
/// </summary>
public enum SkillPhase
{
    Startup,    // 前摇（技能开始到生效之前）
    Active,     // 生效期（伤害/效果生效期间）
    Recovery,   // 后摇（技能效果结束到动画结束）
    Cooldown,   // 冷却（技能可以被新技能完全覆盖）
}
```

### 2.2 技能打断配置（ScriptableObject）

```csharp
using UnityEngine;

/// <summary>
/// 技能打断配置资产：定义每个阶段的可打断性
/// 通过 ScriptableObject 实现配置与代码分离
/// </summary>
[CreateAssetMenu(fileName = "SkillInterruptConfig", menuName = "Game/Skill/Interrupt Config")]
public class SkillInterruptConfig : ScriptableObject
{
    [Header("技能优先级")]
    public SkillPriority priority = SkillPriority.Normal;
    
    [Header("各阶段打断条件")]
    [Tooltip("前摇阶段：可被什么类型的事件打断")]
    public InterruptFlag startupInterruptFlags = InterruptFlag.StandardCast;
    
    [Tooltip("生效阶段：通常很少被打断（如无敌帧）")]
    public InterruptFlag activeInterruptFlags  = InterruptFlag.AllCC;
    
    [Tooltip("后摇阶段：通常最宽松，可被多种输入取消")]
    public InterruptFlag recoveryInterruptFlags = InterruptFlag.Everything;
    
    [Header("连招窗口")]
    [Tooltip("在此阶段可以输入下一个技能（Cancel Window）")]
    public SkillPhase cancelWindowPhase = SkillPhase.Recovery;
    
    [Tooltip("Cancel窗口开始时间（相对当前阶段的进度 0~1）")]
    [Range(0f, 1f)]
    public float cancelWindowStartNormalized = 0f;
    
    [Header("打断保护（韧性）")]
    [Tooltip("启用韧性：低于此伤害值的攻击无法打断此技能")]
    public bool  enablePoise     = false;
    public float poiseThreshold  = 50f;
    
    [Header("被打断后的行为")]
    [Tooltip("被打断后是否立即进入冷却")]
    public bool triggerCooldownOnInterrupt = true;
    
    [Tooltip("被打断后的冷却倍率（0.5 = 只计算一半冷却时间）")]
    [Range(0f, 1f)]
    public float interruptCooldownMultiplier = 0.5f;
    
    /// <summary>
    /// 根据当前技能阶段返回对应的打断标志
    /// </summary>
    public InterruptFlag GetInterruptFlags(SkillPhase phase) => phase switch
    {
        SkillPhase.Startup  => startupInterruptFlags,
        SkillPhase.Active   => activeInterruptFlags,
        SkillPhase.Recovery => recoveryInterruptFlags,
        _                   => InterruptFlag.Everything
    };
    
    /// <summary>
    /// 检查指定的打断源是否能打断当前阶段的技能
    /// </summary>
    public bool CanBeInterrupted(SkillPhase phase, InterruptFlag interruptSource, float damage = 0f)
    {
        // 韧性检查：低于阈值伤害不打断
        if (enablePoise && interruptSource.HasFlag(InterruptFlag.AnyDamage) && damage < poiseThreshold)
            return false;
        
        var flags = GetInterruptFlags(phase);
        return (flags & interruptSource) != InterruptFlag.None;
    }
}
```

## 三、打断管理器核心实现

### 3.1 SkillInterruptManager

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 技能打断管理器：处理技能的打断逻辑、优先级竞争和状态切换
/// 通常每个角色（玩家/Boss/Enemy）各持有一个实例
/// </summary>
public class SkillInterruptManager : MonoBehaviour
{
    // ── 事件 ──────────────────────────────────────────────────────────
    /// <summary>技能被打断时触发：参数为（被打断的技能ID, 打断来源类型）</summary>
    public event Action<string, InterruptFlag> OnSkillInterrupted;
    
    /// <summary>新技能成功进入执行时触发：参数为（新技能ID）</summary>
    public event Action<string> OnSkillStarted;
    
    // ── 当前状态 ──────────────────────────────────────────────────────
    public string       CurrentSkillId    { get; private set; }
    public SkillPhase   CurrentPhase      { get; private set; }
    public float        PhaseProgress     { get; private set; }  // 当前阶段进度 [0,1]
    
    [SerializeField] private bool _debugMode = false;
    
    private SkillInterruptConfig _currentConfig;
    private Coroutine            _currentSkillCoroutine;
    private SkillExecutionContext _currentContext;
    
    // 技能配置表（技能ID → 配置资产）
    private Dictionary<string, SkillInterruptConfig> _configTable 
        = new Dictionary<string, SkillInterruptConfig>();
    
    // ── 公共 API ─────────────────────────────────────────────────────
    
    /// <summary>
    /// 注册技能配置（通常在初始化时批量注册）
    /// </summary>
    public void RegisterSkill(string skillId, SkillInterruptConfig config)
    {
        _configTable[skillId] = config;
    }
    
    /// <summary>
    /// 尝试执行一个技能：如果当前技能可被打断，则执行新技能；否则失败
    /// </summary>
    /// <param name="skillId">技能唯一ID</param>
    /// <param name="context">技能执行上下文（动画、特效、伤害等回调）</param>
    /// <returns>是否成功触发技能</returns>
    public bool TryExecuteSkill(string skillId, SkillExecutionContext context)
    {
        if (!_configTable.TryGetValue(skillId, out var newConfig))
        {
            Debug.LogWarning($"[SkillInterrupt] 技能配置未找到：{skillId}");
            return false;
        }
        
        // 检查是否可以覆盖当前技能
        if (!CanOverrideCurrentSkill(newConfig))
        {
            if (_debugMode)
                Debug.Log($"[SkillInterrupt] 技能 {skillId} 被拒绝：当前 {CurrentSkillId} 不可被覆盖");
            return false;
        }
        
        // 打断当前技能
        if (!string.IsNullOrEmpty(CurrentSkillId))
        {
            InterruptCurrentSkill(InterruptFlag.SkillChain);
        }
        
        // 执行新技能
        StartSkill(skillId, newConfig, context);
        return true;
    }
    
    /// <summary>
    /// 从外部投递打断事件（受击、控制效果等）
    /// </summary>
    /// <param name="interruptSource">打断来源类型</param>
    /// <param name="damage">伤害值（用于韧性判断，CC效果传0）</param>
    /// <returns>是否成功打断了当前技能</returns>
    public bool TryInterruptByEvent(InterruptFlag interruptSource, float damage = 0f)
    {
        if (string.IsNullOrEmpty(CurrentSkillId)) return false;
        if (_currentConfig == null) return false;
        
        bool canInterrupt = _currentConfig.CanBeInterrupted(CurrentPhase, interruptSource, damage);
        
        if (canInterrupt)
        {
            if (_debugMode)
                Debug.Log($"[SkillInterrupt] 技能 {CurrentSkillId} 在 {CurrentPhase} 阶段被 {interruptSource} 打断");
            
            InterruptCurrentSkill(interruptSource);
            return true;
        }
        
        return false;
    }
    
    /// <summary>
    /// 查询当前技能是否处于连招取消窗口（Cancel Window）
    /// </summary>
    public bool IsInCancelWindow()
    {
        if (_currentConfig == null) return false;
        return CurrentPhase == _currentConfig.cancelWindowPhase 
            && PhaseProgress >= _currentConfig.cancelWindowStartNormalized;
    }
    
    // ── 私有实现 ─────────────────────────────────────────────────────
    
    private bool CanOverrideCurrentSkill(SkillInterruptConfig newConfig)
    {
        // 没有当前技能：直接允许
        if (string.IsNullOrEmpty(CurrentSkillId)) return true;
        
        // 优先级检查：新技能优先级必须 >= 当前技能优先级才能覆盖
        if ((int)newConfig.priority < (int)_currentConfig.priority) return false;
        
        // 检查当前阶段是否允许被输入类打断
        // 这里将"尝试执行新技能"视为 SkillChain 或 SpecialSkill 输入
        var inputFlag = newConfig.priority >= SkillPriority.Special 
            ? InterruptFlag.SpecialSkill 
            : InterruptFlag.NormalAttack;
        
        // 连招窗口内特殊处理
        if (IsInCancelWindow() && inputFlag == InterruptFlag.NormalAttack)
            return true;
        
        return _currentConfig.CanBeInterrupted(CurrentPhase, inputFlag);
    }
    
    private void StartSkill(string skillId, SkillInterruptConfig config, SkillExecutionContext ctx)
    {
        CurrentSkillId   = skillId;
        _currentConfig   = config;
        _currentContext  = ctx;
        CurrentPhase     = SkillPhase.Startup;
        PhaseProgress    = 0f;
        
        // 启动技能执行协程
        if (_currentSkillCoroutine != null)
            StopCoroutine(_currentSkillCoroutine);
        _currentSkillCoroutine = StartCoroutine(ExecuteSkillCoroutine(skillId, config, ctx));
        
        OnSkillStarted?.Invoke(skillId);
    }
    
    private void InterruptCurrentSkill(InterruptFlag reason)
    {
        if (_currentSkillCoroutine != null)
        {
            StopCoroutine(_currentSkillCoroutine);
            _currentSkillCoroutine = null;
        }
        
        // 通知技能自身进行清理（撤销已生效的效果等）
        _currentContext?.OnInterrupted?.Invoke(reason);
        
        // 触发冷却
        if (_currentConfig != null && _currentConfig.triggerCooldownOnInterrupt)
        {
            float cd = (_currentContext?.FullCooldown ?? 0f) 
                     * _currentConfig.interruptCooldownMultiplier;
            _currentContext?.StartCooldown?.Invoke(cd);
        }
        
        string prevSkill = CurrentSkillId;
        CurrentSkillId   = null;
        _currentConfig   = null;
        _currentContext  = null;
        
        OnSkillInterrupted?.Invoke(prevSkill, reason);
    }
    
    private System.Collections.IEnumerator ExecuteSkillCoroutine(
        string skillId, SkillInterruptConfig config, SkillExecutionContext ctx)
    {
        // ─ 前摇阶段 ─
        CurrentPhase = SkillPhase.Startup;
        ctx.OnPhaseChanged?.Invoke(SkillPhase.Startup);
        yield return TrackPhase(ctx.StartupDuration);
        
        // ─ 生效阶段 ─
        CurrentPhase = SkillPhase.Active;
        ctx.OnPhaseChanged?.Invoke(SkillPhase.Active);
        ctx.OnActive?.Invoke();  // 触发伤害/效果
        yield return TrackPhase(ctx.ActiveDuration);
        
        // ─ 后摇阶段 ─
        CurrentPhase = SkillPhase.Recovery;
        ctx.OnPhaseChanged?.Invoke(SkillPhase.Recovery);
        yield return TrackPhase(ctx.RecoveryDuration);
        
        // ─ 技能正常结束 ─
        ctx.OnCompleted?.Invoke();
        ctx.StartCooldown?.Invoke(ctx.FullCooldown);
        
        CurrentSkillId  = null;
        _currentConfig  = null;
        _currentContext = null;
        _currentSkillCoroutine = null;
    }
    
    private System.Collections.IEnumerator TrackPhase(float duration)
    {
        if (duration <= 0f) yield break;
        
        float elapsed = 0f;
        while (elapsed < duration)
        {
            elapsed += Time.deltaTime;
            PhaseProgress = Mathf.Clamp01(elapsed / duration);
            yield return null;
        }
        PhaseProgress = 1f;
    }
}

/// <summary>
/// 技能执行上下文：描述技能各阶段的时长和回调
/// </summary>
[Serializable]
public class SkillExecutionContext
{
    public float StartupDuration  = 0.3f;    // 前摇时长（秒）
    public float ActiveDuration   = 0.2f;    // 生效时长（秒）
    public float RecoveryDuration = 0.5f;    // 后摇时长（秒）
    public float FullCooldown     = 5f;      // 完整冷却时间（秒）
    
    // 生命周期回调
    public Action<SkillPhase>  OnPhaseChanged;  // 阶段切换时
    public Action              OnActive;         // 生效阶段开始时（触发伤害/效果）
    public Action              OnCompleted;      // 技能正常结束时
    public Action<InterruptFlag> OnInterrupted;  // 被打断时
    public Action<float>       StartCooldown;    // 触发冷却（参数：冷却时长）
}
```

## 四、连招取消系统（Cancel System）

### 4.1 连招链配置

```csharp
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 连招链定义：哪个技能在哪个阶段可以接哪些技能
/// </summary>
[CreateAssetMenu(fileName = "SkillChainData", menuName = "Game/Skill/Chain Data")]
public class SkillChainData : ScriptableObject
{
    [System.Serializable]
    public class ChainEntry
    {
        public string fromSkillId;          // 来源技能
        public SkillPhase cancelPhase;      // 在哪个阶段可取消
        [Range(0f, 1f)]
        public float cancelWindowStart;     // 取消窗口开始进度
        [Range(0f, 1f)]
        public float cancelWindowEnd = 1f;  // 取消窗口结束进度
        public List<string> toSkillIds;     // 可以接的技能列表
    }
    
    public List<ChainEntry> chainEntries = new List<ChainEntry>();
    
    // 快速查找表（初始化时构建）
    private Dictionary<string, List<ChainEntry>> _lookup;
    
    public void BuildLookup()
    {
        _lookup = new Dictionary<string, List<ChainEntry>>();
        foreach (var entry in chainEntries)
        {
            if (!_lookup.ContainsKey(entry.fromSkillId))
                _lookup[entry.fromSkillId] = new List<ChainEntry>();
            _lookup[entry.fromSkillId].Add(entry);
        }
    }
    
    /// <summary>
    /// 检查当前技能在当前阶段和进度下，能否接入目标技能
    /// </summary>
    public bool CanChain(string fromSkill, string toSkill, SkillPhase phase, float progress)
    {
        if (_lookup == null) BuildLookup();
        if (!_lookup.TryGetValue(fromSkill, out var entries)) return false;
        
        foreach (var entry in entries)
        {
            if (entry.cancelPhase == phase 
                && progress >= entry.cancelWindowStart 
                && progress <= entry.cancelWindowEnd
                && entry.toSkillIds.Contains(toSkill))
            {
                return true;
            }
        }
        return false;
    }
}
```

### 4.2 连招Buffer：宽容时间窗口

真实游戏中，玩家往往会"提前"输入下一个指令。连招Buffer允许在当前技能完成前几帧输入的指令被"保留"并在时机合适时自动触发：

```csharp
/// <summary>
/// 技能输入Buffer：在宽容时间窗口内保存玩家输入，并在合适时机自动触发
/// 实现"早输入"容错，提升操作手感
/// </summary>
public class SkillInputBuffer : MonoBehaviour
{
    [SerializeField] private float _bufferDuration = 0.2f;  // Buffer保留时间（秒）
    [SerializeField] private SkillInterruptManager _interruptMgr;
    
    private struct BufferedInput
    {
        public string SkillId;
        public float  Timestamp;
        public SkillExecutionContext Context;
    }
    
    private readonly Queue<BufferedInput> _buffer = new Queue<BufferedInput>();
    
    private void Update()
    {
        // 清理过期Buffer
        float now = Time.time;
        while (_buffer.Count > 0 && now - _buffer.Peek().Timestamp > _bufferDuration)
            _buffer.Dequeue();
        
        // 尝试消耗Buffer中的输入
        if (_buffer.Count > 0 && _interruptMgr.IsInCancelWindow())
        {
            var input = _buffer.Peek();
            if (_interruptMgr.TryExecuteSkill(input.SkillId, input.Context))
            {
                _buffer.Dequeue(); // 消耗成功，移出Buffer
            }
        }
    }
    
    /// <summary>
    /// 添加技能输入到Buffer（玩家按下技能键时调用）
    /// </summary>
    public void PushSkillInput(string skillId, SkillExecutionContext context)
    {
        // 先尝试立即执行
        if (_interruptMgr.TryExecuteSkill(skillId, context))
            return;
        
        // 立即执行失败→加入Buffer等待合适时机
        // 只保留最新的输入（避免堆叠过多）
        if (_buffer.Count > 0)
            _buffer.Clear();
        
        _buffer.Enqueue(new BufferedInput
        {
            SkillId   = skillId,
            Timestamp = Time.time,
            Context   = context
        });
    }
    
    /// <summary>清空Buffer（如：进入眩晕状态时）</summary>
    public void ClearBuffer() => _buffer.Clear();
}
```

## 五、动画状态机联动

### 5.1 Animator参数约定

```csharp
/// <summary>
/// 技能与Animator状态机的联动层
/// 通过标准化的Animator参数协议，使动画与游戏逻辑解耦
/// </summary>
public class SkillAnimationBridge : MonoBehaviour
{
    [SerializeField] private Animator _animator;
    [SerializeField] private SkillInterruptManager _interruptMgr;
    
    // 标准Animator参数（建议项目统一命名约定）
    private static readonly int PARAM_SkillId       = Animator.StringToHash("SkillId");
    private static readonly int PARAM_Phase         = Animator.StringToHash("SkillPhase");
    private static readonly int PARAM_Interrupted   = Animator.StringToHash("Interrupted");
    private static readonly int PARAM_InterruptType = Animator.StringToHash("InterruptType");
    
    private void Awake()
    {
        _interruptMgr.OnSkillStarted     += OnSkillStarted;
        _interruptMgr.OnSkillInterrupted += OnSkillInterrupted;
    }
    
    private void Update()
    {
        // 每帧更新阶段参数，驱动Animator的BlendTree过渡
        _animator.SetInteger(PARAM_Phase, (int)_interruptMgr.CurrentPhase);
    }
    
    private void OnSkillStarted(string skillId)
    {
        // 触发对应技能动画
        _animator.SetTrigger(skillId);          // 直接用技能ID作Trigger名
        _animator.ResetTrigger(PARAM_Interrupted);
    }
    
    private void OnSkillInterrupted(string skillId, InterruptFlag reason)
    {
        // 根据打断类型播放对应受击/硬直动画
        _animator.SetTrigger(PARAM_Interrupted);
        _animator.SetInteger(PARAM_InterruptType, MapInterruptToAnimType(reason));
    }
    
    private static int MapInterruptToAnimType(InterruptFlag flag) => flag switch
    {
        var f when f.HasFlag(InterruptFlag.Knockback) => 3, // 击退动画
        var f when f.HasFlag(InterruptFlag.Stun)      => 2, // 眩晕动画
        var f when f.HasFlag(InterruptFlag.AnyDamage) => 1, // 普通受击动画
        _                                              => 0  // 默认中断
    };
    
    private void OnDestroy()
    {
        if (_interruptMgr == null) return;
        _interruptMgr.OnSkillStarted     -= OnSkillStarted;
        _interruptMgr.OnSkillInterrupted -= OnSkillInterrupted;
    }
}
```

### 5.2 动画状态机结构设计

```
Animator Controller 层次结构：

[Base Layer]
  ├─ Idle ──────────→ 任意→ SkillEntry (via Trigger)
  ├─ Walk
  └─ SkillStates (Sub State Machine)
       ├─ SkillStartup   (阶段0：前摇)
       │     ↓ SkillPhase=1
       ├─ SkillActive    (阶段1：生效)
       │     ↓ SkillPhase=2
       ├─ SkillRecovery  (阶段2：后摇)
       │
       └─ [ANY STATE] ──Interrupted=true──→ HitReact (Sub SM)
              ├─ HitLight   (InterruptType=1)
              ├─ HitStun    (InterruptType=2)
              └─ HitFlyback (InterruptType=3)
```

## 六、CC控制效果与打断的集成

### 6.1 控制效果系统对接

```csharp
/// <summary>
/// 控制效果（CC）处理器：将CC效果投递给打断系统
/// </summary>
public class CCEffectHandler : MonoBehaviour
{
    [SerializeField] private SkillInterruptManager _interruptMgr;
    [SerializeField] private SkillInputBuffer      _inputBuffer;
    
    // 当前CC状态
    public bool IsStunned  { get; private set; }
    public bool IsFrozen   { get; private set; }
    
    /// <summary>
    /// 施加眩晕效果（来自敌方技能命中）
    /// </summary>
    public void ApplyStun(float duration)
    {
        // 尝试打断当前技能
        bool interrupted = _interruptMgr.TryInterruptByEvent(InterruptFlag.Stun);
        
        if (interrupted || string.IsNullOrEmpty(_interruptMgr.CurrentSkillId))
        {
            // 清空输入Buffer（眩晕期间禁止输入）
            _inputBuffer?.ClearBuffer();
            
            IsStunned = true;
            StartCoroutine(StunCoroutine(duration));
        }
    }
    
    /// <summary>
    /// 施加击退效果
    /// </summary>
    public void ApplyKnockback(Vector3 force, float duration)
    {
        _interruptMgr.TryInterruptByEvent(InterruptFlag.Knockback);
        _inputBuffer?.ClearBuffer();
        
        // 应用物理击退力
        var rb = GetComponent<Rigidbody>();
        if (rb != null) rb.AddForce(force, ForceMode.Impulse);
        
        StartCoroutine(KnockbackRecovery(duration));
    }
    
    /// <summary>
    /// 受到伤害（可能打断技能，取决于技能的打断标志）
    /// </summary>
    public void OnDamageReceived(float damage)
    {
        _interruptMgr.TryInterruptByEvent(InterruptFlag.AnyDamage, damage);
    }
    
    private System.Collections.IEnumerator StunCoroutine(float duration)
    {
        yield return new WaitForSeconds(duration);
        IsStunned = false;
    }
    
    private System.Collections.IEnumerator KnockbackRecovery(float duration)
    {
        yield return new WaitForSeconds(duration);
        // 击退恢复后可以重新输入
    }
}
```

## 八、最佳实践总结

### 8.1 设计原则

| 原则 | 说明 |
|-----|------|
| **配置与逻辑分离** | 打断条件通过 ScriptableObject 配置，策划可独立调整不需改代码 |
| **位标志枚举** | 使用 `[Flags]` 枚举支持灵活的组合查询，O(1) 时间复杂度判断 |
| **阶段分级** | 前摇/生效/后摇对应不同打断宽松度，生效期通常最保护 |
| **韧性系统** | 为BOSS/重要技能提供"抗控"能力，避免玩家CC无限锁死 |
| **Buffer容错** | 输入Buffer让玩家操作更宽松，提升手感而不降低技巧要求 |
| **事件驱动** | 打断结果通过事件通知动画系统/特效系统/UI，保持系统解耦 |

### 8.2 常见问题与解决方案

| 问题 | 根因 | 解决方案 |
|-----|------|---------|
| 技能打断后动画卡在中间 | 动画状态机未响应打断事件 | 在 Interrupted Trigger 触发时强制跳转到受击状态 |
| 连招输入不灵敏 | Buffer时间太短或取消窗口太小 | 增加Buffer时间至0.3s，扩大取消窗口 |
| BOSS被CC无限打断 | 缺少韧性系统 | 为Boss技能开启 enablePoise，并设置 poiseThreshold |
| 多个System争抢打断导致顺序混乱 | 缺少统一打断入口 | 所有打断请求必须通过 SkillInterruptManager，禁止外部直接停止协程 |
| 冷却时间不准确 | 被打断时冷却倍率未正确应用 | 检查 interruptCooldownMultiplier 配置和 StartCooldown 回调 |

### 8.3 平衡调整建议

```
格斗游戏式连招：
  Normal Attack → Recovery阶段 第0帧可Cancel
  增大 Buffer = 3~5帧（约0.05~0.08秒）

RPG/MOBA式读条：
  所有技能Startup阶段对 AnyDamage 敏感
  终极技能Active阶段设置 InterruptFlag.None（无敌帧）

魂系游戏硬直：
  所有攻击对 HeavyDamage 敏感（超过韧性值才打断）
  翻滚设置极短Startup + 生效阶段 InterruptFlag.None
  普通攻击优先级低于翻滚（确保翻滚总能取消攻击）
```

---

技能打断系统是战斗系统中逻辑密度最高的子模块之一，直接决定游戏的打击感与策略深度。通过**位标志枚举**实现灵活的打断条件组合，结合**ScriptableObject**的配置驱动，**输入Buffer**的宽容容错，以及**事件总线**的系统解耦，可以构建出既满足设计需求又易于维护的工业级打断系统。
