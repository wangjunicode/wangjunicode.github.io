## 设计模式的本质

> 设计模式不是可以无脑套用的模板，而是对常见设计问题的经验总结。

使用设计模式的目的只有一个：**让代码更容易被修改**。

---

## 一、创建型模式

### 1.1 工厂模式（Factory）

**问题**：创建对象的方式很复杂，或者需要在运行时决定创建哪种对象。

```csharp
// ❌ 硬编码的对象创建
public class SkillFactory_Bad
{
    public ISkill CreateSkill(int skillId)
    {
        switch (skillId)
        {
            case 1001: return new FireballSkill();
            case 1002: return new IceArrowSkill();
            case 1003: return new LightningSkill();
            // 每次加新技能都要改这里
            default: throw new Exception($"Unknown skill: {skillId}");
        }
    }
}

// ✅ 注册式工厂（对扩展开放，对修改关闭）
public class SkillFactory
{
    private static readonly Dictionary<int, Func<ISkill>> _creators = new();
    
    // 注册（通常在游戏初始化时，或通过 Attribute 自动扫描）
    public static void Register(int skillId, Func<ISkill> creator)
    {
        _creators[skillId] = creator;
    }
    
    public static ISkill Create(int skillId)
    {
        if (_creators.TryGetValue(skillId, out var creator))
            return creator();
        throw new Exception($"Unknown skill: {skillId}");
    }
}

// 初始化注册
static class SkillRegistrar
{
    [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.BeforeSceneLoad)]
    static void RegisterAll()
    {
        SkillFactory.Register(1001, () => new FireballSkill());
        SkillFactory.Register(1002, () => new IceArrowSkill());
        // 新技能只在这里加，不修改工厂本身
    }
}
```

### 1.2 建造者模式（Builder）

**问题**：创建复杂对象需要很多参数，构造函数变得难以维护。

```csharp
// ❌ 参数过多的构造函数（反模式）
var enemy = new Enemy(
    hp: 100, attack: 20, defense: 5, 
    moveSpeed: 3f, attackRange: 2f, detectRange: 15f,
    dropItemId: 1001, dropChance: 0.3f,
    modelPath: "Enemies/Goblin", animName: "Idle",
    isElite: false, isBoss: false
);
// 调用时完全不知道每个参数的含义

// ✅ 建造者模式
public class EnemyBuilder
{
    private readonly EnemyConfig _config = new();
    
    public EnemyBuilder WithHP(int hp)
    { _config.HP = hp; return this; }
    
    public EnemyBuilder WithAttack(int attack)
    { _config.Attack = attack; return this; }
    
    public EnemyBuilder WithMovement(float speed, float detectRange)
    {
        _config.MoveSpeed = speed;
        _config.DetectRange = detectRange;
        return this;
    }
    
    public EnemyBuilder WithDrop(int itemId, float chance)
    {
        _config.DropItemId = itemId;
        _config.DropChance = chance;
        return this;
    }
    
    public EnemyBuilder AsElite()
    { _config.IsElite = true; return this; }
    
    public Enemy Build()
    {
        Validate();
        return new Enemy(_config);
    }
    
    private void Validate()
    {
        if (_config.HP <= 0) throw new ArgumentException("HP must be > 0");
    }
}

// 使用（可读性极佳）
var goblin = new EnemyBuilder()
    .WithHP(100)
    .WithAttack(20)
    .WithMovement(speed: 3f, detectRange: 15f)
    .WithDrop(itemId: 1001, chance: 0.3f)
    .Build();
```

### 1.3 对象池（Object Pool）

**问题**：频繁创建和销毁对象导致 GC 压力。

对象池已在性能优化章节详细讲解，这里补充一个**泛型对象池与 Unity 的集成**：

```csharp
// Unity 2021+ 内置了对象池（UnityEngine.Pool）
using UnityEngine.Pool;

public class BulletManager : MonoBehaviour
{
    [SerializeField] private Bullet _bulletPrefab;
    
    private IObjectPool<Bullet> _pool;
    
    void Awake()
    {
        _pool = new ObjectPool<Bullet>(
            createFunc: CreateBullet,
            actionOnGet: OnGetBullet,
            actionOnRelease: OnReleaseBullet,
            actionOnDestroy: OnDestroyBullet,
            collectionCheck: true,  // 开发时检查重复归还
            defaultCapacity: 20,
            maxSize: 200
        );
    }
    
    private Bullet CreateBullet() => Instantiate(_bulletPrefab, transform);
    private void OnGetBullet(Bullet b) { b.gameObject.SetActive(true); b.Pool = _pool; }
    private void OnReleaseBullet(Bullet b) { b.gameObject.SetActive(false); }
    private void OnDestroyBullet(Bullet b) => Destroy(b.gameObject);
    
    public Bullet Spawn(Vector3 position, Vector3 direction)
    {
        var bullet = _pool.Get();
        bullet.transform.position = position;
        bullet.Initialize(direction);
        return bullet;
    }
}

public class Bullet : MonoBehaviour
{
    public IObjectPool<Bullet> Pool { get; set; }
    
    public void Initialize(Vector3 direction) { /* 初始化 */ }
    
    void OnTriggerEnter(Collider other)
    {
        // 命中后归还对象池
        Pool?.Release(this);
    }
}
```

## 二、结构型模式

### 2.1 组合模式（Composite）

**问题**：需要用统一的方式处理单个对象和对象的集合。

```csharp
// 技能效果的组合
// 单个效果和效果组合都实现相同接口，外部无需区分

public interface ISkillEffect
{
    void Apply(BattleUnit caster, BattleUnit target);
}

// 单个效果
public class DamageEffect : ISkillEffect
{
    private readonly int _damage;
    public DamageEffect(int damage) => _damage = damage;
    public void Apply(BattleUnit caster, BattleUnit target) => target.HP -= _damage;
}

public class SlowEffect : ISkillEffect
{
    private readonly float _duration;
    public SlowEffect(float duration) => _duration = duration;
    public void Apply(BattleUnit caster, BattleUnit target) 
        => target.AddBuff(new SlowBuff(_duration));
}

// 组合效果（可以包含任意数量的子效果）
public class CompositeEffect : ISkillEffect
{
    private readonly List<ISkillEffect> _effects;
    
    public CompositeEffect(params ISkillEffect[] effects)
    {
        _effects = new List<ISkillEffect>(effects);
    }
    
    public void Add(ISkillEffect effect) => _effects.Add(effect);
    
    // 递归应用所有子效果
    public void Apply(BattleUnit caster, BattleUnit target)
    {
        foreach (var effect in _effects)
            effect.Apply(caster, target);
    }
}

// 使用：构建复合技能效果
var iceArrow = new CompositeEffect(
    new DamageEffect(150),         // 造成150伤害
    new SlowEffect(3f),            // 减速3秒
    new FreezeEffect(chance: 0.3f) // 30%几率冰冻
);

// 对外：就是一个 ISkillEffect，不关心内部有多复杂
iceArrow.Apply(caster, target);
```

### 2.2 装饰器模式（Decorator）

**问题**：需要在不修改类的情况下为其添加功能。

```csharp
// 武器伤害的装饰器：运行时叠加效果
public interface IWeapon
{
    int GetDamage();
    void Attack(BattleUnit target);
}

public class BaseSword : IWeapon
{
    public int GetDamage() => 100;
    public void Attack(BattleUnit target) => target.HP -= GetDamage();
}

// 装饰器基类
public abstract class WeaponDecorator : IWeapon
{
    protected readonly IWeapon _wrapped;
    public WeaponDecorator(IWeapon weapon) => _wrapped = weapon;
    public virtual int GetDamage() => _wrapped.GetDamage();
    public virtual void Attack(BattleUnit target) => _wrapped.Attack(target);
}

// 火焰附魔装饰器
public class FlameEnchant : WeaponDecorator
{
    private readonly int _bonusDamage;
    
    public FlameEnchant(IWeapon weapon, int bonusDamage) : base(weapon)
    {
        _bonusDamage = bonusDamage;
    }
    
    public override int GetDamage() => _wrapped.GetDamage() + _bonusDamage;
    
    public override void Attack(BattleUnit target)
    {
        _wrapped.Attack(target);
        // 额外：施加燃烧 Buff
        target.AddBuff(new BurnBuff(duration: 3f, dps: 20));
    }
}

// 暴击装饰器
public class CriticalEnchant : WeaponDecorator
{
    private readonly float _critRate;
    
    public CriticalEnchant(IWeapon weapon, float critRate) : base(weapon)
    {
        _critRate = critRate;
    }
    
    public override void Attack(BattleUnit target)
    {
        if (UnityEngine.Random.value < _critRate)
        {
            // 暴击：双倍伤害
            target.HP -= GetDamage() * 2;
        }
        else
        {
            _wrapped.Attack(target);
        }
    }
}

// 运行时叠加装饰器
IWeapon weapon = new BaseSword();
weapon = new FlameEnchant(weapon, bonusDamage: 50);  // 附魔火焰
weapon = new CriticalEnchant(weapon, critRate: 0.3f); // 附魔暴击

// 最终效果：火焰暴击剑
weapon.Attack(enemy);
```

### 2.3 外观模式（Facade）

**问题**：复杂子系统的接口过于复杂，需要提供简化的接口。

```csharp
// 游戏流程控制：外观模式统一入口
public class GameFacade
{
    private readonly ResourceManager _resources;
    private readonly SceneManager _scenes;
    private readonly NetworkManager _network;
    private readonly AudioManager _audio;
    private readonly SaveSystem _save;
    
    public GameFacade(/* 所有子系统 */) { /* 初始化 */ }
    
    // 简化的操作入口（隐藏了子系统的复杂性）
    public async UniTask StartNewGame(CancellationToken ct)
    {
        // 内部协调多个子系统
        _save.CreateNewSave();
        await _resources.PreloadEssentials(ct);
        await _scenes.LoadAsync("MainLevel", ct);
        _audio.PlayBGM("BattleTheme");
        _network.Initialize();
    }
    
    public async UniTask LoadSavedGame(int saveSlot, CancellationToken ct)
    {
        var saveData = await _save.LoadAsync(saveSlot);
        await _resources.PreloadEssentials(ct);
        await _scenes.LoadAsync(saveData.CurrentScene, ct);
        _audio.PlayBGM(saveData.BGMName);
    }
    
    public void SaveGame(int saveSlot)
    {
        var data = CollectSaveData();
        _save.Save(saveSlot, data);
        // 可能还需要上传到云端等
    }
    
    private SaveData CollectSaveData() => new(); // 收集当前游戏状态
}
```

## 三、行为型模式

### 3.1 策略模式（Strategy）

**问题**：同一行为有多种实现方式，需要在运行时切换。

```csharp
// AI 移动策略
public interface IMoveStrategy
{
    Vector3 GetNextPosition(Vector3 current, Vector3 target, float speed, float deltaTime);
}

public class DirectMoveStrategy : IMoveStrategy
{
    public Vector3 GetNextPosition(Vector3 current, Vector3 target, float speed, float deltaTime)
    {
        Vector3 dir = (target - current).normalized;
        return current + dir * speed * deltaTime;
    }
}

public class NavMeshMoveStrategy : IMoveStrategy
{
    private readonly UnityEngine.AI.NavMeshAgent _agent;
    
    public NavMeshMoveStrategy(UnityEngine.AI.NavMeshAgent agent) => _agent = agent;
    
    public Vector3 GetNextPosition(Vector3 current, Vector3 target, float speed, float deltaTime)
    {
        _agent.SetDestination(target);
        return _agent.nextPosition; // NavMesh 自动计算
    }
}

public class SteeringMoveStrategy : IMoveStrategy
{
    public Vector3 GetNextPosition(Vector3 current, Vector3 target, float speed, float deltaTime)
    {
        // 基于 Steering Behavior 的平滑移动
        // ... 实现避障、群体行为等
        return Vector3.zero; // 省略实现
    }
}

public class EnemyCharacter
{
    private IMoveStrategy _moveStrategy;
    
    // 运行时切换策略
    public void SetMoveStrategy(IMoveStrategy strategy) => _moveStrategy = strategy;
    
    public void MoveTo(Vector3 target)
    {
        transform.position = _moveStrategy.GetNextPosition(
```

### 3.2 观察者模式（Observer）

已在架构设计章节详细讲解事件总线，这里补充**响应式扩展的简单实现**：

```csharp
// 简单的响应式编程模式
public class Observable<T>
{
    private T _value;
    private event Action<T, T> OnChanged; // (oldValue, newValue)
    
    public T Value
    {
        get => _value;
        set
        {
            if (EqualityComparer<T>.Default.Equals(_value, value)) return;
            var old = _value;
            _value = value;
            OnChanged?.Invoke(old, value);
        }
    }
    
    // 链式操作
    public Observable<T> Where(Func<T, bool> predicate)
    {
        var filtered = new Observable<T>();
        OnChanged += (_, newVal) =>
        {
            if (predicate(newVal)) filtered.Value = newVal;
        };
        return filtered;
    }
    
    public Observable<TResult> Select<TResult>(Func<T, TResult> selector)
    {
        var mapped = new Observable<TResult>();
        OnChanged += (_, newVal) => mapped.Value = selector(newVal);
        return mapped;
    }
}

// 游戏应用
var playerHP = new Observable<int> { Value = 100 };

// 当血量低于 30% 时触发低血量特效
playerHP
    .Where(hp => hp < 30)
    .Select(hp => (float)hp / 100f)
    // 订阅低血量百分比
    ;
```

```csharp
// 宏命令：将多个命令组合为一个
public class MacroCommand : ICommand
{
    private readonly List<ICommand> _commands = new();
    
    public void Add(ICommand command) => _commands.Add(command);
    
    public void Execute()
    {
        foreach (var cmd in _commands) cmd.Execute();
    }
    
    public void Undo()
    {
        // 逆序撤销
        for (int i = _commands.Count - 1; i >= 0; i--)
            _commands[i].Undo();
    }
}

// 关卡编辑器中：批量操作
```

### 3.4 职责链模式（Chain of Responsibility）

**问题**：请求需要经过多个处理器，每个处理器决定是否处理或传递。

```csharp
// 输入处理的职责链
public abstract class InputHandler
{
    protected InputHandler _next;
    
    public InputHandler SetNext(InputHandler next)
    {
        _next = next;
        return next; // 链式设置
    }
    
    public abstract bool Handle(InputEvent input);
    
    protected bool PassToNext(InputEvent input)
    {
        return _next?.Handle(input) ?? false;
    }
}

// UI 优先接收输入
public class UIInputHandler : InputHandler
{
    public override bool Handle(InputEvent input)
    {
        if (EventSystem.current.IsPointerOverGameObject())
        {
            // UI 拦截了输入，不传递给游戏
            return true;
        }
        return PassToNext(input); // 没有 UI，传递给下一个处理器
    }
}

// 对话系统接收输入
public class DialogInputHandler : InputHandler
{
    public override bool Handle(InputEvent input)
    {
        if (DialogManager.IsOpen)
        {
            DialogManager.HandleInput(input);
            return true; // 对话框处理了，不再传递
        }
        return PassToNext(input);
    }
}

// 游戏角色接收输入
public class PlayerInputHandler : InputHandler
{
    public override bool Handle(InputEvent input)
    {
        PlayerController.HandleInput(input);
        return true;
    }
}

// 构建输入链
var inputChain = new UIInputHandler();
inputChain
    .SetNext(new DialogInputHandler())
    .SetNext(new PlayerInputHandler());

// 每帧处理输入
void ProcessInput(InputEvent input) => inputChain.Handle(input);
```

### 4.2 脏标记（Dirty Flag）

```csharp
// 只在数据变化时重新计算，避免每帧重复计算
public class CharacterStats
{
    private int _baseAttack = 100;
    private int _cachedFinalAttack;
    private bool _isDirty = true; // 初始需要计算
    
    private readonly List<StatModifier> _modifiers = new();
    
    public int FinalAttack
    {
        get
        {
            if (_isDirty)
            {
                RecalculateStats();
                _isDirty = false;
            }
            return _cachedFinalAttack;
        }
    }
    
    public void AddModifier(StatModifier mod)
    {
        _modifiers.Add(mod);
        _isDirty = true; // 标记需要重新计算
    }
    
    public void RemoveModifier(StatModifier mod)
    {
        if (_modifiers.Remove(mod))
            _isDirty = true;
    }
    
    private void RecalculateStats()
    {
        float multiplier = 1f;
        int flat = 0;
        
        foreach (var mod in _modifiers)
        {
            if (mod.Type == ModifierType.Multiplicative)
                multiplier *= 1f + mod.Value;
            else
                flat += (int)mod.Value;
        }
        
        _cachedFinalAttack = (int)((_baseAttack + flat) * multiplier);
    }
}
```

## 总结

设计模式的核心是**识别变化点，将变化点隔离**：

| 模式 | 解决什么问题 | 游戏典型应用 |
|------|------------|------------|
| 工厂 | 对象创建的复杂性 | 技能/怪物工厂 |
| 建造者 | 复杂对象的构造 | 关卡配置、角色构建 |
| 对象池 | 频繁创建销毁 | 子弹、特效、粒子 |
| 组合 | 树形结构的统一处理 | 技能效果、场景树 |
| 装饰器 | 动态添加功能 | 武器附魔、角色状态 |
| 外观 | 简化复杂接口 | 游戏流程控制 |
| 策略 | 算法的可替换性 | 移动方式、AI 决策 |
| 观察者 | 解耦事件通知 | 游戏事件系统 |
| 命令 | 封装操作为对象 | 撤销/重做、录像回放 |
| 职责链 | 多处理器的请求传递 | 输入处理链 |

> **记住**：模式是工具，不是目标。用最简单的方案解决问题才是工程智慧。
