# 关于C#

## 值类型和引用类型

C#具有通用类型系统。所有类型都派生自`System.Object`，然后值类型派生自`System.ValueType`, 当然`System.ValueType`也是派生自`System.Object`

值类型主要分为struct和enum，所有的内置数值类型都是struct；引用类型主要分为class，interface，delegate和数组

值类型的变量直接包含其数据，引用类型的变量存储对其数据（对象）的引用。

值类型存储在栈中，引用类型存储在堆中（指针存储于栈中）

对于静态类型的变量，无论是值类型还是引用类型，都全部存储于堆中。如果静态类型的变量是引用类型，那么除非将其设置为null，否则该变量指向的对象永远不会被回收。

**基本上栈用于存储临时的数据（局部值类型和引用），当一个方法执行完毕后立刻自动清除；而堆用于存储引用类型的对象和静态的对象，依靠GC回收**

## 堆和栈

在C#中堆表示堆内存，栈的话有两种，一种是与堆相对应的栈内存，另一种是与队列（Queue）相对应的数据结构

以泛型版本为例，Queue<T>队列是先进先出的数据类型；Stack<T>是先进后出的数据类型

内存中的堆区域和栈区域；

**值类型存储于栈中，存放函数的参数、局部变量、返回数据等值，由编译器自动释放。**

**引用类型存储于堆中（对应的指针存储于栈中），由CLR释放**

**栈内存无需我们管理，也不受GC管理。当栈顶元素使用完毕（栈空间内值变量出了作用域后），系统能自动回收的，而堆则需要GC清理。**

string是特殊的引用类型，目标对象不会变动，对目标的任何修改都是创建一个新的字符串实例。

**静态类型的变量，无论是值类型还是引用类型，都全部存储于堆中。**如果静态类型的变量是引用类型，那么除非将其设置为null，否则该变量指向的对象永远不会被回收

```csharp
static Object a=new Object(); 
//此时，NewObject1对象存储于堆上，而它的引用a也存储于堆上。
void Test3
{
  var b=new Object();// 此时，NewObject2对象存储于堆上,而它的引用b存储于栈上。
  a=b;//此时a的引用改为NewObject2，NewObject1再无引用，坐等回收。
}//此时栈被清空，b变量离开作用域失效。

//注意：一个进程只有一个堆，一个线程只有一个栈
```
