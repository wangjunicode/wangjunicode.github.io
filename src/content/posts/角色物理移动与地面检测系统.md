# 角色物理移动与地面检测系统

动作游戏的"手感"，很大程度上来自物理移动系统的质感。角色加速的手感、在斜面上滑落的效果、被打飞后的飞行弧线……这些都由底层物理系统精确控制。本文深入解析一个不依赖 Unity PhysX 的自实现角色物理系统，理解定点数物理与标准物理引擎的异同。

---

## 为什么不用 Unity CharacterController？

Unity 内置了 `CharacterController` 组件，可以处理角色移动和碰撞，为什么还要自己实现？

**原因一：帧同步不兼容**

`CharacterController` 内部使用浮点数，在不同 CPU/平台上可能有微小差异。帧同步战斗要求计算结果完全一致，必须用定点数（`FP`/`TSVector`）实现所有物理计算。

**原因二：控制粒度不足**

自实现的物理系统可以精确控制：
- 什么时候允许被外力击退
- 什么时候锁定高度（如跳跃最高点暂停）
- 特殊的斜面处理（如滚下斜面 vs 站立在斜面）

**原因三：独立于 Unity 版本**

自实现系统不受 Unity 版本更新的影响，行为更稳定。

## PhysicsComponent 核心更新流程

```csharp
public static void EnterFrame(this PhysicsComponent self, FP fixedDeltaTime)
{
    self.PreSimulationInterpolationUpdate(fixedDeltaTime);  // 阶段0：初始化
    self.UpdatePhase1(fixedDeltaTime);                      // 阶段1：地面检测
    self.UpdatePhase2(fixedDeltaTime);                      // 阶段2：速度处理与移动
    self.PostSimulationInterpolationUpdate(fixedDeltaTime); // 阶段3：后处理
}
```

四个阶段严格分离，每个阶段职责清晰：

### 阶段0：预处理

```csharp
public static void PreSimulationInterpolationUpdate(this PhysicsComponent self, FP deltaTime)
{
    // 记录帧初位置（用于帧末计算位移差）
    self.InitialTickPosition = self.TransientPosition;
    self.InitialTickRotation = self.TransientRotation;
    
    // 发布位置变化事件（通知渲染层记录初始位置，用于插值）
    EventSystem.Instance.Publish(self.DomainScene(), new ChangePosition()
    {
        Unit = self.GetParent<Unit>(),
        NewPos = self.TransientPosition
    });
}
```

注意：`TransientPosition` 是物理引擎内部的"临时位置"，在一帧内可能多次变化。只有在 `PostSimulationInterpolationUpdate` 中才最终写入 `Unit.Position`。

### 阶段1：地面检测与台阶处理

```csharp
public static void UpdatePhase1(this PhysicsComponent self, FP deltaTime)
{
    // 防止 NaN 污染（物理计算中的防御性编程）
    if (FP.IsNaN(self.SelfVelocity.x) || ...) 
        self.SelfVelocity = TSVector.zero;
    
    // 同步位置
    self.TransientPosition = unit.Position;
    self.TransientRotation = unit.Rotation;
    
    // 处理外部物理位移（如被击退后的强制移动）
    self.HandleMovePosition(deltaTime);
    
    // 地面检测初始化
    self.InitialGrounded();
    
    // 着地检测和台阶攀爬
    self.GroundedCheckAndSteping(deltaTime);
}
```

`GroundedCheckAndSteping` 执行地面检测（Raycast/Sweep），确定角色是否站在地面上、地面的法线方向、是否可以攀爬台阶。这个信息在速度计算中会用到（在地面上不受重力影响）。

### 阶段2：速度计算与位移

```csharp
public static void UpdatePhase2(this PhysicsComponent self, FP deltaTime)
{
    // 处理旋转
    self.UpdateRotation(ref transientRotation, deltaTime);
    
    // 处理速度变化（复杂的速度合成逻辑）
    var oldVelocity = self.WillSelfVelocity;
    UpdateVelocity(self, ref self.WillSelfVelocity, deltaTime);
    
    // 检测浮空顶点（用于特效触发：跳跃到最高点开始下落）
    if (oldVelocity.y != 0 && self.WillSelfVelocity.y != 0)
    {
        unit.GetComponent<EventDispatcherComponent>()
            .FireEvent(new Evt_FloatApex() { velocityY = TSMath.Abs(self.WillSelfVelocity.y) });
    }
    
    // 速度过小时清零（防止抖动）
    if (self.WillSelfVelocity.magnitude < PhysicsComponent.MinVelocityMagnitude)
    {
        self.WillSelfVelocity = TSVector.zero;
        if (oldVelocity != TSVector.zero)
        {
            self.NormalStop();  // 触发停止事件
        }
    }
    
    // 按速度更新位置
    self.SelfVelocity += self.WillSelfVelocity;
    self.HandleMovementByVelocity(deltaTime);
    
    // 防止角色掉出世界（y < -1000时重置高度）
    if (self.TransientPosition.y <= -1000)
    {
        self.TransientPosition = new TSVector(self.TransientPosition.x, 10, self.TransientPosition.z);
    }
}
```

`WillSelfVelocity` 和 `SelfVelocity` 分离：
- `WillSelfVelocity`：本帧想要的速度增量
- `SelfVelocity`：累计的历史速度（如击退后的惯性）

分离设计让速度来源更清晰：移动请求修改 `WillSelfVelocity`，物理结算累加到 `SelfVelocity`。

## 速度合成：多源速度叠加

```csharp
public static void UpdateVelocity(this PhysicsComponent self, ref TSVector currentVelocity, FP deltaTime)
{
    var move = self.Parent.GetComponent<SimpleMoveComponent>();
    var targetVelocity = move.TargetVelocity;  // 玩家期望的移动速度
    
    // 速度上限检查
    if (self.SpeedLimit > 0 && targetVelocity.magnitude > self.SpeedLimit)
    {
        targetVelocity = targetVelocity.normalized * self.SpeedLimit;
    }
    
    // 分离水平速度和垂直速度
    var currentPlane = TSVector.ProjectOnPlane(willCurrentVelocity, self.CharacterUp);
    var downVelocity = willCurrentVelocity - currentPlane;
    
    // 跑步状态：保证转弯不打滑（速度方向强制对齐前向）
    if (fsmComp.CurrentRootStateTag() == EActionType.Run)
    {
        currentPlane = self.CharacterForward * currentPlane.magnitude;
    }
    
    // 一次性冲量速度（如技能中的瞬间加速）
    var onceVelocity = move.OnceVelocity;
    move.OnceVelocity = self.CachedZeroVector;  // 用后清零
    
    // 地面移动 vs 空中移动逻辑
    if (self.GroundingStatus.IsStableOnGround)
    {
        // 地面：直接设置目标速度，无重力
        targetMovementVelocity = targetVelocity;
    }
    else
    {
        // 空中：保留水平惯性，添加重力
        targetMovementVelocity += gravity * deltaTime;
    }
    
    currentVelocity = targetMovementVelocity + onceVelocity;
}
```

关键技术点：

**① 速度分解**

```csharp
var currentPlane = TSVector.ProjectOnPlane(velocity, CharacterUp);
var downVelocity = velocity - currentPlane;
```

将速度分为水平分量和垂直分量，分别处理。水平分量受移动输入影响，垂直分量受重力和跳跃影响。

**② 跑步防打滑**

```csharp
if (EActionType.Run)
{
    currentPlane = self.CharacterForward * currentPlane.magnitude;
}
```

当角色跑步转弯时，如果不做这个处理，角色可能因为惯性而"打滑"——还在往旧方向漂移。强制将水平速度对齐到当前前向，实现即时转弯。

**③ 一次性冲量**

```csharp
var onceVelocity = move.OnceVelocity;
move.OnceVelocity = self.CachedZeroVector;  // 用完立刻清零
```

`OnceVelocity` 是专门用于"瞬间加速"的临时速度，如翻滚的瞬间加速、技能中的短距冲刺。使用后立刻清零，不留残余影响。

## 击退力的物理实现

受击系统中的击退力，通过物理组件施加：

```csharp
// HurtSystem 中
defender.GetComponent<PhysicsComponent>().Stop();  // 先清除现有运动

var velocity = TSMath.Abs(subskillParam.PowerXz) * hurtComp.AttackInfo.AttackDirection 
             + subskillParam.PowerY * TSVector.up;
simpleMoveComp.AddForceVelocity(velocity);  // 施加外力速度
```

```csharp
// SimpleMoveComponent 接收外力速度
public static void AddForceVelocity(this SimpleMoveComponent self, TSVector velocity)
{
    self.GetComponent<VelocityOnceComponent>().Add(velocity);
}
```

击退力最终通过 `VelocityOnceComponent` 传入物理计算流程：

```csharp
// UpdateVelocity 中的处理
if (!self.GroundingStatus.IsStableOnGround)
{
    // 空中时，外力速度直接参与合成
    currentVelocity += externalForce;
}
```

**击退的物理轨迹：**
- 水平分量：提供飞出距离
- 垂直分量（`PowerY`）：决定"飞起来的高度"
- 重力持续减小垂直速度，角色先上升后下落
- 地面摩擦力（`FrictionAcceleration`）逐渐减小水平速度，角色滑行一段后停止

## 地面摩擦力计算

```csharp
// SimpleMoveComponentSystem 中的摩擦力处理
public static void ApplyFriction(this SimpleMoveComponent self, FP deltaTime)
{
    if (self.AttachedVelocity.magnitude <= 0) return;
    
    // 摩擦力减速
    var friction = self.FrictionAcceleration * self.FrictionRate;
    var decrease = friction * deltaTime;
    
    if (self.AttachedVelocity.magnitude <= decrease)
    {
        self.AttachedVelocity = TSVector.zero;  // 完全停止
    }
    else
    {
        self.AttachedVelocity -= self.AttachedVelocity.normalized * decrease;
    }
}
```

`FrictionRate` 允许动态调整摩擦力：
- `FrictionRate > 1`：更大摩擦力，停得更快（重击效果）
- `FrictionRate < 1`：更小摩擦力，滑行更远（冰面效果）
- `FrictionRate = 0`：无摩擦，无限滑行（特殊技能效果）

## 跳跃系统

```csharp
// 跳跃触发
public static void Jump(this SimpleMoveComponent self)
{
    var gravity = self.Gravity;
    // 使用运动学公式：v² = 2ah，计算跳跃初速度
    // v = sqrt(2 * g * h)
    var jumpVelocity = TSMath.Sqrt(2 * TSMath.Abs(gravity) * CfgManager.tables.TbParam.JumpHeight);
    
    var velocityOnce = self.GetComponent<VelocityOnceComponent>();
    velocityOnce.AddOnce(new TSVector(0, jumpVelocity, 0));  // 向上的初速度
}
```

跳跃高度由物理公式决定，而不是硬编码初速度，保证了在不同重力设置下跳跃高度的一致性。

**浮空顶点检测**：

```csharp
if (oldVelocity.y != 0 && self.WillSelfVelocity.y != 0)
{
    unit.GetComponent<EventDispatcherComponent>().FireEvent(new Evt_FloatApex() { 
        velocityY = TSMath.Abs(self.WillSelfVelocity.y) 
    });
}
```

跳跃最高点（速度 y 从正变为0再变为负）时触发 `Evt_FloatApex` 事件，供技能系统感知（如"到达最高点时触发下落斩"）。

## 防止数值异常：NaN 保护

物理计算中，除以零或其他异常操作可能产生 NaN（Not a Number）。一旦位置变成 NaN，角色会"消失"：

```csharp
public static void UpdatePhase1(this PhysicsComponent self, FP deltaTime)
{
    // 速度 NaN 检查
    if (FP.IsNaN(self.SelfVelocity.x) || FP.IsNaN(self.SelfVelocity.y) || FP.IsNaN(self.SelfVelocity.z))
    {
        self.SelfVelocity = TSVector.zero;
    }
    
    if (FP.IsNaN(self.WillSelfVelocity.x) || ...) 
        self.WillSelfVelocity = TSVector.zero;
    
    if (FP.IsNaN(self.AttachedVelocity.x) || ...) 
        self.AttachedVelocity = TSVector.zero;
}
```

以及防止角色掉出世界：

```csharp
if (self.TransientPosition.y <= -1000)
{
    self.TransientPosition = new TSVector(self.TransientPosition.x, 10, self.TransientPosition.z);
}
```

这些防御性代码看起来简单，但在大型项目中，它们是保证游戏稳定性的重要防线。

## 定点数物理 vs 浮点数物理

使用 `FP`（Fixed-Point 定点数）而非 `float` 的核心原因：

| 特性 | float | FP (定点数) |
|------|-------|------------|
| 跨平台一致性 | ❌ 不同CPU结果可能不同 | ✅ 完全一致 |
| 精度 | 有效位数更多 | 固定精度，需要合理设计 |
| 性能 | 快（硬件加速） | 慢（软件模拟） |
| 帧同步支持 | ❌ | ✅ |

帧同步战斗必须使用定点数。性能损失可以通过减少物理对象数量来弥补。

## 总结

物理移动系统是战斗体验的物质基础：

| 组件 | 职责 |
|------|------|
| PhysicsComponent | 核心物理计算：速度、位置、碰撞 |
| SimpleMoveComponent | 上层移动接口：目标速度、摩擦力 |
| VelocityOnceComponent | 一次性冲量速度（击退、跳跃） |

更新阶段：
1. `PreSimulation`：记录初始状态
2. `UpdatePhase1`：地面检测
3. `UpdatePhase2`：速度计算与位置更新
4. `PostSimulation`：发布位置变化通知

理解了这套系统，你就能精确控制角色在每种情况下的移动行为——无论是流畅的跑步、有冲击感的被击飞，还是精准的斜面行走。
