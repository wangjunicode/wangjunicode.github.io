## 职业发展的真相

```
游戏行业的职业发展规律：

年资 ≠ 能力 ≠ 薪资 ≠ 职级

很多人工作5年，但成长等于1年
原因：
  - 重复相同类型的工作（没有走出舒适区）
  - 没有刻意练习（做完就扔，不复盘）
  - 信息茧房（只看自己领域的知识）
  - 等待机会（而不是创造机会）
```

## 一、各阶段能力要求

### 1.1 初级工程师（0~2年）

```
核心目标：快速建立基础能力，证明能力

技术要求：
  ✅ 熟练使用 Unity，掌握 MonoBehaviour 生命周期
  ✅ 能独立实现需求文档描述的功能
  ✅ 基础 C# 特性（泛型、委托、事件、异步）
  ✅ 能看懂和修改他人代码
  ✅ 基础性能概念（DrawCall、GC、帧率）
  ✅ Git 基本操作
  
软技能要求：
  ✅ 接到任务能按时完成
  ✅ 遇到问题先自己尝试解决（不立即问人）
  ✅ 沟通清晰，进展及时汇报
  
常见问题：
  ❌ 代码只追求"能跑"，不考虑维护性
  ❌ 遇到 Bug 就 Debug.Log，不学习正确的调试方法
  ❌ 只关注自己的模块，不了解整体架构
  
晋升关键：
  完成所有分配任务，主动要求更难的任务
  代码质量稳定提升，减少 Reviewer 的返工要求
```

### 1.2 中级工程师（2~4年）

```
核心目标：独立负责模块，影响周围人

技术要求：
  ✅ 独立设计和实现一个中等复杂度的系统（如技能系统、UI框架）
  ✅ 能做性能 Profile，找出瓶颈并优化
  ✅ 了解常用设计模式，能识别何时使用
  ✅ 对系统性能有直觉（"这样写会有 GC 问题"）
  ✅ 能做 Code Review 并给出有价值的建议
  
软技能要求：
  ✅ 能带一个经验更少的同学（讲解代码、分享知识）
  ✅ 需求分析能力（不只执行，能思考是否合理）
  ✅ 技术文档能力（能写清楚设计文档）
  
晋升关键：
  有一个"代表作"——一个你主导设计的系统，效果被团队认可
  影响不只是代码本身，还有知识分享、帮助他人提升
```

### 1.3 高级工程师（4~7年）

```
核心目标：跨模块架构设计，技术深度领先

技术要求：
  ✅ 能设计整个游戏的某个核心系统（战斗系统、资源系统）
  ✅ 在团队中是某个技术领域的"专家"（被人问到）
  ✅ 能预判技术决策的长期影响
  ✅ 能评估引入新技术的风险和收益
  ✅ 有构建工具的能力（而不只是使用工具）
  
软技能要求：
  ✅ 能推动技术决策（不只执行，能影响方向）
  ✅ 跨部门协作（能与策划/美术/后端高效协作）
  ✅ 有系统性思维（考虑问题从全局出发）
  
晋升关键：
  有超出职级预期的贡献（做了别人认为需要更资深的人才能做的事）
  被团队之外的人知道（技术影响力扩大）
```

### 1.4 技术专家/技术负责人（7年+）

```
核心目标：定义技术方向，建立可持续的技术体系

技术专家路线（Individual Contributor）：
  ✅ 在某个技术领域达到行业领先水平
  ✅ 解决别人解决不了的技术难题
  ✅ 技术影响力跨越公司边界（开源贡献、技术文章）
  ✅ 参与制定技术标准或 Best Practice

技术管理路线（Engineering Manager）：
  ✅ 团队规划（招聘、培养、激励）
  ✅ 技术方向把握（决策什么技术投入）
  ✅ 项目 & 风险管理
  ✅ 向上管理（与产品总监/CTO 沟通）
```

## 二、技术专家 vs 管理路线的选择

### 2.1 两条路的真实情况

```
技术专家路线：
  优势：
    专注技术深度，做自己喜欢的事
    在国内大厂有专属 TC 通道（腾讯 T3-2/T4，阿里 P9/P10）
    相对较少的"政治"和人际关系复杂度
    
  挑战：
    需要持续学习，否则被新人追上
    到一定级别后晋升更难（对影响力要求极高）
    收入天花板相对管理路线低（在大多数公司）

技术管理路线：
  优势：
    更大的决策权和影响力
    收入上限更高（特别是总监及以上）
    职业发展路径多样（技术总监→CTO/VP）
    
  挑战：
    需要接受"从做事到让别人做事"的心理转变
    管理能力需要刻意培养（不是自然获得的）
    技术深度可能因为分心而退步
    "政治"因素增多

选择建议：
  如果你享受解决技术难题，不喜欢协调人际关系 → 技术专家
  如果你享受带领团队，有影响他人的欲望 → 管理路线
  如果还不确定：先走几年再说（30岁前不需要急于决定）
```

## 三、在不同类型公司的成长策略

### 3.1 大公司 vs 小公司/创业公司

```
大公司（腾讯/网易/米哈游等）：
  优势：
    技术体系成熟，有导师带
    平台资源丰富（性能测试/安全/数据分析等）
    简历背书价值高
    同事普遍水平高（学习环境好）
    
  挑战：
    分工极细（你可能只负责一个很小的模块）
    晋升内卷（同水平竞争者多）
    决策链条长（想做的事不一定做得了）
    
  成长策略：
    利用平台资源（听内部分享、参与技术评审）
    主动扩大职责（做完分内事，再要更多）
    建立人脉（和其他部门优秀人才交流）

创业公司/小团队：
  优势：
    全栈式成长（做得广，从渲染到网络什么都碰）
    决策快，能看到自己的工作直接影响产品
    晋升相对容易（做出来就能上去）
    
  挑战：
    技术体系不成熟，需要自己摸索
    没有导师，容易走弯路
    项目可能死掉，平台价值不稳定
    
  成长策略：
    主动学习（在大公司都有人教的东西，这里要自学）
    建立自己的"作品集"（把做过的系统记录下来）
    保持与外部社区的连接（避免信息茧房）
```

## 四、晋升的底层逻辑

### 4.1 晋升的本质

```
很多人误解：晋升 = 做了足够多的年资
真实逻辑：晋升 = 你的价值被更高级别的人认可

影响晋升的因素：
  60%：实际产出和影响力（这个模块/系统的成功）
  30%：可见性（别人是否知道你做了什么）
  10%：沟通和自我展示能力

所以：
  只做好事，不让人知道 → 晋升慢
  只会说不做 → 很快被揭穿，晋升也慢
  做好 + 让对的人知道 → 晋升快
```

### 4.2 建立技术影响力

```
影响力建设方式（从小到大）：

团队内：
  - 每周/双周技术分享（主动分享自己学到的东西）
  - 帮助团队解决技术难题（成为求助对象）
  - 写好技术文档（降低知识壁垒，形成口碑）

公司内：
  - 跨团队的架构评审中发声
  - 参与内部开源项目或通用工具建设
  - 在内部 Wiki/博客发布深度技术文章

公司外：
  - 个人技术博客
  - GitHub 开源项目
  - 技术大会演讲
  - 游戏开发社区活跃（GDC、IndieGameDev 等）
```

## 五、持续学习方法

### 5.1 刻意练习框架

```
有效的学习 ≠ 看书看视频

刻意练习的关键要素：
  1. 设定具体目标（"理解 TBDR" vs "优化一个移动端场景使 DrawCall < 100"）
  2. 即时反馈（跑 Profiler 看结果，而不是只读理论）
  3. 走出舒适区（做有点超出现有能力的事）
  4. 专注（深度学习，不要同时学 10 样东西）

游戏开发的学习方式：
  ✅ 做 Side Project（做一个小游戏原型来实践新技术）
  ✅ 复现经典案例（看别人的分析，自己实现一遍）
  ✅ 分析优秀作品（拆解一个你喜欢的游戏的技术实现）
  ✅ 写博客/分享（输出是最好的学习方式）
```

### 5.2 知识积累的方法

```
每天：
  30分钟：阅读技术文章/博客（推荐 Mastodon 上的游戏开发者社区）
  
每周：
  1篇：技术总结（把本周学到的东西写成文档）
  
每月：
  回顾：这个月做了什么？哪些值得总结？哪些可以做得更好？
  
每季度：
  评估：职业目标是否在推进？哪些技能还需要加强？

推荐资源：
  书籍：
    《游戏引擎架构》（Jason Gregory）
    《3D数学基础：图形与游戏开发》
    《Clean Code》《Clean Architecture》
    
  网站：
    realtimerendering.com（图形渲染）
    gafferongames.com（网络游戏编程）
    gameprogrammingpatterns.com（游戏设计模式，免费在线）
    
  博客：
    Unity 官方博客（unity.com/blog）
    Catlike Coding（Unity Shader 教程）
    刷题：LeetCode（必刷题：面试热题 100）
```

## 总结

职业发展的核心原则：

```
1. 关注成长，而非职级
   职级是成长的结果，不是目标
   
2. 主动而非被动
   不要等待机会，要创造机会
   不要等别人说"你可以尝试"，要主动说"我想尝试"
   
3. 输出放大成长
   学了不写 = 没学
   写博客、做分享、写文档 = 最好的学习方式
   
4. 长期主义
   游戏行业是需要积累的行业
   3年的深度 > 10年的广度（如果10年都是重复劳动）
   
5. 选择平台
   和什么人一起工作，比做什么工作更重要
   身边都是A级的人，你会成长为A级
   身边都是C级的人，很难成为A级
```

> "职业发展不是爬梯子，而是建设一张网。技术能力是核心，但人际关系、影响力、沟通能力缺一不可。"
