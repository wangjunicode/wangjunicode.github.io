# 游戏资产管理：从制作到运行时加载

资产管理是游戏工程化的重要组成部分。不规范的资产会导致包体膨胀、内存超限、加载缓慢等一系列问题。本文系统讲解从资产制作规范到运行时管理的完整体系。

## 一、资产规范体系

```
命名规则：[前缀]_[功能描述]_[后缀]

纹理：
  T_PlayerHero_DiffuseMap         （漫反射贴图）
```

## 二、纹理资产优化

### 2.1 纹理压缩格式选择

```
移动端纹理压缩格式：

iOS（Metal/A-Series GPU）：
  普通纹理：ASTC 6x6（质量/压缩比平衡）
  高质量：ASTC 4x4
  法线贴图：ASTC 4x4

Android（Adreno/Mali/PowerVR）：
  通用：ETC2（OpenGL ES 3.0 原生支持）
  高质量：ASTC 6x6（Android 5.0+ 支持）
  法线贴图：ETC2（2通道）

PC/Console：
  DXT1（无透明）/ DXT5（有透明）
  BC7（最高质量）

压缩比对比（1024x1024 RGBA纹理）：
  未压缩：4MB
  ASTC 6x6：0.7MB（压缩比5.7:1）
  ETC2：2MB（压缩比2:1）
  DXT5：1MB（压缩比4:1）
```

### 2.2 Unity 纹理导入设置

```csharp
// Editor 脚本：批量优化纹理导入设置
using UnityEditor;

public class TextureImportOptimizer : AssetPostprocessor
{
    void OnPreprocessTexture()
    {
        var importer = (TextureImporter)assetImporter;
        string path = assetPath.ToLower();
        
        // UI 纹理
        if (path.Contains("/art/ui/"))
        {
            importer.textureType = TextureImporterType.Sprite;
            importer.mipmapEnabled = false; // UI 不需要 Mipmap
            importer.filterMode = FilterMode.Bilinear;
            
            var platformSettings = new TextureImporterPlatformSettings();
            platformSettings.name = "Android";
            platformSettings.overridden = true;
            platformSettings.format = TextureImporterFormat.ETC2_RGBA8;
            platformSettings.maxTextureSize = 1024;
            importer.SetPlatformTextureSettings(platformSettings);
        }
        
        // 法线贴图
        if (path.Contains("_normalmap") || path.Contains("_normal"))
        {
            importer.textureType = TextureImporterType.NormalMap;
            importer.mipmapEnabled = true;
        }
        
        // 环境纹理（开启 Mipmap，减少远处闪烁）
        if (path.Contains("/art/environment/"))
        {
            importer.mipmapEnabled = true;
            importer.streamingMipmaps = true; // 流式 Mipmap 节省内存
        }
    }
}
```

### 2.3 纹理图集（Atlas）

```
为什么需要图集：
  散图：100个小 Sprite = 100 个 DrawCall
  图集：100个小 Sprite 合并 = 1 个 DrawCall

图集策略：
├── UI 图集：按功能分组（主界面图集/背包图集/战斗HUD图集）
├── 特效图集：相似特效合并
└── 角色图集：同一角色的多张贴图（较少见）

Unity Sprite Atlas 最佳实践：
  - 图集大小不超过 2048x2048（4096在部分设备上有问题）
  - 设置 Padding=2，避免纹理出血
  - 多个图集：按使用频率分（常驻/按需加载）
```

## 三、模型资产优化

### 3.1 LOD（细节层次）设置

```
LOD 配置建议（角色）：

LOD0（0-15m）：最高精度
  三角面数：5000-15000
  骨骼数：50-80
  纹理：1024x1024

LOD1（15-40m）：中等精度
  三角面数：1000-3000
  骨骼数：20-30
  纹理：512x512

LOD2（40-100m）：低精度
  三角面数：200-500
  骨骼数：5-10
  纹理：256x256

LOD3（100m+）：最低精度或 Billboard
  三角面数：50-200
  纹理：128x128

LOD Culled（200m+）：不渲染
```

### 4.2 资产引用管理

```csharp
// 正确的 Addressables 使用方式
public class CharacterLoader : MonoBehaviour
{
    // 使用 AssetReference（Inspector 中配置，避免硬编码路径）
    [SerializeField] private AssetReferenceGameObject _characterPrefabRef;
    
    private AsyncOperationHandle<GameObject> _loadHandle;
    private GameObject _spawnedCharacter;
    
    public async Task LoadAndSpawnCharacter()
    {
        // 异步加载（不阻塞主线程）
        _loadHandle = _characterPrefabRef.LoadAssetAsync<GameObject>();
        
        await _loadHandle.Task;
        
        if (_loadHandle.Status == AsyncOperationStatus.Succeeded)
        {
            _spawnedCharacter = Instantiate(_loadHandle.Result);
        }
        else
        {
            Debug.LogError($"角色加载失败: {_loadHandle.OperationException}");
        }
    }
    
    // 关键：必须在不需要时释放！
    private void OnDestroy()
    {
        if (_spawnedCharacter != null)
            Destroy(_spawnedCharacter);
        
        // 释放资产引用（减少内存占用）
        if (_loadHandle.IsValid())
            Addressables.Release(_loadHandle);
    }
}

// 批量预加载（切换场景前）
public class ScenePreloader
{
    public async Task PreloadSceneAssets(string sceneLabelOrKey)
    {
        // 获取该 Label 下所有资产的下载大小
        var sizeHandle = Addressables.GetDownloadSizeAsync(sceneLabelOrKey);
        await sizeHandle.Task;
        
        long downloadSize = sizeHandle.Result;
        Addressables.Release(sizeHandle);
        
        if (downloadSize > 0)
        {
            // 显示下载进度
            var downloadHandle = Addressables.DownloadDependenciesAsync(sceneLabelOrKey);
            
            while (!downloadHandle.IsDone)
            {
                float progress = downloadHandle.PercentComplete;
                OnDownloadProgress?.Invoke(progress, downloadSize);
                await Task.Yield();
            }
            
            Addressables.Release(downloadHandle);
        }
    }
}
```

## 五、运行时资产生命周期管理

```csharp
// 资产引用计数管理器（防止重复加载/过早释放）
public class AssetReferenceCounter
{
    private Dictionary<string, (AsyncOperationHandle handle, int count)> _references = new();
    
    public async Task<T> Acquire<T>(string address)
    {
        if (_references.TryGetValue(address, out var existing))
        {
            // 增加引用计数
            _references[address] = (existing.handle, existing.count + 1);
            return (T)existing.handle.Result;
        }
        
        // 首次加载
        var handle = Addressables.LoadAssetAsync<T>(address);
        await handle.Task;
        
        if (handle.Status == AsyncOperationStatus.Succeeded)
        {
            _references[address] = (handle, 1);
            return handle.Result;
        }
        
        Addressables.Release(handle);
        return default;
    }
    
    public void Release(string address)
    {
        if (!_references.TryGetValue(address, out var existing)) return;
        
        int newCount = existing.count - 1;
        
        if (newCount <= 0)
        {
            // 引用计数归零：真正释放资产
            Addressables.Release(existing.handle);
            _references.Remove(address);
        }
        else
        {
            _references[address] = (existing.handle, newCount);
        }
    }
}
```

## 六、包体大小控制

```
包体大小优化清单：

纹理（通常占60-70%）：
  ✅ 启用纹理压缩
  ✅ 按平台设置合适的最大尺寸
  ✅ 移除未使用的纹理
  ✅ 使用 Mipmap Streaming

音频（通常占10-20%）：
  ✅ 背景音乐用 Vorbis（OGG）压缩
  ✅ 音效用合适的采样率（16kHz 够用）
  ✅ 启用 Load in Background（流式加载）

代码（通常占5-10%）：
  ✅ IL2CPP + 代码裁剪（Strip Engine Code）
  ✅ 移除未使用的 Package

分析工具：
  Unity Build Report（Build → Open Last Build Report）
  Asset Bundle Browser（可视化 AB 内容）
```

> 💡 **工程师视角**：资产管理是"无聊但至关重要"的工作。包体超限、内存超限往往不是某一个问题造成的，而是数百个细节疏忽的积累。建议每个 Sprint 都跑一次"资产审计"：检查新增资产是否符合规范、是否有未引用的冗余资产、是否有超规格的纹理。预防胜于治疗。
