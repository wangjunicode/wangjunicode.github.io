## 资源加载概述

Unity 中的资源加载涉及四个核心问题：
- **使用形式**：资源如何被游戏对象引用
- **加载方式**：如何将资源从磁盘读入内存
- **缓存机制**：如何避免重复加载同一资源
- **打包策略**：如何将资源打包以支持热更新

## 常用资源路径

| 路径 | 说明 | 是否可写 |
|------|------|---------|
| `Application.dataPath` | 对应 `Assets` 文件夹，编辑器下可用 | 否 |
| `Application.streamingAssetsPath` | `StreamingAssets` 文件夹，打包后可直接访问 | 否 |
| `Application.persistentDataPath` | 持久化路径，可读写，热更新文件存放于此 | ✅ 是 |
| `Application.temporaryCachePath` | 临时缓存路径 | ✅ 是 |

## 四种加载方式对比

### 1. AssetDatabase（仅编辑器）

```csharp
#if UNITY_EDITOR
using UnityEditor;

GameObject prefab = AssetDatabase.LoadAssetAtPath<GameObject>("Assets/Prefabs/Player.prefab");
Instantiate(prefab);
#endif
```

**适用场景**：编辑器工具、自动化脚本  
**优点**：路径直观，支持所有资源类型  
**缺点**：仅在编辑器下可用，不能用于运行时

---

### 2. Resources（不推荐用于新项目）

```csharp
// 同步加载（阻塞主线程）
GameObject prefab = Resources.Load<GameObject>("Prefabs/Player");
Instantiate(prefab);

// 异步加载（不阻塞主线程）
IEnumerator LoadAsync()
{
    ResourceRequest request = Resources.LoadAsync<GameObject>("Prefabs/Player");
    yield return request;
    Instantiate(request.asset);
}

// 卸载
Resources.UnloadAsset(loadedAsset);
Resources.UnloadUnusedAssets();
```

**路径规则**：路径相对于 `Resources` 文件夹，不含扩展名  
**优点**：使用简单，无需额外配置  
**缺点**：
- 所有 Resources 资源都会打入包体，无法增量更新
- 内存中保留一个大的资源清单
- Unity 官方**已不推荐**在新项目中使用

---

### 3. AssetBundle（传统热更方案）

```csharp
// 从本地加载 AB 包
AssetBundle bundle = AssetBundle.LoadFromFile(Application.streamingAssetsPath + "/characters");

// 从 AB 包中加载资源
GameObject prefab = bundle.LoadAsset<GameObject>("Player");
Instantiate(prefab);

// 异步加载
IEnumerator LoadBundleAsync(string bundlePath, string assetName)
{
    var request = AssetBundle.LoadFromFileAsync(bundlePath);
    yield return request;
    
    var bundle = request.assetBundle;
    var assetRequest = bundle.LoadAssetAsync<GameObject>(assetName);
    yield return assetRequest;
    
    Instantiate(assetRequest.asset);
}

// 卸载（true=同时卸载从该Bundle加载的实例，慎用！）
bundle.Unload(false);
```

**常见问题**：
- ⚠️ **依赖缺失**：加载 AB 时如果没有先加载依赖的 AB，资源会显示错误（粉色）
- ⚠️ **内存泄漏**：Unload 时机不当，导致重复加载或资源无法释放
- 需要自己管理依赖关系（通过 `AssetBundleManifest`）

---

### 4. Addressables（推荐方案）

Unity 2018+ 推出的资源管理系统，自动处理依赖和内存：

```csharp
using UnityEngine.AddressableAssets;
using UnityEngine.ResourceManagement.AsyncOperations;

// 通过地址异步加载
async void LoadByAddress()
{
    var handle = Addressables.LoadAssetAsync<GameObject>("Characters/Player");
    await handle.Task;
    
    if (handle.Status == AsyncOperationStatus.Succeeded)
    {
        Instantiate(handle.Result);
    }
    
    // 释放句柄（减少引用计数）
    Addressables.Release(handle);
}

// 通过引用加载（Inspector 拖拽赋值）
[SerializeField] private AssetReference playerRef;

async void LoadByRef()
{
    var handle = playerRef.LoadAssetAsync<GameObject>();
    await handle.Task;
    Instantiate(handle.Result);
}
```

**核心优势**：
- ✅ **自动管理依赖**：加载一个资源会自动加载其依赖
- ✅ **引用计数**：通过 `Addressables.Release` 减少引用计数，自动卸载
- ✅ **支持热更新**：资源可以放在远端，运行时按需下载
- ✅ **地址解耦**：资源路径变化不影响业务代码

## 加载方式选择建议

```
新项目？
    是 → 使用 Addressables（官方推荐）
    
需要热更新？
    是 → Addressables 或 AssetBundle（需自封装）
    否 → Resources（小项目）或 Addressables
    
编辑器工具？
    → AssetDatabase
```

## 资源缓存策略

```csharp
// 简单的资源缓存管理器示例
public class ResourceCache : MonoBehaviour
{
    private static readonly Dictionary<string, Object> _cache = new();
    
    public static T Load<T>(string path) where T : Object
    {
        if (_cache.TryGetValue(path, out var cached))
            return cached as T;
        
        var asset = Resources.Load<T>(path);
        if (asset != null)
            _cache[path] = asset;
        
        return asset;
    }
    
    public static void Clear()
    {
        _cache.Clear();
        Resources.UnloadUnusedAssets();
    }
}
```
