# 游戏自适应画质系统设计：帧预算驱动的动态质量调节完全指南

## 一、为什么需要自适应画质？

移动设备的硬件碎片化是游戏开发面临的核心挑战之一。同一款游戏需要在骁龙 8 Gen3 的旗舰机和骁龙 680 的入门机上都能流畅运行，静态的画质预设（低/中/高）已无法满足需求：

- **旗舰机跑低画质**：浪费 GPU 算力，画面效果平庸
- **中端机强推高画质**：帧率掉到 20 FPS，体验极差
- **温控触发降频**：长时间游戏后设备过热降频，之前设置的画质档位失效

自适应画质系统（Adaptive Quality System，AQS）通过**实时监测帧预算消耗**，动态调节画质参数，让设备始终在最佳性能与最佳视觉质量之间找到平衡点。

## 三、帧预算监控器

帧预算（Frame Budget）是整个系统的数据来源。目标帧率决定了每帧允许消耗的时间上限。

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 帧预算监控器 - 使用滑动窗口统计帧时间
/// </summary>
public class FrameBudgetMonitor
{
    // 滑动窗口大小（帧数）
    private const int WindowSize = 60;

    private readonly Queue<float> _frameTimes = new Queue<float>(WindowSize);
    private float _frameTimeSum;

    // 目标帧时间（毫秒）
    public float TargetFrameMs { get; private set; }

    // 当前平均帧时间（毫秒）
    public float AverageFrameMs { get; private set; }

    // 帧预算利用率 [0, ∞)，1.0 表示刚好满载，>1.0 表示超载
    public float BudgetUtilization => TargetFrameMs > 0
        ? AverageFrameMs / TargetFrameMs
        : 1f;

    // 最近 N 帧中超出预算的帧比例（卡顿率）
    public float StutterRatio { get; private set; }

    private int _stutterCount;

    public FrameBudgetMonitor(int targetFps)
    {
        SetTargetFps(targetFps);
    }

    public void SetTargetFps(int fps)
    {
        TargetFrameMs = 1000f / fps;
    }

    /// <summary>
    /// 每帧调用，传入 Time.unscaledDeltaTime * 1000
    /// </summary>
    public void Tick(float deltaMs)
    {
        if (_frameTimes.Count >= WindowSize)
        {
            float oldest = _frameTimes.Dequeue();
            _frameTimeSum -= oldest;
            if (oldest > TargetFrameMs * 1.5f)
                _stutterCount--;
        }

        _frameTimes.Enqueue(deltaMs);
        _frameTimeSum += deltaMs;

        if (deltaMs > TargetFrameMs * 1.5f)
            _stutterCount++;

        AverageFrameMs = _frameTimeSum / _frameTimes.Count;
        StutterRatio = (float)_stutterCount / _frameTimes.Count;
    }

    public bool IsOverBudget(float threshold = 1.1f)
        => BudgetUtilization > threshold;

    public bool IsUnderBudget(float threshold = 0.75f)
        => BudgetUtilization < threshold;
}
```

## 四、质量等级状态机

系统定义若干**质量等级（QualityTier）**，每个等级对应一组画质参数的偏移量。状态机负责决定当前应处于哪个等级，并控制升级/降级的**冷却时间**，避免频繁抖动。

```csharp
/// <summary>
/// 自适应画质等级（值越大画质越高）
/// </summary>
public enum QualityTier
{
    Minimum = 0,
    Low = 1,
    Medium = 2,
    High = 3,
    Ultra = 4,
}

/// <summary>
/// 质量等级状态机
/// </summary>
public class QualityStateMachine
{
    public QualityTier Current { get; private set; } = QualityTier.Medium;

    // 升级冷却（秒）：防止在升级后立即又被判定为过载
    private float _upgradeCooldown = 5f;
    // 降级冷却（秒）：防止连续降级导致画质断崖
    private float _downgradeCooldown = 2f;

    private float _lastChangeTime = float.MinValue;
    private bool _pendingUpgrade;
    private bool _pendingDowngrade;

    // 降级需要连续 N 帧超载
    private const int DowngradeTriggerFrames = 10;
    private int _overBudgetFrames;

    public event Action<QualityTier, QualityTier> OnTierChanged;

    public void Update(FrameBudgetMonitor monitor)
    {
        float now = Time.realtimeSinceStartup;

        if (monitor.IsOverBudget(1.15f) || monitor.StutterRatio > 0.2f)
        {
            _overBudgetFrames++;
            _pendingUpgrade = false;
        }
        else
        {
            _overBudgetFrames = 0;
        }

        // 判断降级
        if (_overBudgetFrames >= DowngradeTriggerFrames)
        {
            if (now - _lastChangeTime >= _downgradeCooldown)
            {
                TryDecreaseTier(now);
                _overBudgetFrames = 0;
            }
        }
        // 判断升级（帧率充裕且稳定）
        else if (monitor.IsUnderBudget(0.70f) && monitor.StutterRatio < 0.05f)
        {
            if (now - _lastChangeTime >= _upgradeCooldown)
            {
                TryIncreaseTier(now);
            }
        }
    }

    private void TryIncreaseTier(float now)
    {
        if (Current < QualityTier.Ultra)
        {
            var prev = Current;
            Current++;
            _lastChangeTime = now;
            OnTierChanged?.Invoke(prev, Current);
        }
    }

    private void TryDecreaseTier(float now)
    {
        if (Current > QualityTier.Minimum)
        {
            var prev = Current;
            Current--;
            _lastChangeTime = now;
            OnTierChanged?.Invoke(prev, Current);
        }
    }

    /// <summary>
    /// 外部强制设置等级（如初始化时根据设备评分）
    /// </summary>
    public void ForceSetTier(QualityTier tier)
    {
        Current = tier;
        _lastChangeTime = Time.realtimeSinceStartup;
    }
}
```

## 五、动态分辨率缩放器

分辨率是性能影响最显著的画质参数，Unity URP 内置了 `DynamicResolutionHandler`，但我们需要更精细的控制：

```csharp
using UnityEngine;
using UnityEngine.Rendering;
using UnityEngine.Rendering.Universal;

/// <summary>
/// URP 动态分辨率控制器
/// 注意：需要在 URP Asset 中开启 Upscaling Filter
/// </summary>
public class DynamicResolutionController : MonoBehaviour
{
    [Header("分辨率范围")]
    [Range(0.3f, 1.0f)] public float minScale = 0.5f;
    [Range(0.5f, 1.0f)] public float maxScale = 1.0f;

    // 每个质量等级对应的分辨率缩放
    private static readonly float[] TierScales =
    {
        0.50f, // Minimum
        0.65f, // Low
        0.75f, // Medium
        0.85f, // High
        1.00f, // Ultra
    };

    private float _currentScale = 1f;
    private float _targetScale = 1f;
    // 平滑插值速度（越小越慢，避免画面跳变）
    private const float SmoothSpeed = 2f;

    private UniversalRenderPipelineAsset _urpAsset;

    private void Awake()
    {
        _urpAsset = GraphicsSettings.currentRenderPipeline as UniversalRenderPipelineAsset;
    }

    public void SetTargetScale(QualityTier tier)
    {
        int idx = (int)tier;
        _targetScale = Mathf.Clamp(TierScales[idx], minScale, maxScale);
    }

    private void Update()
    {
        if (Mathf.Approximately(_currentScale, _targetScale)) return;

        _currentScale = Mathf.Lerp(_currentScale, _targetScale,
            Time.unscaledDeltaTime * SmoothSpeed);

        ApplyScale(_currentScale);
    }

    private void ApplyScale(float scale)
    {
        // URP 动态分辨率（需要 URP 12+）
        ScalableBufferManager.ResizeBuffers(scale, scale);

#if UNITY_2022_2_OR_NEWER
        if (_urpAsset != null)
        {
            _urpAsset.renderScale = scale;
        }
#endif
    }
}
```

## 六、LOD 偏置动态调节

LOD Bias 控制模型切换 LOD 级别的距离阈值，值越小切换越激进（近处就切低模），性能越好但视觉质量下降：

```csharp
/// <summary>
/// LOD Bias 动态调节器
/// QualitySettings.lodBias 影响全局 LOD 切换距离
/// </summary>
public class LodBiasController
{
    // 每个质量等级对应的 LOD Bias
    // 值越大 = 高LOD显示越远 = 质量更好但性能更差
    private static readonly float[] TierBias =
    {
        0.4f, // Minimum - 激进剔除
        0.6f, // Low
        0.8f, // Medium
        1.0f, // High - 默认值
        1.5f, // Ultra - 高质量
    };

    private float _currentBias;
    private float _targetBias;

    public void SetTier(QualityTier tier)
    {
        _targetBias = TierBias[(int)tier];
    }

    public void Update()
    {
        if (Mathf.Approximately(_currentBias, _targetBias)) return;

        _currentBias = Mathf.Lerp(_currentBias, _targetBias,
            Time.unscaledDeltaTime * 1.5f);

        QualitySettings.lodBias = _currentBias;
    }

    public void ForceApply(QualityTier tier)
    {
        _currentBias = _targetBias = TierBias[(int)tier];
        QualitySettings.lodBias = _currentBias;
    }
}
```

## 七、后处理降级策略

后处理效果（Bloom、DOF、SSAO 等）是 GPU 消耗大户，在性能压力下应按优先级逐步关闭：

```csharp
using UnityEngine.Rendering;
using UnityEngine.Rendering.Universal;

/// <summary>
/// 后处理效果降级控制器
/// 按性能消耗从高到低逐步关闭效果
/// </summary>
public class PostProcessDowngrader
{
    private Volume _globalVolume;

    // 各效果在不同质量等级的开关状态
    // [effect][tier] = enabled
    private static readonly bool[,] EffectMatrix =
    {
        //               Min    Low    Med    High   Ultra
        /* Bloom    */ { false, true,  true,  true,  true  },
        /* DepthOfF */ { false, false, false, true,  true  },
        /* SSAO     */ { false, false, true,  true,  true  },
        /* ChromAbe */ { false, false, false, false, true  },
        /* Vignette */ { true,  true,  true,  true,  true  },
        /* ColorGrd */ { true,  true,  true,  true,  true  },
        /* MotionBl */ { false, false, false, true,  true  },
    };

    private Bloom _bloom;
    private DepthOfField _dof;
    private AmbientOcclusion _ssao;
    private ChromaticAberration _chromAb;
    private Vignette _vignette;
    private ColorGrading _colorGrading;
    private MotionBlur _motionBlur;

    public void Init(Volume volume)
    {
        _globalVolume = volume;
        var profile = volume.sharedProfile;
        profile.TryGet(out _bloom);
        profile.TryGet(out _dof);
        profile.TryGet(out _ssao);
        profile.TryGet(out _chromAb);
        profile.TryGet(out _vignette);
        profile.TryGet(out _colorGrading);
        profile.TryGet(out _motionBlur);
    }

    public void ApplyTier(QualityTier tier)
    {
        int t = (int)tier;
        SetActive(_bloom,       EffectMatrix[0, t]);
        SetActive(_dof,         EffectMatrix[1, t]);
        SetActive(_ssao,        EffectMatrix[2, t]);
        SetActive(_chromAb,     EffectMatrix[3, t]);
        SetActive(_vignette,    EffectMatrix[4, t]);
        SetActive(_colorGrading,EffectMatrix[5, t]);
        SetActive(_motionBlur,  EffectMatrix[6, t]);
    }

    private void SetActive(VolumeComponent comp, bool active)
    {
        if (comp != null) comp.active = active;
    }
}
```

## 九、热度感知节流（Thermal Throttle）

移动设备长时间运行后 CPU/GPU 会过热触发硬件降频，此时应主动降低画质以维持帧率稳定：

```csharp
#if UNITY_ANDROID || UNITY_IOS
using UnityEngine.Device;
#endif

/// <summary>
/// 设备热量感知节流控制器
/// 监测 Unity 的热量状态，在高温时主动降级画质
/// </summary>
public class ThermalThrottleController
{
    // 热量降级后，冷却多少秒才考虑恢复
    private const float CooldownDuration = 30f;

    private float _thermalDowngradedAt = float.MinValue;
    private bool _isThermalLimited;

    public bool IsThermalLimited => _isThermalLimited;

    // 调用方：每秒检查一次即可
    public bool CheckThermal(out int recommendedTierDrop)
    {
        recommendedTierDrop = 0;

#if UNITY_2021_2_OR_NEWER && (UNITY_ANDROID || UNITY_IOS)
        var status = UnityEngine.Device.SystemInfo.thermalState;

        switch (status)
        {
            case ThermalState.Nominal:
            case ThermalState.Fair:
                // 温度正常
                if (_isThermalLimited &&
                    Time.realtimeSinceStartup - _thermalDowngradedAt > CooldownDuration)
                {
                    _isThermalLimited = false;
                    recommendedTierDrop = 0; // 可以恢复
                    return true;
                }
                return false;

            case ThermalState.Serious:
                _isThermalLimited = true;
                _thermalDowngradedAt = Time.realtimeSinceStartup;
                recommendedTierDrop = 1; // 降一档
                return true;

            case ThermalState.Critical:
                _isThermalLimited = true;
                _thermalDowngradedAt = Time.realtimeSinceStartup;
                recommendedTierDrop = 2; // 降两档
                return true;
        }
#endif
        return false;
    }
}
```

## 十、设备性能评估与初始等级

游戏启动时，需要根据设备硬件自动选择初始质量等级：

```csharp
/// <summary>
/// 设备性能评估器 - 综合多维度指标打分
/// </summary>
public static class DevicePerformanceEvaluator
{
    public static QualityTier EvaluateInitialTier()
    {
        int score = 0;

        // 1. GPU 评估
        string gpuName = SystemInfo.graphicsDeviceName.ToLower();
        score += EvaluateGpu(gpuName);

        // 2. CPU 核心数
        score += SystemInfo.processorCount >= 8 ? 3 :
                 SystemInfo.processorCount >= 6 ? 2 :
                 SystemInfo.processorCount >= 4 ? 1 : 0;

        // 3. 系统内存
        score += SystemInfo.systemMemorySize >= 8192 ? 3 :
                 SystemInfo.systemMemorySize >= 4096 ? 2 :
                 SystemInfo.systemMemorySize >= 2048 ? 1 : 0;

        // 4. 显存（图形内存）
        score += SystemInfo.graphicsMemorySize >= 4096 ? 3 :
                 SystemInfo.graphicsMemorySize >= 2048 ? 2 :
                 SystemInfo.graphicsMemorySize >= 1024 ? 1 : 0;

        // 5. 支持特性加分
        if (SystemInfo.supportsComputeShaders) score += 1;
        if (SystemInfo.graphicsShaderLevel >= 50) score += 1;

        // 综合评分映射到质量等级
        return score switch
        {
            >= 12 => QualityTier.Ultra,
            >= 9  => QualityTier.High,
            >= 6  => QualityTier.Medium,
            >= 3  => QualityTier.Low,
            _     => QualityTier.Minimum,
        };
    }

    private static int EvaluateGpu(string gpu)
    {
        // 高端 GPU
        if (gpu.Contains("adreno 7") || gpu.Contains("adreno 8") ||
            gpu.Contains("mali-g7") || gpu.Contains("apple gpu") ||
            gpu.Contains("geforce rtx"))
            return 5;

        // 中高端
        if (gpu.Contains("adreno 6") || gpu.Contains("mali-g5") ||
            gpu.Contains("geforce gtx"))
            return 3;

        // 中端
        if (gpu.Contains("adreno 5") || gpu.Contains("mali-g5"))
            return 2;

        return 1;
    }
}
```

## 十一、总装：AdaptiveQualityManager

将所有子系统整合到统一的 Manager：

```csharp
using UnityEngine;
using UnityEngine.Rendering;

/// <summary>
/// 自适应画质系统总管理器
/// 挂载到场景常驻 GameObject
/// </summary>
public class AdaptiveQualityManager : MonoBehaviour
{
    [Header("目标帧率")]
    public int targetFps = 60;

    [Header("全局后处理 Volume")]
    public Volume globalVolume;

    [Header("调试模式")]
    public bool debugMode = false;

    private FrameBudgetMonitor _budgetMonitor;
    private QualityStateMachine _stateMachine;
    private DynamicResolutionController _resController;
    private LodBiasController _lodController;
    private PostProcessDowngrader _ppDowngrader;
    private ShadowQualityController _shadowController;
    private ThermalThrottleController _thermalController;

    // 热量检查间隔（秒）
    private float _thermalCheckInterval = 3f;
    private float _lastThermalCheck;

    private void Awake()
    {
        Application.targetFrameRate = targetFps;

        _budgetMonitor = new FrameBudgetMonitor(targetFps);
        _stateMachine = new QualityStateMachine();
        _resController = GetComponent<DynamicResolutionController>();
        _lodController = new LodBiasController();
        _ppDowngrader = new PostProcessDowngrader();
        _shadowController = new ShadowQualityController();
        _thermalController = new ThermalThrottleController();

        _shadowController.Init();
        if (globalVolume != null)
            _ppDowngrader.Init(globalVolume);

        // 根据设备评分设定初始质量等级
        var initialTier = DevicePerformanceEvaluator.EvaluateInitialTier();
        _stateMachine.ForceSetTier(initialTier);
        ApplyTier(initialTier);

        _stateMachine.OnTierChanged += (prev, next) =>
        {
            if (debugMode)
                Debug.Log($"[AQS] 质量等级变化: {prev} → {next}");
            ApplyTier(next);
        };
    }

    private void Update()
    {
        _budgetMonitor.Tick(Time.unscaledDeltaTime * 1000f);
        _stateMachine.Update(_budgetMonitor);
        _lodController.Update();

        // 热量检查
        if (Time.realtimeSinceStartup - _lastThermalCheck >= _thermalCheckInterval)
        {
            _lastThermalCheck = Time.realtimeSinceStartup;
            CheckThermal();
        }

        if (debugMode) DrawDebugInfo();
    }

    private void ApplyTier(QualityTier tier)
    {
        _resController?.SetTargetScale(tier);
        _lodController.SetTier(tier);
        _ppDowngrader.ApplyTier(tier);
        _shadowController.ApplyTier(tier);
    }

    private void CheckThermal()
    {
        if (_thermalController.CheckThermal(out int drop) && drop > 0)
        {
            var target = (QualityTier)Mathf.Max(0, (int)_stateMachine.Current - drop);
            _stateMachine.ForceSetTier(target);
            ApplyTier(target);
            Debug.LogWarning($"[AQS] 设备过热，强制降级至 {target}");
        }
    }

    private void DrawDebugInfo()
    {
        Debug.Log($"[AQS] 当前等级={_stateMachine.Current} " +
                  $"帧时={_budgetMonitor.AverageFrameMs:F1}ms " +
                  $"预算利用={_budgetMonitor.BudgetUtilization:P0} " +
                  $"卡顿率={_budgetMonitor.StutterRatio:P0}");
    }
}
```

## 十二、粒子系统动态限制

粒子效果是移动端另一个重要的性能开销来源：

```csharp
/// <summary>
/// 粒子系统数量与质量限制器
/// 通过挂载到粒子 GameObject 或统一注册管理
/// </summary>
public class ParticleQualityLimiter : MonoBehaviour
{
    [Header("各等级最大粒子数倍率")]
    private static readonly float[] MaxParticleRatios =
    {
        0.15f, // Minimum
        0.30f, // Low
        0.50f, // Medium
        0.75f, // High
        1.00f, // Ultra
    };

    private ParticleSystem[] _particles;
    private int[] _originalMaxParticles;

    private void Awake()
    {
        _particles = GetComponentsInChildren<ParticleSystem>(true);
        _originalMaxParticles = new int[_particles.Length];
        for (int i = 0; i < _particles.Length; i++)
            _originalMaxParticles[i] = _particles[i].main.maxParticles;
    }

    public void ApplyTier(QualityTier tier)
    {
        float ratio = MaxParticleRatios[(int)tier];
        for (int i = 0; i < _particles.Length; i++)
        {
            var main = _particles[i].main;
            main.maxParticles = Mathf.Max(1, (int)(_originalMaxParticles[i] * ratio));
        }
    }
}
```

## 十三、性能基准测试

在初次启动时进行简短基准测试，可以获得更准确的初始等级：

```csharp
using System.Collections;
using UnityEngine;

/// <summary>
/// 启动时基准测试（可选）
/// 渲染一组测试场景，测量帧时间
/// </summary>
public class StartupBenchmark : MonoBehaviour
{
    [Tooltip("测试持续时间（秒）")]
    public float benchDuration = 3f;

    public System.Action<QualityTier> OnBenchmarkComplete;

    private float _totalFrameMs;
    private int _frameCount;
    private float _elapsed;
    private bool _running;

    public IEnumerator Run()
    {
        _running = true;
        _elapsed = 0;
        _totalFrameMs = 0;
        _frameCount = 0;

        // 先稳定几帧
        yield return new WaitForSeconds(0.5f);

        while (_elapsed < benchDuration)
        {
            float ms = Time.unscaledDeltaTime * 1000f;
            _totalFrameMs += ms;
            _frameCount++;
            _elapsed += Time.unscaledDeltaTime;
            yield return null;
        }

        float avgMs = _totalFrameMs / _frameCount;
        var tier = avgMs switch
        {
            <= 10f  => QualityTier.Ultra,    // 100+ FPS
            <= 14f  => QualityTier.High,     // 70+ FPS
            <= 20f  => QualityTier.Medium,   // 50+ FPS
            <= 30f  => QualityTier.Low,      // 33+ FPS
            _       => QualityTier.Minimum,
        };

        OnBenchmarkComplete?.Invoke(tier);
    }
}
```

## 十四、最佳实践总结

### 1. 冷却时间设计
- **降级冷却 < 升级冷却**：降级是为了救帧率，要快；升级是锦上添花，可以慢（避免连续升降造成画面闪烁）
- 建议降级冷却 1~2s，升级冷却 5~8s

### 2. 参数调节优先级
按性能影响从大到小：
1. 分辨率缩放（影响最大，视觉质量下降明显）
2. 阴影距离/级联数（GPU 和 CPU 双重影响）
3. SSAO（逐像素计算，开销大）
4. LOD Bias（影响批次和三角形数量）
5. Bloom / DOF（后处理全屏 pass）
6. 粒子数量（Draw Call + 填充率）

### 3. 不要调节所有参数
找出游戏中**性能消耗最大的3~5个参数**重点调节即可，调节过多会让调试变复杂，且用户感知不到微小的画质变化。

### 4. 给玩家保留手动控制
自适应画质是兜底机制，要允许玩家手动锁定画质等级，并在设置界面显示当前等级。

### 5. 移动端热量管理
- 超过 40 分钟持续游戏，任何设备都会开始降频
- 在长局游戏中（如 MOBA），建议在赛前准备阶段主动降低画质以储蓄热量余量
- 合理使用 `Application.targetFrameRate`，30 FPS 的功耗远低于 60 FPS

### 6. 数据监控
在线上采集帧率分布数据，按设备型号分组分析，用于离线调优各设备的初始质量等级映射表。

## 总结

自适应画质系统是一套**数据驱动、反馈闭环**的工程方案。核心要点：

| 模块 | 职责 |
|------|------|
| FrameBudgetMonitor | 实时测量帧时间，提供超载/余量信号 |
| QualityStateMachine | 带冷却的状态转换，防止频繁抖动 |
| DynamicResolution | 最高性价比的画质调节手段 |
| PostProcessDowngrader | 按优先级关闭高开销效果 |
| ThermalThrottleController | 感知硬件热量，主动降级避免更严重降频 |
| DevicePerformanceEvaluator | 启动时快速评估设备初始档位 |

通过分层调节策略，在帧率压力下优先保障流畅度，在性能充裕时逐步提升画质，让每台设备都能达到它最好的游戏体验。
