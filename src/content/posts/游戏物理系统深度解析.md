## 游戏物理的基础认知

```
游戏物理 vs 真实物理：
  真实物理：连续的、精确的
  游戏物理：离散时间步、近似模拟
  
  PhysX（Unity 默认）的工作方式：
    每个 FixedUpdate 步长（默认 0.02s = 50Hz）模拟一次
    连续碰撞检测（CCD）处理高速物体穿透问题
    
游戏物理 ≠ 真实物理的后果：
  隧穿（Tunneling）：高速物体穿过薄墙
  抖动（Jitter）：堆叠物体在静止时轻微震颤
  数值误差：浮点运算累积误差
```

## 一、碰撞体性能优化

### 1.1 碰撞体选型

```
碰撞体性能从好到差：

1. Sphere（球体）：最快，O(1) 相交检测
2. Capsule（胶囊）：很快，角色控制器标配
3. Box（长方体）：快
4. Mesh（网格）：慢！只用于静态物体

使用建议：
  角色：Capsule（高度方向运动流畅）
  子弹/炮弹：Sphere（简单快速）
  载具：多个 Box 组合（比 Mesh 快很多）
  地形：Terrain Collider（有专用优化）
  复杂静态建筑：Mesh Collider（只适合 IsConvex=false 的静态物体）

⚠️ 非凸 Mesh Collider（IsConvex=false）：
  只能与其他碰撞体碰撞，不能两个非凸 Mesh Collider 互相碰撞
  性能最差，仅用于静态障碍物
  
⚠️ 凸 Mesh Collider（IsConvex=true）：
  比非凸快，可以互相碰撞
  适合形状不规则但需要精确碰撞的动态物体
```

### 1.2 Layer Collision Matrix 优化

```
Physics 设置中的 Layer Collision Matrix（碰撞层矩阵）
是最被忽视但效果显著的优化：

问题：默认所有层都互相检测碰撞
      即使玩家子弹和敌人子弹不可能互相影响

优化：
  Player（玩家角色）与哪些层碰撞？
    ✅ Environment（地形/建筑）
    ✅ Enemy（敌人）
    ✅ EnemyBullet（敌人子弹）
    ❌ PlayerBullet（不需要）
    ❌ UI（不需要）
    
  PlayerBullet 与哪些层碰撞？
    ✅ Environment
    ✅ Enemy
    ❌ Player（不会打到自己）
    ❌ EnemyBullet（子弹不互相碰撞）
    
设置位置：
  Edit → Project Settings → Physics → Layer Collision Matrix
  取消不必要的层组合
  
效果：可减少 50%+ 的碰撞检测对数
```

## 二、FixedUpdate 与物理时间步

### 2.1 物理帧率选择

```csharp
// 物理帧率的权衡：

// 默认：FixedUpdate 50Hz（0.02s/步）
Time.fixedDeltaTime = 0.02f;

// 竞技游戏（需要高精度）：推荐 100Hz（0.01s/步）
Time.fixedDeltaTime = 0.01f;

// 手机性能受限：可以降低到 30Hz（0.033s/步）
Time.fixedDeltaTime = 0.033f;

// 注意：
// - 物理帧率高 → 精度高，但 CPU 负担更重
// - 物理帧率低 → 性能好，但碰撞可能不准确，隧穿风险高

// 对于快速移动的子弹，使用 CCD（连续碰撞检测）：
// Rigidbody → Collision Detection → Continuous
// 只对高速小物体开启，代价较高
```

### 2.2 正确使用 FixedUpdate

```csharp
public class PlayerController : MonoBehaviour
{
    private Rigidbody _rb;
    private Vector3 _moveInput;
    
    [SerializeField] private float moveSpeed = 5f;
    [SerializeField] private float jumpForce = 5f;
    
    void Awake() => _rb = GetComponent<Rigidbody>();
    
    // ✅ 在 Update 中读取输入（每帧响应，不丢失输入）
    void Update()
    {
        _moveInput = new Vector3(
            Input.GetAxis("Horizontal"),
            0,
            Input.GetAxis("Vertical")
        );
        
        // 跳跃：在 Update 中检测按键
        if (Input.GetButtonDown("Jump"))
            _pendingJump = true;
    }
    
    private bool _pendingJump;
    
    // ✅ 在 FixedUpdate 中操作 Rigidbody（物理是固定时间步的）
    void FixedUpdate()
    {
        // 施加移动力
        Vector3 velocity = _moveInput * moveSpeed;
        _rb.velocity = new Vector3(velocity.x, _rb.velocity.y, velocity.z);
        
        // 处理跳跃
        if (_pendingJump && IsGrounded())
        {
            _rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
            _pendingJump = false;
        }
    }
    
    // ❌ 错误：在 FixedUpdate 读取 Input.GetButtonDown（可能丢失输入！）
    // void FixedUpdate_Wrong()
    // {
    //     if (Input.GetButtonDown("Jump")) // 可能在两次 FixedUpdate 之间被按下然后松开，导致丢失！
    //         Jump();
    // }
    
    private bool IsGrounded()
    {
        return Physics.Raycast(
            transform.position + Vector3.up * 0.1f,
            Vector3.down, 
            0.3f,
            LayerMask.GetMask("Environment")
        );
    }
}
```

## 三、碰撞回调的性能

### 3.1 OnCollisionEnter vs OnTriggerEnter

```
Collider（碰撞器）vs Trigger（触发器）：

Collider + Rigidbody → 产生物理碰撞（有阻挡效果）
  回调：OnCollisionEnter/Stay/Exit
  包含碰撞信息：碰撞点、法线、相对速度
  
Trigger（Is Trigger = true）→ 只检测进入，不阻挡
  回调：OnTriggerEnter/Stay/Exit
  无碰撞物理响应
  
  Trigger 比 Collider 回调更轻量（不计算碰撞解算）

性能建议：
  拾取物（金币、道具）：用 Trigger（只需检测进入，不阻挡）
  攻击范围（技能AOE）：用 Trigger（不需要物理响应）
  玩家/敌人的身体：用 Collider（需要物理阻挡）
  
Rigidbody 的存在规则：
  至少一方有 Rigidbody，才会触发 OnTriggerEnter
  静态 Collider（无 Rigidbody）触发静态 Trigger → 不回调！
```

### 3.2 批量物理查询

```csharp
// ❌ 低效：每帧 FindObjectsOfType（代价极高）
void Update()
{
    var enemies = FindObjectsOfType<Enemy>(); // 每帧扫描所有对象！
    foreach (var e in enemies) ...
}

// ✅ 高效：Physics.OverlapSphereNonAlloc（不分配内存）
public class AOESkill : MonoBehaviour
{
    // 预分配结果数组（不在热路径分配内存）
    private readonly Collider[] _hitBuffer = new Collider[20];
    
    [SerializeField] private LayerMask _enemyLayer;
    
    public void Activate(Vector3 center, float radius, int damage)
    {
        // NonAlloc 版本：使用预分配数组，零 GC 分配
        int hitCount = Physics.OverlapSphereNonAlloc(
            center, radius, _hitBuffer, _enemyLayer
        );
        
        for (int i = 0; i < hitCount; i++)
        {
            if (_hitBuffer[i].TryGetComponent<Health>(out var health))
                health.TakeDamage(damage);
        }
    }
}

// Physics.XXXNonAlloc 系列：
// OverlapSphereNonAlloc：球形范围检测
// OverlapBoxNonAlloc：盒形范围检测
// RaycastNonAlloc：射线检测（多个命中）
// SphereCastNonAlloc：球形射线检测
```

## 四、角色控制器

### 4.1 Rigidbody vs CharacterController

```
Rigidbody 角色控制：
  优点：与物理世界融合（可以被推开、受力）
  缺点：在斜面/台阶上可能表现不稳定，需要手动处理

CharacterController（Unity 内置）：
  优点：专为角色设计，斜面/台阶/头顶碰撞处理更好
  缺点：不参与物理模拟（被攻击时不会物理飞出去）
  
选择建议：
  第一人称射击（FPS）：CharacterController（移动感更好）
  物理互动丰富的游戏：Rigidbody（可被弹开、受重力等）
  
CharacterController 的陷阱：
  Move() 需要手动实现重力：velocity.y -= gravity * Time.deltaTime
  SimpleMove() 自动处理重力，但不能控制 Y 轴速度
```

## 总结

物理性能优化核心策略：

| 策略 | 收益 |
|------|------|
| 配置 Layer Collision Matrix | ★★★★★ 大幅减少碰撞对 |
| 使用 NonAlloc 物理查询 | ★★★★ 消除热路径 GC |
| 优先选择简单碰撞体形状 | ★★★ 减少计算量 |
| 合理设置物理帧率 | ★★★ 平衡精度与性能 |
| 触发器替代碰撞器 | ★★ 减少物理解算 |
| CCD 只对必要物体开启 | ★★ 避免不必要开销 |
