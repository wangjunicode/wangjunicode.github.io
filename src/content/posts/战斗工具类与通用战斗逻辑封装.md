# 战斗工具类与通用战斗逻辑封装

在战斗系统开发中，有大量逻辑需要在多个地方复用：角色朝向敌人校正、技能的后摇状态判断、轮次结束通知……这些逻辑如果分散在各处，维护成本极高。

本文解析 `BattleUtil` 工具类和相关的战斗辅助机制，展示如何通过良好的封装让战斗代码保持清晰。

---

## 旋转校正系统

在技能开始时，角色需要自动朝向敌方，避免背对敌人发动攻击的尴尬。`RotationCorrection` 处理这个问题：

```csharp
public static void RotationCorrection(Unit unit, 
    AT_RotationCorrection.ERotateTarget rotateTargetType = AT_RotationCorrection.ERotateTarget.Self,
    int angleLimit = 180,   // 只在偏转角度小于此值时才校正
    int frames = 0,         // 0=瞬间旋转，>0=平滑旋转过渡
    bool invert = false)    // 是否背对敌人（特殊攻击用）
{
    var otherTeam = battleScriptComp.GetOtherTeam(unit);
    var enemyTarget = otherTeam?.GetCurMainMember();
    
    // 确定旋转主体和目标
    Unit entity = (rotateTargetType == ERotateTarget.Self) ? unit : enemyTarget;
    Unit target = (rotateTargetType == ERotateTarget.Self) ? enemyTarget : unit;
    
    if (entity != null && target != null)
    {
        var direction = (target.Position - entity.Position);
        direction.y = 0;           // 水平方向，忽略高度差
        direction.Normalize();
        
        FP angle = TSVector.Angle(entity.Forward, direction);
        if (angle <= angleLimit)  // 角度限制：防止过度扭曲（如技能中途才发现偏差太大）
        {
            var rot = TSQuaternion.LookRotation(invert ? -direction : direction, TSVector.up);
            
            if (frames <= 0)
            {
                // 瞬间旋转：适合技能起手瞬间对准
                entity.GetComponent<PhysicsComponent>().SetRotation(rot);
                entity.GetComponent<SimpleMoveComponent>().SetTargetRotation(rot * TSVector.forward, 0);
            }
            else
            {
                // 平滑旋转：在 N 帧内旋转到目标朝向（避免瞬间转身的生硬感）
                entity.GetComponent<SimpleMoveComponent>().SetTargetRotation(
                    rot * TSVector.forward, 
                    frames * EngineDefine.fixedDeltaTime);
            }
        }
    }
}
```

### 设计细节解读

**`angleLimit` 参数**：如果角色朝向与敌人方向偏差超过 `angleLimit` 度，就不校正了。这避免了"技能收招时背对敌人也强制转向"的问题——偏差太大说明是意图攻击侧面/背面，不应该强制校正。

**`frames > 0` 的平滑旋转**：

```csharp
entity.GetComponent<SimpleMoveComponent>().SetTargetRotation(
    rot * TSVector.forward, 
    frames * EngineDefine.fixedDeltaTime);  // 转换为时间（帧数 × 帧间隔）
```

平滑旋转使用 `SimpleMoveComponent` 的旋转插值功能，在指定时间内平滑过渡到目标朝向，视觉上不会突兀。

**`invert` 参数**：某些背击技能需要背对敌人，`invert = true` 时旋转方向取反，角色面向远离敌人的方向。

## 战斗状态标签系统（BattleEvent）

```csharp
public static void BattleEvent(Unit unit, EStateTag state, bool begin)
{
    if (unit.Type == EEntityType.Puppet)
    {
        // 傀儡：通知宿主的 PuppetOwnerComponent
        var puppetComp = unit.RootUnit.GetComponent<PuppetOwnerComponent>();
        puppetComp.SetPuppetState(unit, state, begin);
    }
    else
    {
        // 普通单位：通知 BattleScriptComponent
        var battleSpt = unit.Domain.GetComponent<BattleScriptComponent>();
        if (battleSpt != null)
        {
            battleSpt.SetTLState(unit, state, begin);
        }
    }
}

// 便捷方法
public static void SetAttackRecovery(this Unit unit, bool begin)
{
    BattleEvent(unit, EStateTag.PerformAttack, begin);  // 攻击演出中
    BattleEvent(unit, EStateTag.Recovery, begin);       // 后摇中
}

public static void SetDefRecovery(this Unit unit, bool begin)
{
    BattleEvent(unit, EStateTag.DefenderRecovery, begin);  // 防守方回复中
    BattleEvent(unit, EStateTag.Recovery, begin);
}
```

`EStateTag` 是一个枚举，定义了战斗中的各种"宏观状态"：

```csharp
public enum EStateTag
{
    PerformAttack,    // 正在执行攻击演出
    Recovery,         // 处于技能后摇（恢复期）
    DefenderRecovery, // 防守方处于恢复期
    // ...
}
```

这些状态不是 FSM 的细粒度动作状态（如"第3段攻击"），而是更宏观的战斗流程状态，用于被动技能和战斗脚本的条件判断。

## 后摇检查（CheckRecovery）

技能的"后摇"是指技能动作结束后、角色恢复完全控制前的这段时间。`CheckRecovery` 提供了统一的后摇检测：

```csharp
public static bool CheckRecovery(this Unit unit)
{
    var battleSpt = unit.Domain.GetComponent<BattleScriptComponent>();
    
    // 条件1：有明确的 Recovery 标签
    if (battleSpt.IsInState(unit, EStateTag.Recovery)) return true;
    
    // 条件2：处于 Idle 或 Run 状态（这两个状态表示已经完全恢复）
    var fsmTag = unit.GetComponent<FSMComponent>().Fsm.currentState.tag;
    if (fsmTag is (int)EActionType.Idle or (int)EActionType.Run) return true;
    
    // 条件3：注册了移动的可中断键（能移动说明处于恢复期的某阶段）
    if (unit.GetComponent<SkillComponent>().AbortInput.Contains(EInputKey.Move)) return true;
    
    return false;  // 以上条件都不满足：真正的后摇期（不可中断）
}
```

三层检测机制保证了不同场景下后摇检测的准确性：
- 明确标记的后摇期（`EStateTag.Recovery`）
- 完全恢复到空闲状态
- 部分恢复（可以移动但技能还未完全结束）

## 轮次管理：RoundEnd 与 AttackEnd

```csharp
// 轮次结束
public static void RoundEnd(Unit unit)
{
    if (unit != null && unit.Domain != null)
    {
        var battleSpt = unit.Domain.GetComponent<BattleScriptComponent>();
        if (battleSpt != null && battleSpt.IsInBattle())
        {
            var teamSys = unit.Domain.GetComponent<TeamComponent>();
            var team = teamSys.GetTeamByUnit(unit);
            
            // 只有没有剩余输入（连招机会）且在当前回合才结束
            if (!team.bHasRestInput && team.atkTurn == battleSpt.GetCurTurn())
            {
                team.RemoveSkillOwner(unit, true);  // true=轮次结束
            }
        }
    }
}

// 攻击结束
public static void AttackEnd(Unit unit)
{
    var skillComp = unit.GetComponent<SkillComponent>();
    if (skillComp != null)
    {
        skillComp.CurMoveSkillType = EMoveSkillType.Stay;  // 重置移动技能类型
    }
    
    var battleSpt = unit.Domain.GetComponent<BattleScriptComponent>();
    if (battleSpt != null && battleSpt.IsInBattle())
    {
        var teamSys = unit.Domain.GetComponent<TeamComponent>();
        var team = teamSys.GetTeamByUnit(unit);
        
        if (!team.bHasRestInput && team.atkTurn == battleSpt.GetCurTurn())
        {
            team.RemoveSkillOwner(unit, false);  // false=攻击结束但轮次未必结束
        }
    }
}
```

`bHasRestInput`（还有剩余输入）：控制技能结束后是否继续保持"出手权"。如果玩家在技能结束前输入了下一个技能（预输入），则 `bHasRestInput = true`，本轮继续让玩家操作；否则出手权结束，轮到对手。

`RemoveSkillOwner(unit, isRoundEnd)` 的两种参数含义不同：
- `isRoundEnd = true`：完整轮次结束，触发回合结束事件和被动
- `isRoundEnd = false`：单次攻击结束，轮到队友或继续连招

## 攻击状态检查（CanAttack）

```csharp
public static bool CanAttack(Unit unit)
{
    if (unit == null) return false;
    
    // 死亡检查
    var fsmComp = unit.GetComponent<FSMComponent>();
    if (fsmComp.CurrentRootStateTag() == EActionType.Dead) return false;
    
    // 暂停/冻结检查
    if (unit.PauseOrFreeze) return false;
    
    // 护盾归零（破防状态）检查：根据配置决定是否允许攻击
    var buffComp = unit.GetComponent<BuffComponent>();
    if (buffComp.bDead)
    {
        // 配置了可在破防状态攻击的技能才允许
        return CheckCanAttackOnDead(unit);
    }
    
    return true;
}
```

## 目标锁定工具

```csharp
// 获取当前的攻击目标（对方队伍的当前上场主角）
public static Unit GetEnemyTarget(Unit myUnit)
{
    var battleScriptComp = myUnit.Domain.GetComponent<BattleScriptComponent>();
    var otherTeam = battleScriptComp.GetOtherTeam(myUnit);
    var enemyTarget = otherTeam?.GetCurMainMember();
    return enemyTarget;
}
```

这个简单的函数封装了"找到当前攻击目标"的完整逻辑：
1. 找到当前战斗脚本组件
2. 获取对方队伍
3. 获取对方当前上场的主角色

为什么需要封装？因为这个逻辑分散在很多地方：旋转校正、碰撞检测的目标过滤、AI 决策……集中封装后，如果目标选择逻辑改变（如多目标、随机目标），只需要修改一处。

## 位置查询工具

```csharp
// 获取单位的头部挂点坐标（用于受击特效位置）
public static TSVector GetHeadPosition(Unit unit)
{
    var config = unit.GetConfig<CCharacter>();
    var headHeight = config?.HeadHeight ?? 1.8f;
    return unit.Position + TSVector.up * headHeight;
}

// 获取单位的武器挂点坐标（用于攻击特效位置）
public static TSVector GetWeaponPosition(Unit unit, int weaponSlot = 0)
{
    var attachPoints = unit.GetComponent<AttachPointsComponent>();
    if (attachPoints == null) return unit.Position;
    return attachPoints.GetWorldPosition(weaponSlot);
}
```

## BPDef：战斗点事件 ID 定义

```csharp
public class BPDef
{
    // 战斗生命周期
    public static int TurnStartEvtID = 1001;     // 回合开始
    public static int TurnEndEvtID = 1002;       // 回合结束
    public static int RoundStartEvtID = 1003;    // 轮次开始（多回合中的一轮）
    public static int RoundEndEvtID = 1004;      // 轮次结束
    public static int PerformStartEvtID = 1005;  // 演出开始
    public static int BattleBegin = 1015;        // 战斗开始
    public static int BattleEnd = 1017;          // 战斗结束
    
    // 攻防事件
    public static int AttackStartEvtID = 1009;   // 攻击开始
    public static int AttackEndEvtID = 1010;     // 攻击结束
    public static int PreAttackStartEvtID = 1022; // 攻击前置
    public static int ComboBreakEvtID = 1011;    // 连招中断
    
    // 数值事件（Pre/Post 钩子）
    public static int PreDamage = 39;      // 伤害前（可修改伤害）
    public static int PostDamage = 40;     // 伤害后（统计用）
    public static int PreHealHp = 42;      // 治疗前
    public static int PostHealHp = 43;     // 治疗后
    public static int PreHealGuard = 45;   // 护盾回复前
    
    // 角色出入场事件
    public static int CharacterEnterEvtID = 1029;  // 角色入场
    public static int CharacterExitEvtID = 1030;   // 角色退场
    
    // 出手权事件
    public static int NormalGetExecutantEvtID = 1031;  // 常规获得出手权
    public static int NormalLostExecutantEvtID = 1032; // 常规失去出手权
    public static int ForceGetExecutantEvtID = 1033;   // 抢夺出手权
    public static int ForceLostExecutantEvtID = 1034;  // 被抢夺出手权
}
```

这些 ID 用于被动技能的触发条件配置，如"在 TurnStart 事件时触发回复效果"。通过整数 ID 而非字符串，避免了拼写错误，且查找效率更高。

### Pre/Post 钩子模式

```csharp
// 伤害计算前发布前置事件（可被Buff修改伤害）
EventSystem.Instance.Publish(scene, new Evt_PointEvent
{
    Unit = defender, Team = team,
    EventID = BPDef.PreDamage
});

// 实际计算伤害
var finalDamage = CalcFinalDamage(attacker, defender, rawDamage);

// 伤害完成后发布后置事件（统计、成就、被动触发）
EventSystem.Instance.Publish(scene, new Evt_PointEvent
{
    Unit = defender, Team = team,
    EventID = BPDef.PostDamage
});
```

这套 Pre/Post 钩子让任何系统都可以在伤害计算的关键节点介入：
- `PreDamage`：修改原始伤害（如减伤Buff）
- `PostDamage`：响应伤害发生（统计、特效、被动技能）

## 工具类设计原则

通过 `BattleUtil` 可以总结出几个工具类的设计原则：

**① 单一功能，高度聚焦**

每个方法只做一件事。`RotationCorrection` 只负责朝向校正，不顺带处理攻击。

**② 参数可选，提供默认值**

```csharp
public static void RotationCorrection(Unit unit,
    ERotateTarget rotateTargetType = ERotateTarget.Self,  // 默认自身转
    int angleLimit = 180,                                  // 默认不限制角度
    int frames = 0,                                        // 默认瞬间转
    bool invert = false)                                   // 默认正面朝向
```

使用者可以只传 `unit`，其他都用默认值；或者按需覆盖特定参数。

**③ 空值安全**

```csharp
var enemyTarget = otherTeam?.GetCurMainMember();
```

到处使用空值检查（`?.`），避免因为场景未初始化完全或对象已销毁导致的崩溃。

**④ 傀儡/普通单位的统一接口**

```csharp
if (unit.Type == EEntityType.Puppet)
{
    // 傀儡走特殊路径
} else {
    // 普通单位走标准路径
}
```

调用者不需要知道是傀儡还是普通单位，工具类内部处理差异。

## 总结

战斗工具类是"战斗系统的胶水层"，它将散落在各处的通用逻辑集中管理：

| 功能 | 意义 |
|------|------|
| 旋转校正 | 技能自动面向敌人 |
| 后摇检查 | 统一的后摇状态判断 |
| 轮次/攻击结束 | 出手权管理 |
| 攻击能力检查 | 死亡/暂停/破防的统一判断 |
| BPDef 事件ID | 战斗节点事件的集中定义 |
| Pre/Post 钩子 | 可扩展的数值计算介入点 |

好的工具类设计让团队的每个成员都能快速找到需要的功能，减少重复代码，是代码质量的重要体现。
