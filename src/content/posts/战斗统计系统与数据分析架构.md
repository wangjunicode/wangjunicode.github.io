# 战斗统计系统与数据分析架构

一场战斗结束后，你是否想知道：哪个角色造成了最多伤害？总共格挡了几次？AI 打出了多少会心？这些数据不仅是结算界面的展示内容，也是游戏平衡性分析的重要依据。

本文深入解析战斗统计系统的设计，从数据模型到采集时机，再到最终上报，完整呈现一个商业游戏的统计架构。

---

## 第一性原理：统计系统需要解决什么？

战斗统计面临几个核心问题：

1. **时机分散**：伤害发生在碰撞判定时，格挡发生在受击时，治疗发生在技能结束时——数据来源分散
2. **多维度**：需要按角色（Character）、按回合（Round）、按队伍（Team）统计
3. **实时性 vs 最终性**：结算界面要显示最终数据，但战斗中可能也需要实时查询（如"已造成 X 点伤害"的成就判断）
4. **不影响性能**：统计代码路径不应该影响战斗的核心逻辑性能

## 数据模型设计：三层统计结构

### 角色单回合数据（CharacterRoundData）

```csharp
public class CharacterRoundData
{
    public FP damage;         // 抗压值伤害（队伍护盾）
    public FP hpDamage;       // HP 伤害
    public int styleSkillTime; // 奥义次数（大招使用）
    public int dodgeTime;     // 闪避次数
    public int blockTime;     // 格挡次数
    public int focusTime;     // 会心次数
    public int criticalTime;  // 破招次数
    public bool bOnStage;     // 本回合是否出场
    public FP heal;           // 治疗量
    public int changeTime;    // 切人次数
    public int supportTime;   // 支援次数
    public int createPuppet;  // 召唤傀儡次数
    public int destroyPuppet; // 傀儡销毁次数
    public int reviveTime;    // 复活队友次数
}
```

这是粒度最细的数据——某角色在某回合的表现。

### 角色总数据（CharacterTotalData）

```csharp
public class CharacterTotalData
{
    public FP maxTurnDamage;  // 单回合最高伤害（评估爆发力）
    public FP resetGuild;     // 剩余抗压值（评估韧性）
    public FP totalDamage;    // 总伤害
    public FP sufferre;       // 承受伤害（被打了多少）
    public int breakTime;     // 破防次数
    public FP heal;           // 总治疗量
    public Dictionary<Vector2_Int, int> skillUseTime;  // 技能使用次数（技能ID → 次数）
    public List<Vector2_Int> skillAchievementList;     // 技能成就列表
}
```

### 队伍统计数据（TeamStatisticData）

```csharp
public class TeamStatisticData
{
    public List<TeamRoundData> RoundData;          // 每回合的队伍数据
    public Dictionary<int, CharacterStatisticData> CharacterData; // 每个角色的数据
    public bool bWinner;    // 是否获胜
    public FP ResetHp;      // 剩余血量（结束时）
    public FP HpSufferre;   // 总承伤
}
```

三层结构清晰：
- `TeamStatisticData` > `CharacterStatisticData` > `CharacterRoundData`
- 可以从任何维度查询：某角色总伤害、某回合格挡次数、队伍胜率

## BattleStatisticComponent 的初始化

```csharp
public partial class BattleStatisticComponent : Entity, IAwake, IDestroy
{
    public Dictionary<int, TeamStatisticData> TeamDatas;  // 按 TeamId 索引
    public int winerTeamID;          // 获胜队伍ID
    public DateTime startTime;       // 战斗开始时间
    public DateTime endTime;         // 战斗结束时间
    
    public Dictionary<int, FP> OneTurnDamage;  // 当前回合每个角色的伤害累计
    public int DamageTurn;           // 当前回合索引
    
    public BattleScriptComponent bsComp;  // 战斗脚本（用于获取回合信息）
}
```

`OneTurnDamage` 是一个特殊的"当前回合实时累计器"，在回合结束时将其值写入 `CharacterRoundData.damage`，然后清零准备记录下一回合。这是"流式统计"的典型设计。

## 数据采集：事件驱动模式

统计数据不是通过轮询采集的，而是通过监听各类战斗事件。以伤害统计为例：

```csharp
// 伤害事件（由 HurtSystem 发布）
EventSystem.Instance.Publish(scene, new Evt_DamageDealt
{
    Attacker = attacker,
    Defender = defender,
    Damage = finalDamage,
    DamageType = EHurtType.Normal
});

// 统计系统监听
[EventSystem]
public class OnDamageDealt : BEventSystem<Evt_DamageDealt>
{
    protected override void Run(Scene scene, Evt_DamageDealt args)
    {
        var statComp = scene.GetComponent<BattleStatisticComponent>();
        if (statComp == null) return;
        
        // 找到攻击者所属队伍和角色
        var attackerUnit = args.Attacker;
        var team = teamSys.GetTeamByUnit(attackerUnit);
        
        // 更新当前回合伤害
        var characterId = attackerUnit.ConfigId;
        if (!statComp.OneTurnDamage.ContainsKey(characterId))
            statComp.OneTurnDamage[characterId] = FP.Zero;
        statComp.OneTurnDamage[characterId] += args.Damage;
        
        // 更新角色总数据
        var charData = GetOrCreateCharData(statComp, team.TeamId, characterId);
        charData.TotalData.totalDamage += args.Damage;
    }
}
```

这种事件驱动的采集方式有几个优点：
1. **零侵入**：核心战斗代码不需要知道统计系统的存在
2. **可选**：统计系统可以独立启停，不影响战斗逻辑
3. **扩展性强**：新增统计维度只需添加新的事件监听，不需要修改已有代码

## 回合数据的批量结算

每个回合结束时，将实时累计的 `OneTurnDamage` 写入持久化的回合数据：

```csharp
public static void OnRoundEnd(this BattleStatisticComponent self, int teamId, int roundIndex)
{
    var teamData = self.GetTeamData(teamId);
    
    // 确保回合数据列表有足够容量
    while (teamData.RoundData.Count <= roundIndex)
    {
        teamData.RoundData.Add(new TeamRoundData());
    }
    
    foreach (var (characterId, damage) in self.OneTurnDamage)
    {
        var charData = self.GetOrCreateCharData(teamId, characterId);
        
        // 确保角色回合数据列表有足够容量
        while (charData.RoundData.Count <= roundIndex)
        {
            charData.RoundData.Add(new CharacterRoundData());
        }
        
        var roundData = charData.RoundData[roundIndex];
        roundData.damage += damage;
        
        // 更新最高单回合伤害
        if (damage > charData.TotalData.maxTurnDamage)
        {
            charData.TotalData.maxTurnDamage = damage;
        }
    }
    
    // 清空当前回合累计
    self.OneTurnDamage.Clear();
    self.DamageTurn = roundIndex + 1;
}
```

## 黑板组件：AI 数据共享

`BlackboardComponent` 是一个通用的数据容器，用于不同系统之间共享临时数据（特别是 AI 决策数据）：

```csharp
public class BlackboardComponent : Entity, IAwake, IDestroy, IReset
{
    public Dictionary<string, FP> FPValues = new();        // 数值型变量
    public Dictionary<string, TSVector> VectorValues = new();  // 向量型变量
    public Dictionary<string, object> ObjectValues = new();    // 对象型变量
    public Dictionary<string, List<FP>> ListFPValues = new();  // 数值列表型变量
}
```

黑板的典型用途：

**AI 决策数据**：
```csharp
// AI 行为树将感知到的敌人信息写入黑板
blackboard.FPValues["NearestEnemyDistance"] = distance;
blackboard.ObjectValues["Target"] = nearestEnemy;

// 其他 AI 节点从黑板读取
var target = blackboard.ObjectValues["Target"] as Unit;
```

**技能参数传递**：
```csharp
// 技能开始时设置黑板参数
blackboard.FPValues["AttackMagnification"] = 3.0;

// 技能图中的节点读取参数
var magnification = blackboard.FPValues["AttackMagnification"];
```

**跨帧数据缓存**：
```csharp
// 记录上一帧的攻击结果（用于连击判断）
blackboard.FPValues["LastAttackTime"] = currentTime;
blackboard.ObjectValues["LastAttackTarget"] = lastTarget;
```

`IReset` 接口确保战斗重置时黑板数据被清空，避免上一场战斗的数据污染下一场。

## 战斗中的实时统计查询

某些成就系统需要实时查询统计数据：

```csharp
// 成就检查："单场战斗造成 10000 伤害"
public static bool CheckDamageAchievement(this BattleStatisticComponent self, Unit unit, FP targetDamage)
{
    var characterId = unit.ConfigId;
    var team = teamSys.GetTeamByUnit(unit);
    
    var charData = self.GetCharData(team.TeamId, characterId);
    if (charData == null) return false;
    
    // 历史回合总伤害 + 当前回合未结算伤害
    var totalDamage = charData.TotalData.totalDamage;
    if (self.OneTurnDamage.TryGetValue(characterId, out var currentTurnDamage))
    {
        totalDamage += currentTurnDamage;
    }
    
    return totalDamage >= targetDamage;
}
```

注意需要同时考虑已结算的历史数据和尚未结算的当前回合数据，确保实时查询的准确性。

## 结算数据上报

战斗结束时，将统计数据打包上报服务器：

```csharp
public static BattleResultData BuildResultData(this BattleStatisticComponent self)
{
    var result = new BattleResultData();
    result.WinnerTeamId = self.winerTeamID;
    result.BattleTime = (float)(self.endTime - self.startTime).TotalSeconds;
    
    foreach (var (teamId, teamData) in self.TeamDatas)
    {
        var teamResult = new TeamResultData();
        teamResult.IsWinner = teamData.bWinner;
        teamResult.RemainHp = teamData.ResetHp.AsFloat();
        
        foreach (var (charId, charData) in teamData.CharacterData)
        {
            var charResult = new CharacterResultData();
            charResult.CharacterId = charId;
            charResult.TotalDamage = charData.TotalData.totalDamage.AsFloat();
            charResult.TotalHeal = charData.TotalData.heal.AsFloat();
            charResult.SkillUsages = charData.TotalData.skillUseTime;
            // ... 其他数据
            
            teamResult.CharacterResults.Add(charResult);
        }
        
        result.TeamResults.Add(teamResult);
    }
    
    return result;
}
```

## 性能考量

统计系统会在高频路径（如每次造成伤害）被调用，需要注意性能：

**① 懒加载数据结构**

```csharp
private static CharacterStatisticData GetOrCreateCharData(this BattleStatisticComponent self, int teamId, int charId)
{
    var teamData = self.GetOrCreateTeamData(teamId);
    if (!teamData.CharacterData.TryGetValue(charId, out var charData))
    {
        charData = new CharacterStatisticData();
        teamData.CharacterData[charId] = charData;
    }
    return charData;
}
```

只有当角色第一次有数据时才创建统计对象，避免初始化时为所有可能的角色预创建。

**② 避免字符串 key 的 Dictionary 操作**

黑板使用字符串作为 Dictionary key，每次查找有较大开销。性能敏感路径应该使用整数 ID 或 enum 代替字符串。

**③ 事件系统的异步解耦**

统计系统监听的事件通过 `EventSystem` 分发，如果统计处理逻辑过重，可以考虑放到帧末批量处理（类似渲染的 Late Update）。

## 设计总结

| 组件 | 职责 |
|------|------|
| CharacterRoundData | 单角色单回合的细粒度数据 |
| CharacterTotalData | 单角色全场总数据 |
| TeamStatisticData | 队伍维度的聚合数据 |
| OneTurnDamage | 当前回合实时累计（临时） |
| BlackboardComponent | 跨系统共享的临时数据黑板 |

架构特点：
1. **事件驱动采集**：零侵入核心战斗代码
2. **三层分级存储**：按时间（回合）+ 按空间（角色/队伍）
3. **实时+批量结合**：实时累计当前回合，回合结束批量结算
4. **可插拔**：统计系统可以独立关闭，不影响战斗运行

对于初学者来说，战斗统计系统是学习"观察者模式在游戏中应用"的绝佳案例——它展示了如何在不修改核心代码的前提下，完整采集任意维度的战斗数据。
