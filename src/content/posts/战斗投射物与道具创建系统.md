# 战斗投射物与战斗道具创建系统

在战斗游戏中，很多技能效果涉及创建"独立运动的实体"——飞行中的剑气、落下的陨石、滚动的炸弹……这些统称为"BattleItem"（战斗道具/投射物）。本文深入解析 BattleItem 的完整生命周期，从技能图中的创建，到独立运动，再到命中判定和自我销毁。

---

## 第一性原理：投射物是"自主运动的战斗单元"

投射物本质上是一个**简化版的 Unit**：
- 有位置、朝向（和普通 Unit 相同）
- 有攻击碰撞盒（可以命中敌人）
- 有自己的状态机（控制运动轨迹）
- 有生命周期（命中后或超时后销毁）
- **没有**受击盒（不能被攻击）
- **没有**玩家输入（由状态机自主控制）

用 Unit 的框架实现投射物，最大化复用了碰撞检测、状态机、特效系统等基础设施。

## AT_CreateBattleItemHandler：投射物创建节点

```csharp
public class AT_CreateBattleItemHandler : AActionTaskHandler<AT_CreateBattleItem>
{
    protected override void OnExecute(AT_CreateBattleItem task)
    {
        if (task.opType == ECreateBattleItemOpType.Create)
            CreateBattleItem(task);  // 创建新投射物
        else
            ModifyBattleItem(task);  // 修改已有投射物状态
    }
    
    private void CreateBattleItem(AT_CreateBattleItem task)
    {
        var agent = task.Agent<Entity>();  // 创建者（施法角色）
        
        var unitInfo = new UnitInfo
        {
            UnitId = IdGenerater.Instance.GenerateId(),  // 生成唯一ID
            Type = (int)EEntityType.BattleItem,          // 类型标记为战斗道具
            ConfigId = task.id,        // 道具配置ID（决定使用哪个FSM、哪个技能图）
            Identifier = task.alias,   // 别名（用于其他节点引用此投射物）
            RootUnit = task.Agent<Unit>().RootUnit,  // 记录创建者（用于防止自伤）
        };
        
        if (agent is Unit parentUnit)
        {
            // 基于创建者的位置和朝向计算初始状态
            unitInfo.CreateTurn = parentUnit.GetComponent<AttackComponent>().Turn;
            unitInfo.Forward = parentUnit.Rotation 
                * TSQuaternion.Euler(task.angleOffset)  // 角度偏移（如斜向发射）
                * TSVector.forward;
            unitInfo.Pos = parentUnit.Position 
                + parentUnit.Rotation * task.posOffset;  // 位置偏移（如从武器手部发射）
            unitInfo.Group = parentUnit.Group;  // 继承队伍（友方/敌方）
        }
        
        UnitFactory.Create(agent.DomainScene(), unitInfo, true);
    }
}
```

### 关键设计解析

**① 角度和位置偏移（AngleOffset & PosOffset）**

```csharp
unitInfo.Forward = parentUnit.Rotation 
    * TSQuaternion.Euler(task.angleOffset)  // 在角色朝向基础上再旋转
    * TSVector.forward;

unitInfo.Pos = parentUnit.Position 
    + parentUnit.Rotation * task.posOffset;  // 在角色坐标系中的偏移
```

`task.posOffset` 是在角色**本地坐标系**中的偏移。通过 `parentUnit.Rotation * posOffset`，将本地偏移转换为世界坐标。这意味着：
- 无论角色朝向哪里，`posOffset = (0, 0, 1)` 总是"角色前方1单位"

这让策划可以简单地配置"从角色右手（相对坐标 (1, 0, 0.5)）发射"，不需要担心角色的实际朝向。

**② CreateTurn（回合标记）**

```csharp
unitInfo.CreateTurn = parentUnit.GetComponent<AttackComponent>().Turn;
```

`Turn`（回合）是攻击组件中记录的"本次攻击是在第几回合产生的"。投射物继承这个值，用于判断"是否算本回合的伤害"，以及防止"自伤"（同一回合产生的攻击不会伤害创建者队伍）。

**③ Group（队伍归属）**

```csharp
unitInfo.Group = parentUnit.Group;
```

投射物继承创建者的队伍，保证只会命中敌方单位，不会误伤友方。

## BattleItem 在 UnitFactory 中的特殊处理

```csharp
// UnitFactory.Create 中
if (unit.Type == EEntityType.BattleItem)
{
    // BattleItem 不需要：受击盒、Buff 系统、玩家输入
    // 但需要：攻击盒、碰撞检测、物理移动
    unit.AddComponent<SimpleMoveComponent>();
    unit.AddComponent<PhysicsComponent>();
    unit.AddComponent<FSMComponent>();        // 状态机（控制运动轨迹）
    
    var atkcomp = unit.AddComponent<AttackComponent>();
    atkcomp.Turn = unitInfo.CreateTurn;       // 记录创建回合（防自伤）
    
    unit.AddComponent<ColliderComponent>();   // 碰撞检测（只有攻击盒，没有受击盒）
}
```

与玩家/怪物相比，BattleItem 的组件集合更精简——它是一个"纯粹的攻击者"。

## 投射物的状态机设计

投射物通过 FSM 控制运动行为：

```
Root FSM
  └── Moving（移动中）
       ├── Enter: 设置初始速度
       ├── Update: 更新物理运动
       │    └── 碰撞检测（每帧）
       │         └── 命中敌方 → 触发伤害 → 进入 Die 状态
       └── 超时 → 进入 Die 状态

Die（销毁）
  ├── Enter: 播放消散特效
  └── 延迟几帧后 → 销毁 Unit
```

不同类型的投射物使用不同的 FSM：
- **直线飞行**：匀速直线运动，碰到障碍物/敌人消失
- **抛物线**：初速度 + 重力，落地后触发范围爆炸
- **追踪**：每帧修正方向，朝目标飞行
- **滞留**：落地后停在原地一段时间，持续造成伤害

这些都是通过不同的 FSM 图和 SimpleMoveComponent 的参数组合实现的，不需要为每种类型写单独的代码。

## 投射物的标识符系统（Identifier）

```csharp
unitInfo.Identifier = task.alias;  // 别名，如 "SwordStrike1"
```

技能图中其他节点可以通过别名找到这个投射物：

```csharp
// 查找标识符为 "SwordStrike1" 的子 Unit
public static int SearchEntity(this Unit self, string identifier, List<Unit> results)
{
    foreach (var child in self.Children.Values)
    {
        var unit = child as Unit;
        if (unit?.Identifier == identifier)
        {
            results.Add(unit);
        }
    }
    return results.Count;
}
```

这让技能图可以精确控制特定的投射物：
- "创建投射物 SwordStrike1"
- "等待投射物 SwordStrike1 命中"
- "如果 SwordStrike1 命中了，触发连锁效果"

## 投射物的攻击判定流程

```csharp
// ColliderComponent 中的攻击碰撞检测（每帧）
// （BattleItem 的碰撞盒类型是 Attack）
public static bool CanAttack(this ColliderComponent self, Unit target, 
    out ColliderInfo attackCollider, out ColliderInfo hitCollider)
{
    var atkComp = self.GetParent<Unit>().GetComponent<AttackComponent>();
    
    // 防止多次命中同一目标（同一投射物只伤害每个目标一次）
    if (atkComp.AttackedEnemies.Contains(target)) return false;
    
    // 防自伤（同回合产生的投射物不伤害创建者队伍）
    if (atkComp.Turn == target.GetComponent<AttackComponent>().Turn 
        && SameTeam(self.GetParent<Unit>(), target)) return false;
    
    // OBB 碰撞检测
    return ColliderSystemHelper.Intersects(self.GetParent<Unit>(), attackCollider, target, hitCollider);
}
```

命中后的处理：

```csharp
// 命中事件处理
var hurtComp = targetUnit.GetComponent<HurtComponent>();
hurtComp.AttackInfo = new AttackInfo
{
    Attacker = battleItemUnit.RootUnit,  // 攻击者是创建者，不是投射物本身
    Defender = targetUnit,
    // ...
};

// 通知技能图：投射物命中了目标
battleItemUnit.GetComponent<EventDispatcherComponent>()
    .FireEvent(new Evt_BattleItemHit { Target = targetUnit });
```

注意：`Attacker` 是 `battleItemUnit.RootUnit`（创建者），而不是投射物自身。伤害统计、暴击加成等都基于创建者的数值计算，投射物只是一个"延伸的攻击盒"。

## 多重投射物（散射弹）

一次技能创建多个投射物：

```csharp
// 技能图节点循环创建多个投射物
for (int i = 0; i < projectileCount; i++)
{
    var angleOffset = new TSVector(0, -spreadAngle/2 + spreadAngle * i / (projectileCount-1), 0);
    
    var unitInfo = new UnitInfo { /* ... */ };
    unitInfo.Forward = parentUnit.Rotation 
        * TSQuaternion.Euler(angleOffset)  // 每个投射物方向稍微不同
        * TSVector.forward;
    
    UnitFactory.Create(scene, unitInfo, true);
}
```

通过角度偏移，可以轻松实现"扇形散射"效果。

## 投射物的生命周期清理

```csharp
// 投射物销毁时（Dead 状态退出）
public class BattleItem_DeadActionHandler : AFSMActionTaskHandler<BattleItem_DeadAction>
{
    protected override void OnExecute(BattleItem_DeadAction task)
    {
        var unit = task.Agent<Unit>();
        
        // 播放消散特效
        EventSystem.Instance.Publish(unit.DomainScene(), 
            new Evt_UnitDead { Unit = unit });
        
        // 延迟几帧（特效播放时间）后真正销毁
        unit.GetComponent<UnitTimeComponent>().DelayDestroy(3);
    }
}
```

`DelayDestroy(3)` 延迟3帧后销毁，给特效（如爆炸粒子）留出播放时间，而不是立刻消失。

## 设计总结

战斗道具（BattleItem）系统体现了框架复用的思想：

| 特性 | 实现方式 |
|------|---------|
| 复用 Unit 框架 | 与玩家/怪物共用碰撞、物理、FSM |
| 精简组件 | 只装配攻击相关组件，无受击能力 |
| 坐标系计算 | 使用角色本地坐标系的偏移 |
| 防自伤 | CreateTurn 记录回合，同回合不自伤 |
| 防重复命中 | AttackedEnemies 集合记录命中目标 |
| 别名系统 | Identifier 支持技能图引用特定投射物 |
| 延迟销毁 | DelayDestroy 留出特效播放时间 |

```
技能图执行: AT_CreateBattleItem
    ↓
UnitFactory.Create(EEntityType.BattleItem)
    ↓
BattleItem 进入 Moving 状态
    ↓
每帧: PhysicsComponent 更新位置
      ColliderComponent 检测碰撞
    ↓
命中敌方: 发布 Evt_BattleItemHit
    ↓
伤害计算（以 RootUnit 为攻击者）
    ↓
BattleItem 进入 Die 状态
    ↓
特效播放 + DelayDestroy(3帧)
```

投射物系统的优雅在于：新增一种投射物只需要配置 FSM 和参数，不需要写代码。这是"数据驱动"设计在战斗系统中的又一成功实践。
