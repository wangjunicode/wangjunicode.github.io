## 渲染流水线概述

渲染流水线（Rendering Pipeline）是 GPU 将 3D 场景数据转换为 2D 屏幕图像的完整处理流程。

```
CPU                    GPU
  │                      │
  │  Draw Call           │
  ├─────────────────────►│
  │                      │
  │                  ┌───┴──────────────────────────────────────┐
  │                  │  顶点处理阶段（Vertex Stage）              │
  │                  │  ├── 顶点着色器（Vertex Shader）          │
  │                  │  └── 图元装配（Primitive Assembly）       │
  │                  ├──────────────────────────────────────────┤
  │                  │  光栅化阶段（Rasterization Stage）        │
  │                  │  ├── 三角形裁剪                          │
  │                  │  ├── 透视除法                            │
  │                  │  └── 扫描转换（生成片元）                 │
  │                  ├──────────────────────────────────────────┤
  │                  │  片元处理阶段（Fragment Stage）           │
  │                  │  ├── 片元着色器（Fragment Shader）        │
  │                  │  ├── 深度测试（Depth Test）               │
  │                  │  ├── 模板测试（Stencil Test）             │
  │                  │  └── 混合（Blending）                    │
  │                  └──────────────────────────────────────────┘
  │                      │
  │                  帧缓冲（Frame Buffer）
  │                      │
  │                  显示到屏幕
```

## 顶点着色器（Vertex Shader）

可编程阶段，在 GPU 上并行处理**每个顶点**：

**主要工作**：
- 将顶点坐标从**模型空间**变换到**裁剪空间**（MVP 矩阵变换）
- 计算光照所需的法线、切线等数据
- 传递 UV 坐标等数据给片元着色器

```hlsl
// Unity Shader 中的顶点着色器示例
v2f vert(appdata v)
{
    v2f o;
    // 模型空间 → 裁剪空间（MVP变换）
    o.vertex = UnityObjectToClipPos(v.vertex);
    // 传递 UV
    o.uv = TRANSFORM_TEX(v.uv, _MainTex);
    // 转换法线到世界空间
    o.normal = UnityObjectToWorldNormal(v.normal);
    return o;
}
```

## 坐标空间变换

渲染过程中，顶点经过多次坐标变换：

```
模型空间（Object Space）
    │ × M（Model Matrix / 世界变换矩阵）
    ▼
世界空间（World Space）
    │ × V（View Matrix / 观察变换矩阵）
    ▼
观察空间（View Space / Camera Space）
    │ × P（Projection Matrix / 投影矩阵）
    ▼
裁剪空间（Clip Space / NDC）
    │ ÷ w（透视除法）
    ▼
标准化设备坐标（NDC: -1 到 1）
    │ 视口变换
    ▼
屏幕空间（Screen Space）
```

## 光栅化（Rasterization）

将几何图元（三角形）转换为**屏幕像素**（片元/Fragment）的过程：

1. **三角形裁剪**：裁掉视锥体外的部分
2. **背面剔除**：剔除朝向相机背面的三角形（减少 GPU 工作量）
3. **扫描转换**：遍历三角形覆盖的所有像素，生成片元
4. **插值**：对顶点属性（UV、颜色、法线）进行**重心坐标插值**

## 片元着色器（Fragment Shader）

可编程阶段，处理**每个片元**（屏幕像素）：

```hlsl
// 简单的 Lambert 漫反射光照
fixed4 frag(v2f i) : SV_Target
{
    // 采样贴图
    fixed4 col = tex2D(_MainTex, i.uv);
    
    // 法线（世界空间）
    float3 normal = normalize(i.normal);
    
    // 光照方向
    float3 lightDir = normalize(_WorldSpaceLightPos0.xyz);
    
    // Lambert 漫反射
    float diff = max(0, dot(normal, lightDir));
    col.rgb *= diff * _LightColor0.rgb;
    
    return col;
}
```

## 逐片元测试与混合

片元着色器之后，片元还需通过一系列测试：

| 测试/操作 | 说明 | 用途 |
|----------|------|------|
| **深度测试** | 比较片元深度与深度缓冲 | 遮挡关系 |
| **模板测试** | 比较模板缓冲中的值 | 遮罩、轮廓描边 |
| **混合** | 将片元颜色与帧缓冲混合 | 透明效果 |

```hlsl
// 透明混合设置（Unity Shader）
Blend SrcAlpha OneMinusSrcAlpha  // 标准 Alpha 混合
ZWrite Off                        // 透明物体关闭深度写入
```

## Early-Z 优化

现代 GPU 在片元着色器**之前**就进行深度测试（Early-Z），提前剔除被遮挡的片元：

- **前提**：顶点着色器不修改深度值，片元着色器不使用 `discard`
- **意义**：大幅减少片元着色器的工作量
- **实践**：先渲染不透明物体（从近到远），充分利用 Early-Z

## Overdraw 问题

每个像素被多次绘制称为 Overdraw：

```
Overdraw = 实际绘制片元数 / 屏幕像素数

Overdraw = 1  → 理想情况，每个像素只绘制一次
Overdraw = 3  → 每个像素平均绘制 3 次，GPU 负担较重
```

**减少 Overdraw 的方法**：
- 渲染顺序：不透明物体从近到远（Early-Z 剔除）
- 透明物体：从远到近，合并绘制
- 使用遮挡剔除（Occlusion Culling）
- 减少半透明 UI 层叠
