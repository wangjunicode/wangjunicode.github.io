# 构建与发布系统

## 1. 系统概述

构建系统负责将 Unity 工程打包成可发布的 Android APK / iOS IPA，并配合热更资源的 AssetBundle 打包。本项目基于 ET 框架的 BuildEditor 进行扩展，通过 Editor 菜单工具 + CI 命令行两种方式驱动。

核心功能：
- **平台切换**：一键切换 Android / iOS / Windows
- **编译优化控制**：Debug（附分析器）vs Release 构建
- **Addressables 打包**：资源包生成与 Catalog 更新
- **HybridCLR 热更 DLL 打包**：热更代码编译分离
- **构建耗时记录**：`BuildTimeRecorder` 追踪每步耗时

### 2.3 Editor 可视化窗口

```csharp
// BuildEditor 窗口（简化版）
public class BuildEditor : EditorWindow
{
    private PlatformType platformType;
    private bool clearFolder;
    private bool isBuildExe;
    private bool isContainAB;
    private CodeOptimization codeOptimization = CodeOptimization.Debug;
    
    [MenuItem("ET/Build Tool")]
    public static void ShowWindow()
    {
        GetWindow<BuildEditor>();
    }
    
    private void OnGUI()
    {
        this.platformType = (PlatformType)EditorGUILayout.EnumPopup(platformType);
        this.clearFolder  = EditorGUILayout.Toggle("清理输出目录", clearFolder);
        this.isBuildExe   = EditorGUILayout.Toggle("构建可执行文件", isBuildExe);
        this.isContainAB  = EditorGUILayout.Toggle("包含AssetBundle", isContainAB);
        this.codeOptimization = (CodeOptimization)EditorGUILayout.EnumPopup(
            "代码优化级别", codeOptimization);
        
        // 根据优化级别设置 BuildOptions
        switch (this.codeOptimization)
        {
            case CodeOptimization.Debug:
                // 开发包：启用 Profiler 连接
                this.buildOptions = BuildOptions.Development 
                                  | BuildOptions.ConnectWithProfiler;
                break;
            case CodeOptimization.Release:
                this.buildOptions = BuildOptions.None;
                break;
        }
        
        if (GUILayout.Button("开始构建", GUILayout.Height(40)))
        {
            Build();
        }
    }
}
```

## 3. 宏定义管理

```csharp
// Editor/Assembly/DefineSymbolManager.cs
// 通过 Editor 菜单动态添加/移除编译宏
public static class DefineSymbolManager
{
    // 项目关键宏说明：
    // DEBUG           - 开发调试开关
    // PRODUCTION_ENVIRONMENT - 生产环境（连接正式服）
    // APP_GRAY_UPDATE - 灰度更新开关
    // AA_STAGE        - Addressables Staging 环境
    
    public static void AddDefine(BuildTargetGroup group, string define)
    {
        string defines = PlayerSettings.GetScriptingDefineSymbolsForGroup(group);
        if (!defines.Contains(define))
        {
            PlayerSettings.SetScriptingDefineSymbolsForGroup(
                group, defines + ";" + define);
        }
    }
    
    public static void RemoveDefine(BuildTargetGroup group, string define)
    {
        string defines = PlayerSettings.GetScriptingDefineSymbolsForGroup(group);
        PlayerSettings.SetScriptingDefineSymbolsForGroup(
            group, defines.Replace(define, "").Replace(";;", ";"));
    }
}
```

## 4. 资源分析工具

```csharp
// Editor/ResourcesAnalyze/ResourceSizeAnalyzer.cs
// 分析各资源包体积，找出大文件
public class ResourceSizeAnalyzer : EditorWindow
{
    [MenuItem("ET/资源分析")]
    public static void ShowWindow()
    {
        GetWindow<ResourceSizeAnalyzer>("资源大小分析");
    }
    
    // 扫描 AssetBundle 输出目录，统计各 bundle 大小
    // 列出超过阈值的资源，辅助包体优化
    private void AnalyzeBundles()
    {
        var bundlePath = Application.streamingAssetsPath;
        var files = Directory.GetFiles(bundlePath, "*.bundle", SearchOption.AllDirectories);
        
        var results = files
            .Select(f => new { Path = f, Size = new FileInfo(f).Length })
            .OrderByDescending(r => r.Size)
            .ToList();
        
        // 输出报告到 Editor 控制台
        foreach (var r in results.Take(20))
        {
            Debug.Log($"{Path.GetFileName(r.Path)}: {r.Size / 1024}KB");
        }
    }
}
```

## 5. 构建流程 SOP（标准操作流程）

```
开发完成
    ↓
1. 更新版本号（VersionManager.AppConfig）
    ↓
2. 执行代码热更 DLL 编译
    ↓
3. 执行 Addressables 构建（生成 Bundle + Catalog）
    ↓
4. 运行资源分析，确认无超体资源
    ↓
5. 执行 Unity Build（生成 APK/IPA）
    ↓
6. 上传热更资源到 CDN（ResUpdate 目录）
    ↓
7. 更新 Dolphin 版本配置（发布正式/预发布渠道）
    ↓
8. 测试验证热更流程
```

## 6. Addressables 路径工具

```csharp
// Editor/BuildEditor/AddressablesPathUtility.cs
// 批量修正 Addressables 资产地址（Address），保持与运行时加载Key一致
public static class AddressablesPathUtility
{
    [MenuItem("ET/Addressables/修正所有地址")]
    public static void FixAllAddresses()
    {
        // 遍历所有 AssetGroup
        var settings = AddressableAssetSettingsDefaultObject.Settings;
        foreach (var group in settings.groups)
        {
            foreach (var entry in group.entries)
            {
                // 将绝对路径改为相对路径（Assets/ 开头）
                string address = entry.AssetPath.Replace("Assets/", "");
                entry.SetAddress(address);
            }
        }
        AssetDatabase.SaveAssets();
    }
}
```

## 7. 多渠道与环境切换

项目支持多渠道包（正式服 / 预发布 / 开发服），通过 `DeploymentEnvironment` 资源文件控制：

```csharp
// PatchManager.cs 中读取部署环境
public void ReadDeploymentEnvironment()
{
    TextAsset envTxTFile = Resources.Load<TextAsset>("DeploymentEnvironment");
    if (envTxTFile != null)
    {
        DeploymentEnvironment = envTxTFile.text.Trim();
        // "Review" → 审核包（预发布渠道）
        // "Release" → 正式包
    }
}

// 根据环境选择 CDN 地址
private bool IsReviewEnvironment()
{
    return DeploymentEnvironment.Equals("Review");
}
```

## 9. 常见问题

**Q: 构建时报 HybridCLR 相关错误？**  
A: 检查 `BuildHelper.cs` 中热更 DLL 编译步骤是否在 Unity 构建前完成。热更程序集必须先编译输出，再打包进 StreamingAssets。

**Q: iOS 包在 AppStore 审核被拒？**  
A: 审核包（Review 环境）需要连接预发布渠道，不能用正式渠道内容。确认 `DeploymentEnvironment.txt` 内容为 "Review"。

**Q: AssetBundle 变化导致热更包体过大？**  
A: 检查 Bundle 分组策略，避免高频变动的资源（如 UI 图集）与低频资源（模型）放在同一 Bundle。
