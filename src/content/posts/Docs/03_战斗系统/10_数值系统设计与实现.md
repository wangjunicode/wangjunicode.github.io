# 数值系统设计与实现

## 1. 系统概述

数值系统（NumericSystem）是战斗的核心基础设施，负责管理所有战斗实体的属性数值（HP、攻击、防御、速度等），支持：
- **多段加成**：基础值、绝对值加成、百分比加成、二阶绝对值加成、二阶百分比加成
- **公式驱动**：通过 `[NumericBinder]` 特性将最终值计算逻辑注册到配置公式
- **响应式更新**：数值变化自动触发 Watcher 回调，驱动上层逻辑（如血条UI、死亡判定）
- **定点数精度**：使用 `FP`（Fixed Point）避免浮点误差，确保帧同步一致性

---

## 2. 架构设计

```
NumericComponent（挂在 Unit 上）
    │
    ├── numericInfos: NumericInfoArray   // 所有属性的存储数组
    │       └── NumericInfo[ENumericId]
    │               ├── Base            // 基础值
    │               ├── Add             // 绝对加成
    │               ├── Pct             // 百分比加成 (%)
    │               ├── FinalAdd        // 二阶绝对加成
    │               ├── FinalPct        // 二阶百分比加成
    │               └── Dynamic         // 动态值（实时变化，如当前HP）
    │
    ├── Formula（静态公式类）           // 最终值计算逻辑
    │       └── [NumericBinder] 标记    // 将方法绑定到对应属性ID
    │
    └── NumericWatcher（观察者）        // 数值变化时的响应逻辑
            ├── NumericWatcher_Hp.cs   // HP变化 → 死亡判定/血条更新
            ├── NumericWatcher_Ap.cs   // AP变化
            └── NumericWatcher_Mana.cs // Mana变化
```

### 3.1 NumericComponent 初始化

```csharp
// 位置：Hotfix/Battle/Function/GamePlay/Numeric/NumericComponentSystem.cs
[FriendOf(typeof(NumericComponent))]
public static partial class NumericComponentSystem
{
    [EntitySystem]
    private static void Awake(this NumericComponent self)
    {
        int count = NumericComponent.NumericIdMax;
        if (count < 100) count = 1000;
        // 预分配全部属性存储数组，避免运行时 GC
        self.numericInfos = new NumericInfoArray(new NumericInfo[count]);
    }

    [EntitySystem]
    public static void Reset(this NumericComponent self)
    {
        // 战斗重置时清空所有数值
        self.numericInfos.Clear();
        self.RandExpFTimes.Clear();
        self.bCleared = true;
        // 发布清空事件，让上层系统感知（如UI重置血条）
        EventSystem.Instance.Publish(self.CurrentScene(), 
            new Evt_OnNumericClear() { comp = self });
    }
    
    private static void TryAddNumeric(this NumericComponent self, ENumericId nid)
    {
        int id = (int)nid;
        if (self.numericInfos[id] != null) return;  // 已存在，跳过

        // 从配置表读取该属性定义
        var define = CfgManager.tables.TbNumeric.GetOrDefault(nid);
        if (define == null)
        {
            Log.Error(ZString.Concat("初始化时未找到Numeric:", nid));
            return;
        }
        // ... 创建 NumericInfo 并初始化
    }
}
```

### 3.2 NumericMap —— 战斗公式的数据上下文

```csharp
// 一次攻击/Buff计算时，需要访问自身、敌方、队伍等多方数值
public struct NumericMap
{
    public NumericComponent self;            // 自身属性
    public NumericComponent other;           // 敌方属性
    public NumericComponent selfTeam;        // 己方队伍属性
    public NumericComponent otherTeam;       // 敌方队伍属性
    public NumericComponent selfTeamCurMain; // 己方当前出场角色属性
    public NumericComponent otherTeamCurMain;// 敌方当前出场角色属性

    public NumericMap(Unit owner)
    {
        TeamComponent teamSys = owner.Domain.GetComponent<TeamComponent>();
        self = owner.GetComponent<NumericComponent>();
        selfTeam = teamSys.GetTeamByUnit(owner).GetComponent<NumericComponent>();
        selfTeamCurMain = teamSys.GetTeamByUnit(owner)
                                  .GetCurMainMember()
                                  .GetComponent<NumericComponent>();
        
        // 敌方引用（PVP场景下才有）
        if (owner.Domain.GetComponent<BattleScriptComponent>() != null)
        {
            Unit enemy = owner.GetRelativeEntity(false, true) as Unit;
            other = enemy.GetComponent<NumericComponent>();
            otherTeam = teamSys.GetTeamByUnit(enemy).GetComponent<NumericComponent>();
            otherTeamCurMain = teamSys.GetTeamByUnit(enemy)
                                       .GetCurMainMember()
                                       .GetComponent<NumericComponent>();
        }
    }
}
```

```csharp
        private const int ConstA = 200;
```

### 3.4 NumericWatcher —— 数值变化响应

```csharp
// 位置：Hotfix/Battle/Function/GamePlay/Numeric/NumericWatcher_Hp.cs
// HP 值变化时的响应逻辑
[NumericWatcher(ENumericId.CurHp)]
public class NumericWatcher_CurHp : INumericWatcher
{
    public void Run(NumericComponent numeric, FP oldValue, FP newValue)
    {
        Unit unit = numeric.GetParent<Unit>();
        
        // 1. 更新 HUD 血条（发布事件给表现层）
        EventSystem.Instance.Publish(unit.CurrentScene(), 
            new Evt_OnHpChange { Unit = unit, CurHp = newValue });
        
        // 2. HP 为 0 触发死亡逻辑
        if (newValue <= 0 && oldValue > 0)
        {
            var fsmComp = unit.GetComponent<FSMComponent>();
            fsmComp?.TryManualCondition(EInputKey.Dead);
        }
    }
}
```

## 4. 多段加成的最终值计算

项目采用业界标准的"两段乘法"公式：

```
最终值 = (Base + Add) * (1 + Pct/100) * (1 + FinalPct/100) + FinalAdd
```

**示例：角色攻击力计算**
```
基础攻击力(Base) = 1000
+300攻击力词条(Add) = 300
攻击力+15%词条(Pct) = 15
二阶攻击力+20%(FinalPct) = 20

最终攻击力 = (1000 + 300) * (1 + 0.15) * (1 + 0.20) = 1794
```

这套设计的好处：
- Add段加成是绝对值堆叠（适合装备词条）
- Pct段是一阶百分比（适合技能buff）
- FinalPct是更强力的二阶百分比（适合特殊强化）
- 防止某一段无限叠加导致数值失控

## 5. 定点数 FP 的使用

帧同步游戏中，浮点数在不同平台/CPU上可能产生精度差异，导致客户端逻辑分叉。本项目使用 `TrueSync` 的 `FP`（Fixed Point 定点数）解决此问题。

```csharp
// TrueSync/Fix64.cs - 64位定点数，低16位为小数部分
public struct FP : IEquatable<FP>, IComparable<FP>
{
    // 精度：1/65536 ≈ 0.000015
    // 范围：约 ±131072
    
    public static readonly FP One = new FP(ONE);
    public static readonly FP Zero = new FP(0);
    
    // 所有运算均为整数运算，跨平台完全一致
    public static FP operator +(FP x, FP y) { ... }
    public static FP operator *(FP x, FP y) { ... }
    // TSMath 提供 Sqrt、Sin、Cos 等查表运算
}
```

**使用规范：**
- 所有战斗逻辑中的位置、数值、判定均使用 FP
- 仅表现层（动画插值、特效位置）使用 float
- 接收客户端输入时，将 float 转 FP（乘以10000取整）

## 6. 与 Buff 系统的协作

Buff 系统通过 NumericComponent 修改数值：

```csharp
// Buff 添加攻击力加成
public void AddBuff_AtkAdd(NumericComponent target, int addValue)
{
    target.ModifyPart(ENumericId.Atk, ENumericPart.Add, addValue);
    // 框架自动重算最终攻击力，并触发相关 Watcher
}

// Buff 移除时恢复
public void RemoveBuff_AtkAdd(NumericComponent target, int addValue)
{
    target.ModifyPart(ENumericId.Atk, ENumericPart.Add, -addValue);
}
```

## 7. 常见问题与最佳实践

**Q: 为什么用 FP 而不是 long 直接存储？**  
A: FP 封装了定点数的所有运算符重载，代码可读性与 float 相近。使用 long 需要手动处理精度转换，容易出错。

**Q: 数值修改是线程安全的吗？**  
A: 帧同步游戏所有逻辑在主线程单线程执行，无需额外同步。

**Q: 如何调试某个属性的最终值？**  
A: 使用 `unit.GetComponent<NumericComponent>().GetFinalValue(ENumericId.Atk)` 在 Editor 中实时查看。

**Q: 公式很复杂，如何测试？**  
A: 写单元测试，构造 mock 的 NumericComponent，设置各段数值，验证最终值是否符合预期。

**Q: NumericWatcher 注册顺序会影响执行顺序吗？**  
A: 会。注册顺序即执行顺序。死亡判定的 Watcher 应确保在 UI 更新之后执行，避免界面闪烁。
