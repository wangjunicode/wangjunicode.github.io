# 音频系统设计与实现

## 1. 系统概述

本项目音频系统（VGameAudioManager）基于 Unity 自研封装，支持多种音效类型的分组管理、音量控制、实例池化和 3D 空间音效，并与 ECS 事件系统深度集成，实现"发布事件 → 自动播音"的解耦架构。

**音频类型分类：**
- **SFX**（Sound Effects）：战斗音效、UI 反馈，优先级高，实时响应
- **BGM**（Background Music）：背景音乐，淡入淡出过渡
- **Voice**：剧情对话语音，支持多语言切换
- **Global**：环境音，跨场景持续播放

---

## 2. 架构设计

```
VGameAudioManager（单例，统一入口）
    ├── VGameSndInstanceManager（音效实例池管理）
    │       └── VGameSndInstance[]（音效实例，复用 AudioSource）
    ├── SoundType 分组（SFX/BGM/Voice/Global）
    ├── VolumeType 分组（Main/Music/Sound/Dialog/UI）
    └── WwiseAudioPrefab（高级空间音效，可选 Wwise 后端）

VGameAudioEventManager（事件驱动桥接）
    └── 订阅 ECS 事件 → 调用 VGameAudioManager
```

### 3.2 实例复用池

```csharp
// VGameSndInstanceManager.cs - 音效实例池管理
// 避免频繁 new AudioSource，通过对象池复用
public class VGameSndInstanceManager
{
    private static VGameSndInstanceManager _instance;
    public static VGameSndInstanceManager Instance
    {
        get { return _instance ??= new VGameSndInstanceManager(); }
    }
    
    // 实例池：idle 空闲池 + active 活跃列表
    private Queue<VGameSndInstance> _idlePool = new();
    private List<VGameSndInstance> _activeList = new();
    
    // 从池中取出一个实例
    public VGameSndInstance GetInstance(SoundType type, string soundId)
    {
        VGameSndInstance inst;
        if (_idlePool.Count > 0)
        {
            inst = _idlePool.Dequeue();
            inst.Reset(type, soundId);
        }
        else
        {
            // 池空了，新建一个 AudioSource GameObject
            inst = new VGameSndInstance(type, soundId);
        }
        _activeList.Add(inst);
        return inst;
    }
    
    // 音效播放完毕，归还到池
    public void ReturnInstance(VGameSndInstance inst)
    {
        _activeList.Remove(inst);
        inst.Stop();
        _idlePool.Enqueue(inst);
    }
    
    // 预留 ID（用于异步加载期间防止被其他声音占用槽位）
    public void RemoveReservedId(ulong id)
    {
        // 实现保留槽位逻辑...
    }
}
```

### 3.3 BGM 淡入淡出

```csharp
public class VGameAudioManager
{
    // BGM 淡入淡出切换
    public async ETTask SwitchBGM(string newBGMId, float fadeDuration = 1.0f)
    {
        // 1. 淡出当前 BGM
        if (_currentBGMInst != null)
        {
            await FadeOut(_currentBGMInst, fadeDuration * 0.5f);
            VGameSndInstanceManager.Instance.ReturnInstance(_currentBGMInst);
        }
        
        // 2. 加载新 BGM（Addressables 异步）
        var clip = await YIUILoader.LoadAssetAsync<AudioClip>(GetBGMPath(newBGMId));
        
        // 3. 淡入新 BGM
        _currentBGMInst = VGameSndInstanceManager.Instance.GetInstance(SoundType.BGM, newBGMId);
        _currentBGMInst.Play(clip, volume: 0);
        await FadeIn(_currentBGMInst, fadeDuration * 0.5f);
    }
    
    private async ETTask FadeOut(VGameSndInstance inst, float duration)
    {
        float startVol = inst.Volume;
        float elapsed = 0;
        while (elapsed < duration)
        {
            elapsed += Time.unscaledDeltaTime;
            inst.Volume = Mathf.Lerp(startVol, 0, elapsed / duration);
            await TimerComponent.Instance.WaitFrameAsync();
        }
        inst.Volume = 0;
    }
    
    private async ETTask FadeIn(VGameSndInstance inst, float duration)
    {
        float targetVol = GetGroupVolume(VolumeType.Music);
        float elapsed = 0;
        while (elapsed < duration)
        {
            elapsed += Time.unscaledDeltaTime;
            inst.Volume = Mathf.Lerp(0, targetVol, elapsed / duration);
            await TimerComponent.Instance.WaitFrameAsync();
        }
        inst.Volume = targetVol;
    }
}
```

```csharp
[Event(SceneType.Battle)]

[Event(SceneType.Battle)]
public class Evt_HurtHitEffect_AudioHandler : AEvent<Evt_HurtHitEffect>
{
    protected override void Run(Scene scene, Evt_HurtHitEffect args)
    {
        // 受击音效（根据受击类型：普通/硬直/浮空/格挡破防）

        {
            JumpTextType.Critical  => "sfx_hit_critical",
            JumpTextType.Parry     => "sfx_parry",
            JumpTextType.Miss      => "sfx_miss",
            _                      => "sfx_hit_normal"
        };
        VGameAudioManager.Instance.PlaySFX(sfxId, args.Defender.Position);
    }
}
```

## 6. 空间音效（3D Audio）

```csharp
// VGameAudioPlay.cs - 3D 空间音效播放封装
public static class VGameAudioPlay
{
    // 在世界坐标位置播放 3D 音效
    public static void PlayAtPosition(string soundId, TSVector position)
    {
        var cfg = CfgManager.tables.TbSound.GetOrDefault(soundId);
        if (cfg == null) return;
        
        var inst = VGameSndInstanceManager.Instance.GetInstance(SoundType.SFX, soundId);
        
        // 设置 3D 属性
        inst.SpatialBlend = cfg.SpatialBlend;
        inst.MaxDistance = cfg.MaxDistance;
        
        // 注意：Unity AudioSource 的位置使用 float，这里从 TSVector（定点数）转换
        // 仅表现层（音效）可以用 float，战斗逻辑仍用 FP
        inst.transform.position = new UnityEngine.Vector3(
            position.x.AsFloat(), 
            position.y.AsFloat(), 
            position.z.AsFloat());
        
        inst.Play(GetClip(cfg.AddressKey), cfg.Volume);
    }
}
```

```csharp
// 语音系统支持多语言切换
public class VGameSoundEngine
{
    // 当前语言（与游戏语言设置同步）
    private static string _currentLanguage = "zh_CN";
    
    public static void SetLanguage(string lang)

        _currentLanguage = lang;
```

## 8. 常见问题与最佳实践

**Q: 音效同时播放太多，性能下降？**  
A: 通过 `SetMaxConcurrentSFX` 限制同时播放的 SFX 数量（如战斗中最多 16 个）。超出时，优先级低/距离远的音效被打断。

**Q: 进入战斗时音效加载卡顿？**  
A: 战斗开始前预加载常用音效到 SndInstance 池，避免战斗中动态加载。预加载列表根据 skillId 查配置表确定。

**Q: 剧情语音与字幕不同步？**  
A: 语音时长 `Duration` 存在配置表中，字幕系统读取此字段控制显示时间，不依赖音频播放完成回调，避免加载延迟导致的不同步。

**Q: 切换场景时 BGM 怎么处理？**  
A: SceneSwitchEvent 中统一处理：先淡出当前 BGM，场景加载完成后根据新场景配置淡入对应 BGM。Global 环境音跨场景不停止。

**Q: Wwise 和 Unity 原生音频如何选择？**  
A: 项目通过 `WwiseAudioPrefab` 支持可选 Wwise 后端。Wwise 适合需要复杂音频状态机（如根据战斗烈度动态混音）的场景；普通项目用 Unity 原生即可，维护成本低。
