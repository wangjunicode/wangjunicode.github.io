# UI本地化（L10N）与多语言支持

## 1. 系统概述

本项目 UI 本地化系统（YIUIL10N）基于 LuBan 配置表中的文本 ID 进行查询，通过 `Lang.GetText(id)` 统一获取当前语言的翻译文本，支持多参数格式化，并使用 **ZString** 实现零 GC 字符串拼接。

**设计特点：**
- **ID 驱动**：UI 代码中只使用数字 ID，不直接写中文字符串，方便多语言替换
- **配置表集中管理**：所有文本在 Excel 中统一管理，策划/翻译团队直接修改 Excel
- **零 GC 格式化**：使用 `ZString.Format` 替代 `string.Format`，避免 GC 分配
- **热更新支持**：文本配置作为 .bytes 文件可通过热更推送，无需发新包

## 2. 核心 API

### 2.1 YIUIL10N —— UI 本地化工具类

```csharp
// 位置：Hotfix/UIFunction/VGameUI/YIUIL10N.cs
// 所有 UI 文本查询的统一入口
public static class YIUIL10N
{
    // 简单文本（无参数）
    public static string GetText(int id)
    {
        return Lang.GetText(id);
    }
    
    // 1个参数格式化（零GC）
    // 示例：GetText(LANG_ID_LEVEL_DISPLAY, 15) → "等级：15"
    public static string GetText<T1>(int id, T1 arg1)
    {
        var format = GetText(id);
        if (string.IsNullOrEmpty(format)) return string.Empty;
        return ZString.Format(format, arg1);
    }
    
    // 2个参数（如"攻击力：{0}/{1}"）
    public static string GetText<T1, T2>(int id, T1 arg1, T2 arg2)
    {
        var format = GetText(id);
        if (string.IsNullOrEmpty(format)) return string.Empty;
        return ZString.Format(format, arg1, arg2);
    }
    
    // 3、4、5个参数的重载... （同上模式）
}
```

### 2.2 Lang —— 底层文本查询

```csharp
// Lang.cs（位于 FunctionView/Framework）
// 底层文本 ID → 当前语言字符串 的映射
public static class Lang
{
    // 查询指定 ID 的本地化文本
    // ID 对应 cfg.lang.TbLang 配置表中的行
    public static string GetText(int id)
    {
        var cfg = CfgManager.tables.TbLang.GetOrDefault(id);
        if (cfg == null)
        {
            Log.Warning($"[Lang] 文本ID不存在: {id}");
            return $"[MISSING:{id}]";  // 返回可见的错误标识，方便调试
        }
        
        // 根据当前语言设置选择对应文本列
        return CurrentLanguage switch
        {
            ELanguage.ZhCN => cfg.ZhCN,
            ELanguage.ZhTW => cfg.ZhTW,
            ELanguage.En   => cfg.En,
            ELanguage.Ja   => cfg.Ja,
            ELanguage.Ko   => cfg.Ko,
            _ => cfg.ZhCN
        };
    }
    
    // 当前语言（由设置界面修改，持久化到 PlayerPrefs）
    public static ELanguage CurrentLanguage { get; private set; } = ELanguage.ZhCN;
    
    public static void SetLanguage(ELanguage lang)
    {
        CurrentLanguage = lang;
        PlayerPrefs.SetInt("LanguageSetting", (int)lang);
        // 发布语言切换事件，通知所有已显示的 UI 刷新文本
        EventSystem.Instance.Publish(GameContext.ClientScene, new Evt_LanguageChanged());
    }
}
```

## 3. 与 YIUIFramework 的集成

在 BasePanel 生命周期中处理文本刷新：

```csharp
// 典型的多语言面板（以装备详情为例）
public partial class EquipDetailPanel : BasePanel
{
    protected override void OnCreate()
    {
        // 初始刷新文本
        RefreshTexts();
        
        // 注册语言切换事件（语言切换时自动刷新所有文本）
        EventSystem.Instance.RegisterEvent<Evt_LanguageChanged>(OnLanguageChanged);
    }
    
    protected override void OnDestroy()
    {
        EventSystem.Instance.UnRegisterEvent<Evt_LanguageChanged>(OnLanguageChanged);
    }
    
    private void OnLanguageChanged(Evt_LanguageChanged _) => RefreshTexts();
    
    private void RefreshTexts()
    {
        // 使用 YIUIL10N 查询，支持零 GC 格式化
        m_TitleText.text = YIUIL10N.GetText(LANG.EQUIP_DETAIL_TITLE);
        m_AtkLabel.text  = YIUIL10N.GetText(LANG.EQUIP_DETAIL_ATK);
        
        // 带参数格式化
        // 配置表中：EQUIP_ENHANCEMENT_LEVEL = "强化等级：{0}"
        m_LevelText.text = YIUIL10N.GetText(LANG.EQUIP_ENHANCEMENT_LEVEL, _enhancementLevel);
        
        // 多参数：EQUIP_STAT_RANGE = "攻击力：{0} ~ {1}"
        m_StatRangeText.text = YIUIL10N.GetText(LANG.EQUIP_STAT_RANGE, _minAtk, _maxAtk);
    }
}
```

## 5. ZString 零 GC 格式化原理

```csharp
// 传统 string.Format 每次产生 GC 分配
string text = string.Format("攻击力：{0}", atkValue);  // ❌ 有 GC

// ZString.Format 使用栈上缓冲区，不产生 GC
string text = ZString.Format("攻击力：{0}", atkValue);  // ✅ 零 GC

// ZString 还支持 StringBuilder 模式
using (var sb = ZString.CreateStringBuilder())
{
    sb.Append("名称：");
    sb.Append(name);
    sb.AppendFormat("，等级：{0}", level);
    text.text = sb.ToString();  // 只在最终 ToString 时分配一次
}
```

**为什么 UI 要关注 GC？**  
每帧大量字符串拼接（血量条 "123/456"、伤害数字等）会产生大量短命字符串对象，频繁触发 GC（垃圾回收），导致帧率抖动。使用 ZString 可以彻底消除这类 GC。

## 8. 常见问题与最佳实践

**Q: 文本 ID 不存在时如何调试？**  
A: `Lang.GetText` 返回 `[MISSING:id]` 字符串，在屏幕上直接可见。Release 包中可以改为返回空字符串，避免暴露内部 ID 给玩家。

**Q: 语言切换后有些 UI 没刷新？**  
A: 没有订阅 `Evt_LanguageChanged` 事件的面板不会自动刷新。解决方案：在 YIUIFramework 的 `BasePanel.OnCreate` 中统一注册语言刷新回调，所有子类自动继承。

**Q: 配置表文本很多，加载慢？**  
A: LuBan 的 .bytes 格式比 CSV 小 60~70%，加载速度更快。如果还是慢，考虑拆分成常用文本（战斗/主界面）和不常用文本（设置/帮助）两张表，延迟加载后者。

**Q: 如何支持玩家自定义昵称中的 emoji？**  
A: TMP 支持 Sprite 形式的 emoji（SpriteAsset），但需要提前烘焙。服务端昵称验证需要过滤非法字符，客户端显示时通过 TMP 的富文本标签 `<sprite="emoji" index=0>` 渲染。
