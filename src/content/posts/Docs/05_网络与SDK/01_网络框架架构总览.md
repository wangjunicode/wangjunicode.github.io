# 01_网络框架架构总览

> 面向新入职的同学，帮你建立对整个网络系统的全局认知。读完这篇文档，你应该能够回答"一条消息是怎么从服务器到达玩家手里的"这个问题。

---

## 1. 系统概述

本项目是一款对战类手游，网络系统是整个游戏的"血脉"。它负责：

- **登录鉴权**：玩家通过微信/QQ登录，MSDK负责身份验证
- **资源更新**：游戏启动时通过GCloud Dolphin进行热更新
- **实时战斗同步**：PVP战斗采用帧同步（LockStep）方案，通过GCloud提供的底层UDP/TCP混合传输实现
- **WebSocket长连接**：非战斗场景下，游戏大厅与服务器保持WebSocket长连接推送消息
- **CDN下载**：资源通过腾讯云COS（对象存储）+ CDN加速分发

整个网络层的架构可以分为以下几个模块：

```
┌─────────────────────────────────────────────────────────────┐
│                         游戏业务层                            │
├──────────────┬──────────────┬──────────────┬────────────────┤
│  登录/账号    │  战斗帧同步   │  大厅长连接   │  资源下载/CDN  │
│  (MSDK)     │  (LockStep)  │  (WebSocket) │  (COS/Dolphin) │
├──────────────┴──────────────┴──────────────┴────────────────┤
│                    GCloud SDK 核心层                          │
│   GCloudCore / Connector / NetworkService / HttpDNS         │
├─────────────────────────────────────────────────────────────┤
│                     操作系统网络层                            │
│              TCP / UDP / HTTP / WebSocket                    │
└─────────────────────────────────────────────────────────────┘
```

## 2. 架构设计

### 2.1 模块职责划分

| 模块 | 主要文件 | 职责 |
|------|---------|------|
| GCloud核心 | `GCloud.cs`, `IGCloud.cs` | SDK初始化、全局生命周期管理 |
| 网络状态监控 | `NetworkService.cs`, `INetworkService.cs` | 监听网络切换（WiFi→4G等） |
| 帧同步 | `LockSteper.cs`, `LockStepImpl.cs` | PVP战斗帧数据的收发与管理 |
| 连接器 | `Connector.cs`, `IConnector.cs` | TCP/UDP底层连接管理 |
| DNS优化 | `HttpDns.cs`, `GCloudDNS.cs` | HttpDNS防域名劫持 |
| 登录 | `MSDKLogin.cs`, `MSDK.cs` | 微信/QQ登录、Token管理 |
| 热更新 | `PatchManager.cs` | GCloud Dolphin更新框架驱动 |
| 资源下载 | `ResDownloadWsClient.cs` | 补充资源的WebSocket下载 |
| COS存储 | `CosXmlServer.cs`, `TransferManager.cs` | 腾讯云对象存储上传/下载 |

### 2.2 初始化流程

游戏启动时，网络系统的初始化顺序如下：

```
App启动
  │
  ▼
PatchManager.Start()
  │
  ├─ GCloudCore.Instance.Initialize()  ← 初始化GCloud底层
  │
  ├─ IGCloud.Instance.Initialize(gameId, gameKey)  ← 注册游戏凭证
  │
  ├─ MSDK.Init()  ← 初始化登录模块
  │
  └─ DolphinMgr.StartUpdateService()  ← 启动热更新检测
        │
        ▼
      更新完毕后进入登录场景
        │
        ▼
      MSDK登录 → 获取Token → 进入大厅
        │
        ▼
      WebSocket长连接建立
        │
        ▼
      匹配到对战 → LockStep初始化 → 战斗开始
```

### 2.3 GCloud Connector 连接架构

战斗网络依托GCloud的`Connector`组件。它支持两种模式：

- **TCP模式**：可靠传输，用于关键指令（如登录、结算）
- **UDP + 可靠UDP混合模式**：低延迟传输，用于战斗帧数据

```csharp
// Connector.cs 核心初始化（精简版）
public class GCloudConnector : ApolloObject, IConnector
{
    public GCloudConnector(ConnectorType type, bool manualUpdate, 
                           bool autoReconnect, int modeMask)
    {
        this.manualUpdate = manualUpdate;
        this.autoReconnect = autoReconnect;
        this.connectorType = type;
        this.modeMask = modeMask;
        Connected = false;
    }

    public ConnectorResult Initialize(ConnectorInitInfo initInfo)
    {
        // 调用底层Native创建Connector
        ConnectorErrorCode ret = gcloud_connector_create(
            this.ObjectId, connectorType, manualUpdate, autoReconnect, modeMask);
        // ...
    }
}
```

### 2.4 网络状态监控

`NetworkService`负责监听系统网络状态变化，并通知上层业务：

```csharp
// NetworkService.cs
public class NetworkService : INetworkService
{
    public event NetworkStateChanged NetworkChangedEvent;

    private NetworkService()
    {
        // 订阅底层Tx的网络状态变化事件
        Tx.Instance.TXNetworkChangedEvent += onNetworkStateChanged;
    }

    public NetworkState GetNetworkState()
    {
        return Tx.Instance.GetNetworkState();
    }

    private void onNetworkStateChanged(NetworkState state)
    {
        ADebug.Log("C# NetworkService onNetworkStateChanged state:" + state);
        if (null != NetworkChangedEvent)
        {
            NetworkChangedEvent(state);
        }
    }
}
```

当网络切换时（例如从WiFi切到4G），`NetworkChangedEvent`会被触发，战斗系统可据此发起重连或降级处理。

## 4. 设计亮点

### 4.1 分层架构，职责清晰

整个网络层被分为"业务逻辑层 → GCloud SDK层 → 操作系统网络层"三个层次。业务代码只需调用`LockSteper.Instance.Input(data)`即可发送战斗帧，完全不用关心底层是UDP还是TCP。

### 4.2 接口与实现分离

所有核心组件都有对应的接口：
- `IConnector` / `GCloudConnector`
- `INetworkService` / `NetworkService`
- `LockSteper`（抽象类）/ `LockStepImpl`

这样在单元测试时可以注入Mock实现，不需要真实网络环境。

### 4.3 GCloud Native层调用

`LockStepImpl`通过`[DllImport]`调用底层C++库，零拷贝传输游戏帧数据，性能极高：

```csharp
[DllImport("__Internal")]
private static extern bool gcloud_lockstep_has_logined();

[DllImport("__Internal")]  
private static extern LockStepStatus gcloud_lockstep_get_status();
```

### 4.4 HttpDNS防劫持

使用GCloud提供的HttpDNS，避免运营商DNS污染，确保玩家连接到最近的服务器节点。

## 5. 常见问题与最佳实践

### Q1：为什么有些消息用UDP，有些用TCP？

战斗帧数据对实时性要求极高，能接受偶尔丢帧（下一帧会补全），所以用UDP降低延迟。但玩家的输入指令（如技能释放）使用"可靠UDP"（内置重传机制的UDP），兼顾速度和可靠性。结算等关键消息用TCP，确保100%到达。

### Q2：GCloud SDK 初始化失败怎么排查？

1. 检查GameID和GameKey是否正确（见`PatchManager.cs`头部常量）
2. 检查设备网络权限（Android需要`INTERNET`权限）
3. 查看GCloud日志（调用`SetLogger(LogPriority.Debug, null)`后在Logcat/Console过滤`GCloud`关键词）

### Q3：如何在本地测试网络功能？

开发阶段可以在项目中设置`DEBUG`宏，系统会连接到测试服通道（`DOPHIN_PRE_UPDATEURL`），不会影响线上玩家。

### Q4：网络层需要主线程驱动吗？

是的。`GCloudConnector`设置了`manualUpdate=true`时，需要在Unity的`Update()`方法中调用`DriveUpdateService()`来触发网络回调，确保所有网络事件在主线程执行，避免线程安全问题。

```csharp
// PatchManager.Update()
void Update()
{
    if (isUpdating && mgr != null)
    {
        mgr.DriveUpdateService(); // 必须在主线程调用
    }
}
```

### Q5：如何监控当前网络状态？

```csharp
// 订阅网络状态变化
NetworkService.Instance.NetworkChangedEvent += (state) => {
    if (state == NetworkState.Disconnected)
    {
        // 触发重连逻辑
    }
};

// 主动查询
NetworkState current = NetworkService.Instance.GetNetworkState();
```

## 6. 总结

本项目的网络架构以**GCloud SDK**为核心底层，向上提供帧同步、连接管理、网络监控等能力，再往上由业务层的MSDK、PatchManager、ResDownloadWsClient等模块各司其职。整体设计遵循"接口抽象、分层解耦、Native高性能"的原则，是腾讯游戏标准网络架构的典型实现。

理解了这个总览，你就有了学习后续每个子系统的地图。
