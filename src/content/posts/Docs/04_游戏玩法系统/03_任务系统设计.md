## 1. 系统概述

任务系统（Task System）是引导玩家逐步探索游戏内容、完成目标并获得奖励的核心激励模块。在本项目中，任务系统以**剧本日程（Cultivation Schedule）**为核心载体，并结合新手引导（NewbieGuide）机制，共同构成玩家的目标指引体系。

在代码层面，`Task` 目录当前为空目录（预留扩展），实际的"任务-目标-奖励"逻辑主要分布在以下位置：

| 模块 | 路径 | 说明 |
|---|---|---|
| 新手引导 | `PlayerComponentSystem.cs` | 新手引导任务数据同步 |
| 养成日程 | `CultivationComponentSystem.ScriptSchedule.cs` | 每日/每周任务-日程安排 |
| 日程历史 | `CultivationComponentSystem.ScriptHistory.cs` | 任务完成历史记录 |
| 系统开关 | `PlayerComponentSystem.cs - SystemEntries` | 功能解锁/任务开关 |
| 剧情触发 | `StoryLogicCondition.cs` | 任务条件判断与触发 |

## 2. 架构设计

### 2.1 任务体系总览

本项目的任务分为三个层次：

```
任务体系
├── 新手引导任务（NewbieGuide）
│     ├── 数据：PlayerComponent.NewbieGuideData
│     ├── 完成标记：PlayerComponent.NewbieGuideCompeteFlag（BitArray）
│     └── 配置表：TbNewbieGuideConf
│
├── 剧本日程任务（CultivationSchedule）
│     ├── 当日任务（DayArrangement）：安排直播/训练/休息
│     ├── 夜晚任务（NightArrangement）：疗愈安排
│     ├── 比赛任务（MatchSchedule）：出战赛事
│     └── 历史记录：CultivationHistoryData
│
└── 系统功能任务（SystemEntry）
      ├── 数据：PlayerComponent.SystemEntries
      ├── 状态：ESystemEntryStatus（Locked/Unlocked/...）
      └── 驱动：服务端下发 PlayerSystemStatusInfoMsg
```

### 2.2 新手引导数据结构

```csharp
// 新手引导任务项
class NewbieGuideData
{
    int GuideID;        // 引导任务 ID
    bool IsCompleted;   // 是否已完成
    // ...
}

// PlayerComponent 中
List<NewbieGuideData>   NewbieGuideData;          // 引导列表
BitArray                NewbieGuideCompeteFlag;    // 按位存储的完成标记（节省内存）
```

使用 `BitArray` 而不是 `bool[]` 的原因：大量的新手引导步骤（通常有数百个）用 BitArray 存储，每个 bool 仅占 1 bit，大幅节省内存和序列化开销。

### 2.3 养成日程作为"任务"的设计

在养成剧本玩法中，每个"日程安排"实质上就是一个**阶段性任务**：

```
CultivationScheduleData（日程数据）
├── ScheduleID          当前日程 ID
├── StartScheduleID     起始日程 ID
├── LastMatchScheduleID 最后一个比赛日程 ID
├── EndScheduleID       结尾日程 ID
└── ScheduleDay         已过天数（任务进度）

日程类型（EScheduleType）：
├── Normal     养成日（安排直播/训练/休息）
└── Match      比赛日（出战）
```

每个日程的完成条件通过 `CultivationUtil.IsScheduleArrangeReady` 判断：只有当日所有安排确认后，才能推进到下一个日程（下一个"任务"）。

### 2.4 任务条件驱动：StoryLogicCondition

`StoryLogicCondition` 是整个任务/剧情触发判断的核心，它通过 `switch-case` 将条件枚举映射到具体的判断函数：

```
condition 枚举 → 判断函数
├── SL_SCRIPT        → 剧本进度条件
├── SL_GV            → 全局变量条件
├── SL_BATTLE        → 战斗结果条件
├── SL_SCHEDULE_CHARACTER → 日程安排角色条件
├── SL_IN_TEAM       → 队伍成员条件
├── ScheduleNum      → 日程天数条件
├── SL_PRESSURE_CHARACTER → 角色压力条件
├── SL_RELATION_CHARACTER → 角色关系条件
└── SL_STATE_CHARACTER    → 角色状态条件
```

### 2.5 系统功能解锁（任务驱动开关）

```csharp
// SystemEntryData
class SystemEntryData
{
    ESystemType         ID;      // 系统类型（如背包、公会、商城...）
    ESystemEntryStatus  Status;  // Locked/Unlocked/Hidden...
}
```

当玩家完成特定任务或到达特定等级时，服务端推送 `PlayerSystemStatusInfoMsg`，客户端通过 `SyncSystemInfoFromLogin` 或相关通知更新 `SystemEntries`，对应系统的入口按钮才会解锁显示。

### 3.1 新手引导数据同步

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Player\PlayerComponentSystem.cs

/// <summary>
/// 从登录数据同步玩家信息（包含新手引导）
/// </summary>
public static void SyncPlayerInfoFromLogin(this PlayerComponent self, PlayerInfoMsg info)
{
    if (info == null)
    {
        Log.LogError("[Player][Network] 返回的玩家数据为空");
        return;
    }

    // 基础信息
    self.RegistDatetime = (int)info.RegistDate;
    self.GID    = info.Gid;
    self.Name   = info.Name;
    self.Level  = info.Level;
    self.Gender = (EnumGenderType)info.Gender;

    // 体力信息
    self.PlayerPhysicalInfo = new PlayerPhysicalInfo
    {
        Physical         = info.PlayerPhysicalInfo?.Physical         ?? 0,
        LastPhysicalTime = info.PlayerPhysicalInfo?.LastPhysicalTime ?? 0,
        MaxPhysical      = info.PlayerPhysicalInfo?.MaxPhysical      ?? 0,
    };

    // 新手引导任务列表
    self.NewbieGuideData = ServerDataConverter.ToList<NewbieGuideMsg, NewbieGuideData>(
        info.NewbieGuideList, self.ConvertToNewbieGuide);

    // 用 BitArray 存储完成标记，节省内存
    self.NewbieGuideCompeteFlag = new BitArray(info.NewbieGuideCompeteFlag.ToByteArray());
}

/// <summary>
/// 同步系统功能开关（任务驱动的功能解锁）
/// </summary>
public static void SyncSystemInfoFromLogin(this PlayerComponent self, PlayerSystemInfoMsg info)
{
    self.SystemEntries = ServerDataConverter.ToMap(
        info?.SystemInfoList,
        GetSystemEntryFromServer,
        x => (int)x.ID);
}

private static SystemEntryData GetSystemEntryFromServer(PlayerSystemStatusInfoMsg info)
{
    return new SystemEntryData
    {
        ID     = (ESystemType)info.SystemId,
        Status = (ESystemEntryStatus)info.Status,
    };
}
```

```csharp
/// <summary>
/// 判断当前日程是否为首个日程
/// </summary>
public static bool IsFirstSchedule(this CultivationComponent self)
{
    var data = self.GetScheduleData();
    return data != null && self.GetScheduleID() == data.StartScheduleID;
}

/// <summary>
/// 判断当前日程是否为最后一个比赛日程
/// </summary>
public static bool IsLastMatchSchedule(this CultivationComponent self)
{
    var data = self.GetScheduleData();
    return data != null && self.GetScheduleID() == data.LastMatchScheduleID;
}

/// <summary>
/// 获取经过的日程天数（任务进度）
/// </summary>
public static int GetScheduleDay(this CultivationComponent self)
{
    var data = self.GetScheduleData();
    return data?.ScheduleDay ?? 0;
}

/// <summary>
/// 判断是否为白天安排阶段（任务可接受状态）

{
    return scheduleStage == (int)EScheduleStageType.DayArrangement;
```

### 3.3 任务条件判断（StoryLogicCondition）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Story\StoryLogic\StoryLogicCondition.cs

public static bool Evaluate(StoryComponent comp, condition conditionName, string conditionParam)
{
    switch (conditionName)
    {
        case condition.None:
            return true;                     // 无条件，直接通过

        case condition.SL_GV:
            return SL_GV(comp, conditionParam);   // 检查全局变量

        case condition.SL_SCRIPT:
            return SL_SCRIPT(comp, conditionParam); // 检查剧本进度

        case condition.SL_BATTLE:
            return SL_BATTLE(comp, conditionParam); // 检查战斗结果

        case condition.SL_SCHEDULE_CHARACTER:
            return SL_SCHEDULE_CHARACTER(comp, conditionParam); // 日程角色条件

        case condition.SL_IN_TEAM:
            return SL_IN_TEAM(comp, conditionParam); // 队伍成员条件

        case condition.ScheduleNum:
            return ScheduleNum(comp, conditionParam); // 日程天数条件

        case condition.SL_PRESSURE_CHARACTER:
            return SL_PRESSURE_CHARACTER(comp, conditionParam); // 角色压力条件

        case condition.SL_RELATION_CHARACTER:
            return SL_RELATION_CHARACTER(comp, conditionParam); // 角色关系

        case condition.SL_STATE_CHARACTER:
            return SL_STATE_CHARACTER(comp, conditionParam); // 角色状态

        default:
            Log.LogError("条件 = {0} 未开发", conditionName.ToString());
            return false;
    }
}
```

### 3.4 历史记录（任务完成存档）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Cultivation\
//   CultivationComponentSystem.ScriptHistory.cs

/// <summary>
/// 记录角色安排历史数据（任务完成后存档）
/// </summary>
public static void RecordCharacterScheduleHistory(this CultivationComponent self)
{
    // 比赛日不记录安排历史
    if (CultivationUtil.IsMatchScheduleType(self.GetScheduleType()))
        return;

    var (_, scheduleHistory) = self.GetOrAddScheduleHistory();
    if (scheduleHistory == null) return;

    var characterScheduleHistories = new List<CultivationCharacterScheduleHistoryData>();
    foreach (var character in self.GetCharacters())
    {
        var arrangeHistories = new List<CultivationCharacterArrangeHistoryData>();

        // 依次记录直播/训练/休息安排
        RecordSchedule(self.GetLiveSchedule(character.LiveID));
        RecordSchedule(self.GetTrainingSchedule(character.TrainingID));
        RecordSchedule(self.GetRestSchedule(character.RestID));

        characterScheduleHistories.Add(new CultivationCharacterScheduleHistoryData
        {
            CharacterIP             = character.IP,
            StoryCharacterID        = character.StoryCharacterID,
            ArrangeHistories        = arrangeHistories,
            IsLiveHost              = character.IsLiveHost,
        });

        continue;

        // 本地函数：记录单个安排
        void RecordSchedule(IScheduleDetailData scheduleData)
        {
            if (scheduleData == null) return;
            arrangeHistories.Add(new CultivationCharacterArrangeHistoryData
            {
                ScheduleType    = scheduleData.ScheduleType,
                ScheduleSubType = scheduleData.ScheduleSubType,
                ScheduleID      = scheduleData.ScheduleID,
            });
        }
    }

    scheduleHistory.CharacterScheduleHistories = characterScheduleHistories;
    // 同步记录日程观看角色
    scheduleHistory.ScheduleWatchingCharacter = self.GetScheduleWatchingCharacterIP();
}
```

## 4. 设计亮点

### 4.1 BitArray 高效存储新手引导完成标记
将数百个布尔标记压缩到一个 BitArray 中，内存占用缩减为原来的 1/8，序列化到网络协议时字节数也大幅减少。这是业界常见的优化手段，尤其适用于大量 bool 型标记的场景。

### 4.2 配置表驱动的任务条件
`StoryLogicCondition.Evaluate` 接受的参数 `conditionParam` 是字符串（以 `|` 分隔多个参数），策划可在配置表中自由组合条件，无需程序修改代码即可设计复杂的任务触发规则。

### 4.3 日程状态机
养成日程的推进本质上是一个**有限状态机**（FSM）：
```
DayArrangement → 各活动执行 → NightArrangement → 疗愈执行 → 下一个日程
```
通过 `EScheduleStageType` 枚举严格控制状态转换，杜绝了乱序执行的可能性。

### 4.4 历史记录作为任务凭证
每个完成的日程都会写入 `CultivationHistoryData`，既作为服务端结算的凭证，也支持玩家在"回忆"界面查看自己的养成历程，兼顾功能性和玩法体验。

## 5. 常见问题与最佳实践

### Q1: 如何新增一种任务条件类型？
1. 在 `condition` 枚举中添加新值（配置表同步更新）；
2. 在 `StoryLogicCondition.Evaluate` 的 switch 中添加对应 case；
3. 实现对应的判断函数 `private static bool YourCondition(StoryComponent comp, string param)`；
4. 在配置表中的条件字段填入新枚举名和参数即可触发。

### Q2: 新手引导的进度如何持久化？
新手引导的完成标记由 `NewbieGuideCompeteFlag`（BitArray）在本地维护，并在关键节点（通常是步骤完成时）通过网络协议同步到服务端。客户端不自行存储到本地文件，始终以服务端数据为权威来源。

### Q3: 日程天数（ScheduleDay）如何计算？
`ScheduleDay` 由服务端在每次日程推进时计算并下发，客户端只负责展示。`IsFirstSchedule()` 和 `IsLastSchedule()` 是客户端根据 ID 比较实现的快捷判断，并不自行计算天数。

### Q4: 任务奖励是如何发放的？
任务完成后，服务端通过 `ZoneInfoChangeNotify` 通知背包数据变更（或通过邮件发放），客户端的 `BackpackComponentSystem.SyncItemListFromInfoChangeNotify` 处理后，触发 `Evt_BackpackDataUpdated` 事件，UI 层监听该事件展示奖励弹窗。

### Q5: 如何查看当前系统功能是否已解锁？
```csharp
// 检查指定系统是否已解锁
var systemEntry = player.SystemEntries.GetValueOrDefault((int)ESystemType.YourSystem);
bool isUnlocked = systemEntry?.Status == ESystemEntryStatus.Unlocked;
```
如果 `SystemEntries` 中不存在对应记录，说明服务端尚未下发，应视为未解锁。
