# 06 活动与运营系统框架

## 1. 系统概述

活动与运营系统是游戏保持长期活跃度的核心驱动力，通常包括：
- **限时活动**：节日活动、签到、累充活动等；
- **版本运营**：新版本功能的分批开放；
- **赛季制度**：PVP 赛季、养成剧本更新；
- **系统功能开关**：控制哪些功能对哪些玩家开放。

在本项目中，活动与运营系统以 **系统功能开关（SystemEntry）** 为基础骨架，结合 **养成剧本（CultivationScript）** 作为核心运营周期载体，配合 **延迟通知（DelayedNotification）** 机制实现时间节点触发。

## 2. 架构设计

### 2.1 系统功能开关（SystemEntry）

`SystemEntry` 是本项目运营系统的"总开关"机制，每个游戏功能都有一个对应的枚举值，服务端控制其开放状态：

```csharp
// 系统功能入口数据
class SystemEntryData
{
    ESystemType         ID;      // 系统 ID（背包、商城、公会、活动...）
    ESystemEntryStatus  Status;  // Locked / Unlocked / Hidden / Coming
}
```

**运营控制流程：**
```
运营配置（后台）改变功能状态
         │
         ▼
服务端推送 PlayerSystemStatusInfoMsg
         │
         ▼
SyncSystemInfoFromLogin / 实时通知处理
         │
         ▼
PlayerComponent.SystemEntries 更新
         │
         ▼
UI 层查询 SystemEntries，决定按钮显隐/解锁/预告
```

### 2.2 养成剧本作为运营周期

每个养成剧本（`CultivationScript`）代表一段完整的运营周期（类比"赛季"），包含：

```
CultivationCycleScriptDetailData（循环本）
├── ScriptID          剧本 ID（运营周期标识）
├── Name              剧本名称（版本名称）
├── StartTime         开始时间（活动开始）
├── EndTime           结束时间（活动结束）
├── Status            EScriptStatus（未开始/进行中/已结束）
└── ...

CultivationStoryScriptDetailData（剧情本）
├── ScriptID          剧本 ID
├── IsUnlocked        是否已解锁（玩家条件）
└── ...
```

通过 `CultivationComponentSystem.Entry.cs` 提供的查询接口，UI 层可以获取所有"当前开放的活动剧本"列表，展示活动大厅。

### 2.3 延迟通知系统（DelayedNotification）

延迟通知机制用于在特定时间点推送给玩家（如"距活动结束还有1小时"的提醒）：

```
DelayedNotification
├── 定时器管理：基于 ETTask 协程延迟执行
├── 通知内容：文字提示 / 弹窗 / 红点
└── 触发时机：由服务端时间戳驱动，避免客户端时钟偏差
```

### 2.4 活动系统的标准架构模式

根据项目现有代码规律，活动系统的标准扩展模式为：

```
活动系统架构（推荐范式）
├── XxxActivityComponent              活动数据组件
│     ├── IsActive: bool              是否正在进行
│     ├── StartTime / EndTime: long   活动时间
│     ├── Progress: int               进度
│     └── RewardClaimed: BitArray     已领取奖励标记
│
├── XxxActivityComponentSystem        活动逻辑系统
│     ├── RequestActivityInfo()       请求活动数据
│     ├── SyncFromServer()            同步服务端数据
│     ├── ClaimReward(index)          领取奖励
│     └── IsRewardClaimed(index)      检查是否已领取
│
└── XxxActivityEvtHandler             活动事件处理
      └── 监听活动状态变化事件，驱动 UI 更新
```

## 3. 核心代码展示

### 3.1 系统功能开关同步

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Player\PlayerComponentSystem.cs

/// <summary>
/// 同步系统功能开关（运营核心：控制所有功能的开放状态）
/// </summary>
public static void SyncSystemInfoFromLogin(this PlayerComponent self, PlayerSystemInfoMsg info)
{
    // 将服务端下发的系统状态列表转换为本地字典（key=SystemID）
    self.SystemEntries = ServerDataConverter.ToMap(
        info?.SystemInfoList,
        GetSystemEntryFromServer,
        x => (int)x.ID);
}

private static SystemEntryData GetSystemEntryFromServer(PlayerSystemStatusInfoMsg info)
{
    return new SystemEntryData
    {
        ID     = (ESystemType)info.SystemId,
        Status = (ESystemEntryStatus)info.Status,
    };
}
```

### 3.2 活动剧本列表获取

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Cultivation\
//   CultivationComponentSystem.Entry.cs

/// <summary>
/// 获取所有满足条件的循环本（活动列表查询）
/// </summary>
public static IEnumerable<CultivationCycleScriptDetailData> GetCycleScripts(
    this CultivationComponent self,
    Func<CultivationCycleScriptDetailData, bool> predicate = null)
{
    var scripts = self.CycleScripts;
    if (scripts == null)
    {
        Log.LogError("[Cultivation] 养成入口循环本列表为空");
        yield break;
    }

    foreach (var script in scripts)
    {
        // 满足条件（如：只返回"进行中"的活动）才 yield
        if (predicate == null || predicate(script))
            yield return script;
    }
}

/// <summary>
/// 获取首个满足条件的循环本（当前活动快捷入口）
/// </summary>
public static CultivationCycleScriptDetailData GetCycleScript(
    this CultivationComponent self,
    Func<CultivationCycleScriptDetailData, bool> predicate = null)
{
    foreach (var script in self.GetCycleScripts(predicate))
        return script;
    return null;
}

/// <summary>
/// 按 ID 精确获取剧本（活动详情页）
/// </summary>
public static CultivationCycleScriptDetailData GetCycleScript(
    this CultivationComponent self, int scriptID)
{
    return self.GetCycleScript(x => x.ScriptID == scriptID);
}

/// <summary>
/// 获取所有满足条件的剧情本（解锁剧情活动）
/// </summary>
public static IEnumerable<CultivationStoryScriptDetailData> GetStoryScripts(
    this CultivationComponent self,
    Func<CultivationStoryScriptDetailData, bool> predicate = null)
{
    var scripts = self.StoryScripts;
    if (scripts == null)
    {
        Log.LogError("[Cultivation] 养成入口剧情本列表为空");
        yield break;
    }

    foreach (var script in scripts)
    {
        if (predicate == null || predicate(script))
            yield return script;
    }
}
```

### 3.3 请求活动剧本列表（网络层）

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Cultivation\
//   CultivationComponentSystem.Server.cs

/// <summary>
/// 请求剧本（活动）列表
/// </summary>
public static async ETTask<bool> RequestCultivateScriptInfo(this CultivationComponent self)
{
    Log.LogInfo("[Cultivation][Network] 协议 ZoneCsCultivateScriptInfo 请求开始");

    var req = new ZoneCultivateScriptInfoReq();
    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneCultivateScriptInfoResp>(
            (uint)ZoneClientCmd.ZoneCsCultivateScriptInfo, req);

    if (nr.IsCompleteSuccess)
    {
        Log.LogInfo("[Cultivation][Network] 协议 ZoneCsCultivateScriptInfo 接收成功");
        self.SyncFromCultivateScriptInfo(nr.Data);
        return true;
    }

    Log.LogError("[Cultivation][Network] 协议 ZoneCsCultivateScriptInfo 接收失败");
    return false;
}

/// <summary>
/// 服务端实时推送养成数据变更
/// </summary>
[MessageHandler(SceneType.Client)]
public class ZoneCultivateNotifyHandler : MessageHandler<Scene, ZoneCultivateNotify>
{
    protected override async ETTask Run(Scene scene, ZoneCultivateNotify resp)
    {
        await ETTask.CompletedTask;
        Log.LogInfo("[Cultivation][Network] 养成通知接收成功");

        var comp = scene.Cultivation();
        comp.SyncFromCultivateNotify(resp);   // 实时更新活动状态
    }
}
```

## 4. 设计亮点

### 4.1 服务端权威的活动时间控制
所有活动的开始/结束时间由服务端时间戳控制，客户端只做展示换算，杜绝了因客户端本地时间被篡改而提前参与活动的风险。

### 4.2 配置表驱动的活动内容
活动奖励、活动规则等内容全部来自配置表（`TbCultivation*` 系列），运营人员修改配置后无需发版即可调整活动内容，与代码完全解耦。

### 4.3 `IEnumerable` + 谓词过滤的活动列表接口
活动列表的查询接口接受 `Func<T, bool> predicate` 谓词参数，调用方可以灵活定义过滤条件（只取"进行中"、只取"未参与"等），接口设计极具扩展性，无需为每种场景单独写一个查询方法。

### 4.4 系统开关的细粒度控制
`ESystemEntryStatus` 支持多种状态（不只是 Locked/Unlocked），还可以设置为 `Hidden`（玩家看不到入口）或 `Coming`（显示"即将开放"预告），为运营提供了精细化控制能力。

## 5. 常见问题与最佳实践

### Q1: 如何添加一个全新的限时活动？
1. 策划在配置表中定义活动 ID、时间、奖励等；
2. 服务端实现活动逻辑和状态管理；
3. 在 `ESystemType` 中添加新活动的枚举值；
4. 客户端创建 `XxxActivityComponent` 和 `XxxActivityComponentSystem`；
5. 在登录/活动大厅界面通过 `RequestActivityInfo` 拉取活动数据；
6. UI 根据 `SystemEntries` 决定是否显示活动入口。

### Q2: 活动结束后数据如何清理？
- 如果是周期性活动（如每周签到），服务端在新周期开始时重置数据并通知客户端；
- 客户端在收到重置通知后，清空对应 Component 的数据；
- 为避免遗留旧数据，建议在 Component 的 `Awake` 时初始化为安全默认值。

### Q3: 如何判断当前是否有正在进行的活动？
```csharp
// 方法一：通过 SystemEntry 检查
var entry = player.SystemEntries.GetValueOrDefault((int)ESystemType.YourActivity);
bool isActive = entry?.Status == ESystemEntryStatus.Unlocked;

// 方法二：通过剧本状态检查
var activeScript = cultivation.GetCycleScript(
    s => s.Status == EScriptStatus.InProgress);
bool hasActiveActivity = activeScript != null;
```

### Q4: 运营活动的红点提示如何实现？
1. 订阅活动数据更新事件；
2. 在事件处理中检查是否有未领取奖励（遍历 `RewardClaimed` BitArray）；
3. 通过全局红点管理系统（通常是 `RedDotComponent`）设置对应节点的红点状态；
4. UI 层绑定红点节点，数据驱动显示/隐藏。

### Q5: PVP 赛季切换如何处理？
赛季切换由服务端推送赛季更新通知，客户端重新请求 `PVPMainInfoReq` 刷新赛季数据。为避免玩家在赛季切换时界面残留旧数据，需要在 `PVPArenaComponent.Clear()` 中完整清空所有赛季相关字段。
