## 1. 系统概述

背包系统（Backpack System）负责管理玩家持有的所有**道具（Item）**，包括：
- **虚拟道具（Virtual Items）**：货币类，如金币、钻石、体力值，**不占格子**，直接以数量存储；
- **实体道具（Bag Items）**：占据背包格子的普通道具，如材料、道具卡、召唤券等。

## 2. 架构设计

### 2.1 数据结构

```
BackpackComponent（背包组件）
├── VirtualItems: Dictionary<int, ItemData>   虚拟道具（key=ItemID）
└── BagItems:     Dictionary<int, ItemData>   实体道具（key=ItemID，内部以多层 Map 管理）

ItemData（道具数据）
├── ItemID:   int       道具配置 ID
├── Pos:      int       背包槽位（服务端分配）
├── Count:    long      数量
└── ...（其他扩展属性）
```

### 2.2 虚拟道具与实体道具的区分

```csharp
// 通过 ItemHelper.IsVirtualItem(itemId) 判断
// 虚拟道具：货币、体力等，通过 SetVirtualItemCount 直接更新数量
// 实体道具：有槽位概念，通过 BagItems 字典管理，数量为 0 时移除
```

### 2.3 道具数据更新流程

```
服务端推送 ZoneInfoChangeNotify
         │
         ▼
BackpackComponentSystem.SyncItemListFromInfoChangeNotify()
  ├── 遍历 ItemNotify 列表
  ├── 判断是虚拟道具还是实体道具
  ├── 虚拟道具 → SetVirtualItemCount()
  ├── 实体道具已存在 → 更新数量（数量<=0 则移除）
  └── 实体道具不存在且数量>0 → 新增到 BagItems
         │
         ▼
发布 Evt_BackpackDataUpdated { UpdatedItemIds }
         │
         ▼
UI 层监听事件，刷新背包界面
```

### 2.4 奖励通知（AwardNotify）

`AwardNotify` 是一个专用的奖励展示数据结构，分为三类：
- `AwardItem`：直接发放到背包的奖励
- `MailItem`：通过邮件发放的奖励（背包满或特殊情况）
- `ShowItem`：仅展示但不实际发放的道具（如战斗过场动画中的收集物）

## 3. 核心代码展示

### 3.1 登录时同步道具列表

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Backpack\BackpackComponentSystem.cs

/// <summary>
/// 从登录数据同步物品列表
/// </summary>
public static void SyncItemList(
    this BackpackComponent self,
    RepeatedField<ItemMsg> virtualItems,
    RepeatedField<ItemMsg> mainBagItemList)
{
    // 虚拟道具：以 ItemID 为 key 建立字典
    self.VirtualItems = ServerDataConverter.ToMap(
        virtualItems,
        ServerDataConverter.GetItemData,
        x => x.ItemID);

    // 实体道具：通过专用方法建立多层 Map（支持堆叠）
    self.BagItems = ServerDataConverter.ToItemMap(
        mainBagItemList,
        ServerDataConverter.GetItemData,
        x => x.ItemID);
}
```

### 3.2 实时更新道具数据

```csharp
/// <summary>
/// 同步服务端推送的道具变更通知
/// </summary>
public static void SyncItemListFromInfoChangeNotify(
    this BackpackComponent self, ZoneInfoChangeNotify info)
{
    var itemList = info?.ItemNotify;
    if (itemList == null) return;

    var updatedIds = new List<int>();
    if (self.BagItems == null)
        self.BagItems = new Dictionary<int, ItemData>();

    foreach (var item in itemList)
    {
        if (item != null && item.Id > 0)
            updatedIds.Add(item.Id);

        if (ItemHelper.IsVirtualItem(item.Id))
        {
            // 虚拟道具：直接设置数量
            self.SetVirtualItemCount(item.Id, item.Count);
        }
        else if (self.BagItems.ContainsKey(item.Id))
        {
            if (item.Count <= 0)
            {
                // 数量归零：从背包移除
                self.BagItems.Remove(item.Id);
            }
            else
            {
                // 更新已有道具
                ServerDataConverter.GetItemData(item, self.BagItems[item.Id]);
            }
        }
        else
        {
            // 新增道具（只有数量 > 0 才加入）
            if (item.Count > 0)
                self.BagItems.Add(item.Id, ServerDataConverter.GetItemData(item));
        }
    }

    // 通知 UI 刷新
    self.ClientScene().GetComponent<EventDispatcherComponent>()
        ?.FireEvent(new Evt_BackpackDataUpdated
        {
            UpdatedItemIds = updatedIds
        });
}
```

### 3.3 使用道具（网络请求）

```csharp
/// <summary>
/// 请求使用道具
/// </summary>
public static async ETTask<bool> RequestItemUse(
    this BackpackComponent self, ItemData item, int count)
{
    Log.LogInfo("[Backpack][Network] 协议 ZoneCsItemUse 请求开始");

    // 记录使用前的体力，用于后续奖励展示
    var energyCount = self.Player().GetCurrentEnergyCount();

    var req = new ZoneItemUseReq
    {
        Pos      = item.Pos,      // 服务端槽位标识
        UseCount = count,
    };

    var nr = await self.ClientScene()
        .GetComponent<NetworkComponent>()
        .SendAsync<ZoneItemUseResp>((uint)ZoneClientCmd.ZoneCsItemUse, req);

    if (nr.IsCompleteSuccess)
    {
        Log.LogInfo("[Backpack][Network] 协议 ZoneCsItemUse 接收成功");

        var useCount = nr.Data?.UseCount ?? 0;

        // 发布道具使用事件（包含奖励信息，供 UI 弹窗展示）
        EventSystem.Instance.Publish(self.ClientScene(), new Evt_BackpackItemUsed
        {
            ItemID        = item.ItemID,
            UseCount      = useCount,
            AwardNotify   = nr.Data?.AwardNotify,
            PrevEnergyCount = energyCount,
        });
        return true;
    }

    Log.LogError("[Backpack][Network] 协议 ZoneCsItemUse 接收失败");
    return false;
}
```

### 3.4 奖励通知数据查询

```csharp
// X:\UnityProj\Assets\Scripts\Hotfix\Function\GamePlay\Player\Backpack\AwardNotifyExtensions.cs

/// <summary>
/// 判断奖励通知中是否包含指定道具（三个来源都检查）
/// </summary>
public static bool ContainsItem(this AwardNotify self, int itemID)
{
    return ContainsItem(self.AwardItem, itemID) ||
           ContainsItem(self.MailItem,  itemID) ||
           ContainsItem(self.ShowItem,  itemID);
}

/// <summary>
/// 获取奖励通知中指定道具的总数量
/// </summary>
public static long GetItemTotalCount(this AwardNotify self, int itemID)
{
    var count = 0L;
    count += GetItemTotalCount(self.AwardItem, itemID);
    count += GetItemTotalCount(self.MailItem,  itemID);
    count += GetItemTotalCount(self.ShowItem,  itemID);
    return count;
}

/// <summary>
/// 获取指定道具的变更数量（增减量，用于飘字效果）
/// </summary>
public static long GetItemChangeCount(this AwardNotify self, int itemID)
{
    var count = 0L;
    count += GetItemChangeCount(self.AwardItem, itemID);
    count += GetItemChangeCount(self.MailItem,  itemID);
    count += GetItemChangeCount(self.ShowItem,  itemID);
    return count;
}

private static bool ContainsItem(IReadOnlyList<ItemMsg> items, int itemID)
{
    if (items == null) return false;
    foreach (var item in items)
        if (item.Id == itemID) return true;
    return false;
}
```

## 4. 设计亮点

### 4.1 虚拟道具与实体道具分离存储

将货币类道具（虚拟）和普通背包道具（实体）分开存储，是因为两者的更新逻辑不同：
- 虚拟道具只有数量变化，用简单的 `SetCount` 即可；
- 实体道具有槽位概念，需要处理新增/更新/删除三种情况。

分离存储使代码逻辑清晰，也便于 UI 层分别订阅不同的更新事件。

### 4.2 增量更新而非全量刷新

`SyncItemListFromInfoChangeNotify` 只处理**变更的道具**，而不是每次都重新同步全部背包数据。这显著减少了每帧/每次操作的数据处理量，对性能友好。

### 4.3 事件驱动的 UI 更新

背包数据更新后，通过 `Evt_BackpackDataUpdated { UpdatedItemIds }` 通知 UI 层，UI 层只需刷新 `UpdatedItemIds` 中指定的格子，而不是全部重绘，进一步减少渲染开销。

### 4.4 AwardNotify 三路汇总

奖励通知拆分为 `AwardItem/MailItem/ShowItem` 三路，应对不同的发放场景（背包满发邮件、演出效果仅展示不入库），扩展方法统一汇总三路数据，上层代码无需关心具体来源。

### 4.5 使用前记录体力快照

在调用 `RequestItemUse` 前先记录 `PrevEnergyCount`，用于在奖励弹窗中展示"使用恢复丹前后体力变化"的对比，提升玩家操作反馈体验。

## 5. 常见问题与最佳实践

### Q1: 如何判断玩家是否拥有指定道具？
```csharp
// 虚拟道具
long count = backpack.GetVirtualItemCount(itemId);

// 实体道具
bool has = backpack.BagItems.ContainsKey(itemId);
long bagCount = backpack.BagItems.TryGetValue(itemId, out var data) ? data.Count : 0;
```

### Q2: 道具使用失败会不会扣道具？
不会。`RequestItemUse` 是 C/S 交互，由**服务端权威判断**。服务端返回失败（`!nr.IsCompleteSuccess`）时，客户端不修改任何本地数据，等待下一次服务端推送来同步正确状态。

### Q3: 背包数据在什么时候初始化？
登录成功后，服务端在登录响应（`ZoneCsLogin` 回包）中携带完整的背包数据，客户端调用 `SyncItemList` 一次性初始化。后续所有变更通过 `ZoneInfoChangeNotify` 增量推送。

### Q4: 为什么槽位（Pos）的概念只在服务端使用？
客户端背包以 `ItemID` 作为 key 存储，实际上是无序的；`Pos` 只在发起"使用道具"请求时传给服务端，服务端用来定位玩家背包中的具体道具。客户端的 UI 排序由 UI 层自行处理，与服务端槽位无关。

### Q5: 如何添加一种新的道具使用效果？
1. 服务端实现对应的道具使用逻辑并下发 `AwardNotify`；
2. 客户端监听 `Evt_BackpackItemUsed` 事件，根据 `ItemID` 判断是否需要特殊处理；
3. 如需展示特殊动效或界面，在事件处理器中发布对应的 UI 事件即可；
4. **背包系统本身不需要修改**，符合开闭原则。
