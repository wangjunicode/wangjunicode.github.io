# 战斗状态动作处理器与格挡弹反机制

在动作游戏中，格挡是核心战斗机制之一。普通格挡减伤但仍会被打飞，完美格挡（parry）不仅无伤还能弹反敌人，会心弹反则造成额外伤害。这些机制背后，是一套精密的状态动作处理器系统。

本文深入解析 `FSMActionTaskHandler` 模式以及各个战斗状态的具体实现。

---

## 第一性原理：状态动作处理器的职责分层

在 FSM 系统中，状态的"显示什么"（节点图配置）和"执行什么"（处理器代码）是分离的：

```
FSM 节点图（策划配置）
    └── State Node：格挡状态
         └── ActionTask：ParryStateAction（数据节点）
              └── AFSMActionTaskHandler（执行者）
                       └── ParryStateActionHandler.OnExecute()
```

`ActionTask`：数据节点，包含这个动作需要的参数（如按键类型、跳字类型、技能配置）

`AFSMActionTaskHandler`：执行者，实现 `OnExecute` 和 `OnStop`，包含具体的业务逻辑

这种分离让：
- 策划可以在节点图中配置"进入格挡状态时执行 ParryStateAction"
- 程序在 `ParryStateActionHandler` 中实现格挡的具体逻辑
- 两者互不依赖，分别迭代

## ParryStateActionHandler：格挡状态处理器

```csharp
public class ParryStateActionHandler : AFSMActionTaskHandler<ParryStateAction>
{
    protected override void OnExecute(ParryStateAction task)
    {
        // 根据进入格挡状态的类型，设置不同的参数
        switch (task.ActionType)
        {
            case EActionType.Parry:
                // 普通格挡：受伤但减少击退
                task.InputKey = EInputKey.Hurt_Parry;
                task.JumpText = JumpTextType.BlockDamage;
                task.SubSkillCfg = true;   // 使用技能配置的受击参数
                break;
            case EActionType.SwordPlay:
                // 完美格挡（剑术弹反）：完美时机触发，弹反攻击者
                task.InputKey = EInputKey.SwordPlay;
                task.JumpText = JumpTextType.PerfectBlocked;
                task.SubSkillCfg = false;  // 不使用受击参数，自有动画
                break;
            case EActionType.CriticalHurt:
                // 会心弹反：弹反时触发会心效果
                task.InputKey = EInputKey.Hurt_Critical;
                task.JumpText = JumpTextType.CriticalDamage;
                task.SubSkillCfg = true;
                break;
        }
        
        base.OnExecute(task);  // 基类通用初始化
        var unit = task.Agent<Unit>();
        
        // 注册受击事件监听（用于实时响应攻击命中时的格挡处理）
        unit.GetComponent<EventDispatcherComponent>()
            .RegisterEvent<Evt_HurtHit>(task.OnHit);

        // 查找当前角色对应的技能配置
        var vkeySkillConf = CfgManager.tables.TbVKeySkill.GetOrDefault(unit.ConfigId);
        unit.GetComponent<PhysicsComponent>().Stop();  // 停止现有运动
        
        if (vkeySkillConf != null)
        {
            if (vkeySkillConf.SkillMap.TryGetValue(task.InputKey, out var skillID))
            {
                var hurtComp = unit.GetComponent<HurtComponent>();
                var subskillConfig = hurtComp.AttackInfo.SubSkillConfigClone;
                
                // 启动格挡技能（播放格挡动画、粒子等）
                HurtSystem.CastSkill(task, unit, skillID, subskillConfig).Coroutine();
                
                // 如果已经命中了（同帧发生的情况），立刻处理
                if (hurtComp.AttackInfo.Hit)
                {
                    task.OnHit(default);
                }
            }
        }
    }

    protected override void OnStop(ParryStateAction task, bool interrupted)
    {
        var unit = task.Agent<Unit>();
        // 离开格挡状态时，注销受击监听
        unit.GetComponent<EventDispatcherComponent>()
            .UnRegisterEvent<Evt_HurtHit>(task.OnHit);
        base.OnStop(task, interrupted);
    }
}
```

### 关键细节解析

**① 三种格挡类型**

| 类型 | 触发条件 | 效果 | 跳字 |
|------|---------|------|------|
| Parry | 普通格挡 | 减伤 + 微击退 | BlockDamage |
| SwordPlay | 完美格挡窗口内 | 无伤 + 弹反攻击者 | PerfectBlocked |
| CriticalHurt | 完美弹反的会心版本 | 无伤 + 弹反 + 暴击 | CriticalDamage |

**② 受击事件监听时序**

```csharp
// 进入格挡状态时：注册
unit.GetComponent<EventDispatcherComponent>().RegisterEvent<Evt_HurtHit>(task.OnHit);

// 离开格挡状态时：注销
unit.GetComponent<EventDispatcherComponent>().UnRegisterEvent<Evt_HurtHit>(task.OnHit);
```

格挡状态"有效期"正好等于注册到注销这段时间。在此期间，任何命中都会触发 `OnHit`，执行格挡的具体处理（减伤、弹反等）。超出这段时间的命中，按正常受击处理。

**③ 同帧命中的特殊处理**

```csharp
// 已经命中了（同帧发生），立刻处理
if (hurtComp.AttackInfo.Hit)
{
    task.OnHit(default);
}
```

攻击可能在同一帧内完成：碰撞检测发现命中 → 进入格挡状态 → 已经命中了。这种情况下不等下一帧的事件，立刻执行格挡处理。

## 格挡命中处理（OnHit）

```csharp
public static void OnHit(this ParryStateAction task, Evt_HurtHit evtHurt)
{
    var unit = task.Agent<Unit>();
    var hurtComp = unit.GetComponent<HurtComponent>();
    var attacker = hurtComp.AttackInfo.Attacker;
    
    // 处理格挡受击：顿帧 + 特效，但不切换受击状态
    HurtSystem.HurtMoveAndFreezeNoCastSkill(
        attacker, unit,
        task.JumpText,
        task.SubSkillCfg ? evtHurt.SubSkill : null,  // 完美格挡不用受击配置
        true,   // enableEffect：显示特效
        true,   // enableFreeze：允许顿帧
        true,   // enableHitPower：允许击退（但格挡状态的击退通常很小）
        EHurtResult.ParryHurt
    );
}
```

`HurtMoveAndFreezeNoCastSkill` 是专为"格挡中受击"设计的变体：
- **不切换技能/动画**：角色保持在格挡动画中，而不是切换到受击动画
- **只处理物理效果**：顿帧、轻微击退
- 通过 `EHurtResult.ParryHurt` 标记结果，让伤害计算系统知道此次命中是被格挡的

## HurtActionHandler：受击状态处理器

```csharp
public class HurtActionHandler : AFSMActionTaskHandler<HurtAction>
{
    private readonly FP FloatHeight = FP.Half;  // 0.5 单位高度
    
    protected override void OnExecute(HurtAction task)
    {
        base.OnExecute(task);
        var unit = task.Agent<Unit>();
        var hurtComp = unit.GetComponent<HurtComponent>();
        unit.GetComponent<PhysicsComponent>().Stop();
        
        var subskillConfig = hurtComp.AttackInfo.SubSkillConfigClone;
        if (subskillConfig != null)
        {
            var subskillParam = subskillConfig.GetSubSkillParam(
                unit.GetComponent<FSMComponent>().CurrentLeafStateTag());
            
            // 决定进入哪种受击状态：浮空 or 硬直
            bool shouldFloat = subskillParam.PowerY > 0  // 有向上的击退力
                            || unit.Position.y >= FloatHeight;  // 已经在空中
            
            if (shouldFloat)
            {
                unit.GetComponent<FSMComponent>().TryManualCondition(EInputKey.Hurt_Float);  // 进入浮空受击
            }
            else
            {
                unit.GetComponent<FSMComponent>().TryManualCondition(EInputKey.Hurt_Stiff);  // 进入地面硬直
            }
        }
        
        task.EndAction();  // 立刻结束，让状态机切换到对应子状态
    }
}
```

受击状态处理器做了一个关键决策：**浮空受击 vs 地面硬直**。

判断逻辑：
1. 攻击有向上击退力（`PowerY > 0`）→ 进入浮空
2. 角色已经离地（`Position.y >= 0.5`）→ 进入浮空
3. 否则 → 地面硬直

这两种受击状态对应不同的动画（浮空有特殊的飞起动画）和不同的后续状态（浮空后要落地、起身）。

## DeadActionHandler：死亡状态处理器

```csharp
public class DeadActionHandler : AFSMActionTaskHandler<DeadAction>
{
    protected override void OnExecute(DeadAction task)
    {
        base.OnExecute(task);
        var unit = task.Agent<Unit>();
        var fsmComp = unit.GetComponent<FSMComponent>();
        
        // 增加死亡计数
        fsmComp.DeadCnt++;
        var firstDead = fsmComp.DeadCnt <= 1;
        
        // 通过黑板告知技能图是否是第一次死亡
        unit.GetComponent<BlackboardComponent>().SetInt("firstDead", firstDead ? 1 : 0);
        
        var vkeySkillConf = CfgManager.tables.TbVKeySkill.GetOrDefault(unit.ConfigId);
        unit.GetComponent<PhysicsComponent>().Stop();
        
        if (vkeySkillConf != null)
        {
            if (vkeySkillConf.SkillMap.TryGetValue(EInputKey.Dead, out var skillID))
            {
                var hurtComp = unit.GetComponent<HurtComponent>();
                var subskillConfig = hurtComp.AttackInfo.SubSkillConfigClone;
                HurtSystem.CastSkill(task, unit, skillID, subskillConfig).Coroutine();
            }
        }
    }
}
```

`DeadCnt`（死亡计数）很有意思：计数器而非布尔值。

原因：某些角色有"复活"机制，死亡后可以复活，然后再次死亡。如果是布尔值，无法区分是第一次死亡还是第二次。

第一次死亡 (`firstDead = true`) 可能有特殊的死亡动画或触发特殊被动技能（"临死"效果）；后续死亡则使用普通死亡动画。

## FloatStateActionHandler：浮空状态处理器

```csharp
public class FloatStateActionHandler : AFSMActionTaskHandler<FloatStateAction>
{
    protected override void OnExecute(FloatStateAction task)
    {
        base.OnExecute(task);
        var unit = task.Agent<Unit>();
        var hurtComp = unit.GetComponent<HurtComponent>();
        var subskillConfig = hurtComp.AttackInfo.SubSkillConfigClone;
        
        // 启动浮空受击技能（播放飞起动画）
        HurtSystem.CastSkill(task, unit, skillID, subskillConfig).Coroutine();
        
        // 注册落地检测（角色落地时触发 LieDown）
        unit.GetComponent<EventDispatcherComponent>()
            .RegisterEvent<Evt_UnitGrounded>(task.OnGrounded);
    }
    
    protected override void OnStop(FloatStateAction task, bool interrupted)
    {
        unit.GetComponent<EventDispatcherComponent>()
            .UnRegisterEvent<Evt_UnitGrounded>(task.OnGrounded);
        base.OnStop(task, interrupted);
    }
}

public static class FloatStateActionSystem
{
    public static void OnGrounded(this FloatStateAction task, Evt_UnitGrounded evt)
    {
        // 落地时切换到倒地状态
        var unit = task.Agent<Unit>();
        unit.GetComponent<FSMComponent>().TryManualCondition(EInputKey.LieDown);
    }
}
```

浮空状态必须感知"何时落地"，通过监听 `Evt_UnitGrounded` 事件实现——物理系统检测到角色接触地面时发布此事件，浮空状态处理器收到后切换到倒地状态。

## 完美格挡的时机检测

```csharp
public class DogeStateActionHandler : AFSMActionTaskHandler<DogeStateAction>
{
    protected override void OnExecute(DogeStateAction task)
    {
        // ... 
        // 进入闪避状态时：注册"预判攻击"检测
        colliderComp.SetPreDogeColliders(...);  // 激活闪避预判碰撞盒
        
        // 闪避状态内，持续检测是否有攻击进入判定范围
        // 如果在完美闪避窗口内检测到攻击 → 触发完美闪避效果
    }
}
```

完美格挡/完美闪避的时机检测原理：
1. 进入格挡/闪避状态时，激活专用的"预判碰撞盒"（与受击盒不同）
2. 技能节点每帧检测这个预判盒是否与攻击盒重叠
3. 如果在特定时间窗口内（"完美格挡窗口"）检测到重叠 → 触发完美格挡效果
4. 窗口外的重叠 → 普通格挡

这套机制让"完美格挡"真正依赖玩家的时机把握，而不是简单的随机或固定判定。

## TbVKeySkill：配置驱动的技能绑定

```csharp
// 配置表中的按键-技能映射（策划可配）
var vkeySkillConf = CfgManager.tables.TbVKeySkill.GetOrDefault(unit.ConfigId);

if (vkeySkillConf.SkillMap.TryGetValue(EInputKey.Hurt_Parry, out var skillID))
{
    // 播放该角色的格挡受击技能（动画、特效）
    HurtSystem.CastSkill(task, unit, skillID, subskillConfig).Coroutine();
}
```

每个角色的格挡动画、死亡动画等都通过配置表绑定，而不是硬编码。这样：
- 不同角色可以有不同的格挡动画风格（重剑格挡 vs 轻盾格挡）
- 添加新角色不需要修改处理器代码
- 策划可以直接调整每个角色的受击技能 ID

## 处理器的注册机制

`AFSMActionTaskHandler` 是所有状态动作处理器的基类：

```csharp
public abstract class AFSMActionTaskHandler<T> where T : ActionTask
{
    protected abstract void OnExecute(T task);  // 进入状态时执行
    protected virtual void OnStop(T task, bool interrupted) { }  // 离开状态时执行
    
    // 基类注册逻辑：将处理器与 ActionTask 类型关联
    public static void Register()
    {
        AFSMActionTaskHandlerFactory.Register<T>(new ParryStateActionHandler());
    }
}
```

系统启动时扫描所有处理器类，自动注册。当 FSM 节点执行 `ParryStateAction` 时，系统自动找到对应的 `ParryStateActionHandler` 并调用。

## 设计总结

状态动作处理器系统的核心价值：

| 特性 | 实现方式 |
|------|---------|
| 配置与代码分离 | ActionTask（数据）+ Handler（执行）分层 |
| 事件驱动的状态感知 | 注册/注销事件监听随状态生命周期 |
| 配置驱动的技能绑定 | TbVKeySkill 表，不同角色不同技能 |
| 状态子类型决策 | 运行时判断进入浮空还是地面硬直 |
| 复活状态追踪 | DeadCnt 计数而非布尔值 |

```
FSM 状态转换
    ↓
ActionTask 节点被激活
    ↓
AFSMActionTaskHandler.OnExecute
    ├── 停止物理运动
    ├── 注册状态内事件监听
    ├── 播放对应技能/动画
    └── 可能立刻决策子状态（浮空/硬直）

状态转换离开时
    ↓
AFSMActionTaskHandler.OnStop
    └── 注销事件监听（必须！防止内存泄漏）
```

这套系统让战斗状态的行为完全可配置、可扩展，是商业动作游戏开发中的重要工程实践。
