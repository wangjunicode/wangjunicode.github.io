# 09_多平台热更差异处理

> 面向刚入行的毕业生 · 建议搭配 PatchManager.cs 和 ClientBridge.cs 阅读

---

## 1. 系统概述

移动游戏需要同时支持 Android 和 iOS 两个平台，而这两个平台在热更新方面存在显著差异。本文档梳理本项目在多平台热更处理上的具体实现和设计决策。

### 1.1 Android vs iOS 热更差异一览

| 维度 | Android | iOS |
|------|---------|-----|
| 程序包更新 | 下载 APK → 自动安装 | 跳转 App Store |
| 资源更新 | 直接下载到 P 目录 | 直接下载到 P 目录 |
| 代码热更 | 完全支持（Assembly.Load） | 完全支持（HybridCLR） |
| 文件系统 | 区分大小写 | 区分大小写（注意！） |
| 包体限制 | 宽松 | App Store 审核敏感 |
| 渠道策略 | 多渠道（各大应用商店） | 单渠道（App Store） |
| 灰度支持 | 支持（Dolphin） | 有限支持 |
| 预发布环境 | 通过文件切换 | 通过环境判断 |

### 1.2 编辑器（Windows/Mac）的特殊处理

编辑器作为开发环境，热更新流程被跳过，但保留了 Addressables 资源管理能力，以保证开发体验接近发布版本。

## 3. 核心代码展示

### 3.1 isSkipPatch 平台差异

```csharp
// PatchManager.cs - 各平台默认热更开关
#if !UNITY_EDITOR && UNITY_ANDROID
    public bool isSkipPatch = false;  // Android 真机：不跳过热更
#else
    public bool isSkipPatch = true;   // iOS、Editor、Windows：跳过热更
#endif
```

**注意**：iOS 目前走 `else` 分支，`isSkipPatch = true`！如果 iOS 需要热更，必须修改此处，或通过 Deep Link 参数 `Enable_Patch=true` 动态开启。

### 3.4 CDN URL 的平台差异（审核机制）

```csharp
// PatchManager.cs
private void AppUpdate()
{
    // ...
#if PRODUCTION_ENVIRONMENT
    #if UNITY_IOS
        // iOS：通过 isAuditVersion 或 DeploymentEnvironment 判断审核/正式服
        info.gameUpdateUrl = IsReviewEnvironment() ? DOPHIN_PRE_UPDATEURL : DOPHIN_UPDATEURL;
    
    #elif UNITY_ANDROID
        // Android：额外支持通过本地文件强制切换到预发布环境
        var predownloadPath = Path.Combine(Application.persistentDataPath, "predownloadtest.txt");
        if (File.Exists(predownloadPath))
        {
            info.gameUpdateUrl = DOPHIN_PRE_UPDATEURL; // 强制预发布
        }
        else
        {
            info.gameUpdateUrl = IsReviewEnvironment() ? DOPHIN_PRE_UPDATEURL : DOPHIN_UPDATEURL;
        }
    #endif
#endif
}
```

**iOS 审核机制说明**：App Store 审核时，游戏需要连接到审核环境（预发布服），避免审核员看到尚未发布的内容。`IsReviewEnvironment()` 通过读取 `Resources/DeploymentEnvironment.txt` 来判断：
- 文件内容为 `"Review"` → 使用 `DOPHIN_PRE_UPDATEURL`（审核/预发布 CDN）
- 其他 → 使用 `DOPHIN_UPDATEURL`（正式 CDN）

### 3.5 APK 安装的 Android 专属实现

```csharp
// PatchManager.cs - Android 专属：调用 Java 层安装 APK
private bool InstallApk(string path)
{
    AndroidJavaClass jc = new AndroidJavaClass("com.unity3d.player.UnityPlayer");
    if (jc == null) return false;

    AndroidJavaObject m_jo = jc.GetStatic<AndroidJavaObject>("currentActivity");
    if (m_jo == null) return false;

    // 调用腾讯 Dolphin 的 Java 安装接口
    AndroidJavaClass clazz = new AndroidJavaClass("com.tencent.gcloud.dolphin.CuIIPSMobile");
    if (clazz == null) return false;

    int result = clazz.CallStatic<int>("installAPK", path, m_jo);
    
    PatchLog.Info(result != 0 ? $"InstallApk Fail: {result}" : "InstallApk Success");
    return result == 0;
}

// Android 专属：从后台恢复时重新触发安装（用户可能在安装过程中切出 App）
void OnApplicationPause(bool pause)
{
    if (!pause && !string.IsNullOrEmpty(m_ApkPath))
    {
        OnNoticeInstallApk(m_ApkPath);
    }
}
```

### 3.7 DLL 读取的平台差异（Android 真机特殊处理）

```csharp
// ClientBridge.cs
public static byte[] ReadDllBytes(string path, bool decrypt = true)
{
    string fullPath = Path.Combine(GetHotUpdateDllPath(), path);

    if (!File.Exists(fullPath))
    {
        fullPath = Path.Combine(Application.streamingAssetsPath, "HotUpdateDll", path);
    }
    else
    {
#if !UNITY_EDITOR && UNITY_ANDROID
        // Android 真机：persistentDataPath 的文件需要 "file:///" 前缀
        fullPath = ZString.Concat("file:///", fullPath);
#endif
    }

    return ReadAndDecryption(fullPath, decrypt);
}
```

```csharp
// 文件读取的平台兼容性
static MemoryStream ReadFileToMemoryStream(string path, bool logError = false)
{
#if UNITY_EDITOR || UNITY_64
    // 编辑器和 64 位平台：直接 File.ReadAllBytes
    if (File.Exists(path))
    {
        var content = File.ReadAllBytes(path);
        return new MemoryStream(content, 0, content.Length, true, true);
    }
#else
    // 其他平台：使用 UnityWebRequest（兼容 Android APK 内的 StreamingAssets）
    var www = UnityWebRequest.Get(path);
    www.SendWebRequest();
    while (!www.isDone) { }
    
    if (!string.IsNullOrEmpty(www.error))
    {
        Debug.LogError(www.error);
        return null;
    }
    
    var content = www.downloadHandler.data;
    var ms = new MemoryStream(content, 0, content.Length, true, true);
    www.Dispose();
    return ms;
#endif
}
```

**为什么 Android StreamingAssets 需要 UnityWebRequest？** Android APK 是 zip 格式，`StreamingAssets` 目录的文件被打包在 APK 内部，`File.ReadAllBytes` 无法直接访问，必须通过 `UnityWebRequest` 解压读取。

## 4. 设计亮点

### 4.1 环境切换的"后门"机制（仅 Android）

通过在手机的 `persistentDataPath` 下创建 `predownloadtest.txt` 文件，可以让线上正式包自动连接预发布 CDN：

```
手机本地文件系统：
/data/data/<包名>/files/predownloadtest.txt  ← 存在该文件则切换到预发布
```

这个机制让测试人员无需重新打包即可在正式包上验证预发布内容，非常实用。

### 4.2 iOS App Store 审核自动适配

通过 `Resources/DeploymentEnvironment` 文本资源，iOS 审核包自动使用预发布 CDN，避免审核员看到尚未正式发布的游戏内容，降低审核风险。

### 4.3 编辑器平台保持一致性

编辑器虽然跳过了热更（`isSkipPatch = true`），但 Addressables 的路径重定向依然生效，开发者可以在编辑器中测试热更资源：

```csharp
// TransformResourcePath 在编辑器中同样生效

// Addressables 就会优先使用热更版本
```

## 5. 常见问题与最佳实践

### Q1：Android Debug 包和 Release 包有什么区别？
**A**：
- Debug 包（`#if DEBUG`）：使用 `UPDATECHANNELID_DEV = 839714826`，连接开发渠道，方便测试热更
- Release 包（`#if !DEBUG`）：使用 `UPDATECHANNELID_PUB = 1879060255`，连接正式渠道

在 Unity Build Settings 中，勾选 "Development Build" 对应 Debug 宏。

### Q2：iOS 为什么不能直接安装 IPA？
**A**：Apple 的 iOS 系统出于安全考虑，不允许应用自行安装其他应用。只能通过 App Store 或企业证书分发。因此 iOS 的大版本更新只能引导用户去 App Store 更新，而不像 Android 可以静默下载安装 APK。

### Q3：多平台 Bundle 必须分开吗？
**A**：必须。Android 和 iOS 的 AssetBundle 格式不兼容（图形 API、纹理格式、着色器编译不同）。Addressables 在打包时会为每个平台生成独立的 Bundle 文件，存放在不同的平台目录（Android/、iOS/）下。

### Q4：在 Windows 编辑器上开发，如何测试 Android 版本的热更 DLL？
**A**：使用 `HotUpdatePushWindow` 中的平台选择功能：
1. 在窗口中选择平台为 Android
2. 点击"热更编译"（调用 HybridCLR 交叉编译 Android DLL）
3. 将编译产物推送到 Android 真机测试

注意：Windows 编辑器不能直接运行 Android 的 HybridCLR DLL，交叉编译只能在真机上验证。

### Q5：如何处理 iOS/Android 版本不同步的情况？
**A**：AppConfig 中为 iOS 和 Android 分别维护版本号：
```csharp
public class AppConfig
{
    public string AndroidSourceVersion { get; set; }
    public string iOSSourceVersion { get; set; }
}
```
可以对 iOS 和 Android 分别推送不同版本的热更包，互不影响。在 GCloud 控制台也可以对不同平台配置不同的 ChannelId 进行独立管理。
