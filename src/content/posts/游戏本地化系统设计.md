## 为什么本地化比想象中复杂

```
表面上看，本地化 = 翻译文字

实际上包含：
  ✦ 文本翻译：语言文本
  ✦ 字体适配：中文/阿拉伯语/泰语需要不同字体
  ✦ UI 布局：有些语言文字比中文长 2~3 倍（德语！）
  ✦ 图片本地化：带文字的图片需要为每种语言单独准备
  ✦ 音频本地化：配音、语音包
  ✦ 文化差异：颜色含义、数字禁忌、节日习俗
  ✦ 文字方向：阿拉伯语是从右到左（RTL）
  ✦ 数字格式：欧洲用"1.000,00"，美国用"1,000.00"
  ✦ 日期格式：中国 2024年3月15日，美国 March 15, 2024
```

## 一、Unity Localization 包

### 1.1 安装与配置

```
安装：
  Package Manager → Unity Registry → 搜索 "Localization" → Install

配置：
  1. Edit → Project Settings → Localization
  2. 创建一个 Locale（语言）集合
  3. 添加支持的语言：Simplified Chinese (zh-Hans), English (en), 等

创建字符串表（String Table）：
  Assets → Create → Localization → String Table Collection
  - 给每个 Key 分配不同语言的翻译值
```

### 1.2 代码中使用本地化字符串

```csharp
using UnityEngine;
using UnityEngine.Localization;
using UnityEngine.Localization.Settings;
using UnityEngine.Localization.Tables;

// 方式1：异步获取（推荐）
public async UniTask<string> GetLocalizedStringAsync(string tableName, string key)
{
    var table = await LocalizationSettings.StringDatabase
        .GetTableAsync(tableName).Task;
    var entry = table.GetEntry(key);
    return entry?.GetLocalizedString() ?? key;
}

// 方式2：同步获取（简单但第一次可能慢）
public string GetLocalizedString(string tableName, string key)
{
    return LocalizationSettings.StringDatabase
        .GetLocalizedString(tableName, key);
}

// 方式3：带参数的本地化（使用 Smart String 格式）
// 在 String Table 中设置："{name} 获得了 {amount} 金币"
public string GetLocalizedStringWithArgs(string key, params object[] args)
{
    return LocalizationSettings.StringDatabase
        .GetLocalizedString("GameUI", key, args);
}

// 使用：
var msg = GetLocalizedStringWithArgs("GoldPickup", 
    new Dictionary<string, object> { ["name"] = playerName, ["amount"] = 100 });

// 方式4：LocalizedString 组件（Inspector 中配置）
[SerializeField] private LocalizedString _titleString;

void Start()
{
    // 订阅语言变化事件（语言切换时自动更新）
    _titleString.StringChanged += OnStringChanged;
    _titleString.RefreshString();
}

void OnStringChanged(string newValue)
{
    titleText.text = newValue;
}
```

## 二、本地化管理器架构

### 2.1 自定义本地化管理器

```csharp
/// <summary>
/// 游戏本地化管理器：封装 Unity Localization，提供统一接口
/// </summary>
public class LocalizationManager : Singleton<LocalizationManager>
{
    // 当前语言变化时通知
    public event Action<SystemLanguage> OnLanguageChanged;
    
    // 支持的语言列表
    public static readonly SystemLanguage[] SupportedLanguages = 
    {
        SystemLanguage.ChineseSimplified,
        SystemLanguage.ChineseTraditional,
        SystemLanguage.English,
        SystemLanguage.Japanese,
        SystemLanguage.Korean,
    };
    
    public SystemLanguage CurrentLanguage { get; private set; }
    
    protected override void Init()
    {
        // 从 PlayerPrefs 读取上次选择的语言
        string savedLang = PlayerPrefs.GetString("Language", "");
        
        if (System.Enum.TryParse<SystemLanguage>(savedLang, out var savedLanguage))
            SetLanguage(savedLanguage);
        else
            SetLanguage(GetSystemLanguage()); // 使用系统语言
    }
    
    /// <summary>
    /// 切换语言
    /// </summary>
    public async UniTask SetLanguageAsync(SystemLanguage language, CancellationToken ct = default)
    {
        if (CurrentLanguage == language) return;
        
        CurrentLanguage = language;
        
        // 保存到 PlayerPrefs
        PlayerPrefs.SetString("Language", language.ToString());
        PlayerPrefs.Save();
        
        // 通知 Unity Localization 系统切换
        var locale = GetLocale(language);
        if (locale != null)
        {
            LocalizationSettings.SelectedLocale = locale;
            // 等待语言加载完成
            await LocalizationSettings.InitializationOperation.Task;
        }
        
        OnLanguageChanged?.Invoke(language);
    }
    
    /// <summary>
    /// 获取本地化字符串
    /// </summary>
    public string Get(string key, string tableName = "GameUI")
    {
        return LocalizationSettings.StringDatabase.GetLocalizedString(tableName, key);
    }
    
    /// <summary>
    /// 获取带参数的本地化字符串
    /// </summary>
    public string Get(string key, params object[] args)
    {
        return LocalizationSettings.StringDatabase.GetLocalizedString("GameUI", key, args);
    }
    
    private SystemLanguage GetSystemLanguage()
    {
        var lang = Application.systemLanguage;
        // 如果系统语言不在支持列表中，使用英语
        return System.Array.IndexOf(SupportedLanguages, lang) >= 0 
            ? lang 
            : SystemLanguage.English;
    }
    
    private Locale GetLocale(SystemLanguage language)
    {
        // 映射到 Unity Localization 的 Locale
        return language switch
        {
            SystemLanguage.ChineseSimplified => GetLocaleByCode("zh-Hans"),
            SystemLanguage.ChineseTraditional => GetLocaleByCode("zh-Hant"),
            SystemLanguage.English => GetLocaleByCode("en"),
            SystemLanguage.Japanese => GetLocaleByCode("ja"),
            SystemLanguage.Korean => GetLocaleByCode("ko"),
            _ => GetLocaleByCode("en")
        };
    }
    
    private Locale GetLocaleByCode(string code)
    {
        return LocalizationSettings.AvailableLocales.Locales
            .FirstOrDefault(l => l.Identifier.Code == code);
    }
    
    private void SetLanguage(SystemLanguage language)
    {
        CurrentLanguage = language;
        var locale = GetLocale(language);
        if (locale != null)
            LocalizationSettings.SelectedLocale = locale;
    }
}
```

## 三、字体与文字排版适配

### 3.1 字体管理

```csharp
/// <summary>
/// 字体管理器：根据语言自动切换字体
/// </summary>
[CreateAssetMenu(fileName = "FontSettings", menuName = "Localization/Font Settings")]
public class FontSettings : ScriptableObject
{
    [System.Serializable]
    public struct LanguageFont
    {
        public SystemLanguage Language;
        public TMP_FontAsset RegularFont;
        public TMP_FontAsset BoldFont;
        public TMP_FontAsset TitleFont;
    }
    
    [SerializeField] private LanguageFont[] _languageFonts;
    [SerializeField] private LanguageFont _fallbackFont; // 兜底字体
    
    public TMP_FontAsset GetFont(SystemLanguage language, FontStyle style = FontStyle.Normal)
    {
        var lang = System.Array.Find(_languageFonts, f => f.Language == language);
        if (lang.Language != language) lang = _fallbackFont;
        
        return style switch
        {
            FontStyle.Bold => lang.BoldFont ?? lang.RegularFont,
            _ => lang.RegularFont
        };
    }
}

// LocalizedText 组件：自动跟随语言变化
[RequireComponent(typeof(TextMeshProUGUI))]
public class LocalizedText : MonoBehaviour
{
    [SerializeField] private string _tableKey;
    [SerializeField] private string _key;
    [SerializeField] private FontSettings _fontSettings;
    
    private TextMeshProUGUI _text;
    
    void Awake()
    {
        _text = GetComponent<TextMeshProUGUI>();
        LocalizationManager.Instance.OnLanguageChanged += OnLanguageChanged;
        RefreshText();
    }
    
    void OnDestroy()
    {
        if (LocalizationManager.Instance != null)
            LocalizationManager.Instance.OnLanguageChanged -= OnLanguageChanged;
    }
    
    private void OnLanguageChanged(SystemLanguage language)
    {
        RefreshText();
        
        // 切换字体
        if (_fontSettings != null)
            _text.font = _fontSettings.GetFont(language);
    }
    
    private void RefreshText()
    {
        _text.text = LocalizationManager.Instance.Get(_key, _tableKey);
    }
}
```

### 3.2 阿拉伯语 RTL 支持

```csharp
// TextMesh Pro 支持 RTL（Right-to-Left）
public class RTLSupport : MonoBehaviour
{
    private TextMeshProUGUI _text;
    
    void Start()
    {
        _text = GetComponent<TextMeshProUGUI>();
        LocalizationManager.Instance.OnLanguageChanged += OnLanguageChanged;
        SetRTLMode(LocalizationManager.Instance.CurrentLanguage);
    }
    
    void OnLanguageChanged(SystemLanguage language) => SetRTLMode(language);
    
    void SetRTLMode(SystemLanguage language)
    {
        bool isRTL = language == SystemLanguage.Arabic || language == SystemLanguage.Hebrew;
        _text.isRightToLeftText = isRTL;
        
        // 对 RTL 语言，文字对齐方式也需要调整
        if (isRTL)
            _text.alignment = TextAlignmentOptions.Right;
        else
            _text.alignment = TextAlignmentOptions.Left;
    }
}
```

## 四、本地化工作流程

### 4.1 翻译工作流

```
本地化工作流程（以 Excel 为中心）：

1. 程序导出字符串
   代码/Inspector → 扫描所有 LocalizedString 的 Key → 导出为 Excel
   
2. 交给翻译团队
   Excel 格式（Key | 中文 | English | 日本語 | 한국어）
   
3. 翻译团队填写
   - 每种语言一列
   - 保留 {参数名} 占位符不翻译
   
4. 导入回 Unity
   Excel → Import → 自动更新 String Table
   
5. UI 测试
   - 检查是否有文字溢出
   - 检查是否有未翻译（显示为 Key 的）
   - RTL 语言检查布局
```

### 4.2 本地化 CI 检查

```python
# localization_check.py：检查本地化覆盖率的脚本
import json
import os

def check_localization_coverage():
    """检查所有语言的翻译覆盖率"""
    
    # 加载字符串表
    tables = load_string_tables("Assets/Localization")
    
    base_lang = "zh-Hans"  # 基准语言
    base_keys = set(tables[base_lang].keys())
    
    results = {}
    for lang, entries in tables.items():
        if lang == base_lang:
            continue
        
        lang_keys = set(entries.keys())
        missing = base_keys - lang_keys
        empty = {k for k, v in entries.items() if not v.strip()}
        
        coverage = (len(base_keys) - len(missing)) / len(base_keys)
        results[lang] = {
            "coverage": coverage,
            "missing": list(missing)[:10],  # 只显示前10个
            "empty_values": list(empty)[:10]
        }
        
        print(f"{lang}: {coverage:.1%} ({len(missing)} missing, {len(empty)} empty)")
        
        if coverage < 0.95:  # 覆盖率低于 95%，CI 失败
            print(f"❌ {lang} 覆盖率不足！")
            return False
    
    print("✅ 所有语言覆盖率检查通过")
    return True

if __name__ == "__main__":
    if not check_localization_coverage():
        exit(1)
```

## 五、本地化常见坑点

### 5.1 文字长度问题

```
中文 "确认" = 2 个字（约 20px）
英文 "Confirm" = 7 个字母（约 50px）
德文 "Bestätigen" = 10 个字母（约 70px）
俄文 "Подтвердить" = 12 个字母（约 80px）

解决方案：
1. UI 设计阶段就考虑多语言，留足空间
2. 使用 AutoSize 字体大小（TMP 的自动缩小功能）
3. 对特别长的语言允许换行
4. 和 UI 设计师协商：按英文长度 × 1.5 设计按钮宽度
```

```csharp
// 自动字体大小：当文字放不下时自动缩小
var text = GetComponent<TextMeshProUGUI>();

text.fontSizeMin = 8;   // 最小字号
text.fontSizeMax = 24;  // 最大字号
```

```
英文有单复数：1 item, 2 items
俄语有三种形式：1 предмет, 2 предмета, 5 предметов
阿拉伯语有六种形式！

Smart String 的复数支持：
```

### 5.3 日期/数字格式

```csharp
// ❌ 硬编码格式
string dateStr = DateTime.Now.ToString("yyyy年MM月dd日"); // 只适合中文

// ✅ 使用当前语言的格式

CultureInfo GetCultureInfo(SystemLanguage language)
{
    return language switch
    {
        SystemLanguage.ChineseSimplified => new CultureInfo("zh-CN"),
        SystemLanguage.ChineseTraditional => new CultureInfo("zh-TW"),
        SystemLanguage.English => new CultureInfo("en-US"),
        SystemLanguage.Japanese => new CultureInfo("ja-JP"),
        _ => CultureInfo.InvariantCulture
    };
}
```

## 总结

本地化系统的工程要点：

1. **从立项开始就考虑本地化**：后期加本地化代价极高
2. **Key 命名规范**：`模块名.组件名.语义`，如 `Battle.DamageLog.CriticalHit`
3. **预留 UI 空间**：按英文长度 ×1.5 设计布局
4. **自动化检查**：CI 阶段检查翻译覆盖率，防止漏翻
5. **翻译 = 工具链**：导出、协作、导入要全流程自动化，减少人工操作

**对技术负责人的建议**：本地化是跨职能协作（程序/UI/翻译/测试），需要尽早建立工作流程并形成文档，避免后期混乱。
