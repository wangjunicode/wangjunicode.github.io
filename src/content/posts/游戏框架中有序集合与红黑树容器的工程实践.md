# 游戏框架中有序集合与红黑树容器的工程实践

## 一、为什么游戏需要自己实现有序集合

Unity 早期版本的 .NET 运行时并不包含 `SortedSet<T>` 和完整的 `SortedDictionary<T,V>`，或者其行为与服务端 .NET 存在细微差异。ET 框架选择将 .NET BCL 的源码直接复制到项目中，原因如下：

1. **Unity 版本锁定**：不同 Unity 版本内置 Mono/.NET 版本差异大，自带的容器类可能缺少某些方法。
2. **热更新兼容**：HybridCLR 热更新程序集中直接引用自定义命名空间下的 `ET.SortedSet<T>`，可以在热更包中随时修改容器行为而不受引擎约束。
3. **行为一致性**：服务端（.NET 7+）和客户端（Unity Mono）使用同一份代码，避免跨端逻辑差异。

## 二、红黑树基础原理

`SortedSet<T>` 和 `SortedDictionary<TKey,TValue>` 都基于**红黑树（Red-Black Tree）**实现。

### 2.1 红黑树四条性质

```
1. 每个节点是红色或黑色
2. 所有叶节点（nil）是黑色
3. 红节点的两个子节点必须是黑色（不能有连续红节点）
4. 任意节点到其后代叶节点的每条路径，黑色节点数相同（黑高相等）
```

这四条性质保证了红黑树的**高度 ≤ 2·log₂(n+1)**，因此 Add/Remove/Contains 的时间复杂度都是 **O(log n)**。

### 2.2 框架中的节点颜色与旋转类型

```csharp
public enum NodeColor : byte
{
    Black,
    Red
}

public enum TreeRotation : byte
{
    Left,
    LeftRight,
    Right,
    RightLeft
}
```

使用 `byte` 枚举而非 `bool`，既节省内存（Node 数量可能很大），又通过命名提升可读性。四种旋转类型对应红黑树调整的全部情况：

| 旋转类型 | 触发条件 | 说明 |
|----------|----------|------|
| `Left` | 父右子为红 | 单左旋 |
| `Right` | 父左子为红 | 单右旋 |
| `LeftRight` | 父左子右孙为红 | 先左旋后右旋 |
| `RightLeft` | 父右子左孙为红 | 先右旋后左旋 |

### 2.3 插入平衡核心逻辑

```csharp
private void InsertionBalance(Node current, ref Node parent,
    Node grandParent, Node greatGrandParent)
{
    bool parentIsOnRight = grandParent.Right == parent;
    bool currentIsOnRight = parent.Right == current;

    Node newChildOfGreatGrandParent;
    if (parentIsOnRight == currentIsOnRight)
    {
        // 同向：单旋转
        newChildOfGreatGrandParent = currentIsOnRight
            ? grandParent.RotateLeft()
            : grandParent.RotateRight();
    }
    else
    {
        // 异向：双旋转
        newChildOfGreatGrandParent = currentIsOnRight
            ? grandParent.RotateLeftRight()
            : grandParent.RotateRightLeft();
        parent = greatGrandParent;
    }

    grandParent.ColorRed();
    newChildOfGreatGrandParent.ColorBlack();
    ReplaceChildOrRoot(greatGrandParent, grandParent, newChildOfGreatGrandParent);
}
```

这是标准的**自顶向下插入**策略：在搜索路径上遇到 4-节点时主动分裂（`Split4Node`），确保插入位置的父节点不是 4-节点，避免回溯。

## 三、SortedSet 的核心功能

### 3.1 基本操作

```csharp
var set = new SortedSet<int>();
set.Add(5);    // O(log n)
set.Add(3);
set.Add(8);
set.Add(1);

// 自动有序遍历：1 3 5 8
foreach (var item in set) Console.WriteLine(item);

set.Remove(3);         // O(log n)
bool has = set.Contains(5); // O(log n)
int min = set.Min;     // O(log n)，最小值
int max = set.Max;     // O(log n)，最大值
```

### 3.2 高效集合运算

```csharp
var a = new SortedSet<int> { 1, 2, 3, 4 };
var b = new SortedSet<int> { 3, 4, 5, 6 };

a.UnionWith(b);          // a = {1,2,3,4,5,6}  并集
a.IntersectWith(b);      // a = {3,4}           交集
a.ExceptWith(b);         // a = {1,2}           差集
a.SymmetricExceptWith(b); // a = {1,2,5,6}      对称差

// 针对同比较器的 SortedSet，使用合并排序优化，O(M+N)
// 普通 IEnumerable 情况下降为 O(M·log N)
```

### 3.3 视图切片 GetViewBetween

这是 `SortedSet` 的一个独特特性，在游戏中非常实用：

```csharp
var timers = new SortedSet<long> { 100, 200, 300, 500, 800, 1000 };

// 获取 200ms ~ 600ms 范围内的定时器（返回视图，非副本）
var view = timers.GetViewBetween(200, 600);
// view = {200, 300, 500}

// 视图是实时的：对原集合的修改会反映到视图中
timers.Add(400);
// view 现在是 {200, 300, 400, 500}
```

视图通过 `TreeSubSet`（内部类）实现，记录范围边界 `[Min, Max]`，迭代时只遍历范围内节点，**无需复制数据**。

### 3.4 从排序数组高效构造

```csharp
// ConstructRootFromSortedArray：将有序数组直接构造为平衡红黑树
// 时间复杂度 O(n)，比逐个 Add 的 O(n log n) 更高效
var data = new int[] { 1, 2, 3, 4, 5, 6, 7 };
var set = new SortedSet<int>(data); // 内部走 ConstructRootFromSortedArray
```

构造算法取数组中点为根，递归处理左右两半，奇数段用黑节点，偶数段右中间元素用红节点补齐，保证树的平衡性。

## 四、TreeSet — 严格去重的 SortedSet

```csharp
public sealed class TreeSet<T> : SortedSet<T>
{
    internal override bool AddIfNotPresent(T item)
    {
        bool ret = base.AddIfNotPresent(item);
        if (!ret)
        {
            throw new ArgumentException($"SR.Argument_AddingDuplicate, {item}");
        }
        return ret;
    }
}
```

`TreeSet<T>` 只重写了 `AddIfNotPresent`：当元素已存在时，`SortedSet` 返回 `false` 静默忽略，而 `TreeSet` 则**抛异常**。

这是 `SortedDictionary` 的内部实现依赖：字典语义下，重复 Key 是编程错误而非正常情况，应及时暴露而非静默丢弃。

## 五、SortedDictionary — 基于红黑树的有序字典

### 5.1 内部结构

```csharp
public class SortedDictionary<TKey, TValue>
    : IDictionary<TKey, TValue>, IDictionary, IReadOnlyDictionary<TKey, TValue>
{
    private readonly TreeSet<KeyValuePair<TKey, TValue>> _set;
    // ...
}
```

`SortedDictionary` 并不直接持有红黑树节点，而是**将 `KeyValuePair<TKey,TValue>` 存入 `TreeSet`**，通过 `KeyValuePairComparer` 只比较 Key 部分：

```csharp
public sealed class KeyValuePairComparer : Comparer<KeyValuePair<TKey, TValue>>
{
    internal IComparer<TKey> keyComparer;

    public override int Compare(KeyValuePair<TKey, TValue> x, KeyValuePair<TKey, TValue> y)
    {
        return keyComparer.Compare(x.Key, y.Key);
    }
}
```

这样复用了 `SortedSet` 的全部红黑树逻辑，实现了 **DRY 原则**。

### 5.2 按序遍历

```csharp
var dict = new SortedDictionary<string, int>
{
    ["banana"] = 2,
    ["apple"]  = 1,
    ["cherry"] = 3
};

// 自动按 Key 的字母序输出
foreach (var kv in dict)
{
    Console.WriteLine($"{kv.Key}: {kv.Value}");
}
// apple: 1
// banana: 2
// cherry: 3
```

### 5.3 修改已有 Value（不触发重排）

```csharp
// 更新 Value 时直接修改节点的 Item 字段，无需删除再插入
node.Item = new KeyValuePair<TKey, TValue>(node.Item.Key, value);
_set.UpdateVersion(); // 版本号递增，使现有枚举器失效
```

这是一个精心设计的优化：修改 Value 不影响 Key 的排序，所以节点位置不变，仅更新 `Item` 即可。

## 六、游戏中的典型应用场景

### 6.1 定时器调度

```csharp
// 用 SortedSet<long> 存储所有触发时间，快速找到最近到期的定时器
private SortedSet<long> _timerSet = new SortedSet<long>();

void AddTimer(long triggerTime)
{
    _timerSet.Add(triggerTime);
}

long GetNextTrigger()
{
    return _timerSet.Min; // O(log n)
}

void Tick(long now)
{
    // 获取所有已到期的定时器（视图切片，无复制）
    var expired = _timerSet.GetViewBetween(0, now);
    foreach (var t in expired.ToList())
    {
        FireTimer(t);
        _timerSet.Remove(t);
    }
}
```

### 6.2 排行榜

```csharp
// 用 SortedDictionary<int, List<Player>> 按分数排序
// Key = 分数（降序需要自定义 Comparer），Value = 同分玩家列表
var leaderboard = new SortedDictionary<int, List<Player>>(
    Comparer<int>.Create((a, b) => b.CompareTo(a)) // 降序
);
```

### 6.3 技能优先队列

```csharp
// 按执行优先级排序的技能队列
// priority 相同时按插入顺序（加 id 后缀区分）
var skillQueue = new SortedDictionary<long, SkillRequest>();
// Key = priority * 1000000 + skillId，保证唯一性
```

### 6.4 地图区域对象管理

```csharp
// 按 X 坐标排序的实体集合，快速查询视野范围内的对象
var entities = new SortedDictionary<float, Entity>();
var inView = entities
    .GetViewBetween(playerX - viewRange, playerX + viewRange); // 近似
```

## 七、与 Dictionary / HashSet 的选型对比

| 特性 | `Dictionary` / `HashSet` | `SortedDictionary` / `SortedSet` |
|------|--------------------------|----------------------------------|
| 插入/删除 | O(1) 均摊 | O(log n) |
| 查找 | O(1) 均摊 | O(log n) |
| 有序遍历 | 不保证顺序 | ✅ 始终有序 |
| Min/Max | 需全遍历 O(n) | O(log n) |
| 范围查询 | 不支持 | ✅ `GetViewBetween` |
| 内存占用 | 哈希表（稀疏）| 红黑树节点（每节点约 40 字节） |

**选型建议：**
- 需要频繁 Min/Max 查询 → `SortedSet` 或 `SortedDictionary`
- 需要范围遍历（如定时器、地图范围） → `SortedSet.GetViewBetween`
- 只需快速存取，不关心顺序 → `Dictionary` / `HashSet`（性能更好）
- 定时器场景中元素数量 < 1000 → `SortedSet`；数量极大时考虑跳表或 Fibonacci Heap


通过 `#if UNITY` 条件编译（注释掉的形式，表示已合并到主代码），补充了高版本 .NET 才有的接口，使代码在低版本 Unity Mono 运行时中也能通过编译。

## 九、总结

ET 框架的 `SortedSet<T>` 和 `SortedDictionary<TKey,TValue>` 是完整的红黑树实现移植，核心设计亮点：

1. **红黑树自包含**：不依赖 BCL，跨 Unity 版本稳定运行
2. **TreeSet 严格去重**：字典语义下的内部实现，重复 Key 立即报错
3. **KVP 复用策略**：`SortedDictionary` 将 KVP 存入 `TreeSet`，极致复用红黑树逻辑
4. **视图切片零拷贝**：`GetViewBetween` 返回 `TreeSubSet` 视图，无内存开销
5. **有序构造优化**：从排序数组构造时直接建树，O(n) vs 逐个插入的 O(n log n)

在游戏工程中，`SortedSet` 是定时器、技能队列、范围查询等场景的首选数据结构，理解其红黑树本质有助于在性能敏感路径上做出正确选型。
