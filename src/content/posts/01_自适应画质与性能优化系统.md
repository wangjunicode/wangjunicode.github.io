# 自适应画质与性能优化系统（AdaptiveLOD）

## 1. 系统概述

移动端游戏面临的最大挑战之一是**机型碎片化**——高端旗舰和低端机器的性能差距可能超过10倍。本项目通过 **AdaptiveLodMgr**（自适应 LOD 管理器）在运行时动态感知设备性能，自动调整画质档位，确保所有机型流畅运行。

**核心策略：**
- 实时监控 FPS 和发热状态
- FPS 低于阈值时自动降配（降分辨率、关闭特效）
- FPS 稳定后自动升配恢复画质
- 发热严重时强制降配（保护设备）

```

    ├── AdaptiveLodScoreMgr（评分系统，决策降配/升配）

    ├── AndroidDeviceProfileSelector（Android 型号匹配）
    ├── IOSDeviceProfileSelector（iOS 型号匹配）
    └── PerformancePlatformConfig（各档位配置数据）
```

## 3. FPS 监控实现

### 3.1 PerformanceTools

```csharp
// 位置：Hotfix/ModelView/Framework/Profile/PlatformPerformance/Runtime/PerformanceTools.cs
public class PerformanceTools : MonoBehaviour, IDeviceBatteryEventListener
{
    // FPS 采样配置
    public float updateInterval = 0.5f;  // 每 0.5s 更新一次 FPS 均值
    public float fps { get; private set; }      // 当前均值 FPS
    public float frameMS { get; private set; }  // 均值帧时间（ms）
    
    private FPSChangeRateController _controller;  // FPS 变化率计算器
    private float elapsedIntervalTime;
    private int intervalFrameCount;
    
    private void Update()
    {
        FPSTick();
        _controller?.Enqueue(fps);
        ReportThermalStatus();  // 定时上报发热状态到 SDK
    }
    
    // FPS 均值统计（滑动窗口，0.5s 刷新一次）
    private void FPSTick()
    {
        intervalFrameCount++;
        elapsedIntervalTime += Time.unscaledDeltaTime;
        
        if (elapsedIntervalTime >= updateInterval)
        {
            fps = intervalFrameCount / elapsedIntervalTime;
            frameMS = elapsedIntervalTime * 1000f / intervalFrameCount;
            
            intervalFrameCount = 0;
            elapsedIntervalTime = 0;
        }
    }
    
    // DeviceBattery SDK 回调：发热状态变化
    public void UpdateBatteryEvent(EDeviceThermalStatus thermalStatus, 
        bool isCharging, bool shouldNotic = false)
    {
        CurrrentThermalStatus = thermalStatus;
        Log.LogInfo("发热状态变化: {0}", thermalStatus);
        // 通知 AdaptiveLodMgr 触发降配检查
        AdaptiveLodMgr.Instance?.UpdateBatteryEvent(thermalStatus, isCharging, shouldNotic);
    }
}
```

### 3.2 FPS 变化率计算器

```csharp
// 位置：Hotfix/ModelView/Framework/Profile/AdaptiveLodMgr/FPSChangeRateController.cs
// 设计思路：不只看当前 FPS，还看 FPS 的变化趋势（防止偶发抖动误触发降配）
public class FPSChangeRateController
{
    private List<float> fpsList = new List<float>(); // 滑动队列
    private int times = 5;          // 统计 5 秒内的 FPS 变化
    private int fspListLen;         // 队列最大长度 = targetFPS * times
    
    public void Enqueue(float fpsValue)
    {
        if (fpsList.Count >= fspListLen)
        {
            fpsList.RemoveAt(0);  // 超出容量，移除最老数据
        }
        fpsList.Add(fpsValue);
    }
    
    // 获取 FPS 变化率（差分求和）
    // 变化率大 = FPS 波动剧烈，可能是帧峰；变化率小 = 稳定
    // 降配判断：avg_fps <= 基准帧率 * 0.8
    // 升配判断：avg_fps >= 目标帧率 && 变化率 <= 阈值（FPS 稳定了再升配）
    public float GetFPSChangeRate()
    {
        if (fpsList.Count < fspListLen) return -1;
        
        float s = 0;
        for (int i = 0; i < fpsList.Count - 1; i++)
        {
            s += (fpsList[i] - fpsList[i + 1]);
        }
        return s;
    }
    
    // 区间内 FPS 均值
    public float GetAvgFPS()
    {
        if (fpsList.Count < fspListLen) return -1;
        float sum = 0;
        foreach (var f in fpsList) sum += f;
        return sum / fpsList.Count;
    }
}
```

## 4. 触发条件系统

### 4.1 条件基类

```csharp
// ConditionLogicBase.cs
public abstract class ConditionLogicBase
{
    protected TriggerData data;  // 触发阈值配置
    
    // 是否触发降配（FPS 掉到阈值以下）
    public abstract bool IsAttachDown();
    
    // 是否触发升配（FPS 回升到稳定）
    public abstract bool IsAttachUp();
}
```

### 4.2 FPS 均值条件

```csharp
// ConditionLogicFPSAvg.cs
public class ConditionLogicFPSAvg : ConditionLogicBase
{
    public override bool IsAttachDown()
    {
        float fps = PerformanceTools.Instance.GetAvgFPS();
        // FPS 低于 triggerDownValue（如 45）→ 触发降配
        return fps > 0 && fps <= data.triggerDownValue;
    }
    
    public override bool IsAttachUp()
    {
        float fps = PerformanceTools.Instance.GetAvgFPS();
        // FPS 高于 triggerUpValue（如 56）→ 触发升配
        return fps >= data.triggerUpValue;
    }
}
```

## 5. 降配特性实现

### 5.1 渲染分辨率降配

```csharp
// LodFeatureRenderScale.cs - 降低渲染分辨率到原来的 70%
public class LodFeatureRenderScale : LodFeatureBase
{
    private float originalRenderScale = 1.0f;
    
    public override void AutoDown()
    {
        base.AutoDown();
        if (GraphicsSettings.currentRenderPipeline is not UniversalRenderPipelineAsset renderPipeline)
            return;
        
        originalRenderScale = renderPipeline.renderScale;
        float targetRenderScale = originalRenderScale * 0.7f;  // 降到 70%
        
        // 如果开启了超分（FSR/DLSS），通过超分 API 设置
        SuperResolutionQualitySetting.SetRenderScale(targetRenderScale);
        Log.LogInfo("[AdaptiveLod] 降配渲染分辨率: {0}", targetRenderScale);
    }
    
    public override void AutoUp()
    {
        base.AutoUp();
        // 恢复原始分辨率
        SuperResolutionQualitySetting.SetRenderScale(originalRenderScale);
        Log.LogInfo("[AdaptiveLod] 升配渲染分辨率: {0}", originalRenderScale);
    }
}
```

    {
        originalTargetFPS = Application.targetFrameRate;

    }

启动时根据设备型号预设画质档位：

```csharp
// DeviceProfileManager.cs
public class DeviceProfileManager
{
    // 启动时选择设备档位
    public static EDeviceProfile GetDeviceProfile()
    {
    #if UNITY_ANDROID
        return AndroidDeviceProfileSelector.Select();
    #elif UNITY_IOS
        return IOSDeviceProfileSelector.Select();
    #else
        return EDeviceProfile.High;  // PC 默认高档
    #endif
    }
}

// Android 档位选择（基于 GPU 型号）
public class AndroidDeviceProfileSelector : IDeviceProfileSelector
{
    public static EDeviceProfile Select()
    {
        string gpu = SystemInfo.graphicsDeviceName.ToLower();
        
        if (gpu.Contains("adreno 7") || gpu.Contains("mali-g7"))
            return EDeviceProfile.Ultra;    // 旗舰 GPU
        else if (gpu.Contains("adreno 6") || gpu.Contains("mali-g5"))
            return EDeviceProfile.High;
        else if (gpu.Contains("adreno 5"))
            return EDeviceProfile.Medium;
        else
            return EDeviceProfile.Low;      // 老旧设备
    }
}
```

## 8. 超分辨率集成（FSR/DLSS）

```csharp
// SuperResolutionQualitySetting.cs
// 集成超分辨率技术（FSR - AMD FidelityFX）
public static class SuperResolutionQualitySetting
{
    // 设置渲染分辨率（配合超分，实际 1080p 可以降到 720p 渲染，超分输出 1080p）
    public static void SetRenderScale(float scale)
    {
        if (GraphicsSettings.currentRenderPipeline is UniversalRenderPipelineAsset urp)
        {
            // Clamp 到合理范围
            scale = Mathf.Clamp(scale, 0.5f, 1.0f);
            urp.renderScale = scale;
            
            // 同时更新超分质量档
            // scale >= 0.9: Quality
            // scale >= 0.75: Balanced
            // scale >= 0.6: Performance
            // scale < 0.6: Ultra Performance
        }
    }
}
```

## 10. 常见问题与最佳实践

**Q: 降配/升配来回抖动怎么办？**  
A: 使用滞回（Hysteresis）设计：降配阈值和升配阈值有一定间隔（如降配在 FPS<45，升配在 FPS>56），避免在临界点反复切换。

**Q: 每次发热都要降配，用户体验差？**  
A: 发热时只降一次，降配后稳定在新档位。给用户一个"已为您优化性能"的小提示，而不是频繁切换。

**Q: 高端机开了降配后能升回来吗？**  
A: 自动升配在 FPS 稳定超过目标帧率的 90% 且持续 5 秒后触发。也可以在设置界面手动切换画质档位。

**Q: 如何测试自适应降配？**  
A: 在 Editor 中通过 `AdaptiveLodMgrDebug` 窗口手动模拟降配，或在低端设备上用 ADB shell 强制降低 GPU 频率触发降配逻辑。

**Q: 分帧处理（splitFrameCount=60）是什么？**  
A: 避免每帧都检查降配条件（浪费 CPU），改为每 60 帧检查一次。对于变化缓慢的发热状态，这个频率足够了。
