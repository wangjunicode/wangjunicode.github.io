# 被动技能系统的槽位与触发机制

被动技能（Passive Skill）是相对于主动技能而言，不需要玩家手动触发、在满足特定条件时自动生效的技能。从简单的"攻击时有20%概率造成额外伤害"到复杂的"当队伍中有两个特定角色时，全队攻击力提升30%"，被动技能是游戏深度的重要来源。

本文深入解析一套工业化的被动技能系统设计。

---

## 第一性原理：被动技能的本质

被动技能本质上是：**监听特定事件，在事件发生时，如果满足特定条件，执行特定效果**。

```
被动技能 = 触发条件（On Attack / On Hit / On Death...）
         + 生效条件（HP < 50% / Team has X character / 概率 Y%...）
         + 效果（Add Buff / Increase Damage / Restore HP...）
```

关键挑战：
1. 同一角色有多个被动技能，需要管理它们的生命周期
2. 被动技能可以有"充能"（限制触发次数）
3. 某些被动只在特定槽位（上场/备战/队伍特性）激活
4. 被动技能本身也可以添加 Buff，Buff 结束时被动也应该清理

---

## 槽位系统（PassiveSlot）

每个 Unit 的被动技能通过"槽位"组织：

```csharp
public class PassiveSlotComponent : Entity
{
    public Dictionary<EPSkillSlotType, PassiveSlot> Slots;
}

public enum EPSkillSlotType
{
    Character,  // 角色自带的被动技能（始终激活）
    BackUp,     // 备战位的被动（只有在备战时才激活）
    Tactic,     // 战术卡（特殊类型）
    Scene,      // 场景被动（Director 使用）
    Team,       // 队伍特性被动
}
```

不同槽位有不同的激活规则：

```csharp
[EntitySystem]
private static void Awake(this PassiveSkillInstance self, PassiveSkillInfo data, BlackboardSource _args)
{
    self.skill = data;
    self.curCharge = 0;
    self.useTime = 0;
    self.AddComponent<EventDispatcherComponent>();  // 用于被动技能内部的事件通信
    
    // 非备战/战术槽位的被动技能立刻激活
    if (self.SlotType() != EPSkillSlotType.BackUp && self.SlotType() != EPSkillSlotType.Tactic)
    {
        self.Invoke();  // 立刻生效
    }
    // 备战/战术槽位的被动需要等特定条件（如切换到后排）才激活
}
```

## PassiveSkillInstance：被动技能实例

```csharp
public class PassiveSkillInstance : Entity, IAwake<PassiveSkillInfo, BlackboardSource>, IDestroy
{
    public PassiveSkillInfo skill;       // 技能信息（ID、等级）
    public BlackboardSource extraArgs;   // 额外参数（从外部传入）
    public SimpleBuff buff;              // 关联的 Buff 实例（被动技能通常通过 Buff 图执行）
    public int curCharge;               // 当前充能次数
    public int useTime;                 // 已使用次数
    
    public Dictionary<long, long> buffViewHandlers;  // 关联的 BuffView 句柄（用于清理）
}
```

每个被动技能实例持有一个 `SimpleBuff`——被动技能最终是通过 Buff 系统驱动的。这意味着被动技能可以使用所有 Buff 系统的能力（节点图、黑板、条件分支）。

## 激活（Invoke）流程

```csharp
public static void Invoke(this PassiveSkillInstance self)
{
    var bsComp = self.GetOwner().CurrentScene().GetComponent<BattleScriptComponent>();
    
    // 检查是否需要延迟激活（如战斗开始前的初始化阶段）
    if (bsComp.IsDelayPSkillInvoke())
    {
        var mgr = self.DomainScene().GetComponent<PassiveSkillMgrComponent>();
        mgr.AddWaitPSkill(self);  // 加入等待队列
        return;
    }
    
    // 直接激活
    self.InvokeNow();
}

private static void InvokeNow(this PassiveSkillInstance self)
{
    var pSkillConf = CfgManager.tables.TBPassiveSkill.Get(self.skill.skillID, self.skill.level);
    if (pSkillConf == null) return;
    
    // 通过被动技能管理器添加 Buff
    var mgr = self.DomainScene().GetComponent<PassiveSkillMgrComponent>();
    
    // 获取或创建关联的 Buff
    var owner = self.GetOwner();
    var buffComp = owner.GetComponent<BuffComponent>();
    
    // 添加被动技能对应的 Buff（Buff 图执行具体效果）
    self.buff = buffComp.AddBuff(
        pSkillConf.GraphId.GraphId,  // Buff 图 ID
        self.skill.level,
        extraArgs: self.extraArgs,   // 额外参数
        addedby: self                // 来源：此被动技能实例
    );
}
```

**延迟激活的必要性**：

战斗开始时，可能需要先完成所有角色的初始化（数值、Buff）再激活被动技能，否则被动技能在激活时可能读到不完整的数值。`IsDelayPSkillInvoke()` 检查是否处于"初始化批处理"阶段，如果是，就把被动技能加入等待队列，等批处理完成后统一激活。

## 被动技能事件触发流程

被动技能通过 `TG_PSkillEvent` 节点监听特定事件：

```csharp
// TG_PSkillEvent 节点在 Buff 图启动时注册事件监听
public override void OnPostGraphStarted(TG_PSkillEvent flowNode)
{
    var BuffGraph = flowNode.rootGraph as BuffGraph;
    var ownerPSkill = BuffGraph.buff.ownerSkill;  // 找到所属的被动技能实例
    
    // 在被动技能实例的 EventDispatcher 上注册监听
    ownerPSkill.GetComponent<EventDispatcherComponent>()
        .RegisterEvent<Evt_PSkillEvent>(flowNode.OnFireEvent);
}
```

当战斗中发生事件（如攻击命中）时：

```csharp
// 在攻击成功后发布事件
public static void FirePSkillEvent(this Unit unit, EPSkillEventTag tag, int skillId = 0, int level = 0)
{
    var evt = new Evt_PSkillEvent
    {
        owner = unit,
        pSkillEvent = tag,
        skillID = skillId,
        level = level
    };
    
    // 触发所有被动技能实例的事件分发器
    var pSkillMgr = unit.DomainScene().GetComponent<PassiveSkillMgrComponent>();
    foreach (var pSkillInstance in pSkillMgr.GetActiveInstances(unit))
    {
        pSkillInstance.GetComponent<EventDispatcherComponent>().FireEvent(evt);
    }
}
```

## 触发条件检查（CanTriggerEvent）

被动技能在触发前需要检查是否满足触发条件：

```csharp
public static bool CanTriggerEvent(this PassiveSkillInstance self)
{
    var buffComp = self.GetOwner().GetComponent<BuffComponent>();
    var pSkillConf = CfgManager.tables.TBPassiveSkill.Get(self.skill.skillID, self.skill.level);
    
    if (buffComp != null && pSkillConf != null)
    {
        // 默认情况下，角色"死亡"（护盾归零）后不触发被动
        // 除非配置了 EnableOnDead 标志
        return (!buffComp.bDead || ((pSkillConf.EffectType & EPSkillEffectType.EnableOnDead) != 0));
    }
    return true;
}
```

`EPSkillEffectType.EnableOnDead` 是一个特殊标志，允许某些被动技能在角色"倒下"（护盾归零但还没真正死亡）时仍然触发。这用于实现"临死反击"类的被动效果。

在 `TG_PSkillEvent` 节点的触发检查中：

```csharp
public static void OnFireEvent(this TG_PSkillEvent self, Evt_PSkillEvent argv)
{
    // 检查事件标签是否匹配
    CPSkillEvent pointConf = CfgManager.tables.TBPSkillEvent.GetOrDefault(evtID);
    if (pointConf == null || !self.HasTag(pointConf)) return;
    
    // 检查前后排条件
    var owner = self.graphAgent as Entity;
    bool bTriggerInBackUp = self.bTriggerInBackUp.value;
    
    if (owner is Unit)
    {
        var unit = owner as Unit;
        var team = teamSys.GetTeamByUnit(unit);
        
        // 后排是否可触发的条件判断
        if ((bTriggerInBackUp && team == argv.owner.GetRelativeEntity(true, false)) 
            || team.GetCurMainMember() == owner)
        {
            bCheckTrigger = true;
        }
        else
        {
            bCheckTrigger = false;
        }
    }
    
    if (bCheckTrigger)
    {
        self.exec.Call(new Flow());  // 触发 Buff 图的后续节点
    }
}
```

前后排判断是一个关键设计：默认情况下，被动技能只有在角色"上场"时才触发。但某些被动技能（`bTriggerInBackUp = true`）即使在后排也能触发，这实现了"后排辅助效果"。

## 充能系统

```csharp
// 触发被动技能时消耗充能
public static bool TryConsumeCharge(this PassiveSkillInstance self)
{
    var pSkillConf = CfgManager.tables.TBPassiveSkill.Get(self.skill.skillID, self.skill.level);
    
    if (pSkillConf.MaxCharge <= 0) return true;  // 无充能限制
    
    if (self.curCharge <= 0) return false;  // 充能耗尽
    
    self.curCharge--;
    return true;
}

// 补充充能（如通过Buff恢复）
public static void AddCharge(this PassiveSkillInstance self, int count)
{
    var pSkillConf = CfgManager.tables.TBPassiveSkill.Get(self.skill.skillID, self.skill.level);
    self.curCharge = TSMath.Min(self.curCharge + count, pSkillConf.MaxCharge);
}
```

充能系统让被动技能可以有"使用次数限制"，如"战斗开始时获得3次充能，每次命中消耗1次，触发特殊效果"。

## 清理流程

当被动技能失效时，需要清理所有关联数据：

```csharp
public static void RemoveLogic(this PassiveSkillInstance self)
{
    // 1. 清理关联的 BuffView（视觉效果）
    foreach (long handler in self.buffViewHandlers.Values)
    {
        self.GetOwner().GetComponent<BuffViewComponent>().RemoveBuff(handler);
    }
    self.buffViewHandlers.Clear();
    
    // 2. 从等待队列移除（如果还在等待激活）
    var mgr = self.DomainScene().GetComponent<PassiveSkillMgrComponent>();
    mgr.RemoveWaitPSkill(self);
    
    // 3. 移除所有由此被动技能添加的 Buff
    if (self.buff != null)
    {
        var buffComp = self.GetOwner().GetComponent<BuffComponent>();
        buffComp.RemoveAllBuffByAttachmentImmediately(self);  // 批量移除关联Buff
        self.buff = null;
    }
    
    // 4. 重置充能
    self.curCharge = 0;
}
```

`RemoveAllBuffByAttachmentImmediately` 通过 `ownerSkill == self` 的条件，精确移除所有由此被动技能添加的 Buff，不影响其他来源的 Buff。

## 被动技能管理器（PassiveSkillMgrComponent）

```csharp
public class PassiveSkillMgrComponent : Entity
{
    // 全局活跃的被动技能缓存（按实体分组）
    private Dictionary<Entity, List<PassiveSkillInstance>> _activeCache;
    private bool _isDirty;  // 缓存是否需要刷新
    
    public void SetPSkillCacheDirty()
    {
        _isDirty = true;  // 标记缓存失效（有被动技能增减时）
    }
    
    // 获取某实体的所有活跃被动技能
    public List<PassiveSkillInstance> GetActiveInstances(Entity owner)
    {
        if (_isDirty) RefreshCache();  // 懒刷新
        if (_activeCache.TryGetValue(owner, out var list)) return list;
        return EmptyList;
    }
}
```

使用懒刷新（`SetPSkillCacheDirty` + 访问时刷新）避免了每次增减被动技能时都重建缓存，同时保证访问时数据是最新的。

## 被动技能设计总结

```
被动技能激活流程：
  PassiveSlotComponent.AddPSkill(skillId)
      ↓
  PassiveSkillInstance.Awake（创建实例）
      ↓
  PassiveSkillInstance.Invoke（激活）
      ↓
  BuffComponent.AddBuff（创建 Buff 图实例）
      ↓
  BuffGraph 启动（TG_PSkillEvent 注册事件监听）

被动技能触发流程：
  战斗事件发生（如 OnAttack）
      ↓
  FirePSkillEvent（广播事件）
      ↓
  TG_PSkillEvent.OnFireEvent（事件处理）
      ├── 标签匹配检查
      ├── 前后排条件检查  
      └── Exec（触发效果节点流）
              ↓
          具体效果（AddBuff / ModifyNumeric / CastSkill...）
```

| 特性 | 实现 |
|------|------|
| 槽位管理 | EPSkillSlotType 分类激活规则 |
| 事件监听 | TG_PSkillEvent 节点注册/注销 |
| 前后排支持 | bTriggerInBackUp 开关 |
| 充能限制 | curCharge 计数器 |
| 延迟激活 | IsDelayPSkillInvoke + 等待队列 |
| 精确清理 | AddedBy 引用追踪 |
| 缓存优化 | 懒刷新活跃实例列表 |

被动技能系统是"数据驱动"设计哲学的完美体现——绝大多数被动技能效果都通过 Buff 图配置实现，不需要为每个被动技能写专门的代码，这是商业游戏扩展性的关键所在。
