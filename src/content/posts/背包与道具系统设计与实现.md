# 背包与道具系统设计与实现

## 1. 系统概述

背包系统（BackpackComponent）负责管理玩家持有的所有道具，包括虚拟物品（货币、体力等不占背包格子的资源）和实体物品（装备、消耗品等）。系统从服务器下发数据，客户端本地缓存，通过 `ZoneInfoChangeNotify` 增量同步。

**架构特点：**
- **虚拟物品** 与 **实体物品** 分开存储（不同查询逻辑）
- **增量同步**：服务器下发变更列表，客户端按 ID merge，不全量刷新
- **事件驱动 UI 更新**：数据变更后发布 `Evt_BackpackDataUpdated`，UI 层响应刷新

## 3. 核心代码解析

### 3.1 初始化同步（登录时全量）

```csharp
// 位置：Hotfix/Function/GamePlay/Player/Backpack/BackpackComponentSystem.cs
[FriendOf(typeof(BackpackComponent))]
public static class BackpackComponentSystem
{
    // 登录时服务器下发全量物品列表
    public static void SyncItemList(this BackpackComponent self,
        RepeatedField<ItemMsg> virtualItems,
        RepeatedField<ItemMsg> mainBagItemList)
    {
        // 一次性转换并缓存到字典（Key = ItemID）
        self.VirtualItems = ServerDataConverter.ToMap(
            virtualItems, 
            ServerDataConverter.GetItemData, 
            x => x.ItemID);  // 以 ItemID 为字典键
            
        self.BagItems = ServerDataConverter.ToItemMap(
            mainBagItemList, 
            ServerDataConverter.GetItemData, 
            x => x.ItemID);
    }
```

### 3.2 增量同步（实时变更）

```csharp
    // 收到服务器推送的 ZoneInfoChangeNotify 时更新本地数据
    public static void SyncItemListFromInfoChangeNotify(
        this BackpackComponent self, ZoneInfoChangeNotify info)
    {
        var itemList = info?.ItemNotify;
        if (itemList == null) return;

        var updatedIds = new List<int>();
        if (self.BagItems == null)
        {
            self.BagItems = new Dictionary<int, ItemData>();
        }
        
        foreach (var item in itemList)
        {
            if (item != null && item.Id > 0)
                updatedIds.Add(item.Id);

            // 虚拟物品：只更新数量
            if (ItemHelper.IsVirtualItem(item.Id))
            {
                self.SetVirtualItemCount(item.Id, item.Count);
            }
            // 实体物品已存在：更新或删除
            else if (self.BagItems.ContainsKey(item.Id))
            {
                if (item.Count <= 0)
                {
                    // 数量归零 = 道具消耗完毕，从背包移除
                    self.BagItems.Remove(item.Id);
                }
                else
                {
                    // 在原有数据上更新（复用对象，减少GC）
                    ServerDataConverter.GetItemData(item, self.BagItems[item.Id]);
                }
            }
            // 实体物品不存在：新获得道具
            else
            {
                if (item.Count > 0)
                {
                    self.BagItems.Add(item.Id, ServerDataConverter.GetItemData(item));
                }
            }
        }

        // 发布数据变更事件，通知 UI 层刷新（仅刷新变动的物品，不全量重绘）
        self.ClientScene()
            .GetComponent<EventDispatcherComponent>()
            ?.FireEvent(new Evt_BackpackDataUpdated
            {
                UpdatedItemIds = updatedIds
            });
    }
```

### 3.3 虚拟物品操作

```csharp
    // 虚拟物品数量查询
    public static int GetVirtualItemCount(this BackpackComponent self, int itemId)
    {
        if (self.VirtualItems == null) return 0;
        return self.VirtualItems.TryGetValue(itemId, out var data) ? data.Count : 0;
    }
    
    // 虚拟物品数量设置（由增量同步调用）
    public static void SetVirtualItemCount(this BackpackComponent self, int itemId, int count)
    {
        if (self.VirtualItems == null)
        {
            self.VirtualItems = new Dictionary<int, ItemData>();
        }
        if (self.VirtualItems.ContainsKey(itemId))
        {
            self.VirtualItems[itemId].Count = count;
        }
        else if (count > 0)
        {
            self.VirtualItems[itemId] = new ItemData { Id = itemId, Count = count };
        }
    }
```

## 5. ServerDataConverter —— 服务端数据转换

将 Protobuf 消息转换为客户端数据结构：

```csharp
// Hotfix/Function/GamePlay/Player/ServerDataConverter.cs
public static class ServerDataConverter
{
    // 转换单条物品数据
    public static ItemData GetItemData(ItemMsg msg)
    {
        return new ItemData
        {
            Id    = msg.Id,
            Count = (int)msg.Count,
            // 其他字段...
        };
    }
    
    // 复用现有对象更新（减少GC，重要优化！）
    public static void GetItemData(ItemMsg msg, ItemData existing)
    {
        existing.Id    = msg.Id;
        existing.Count = (int)msg.Count;
        // 其他字段...
    }
    
    // 批量转为字典
    public static Dictionary<TKey, TValue> ToMap<T, TKey, TValue>(
        IEnumerable<T> source,
        Func<T, TValue> convert,
        Func<TValue, TKey> keySelector)
    {
        var dict = new Dictionary<TKey, TValue>();
        foreach (var item in source)
        {
            var value = convert(item);
            dict[keySelector(value)] = value;
        }
        return dict;
    }
}
```

## 6. 道具获得与消耗流程

```
玩家操作（使用道具/战斗结算）
    ↓
客户端发送请求给服务器（Protobuf Request）
    ↓
服务器校验合法性并扣除/添加物品
    ↓
服务器推送 ZoneInfoChangeNotify（含变更物品列表）
    ↓
BackpackComponentSystem.SyncItemListFromInfoChangeNotify()
    ↓（在本地更新物品数据）
发布 Evt_BackpackDataUpdated（含变更物品ID列表）
    ↓
UI 事件处理器响应，仅刷新变更的道具格子
```

## 7. 常见问题与最佳实践

**Q: 客户端可以直接修改背包数据吗？**  
A: 不可以。客户端只负责显示，所有物品变更必须经服务器验证后推送。直接修改本地数据会在下次同步时被覆盖，且属于客户端作弊。

**Q: 背包格子满了如何处理？**  
A: 服务器端判断背包容量，满时返回错误码，客户端弹出"背包已满"提示。背包扩展（购买格子）也走服务器请求。

**Q: 虚拟物品为什么不占背包格子？**  
A: 虚拟物品（金币、体力、钻石等）数量庞大且没有独立性，放在独立的 `VirtualItems` 字典，不参与背包格子渲染，UI 展示在资源栏而非背包界面。

**Q: 跨场景后背包数据会丢失吗？**  
A: 不会。`BackpackComponent` 挂在 `ClientScene` 下（持久化场景），不会随战斗场景销毁。
