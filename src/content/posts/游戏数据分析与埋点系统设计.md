## 为什么游戏需要数据分析

```
没有数据时的决策方式：
  策划："我觉得这关太难了"
  程序："我觉得没问题"
  → 靠主观感受，没有共识

有数据后的决策方式：
  第3关通关率 = 42%（行业均值 60%）
  卡关位置：Boss 第二阶段 (78% 的失败发生在这里)
  平均尝试次数：11.3 次（健康值：3~7次）
  → 数据明确指向"第3关Boss第二阶段需要调难度"
```

数据分析的价值：
- 发现肉眼看不到的问题（留存率、付费转化漏斗）
- 验证产品决策（A/B 测试）
- 指导内容迭代（哪些关卡最受欢迎）
- 支撑商业决策（哪个活动ROI最高）

## 一、埋点设计方法论

### 1.1 事件模型规范

```
埋点事件结构（AARRR 模型）：

Acquisition（获取）：用户从哪里来
  event: app_install
  event: tutorial_start
  
Activation（激活）：用户是否完成核心体验
  event: tutorial_complete
  event: first_battle_win
  
Retention（留存）：用户是否回来
  event: daily_login
  event: session_start / session_end
  
Revenue（收入）：用户是否付费
  event: shop_open
  event: purchase_start / purchase_success / purchase_fail
  
Referral（传播）：用户是否分享
  event: share_result
  event: invite_friend
```

### 1.2 事件命名规范

```
事件命名：动词_名词
  ✅ battle_start, battle_end, skill_use, item_buy
  ❌ startBattle, BattleStart, battle (太模糊)

属性命名：名词 或 名词_修饰词
  ✅ player_level, enemy_type, skill_id, is_success
  ❌ playerLevel, EnemyType

公共属性（每个事件都携带）：
  user_id: 用户唯一标识
  session_id: 会话标识（同一次打开游戏为同一个 session）
  device_id: 设备标识
  platform: iOS / Android / PC
  app_version: 游戏版本
  timestamp: 事件时间戳（UTC）
  level: 玩家当前等级
```

## 二、埋点 SDK 架构设计

### 2.1 事件收集器

```csharp
/// <summary>
/// 游戏埋点管理器
/// </summary>
public class Analytics : Singleton<Analytics>
{
    private const int BATCH_SIZE = 50;          // 批量上报数量
    private const float FLUSH_INTERVAL = 30f;   // 定时上报间隔（秒）
    
    private readonly Queue<AnalyticsEvent> _eventQueue = new();
    private string _sessionId;
    private float _lastFlushTime;
    
    // 公共属性（每个事件都携带）
    private Dictionary<string, object> _commonProperties;
    
    protected override void Init()
    {
        _sessionId = Guid.NewGuid().ToString("N");
        
        _commonProperties = new Dictionary<string, object>
        {
            ["user_id"] = GetUserId(),
            ["session_id"] = _sessionId,
            ["device_id"] = SystemInfo.deviceUniqueIdentifier,
            ["platform"] = Application.platform.ToString(),
            ["app_version"] = Application.version,
            ["device_model"] = SystemInfo.deviceModel,
            ["os_version"] = SystemInfo.operatingSystem,
        };
    }
    
    /// <summary>
    /// 上报事件
    /// </summary>
    public void Track(string eventName, Dictionary<string, object> properties = null)
    {
        var mergedProperties = new Dictionary<string, object>(_commonProperties)
        {
            ["timestamp"] = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds(),
            ["player_level"] = GetPlayerLevel(),
        };
        
        if (properties != null)
        {
            foreach (var kv in properties)
                mergedProperties[kv.Key] = kv.Value;
        }
        
        var evt = new AnalyticsEvent
        {
            EventName = eventName,
            Properties = mergedProperties
        };
        
        _eventQueue.Enqueue(evt);
        
        // 队列满了，立即上报
        if (_eventQueue.Count >= BATCH_SIZE)
            FlushAsync().Forget();
    }
    
    void Update()
    {
        // 定时上报
        if (Time.time - _lastFlushTime >= FLUSH_INTERVAL && _eventQueue.Count > 0)
            FlushAsync().Forget();
    }
    
    private async UniTaskVoid FlushAsync()
    {
        _lastFlushTime = Time.time;
        
        if (_eventQueue.Count == 0) return;
        
        // 取出要上报的事件
        var batch = new List<AnalyticsEvent>();
        while (_eventQueue.Count > 0 && batch.Count < BATCH_SIZE)
            batch.Add(_eventQueue.Dequeue());
        
        try
        {
            await UploadEvents(batch);
        }
        catch (Exception e)
        {
            Debug.LogError($"[Analytics] Upload failed: {e.Message}");
            
            // 上报失败，放回队列（简单重试）
            foreach (var evt in batch)
                _eventQueue.Enqueue(evt);
        }
    }
    
    private async UniTask UploadEvents(List<AnalyticsEvent> events)
    {
        var payload = JsonConvert.SerializeObject(new { events });
        
        // 调用埋点服务 API
        using var request = UnityWebRequest.Post(_endpointUrl, payload);
        request.SetRequestHeader("Content-Type", "application/json");
        request.SetRequestHeader("Authorization", $"Bearer {_apiKey}");
        
        await request.SendWebRequest().ToUniTask();
        
        if (request.result != UnityWebRequest.Result.Success)
            throw new Exception(request.error);
    }
    
    void OnApplicationPause(bool paused)
    {
        if (paused)
        {
            Track("session_pause");
            FlushAsync().Forget(); // 切后台时强制上报
        }
        else
        {
            Track("session_resume");
        }
    }
    
    void OnApplicationQuit()
    {
        Track("session_end", new Dictionary<string, object>
        {
            ["session_duration"] = Time.time // 本次 session 时长
        });
        // 注意：退出时异步上报可能不完整，这里用同步方式
        FlushSync();
    }
    
    private void FlushSync()
    {
        // 退出时同步上报剩余事件（不等异步）
        // 实际项目中可以先写入本地文件，下次启动时上报
    }
    
    private string GetUserId() => PlayerPrefs.GetString("UserId", CreateNewUserId());
    private string CreateNewUserId()
    {
        var id = Guid.NewGuid().ToString("N");
        PlayerPrefs.SetString("UserId", id);
        return id;
    }
    private int GetPlayerLevel() => GameDataManager.Instance?.PlayerLevel ?? 0;
    
    [SerializeField] private string _endpointUrl;
    [SerializeField] private string _apiKey;
}
```

### 2.2 语义化埋点 API

```csharp
/// <summary>
/// 语义化埋点接口：按业务场景封装埋点调用
/// </summary>
public static class GameAnalytics
{
    // 战斗相关
    public static void TrackBattleStart(int levelId, string difficulty)
    {
        Analytics.Instance.Track("battle_start", new()
        {
            ["level_id"] = levelId,
            ["difficulty"] = difficulty,
            ["attempt_count"] = GetAttemptCount(levelId),
        });
    }
    
    public static void TrackBattleEnd(int levelId, bool isWin, float duration, int score)
    {
        Analytics.Instance.Track("battle_end", new()
        {
            ["level_id"] = levelId,
            ["is_win"] = isWin,
            ["duration_seconds"] = duration,
            ["score"] = score,
            ["hp_remaining_percent"] = PlayerHealth.Instance.Percentage,
        });
    }
    
    public static void TrackSkillUse(int skillId, string targetType, bool isHit)
    {
        Analytics.Instance.Track("skill_use", new()
        {
            ["skill_id"] = skillId,
            ["target_type"] = targetType,
            ["is_hit"] = isHit,
        });
    }
    
    // 经济系统
    public static void TrackItemPurchase(int itemId, string itemName, int price, string currency)
    {
        Analytics.Instance.Track("item_purchase", new()
        {
            ["item_id"] = itemId,
            ["item_name"] = itemName,
            ["price"] = price,
            ["currency"] = currency, // "gold" 或 "diamond"
        });
    }
    
    // 漏斗事件
    public static void TrackFunnelStep(string funnelName, int step, string stepName)
    {
        Analytics.Instance.Track("funnel_step", new()
        {
            ["funnel_name"] = funnelName,
            ["step"] = step,
            ["step_name"] = stepName,
        });
    }
    
    private static int GetAttemptCount(int levelId)
    {
        string key = $"LevelAttempts_{levelId}";
        int count = PlayerPrefs.GetInt(key, 0) + 1;
        PlayerPrefs.SetInt(key, count);
        return count;
    }
}

// 在游戏代码中使用（干净整洁）

}

}
```

## 三、关键指标体系

### 3.1 游戏核心指标

```
留存指标：
  次日留存（Day 1 Retention）：> 40% 为优秀，< 20% 有问题
  三日留存（Day 3 Retention）：> 25% 为优秀
  七日留存（Day 7 Retention）：> 15% 为优秀
  
付费指标：
  DAU（日活跃用户）
  ARPU（每用户平均收入 = 总收入 / DAU）
  ARPPU（每付费用户平均收入 = 总收入 / 付费用户数）
  付费率（付费用户 / 总用户）：手游行业均值约 2~5%

体验指标：
  平均 Session 时长：健康游戏通常 15~30 分钟
  关卡通关率：过高或过低都有问题（70~85% 为合理区间）
  
技术指标：
  崩溃率：< 0.1% 为优秀，> 1% 严重问题
  ANR 率（Android Not Responding）：< 0.1%
  帧率分布：60fps 设备占比、平均帧率
```

### 3.2 数据驱动决策示例

```
场景：新手关卡流失分析

数据发现：
  关卡1完成率：91%（正常）
  关卡2完成率：86%（正常）
  关卡3完成率：43%（异常！）
  关卡4完成率：78%（又正常了）

深挖关卡3数据：
  平均尝试次数：14.2 次（过高！行业期望：3~7次）
  失败分布：
    第1阶段失败：8%
    第2阶段失败：15%
    Boss战失败：77% （← 明显卡点）
  平均卡关时间：23分钟 （← 玩家会在这关放弃）

行动方案：
  1. 降低Boss第二阶段伤害 20%
  2. 增加Boss战前的回复站
  3. 调整 Boss 行为模式（减少连续技概率）
  
调整效果验证（A/B测试后）：
  旧版：通关率 43%，次日留存 22%
  新版：通关率 71%，次日留存 31%（+9个百分点！）
```

```csharp
public class PrivacyManager : MonoBehaviour
```

## 总结

数据分析体系建设路线：

```
阶段一（初期）：接入第三方分析平台
  推荐：Firebase Analytics（免费，Google 生态）
       Adjust / AppsFlyer（归因分析）
  成本：低，接入快

阶段二（成长期）：建立自研埋点基础设施
  自建上报服务（Nginx + Kafka + Flink + ClickHouse）
  建立标准化的事件模型
  成本：中，需要后端团队配合

阶段三（成熟期）：数据中台与智能化
  A/B 测试平台
  实时数据看板
  预测模型（流失预测、付费意愿预测）
  成本：高，需要专门的数据团队
```

**技术负责人的任务**：
1. 与产品团队对齐：分析什么数据，期望什么结论
2. 制定埋点规范：命名、格式、公共属性
3. 建设可观测性：不只是业务指标，还要监控技术指标（崩溃、帧率）
4. 数据驱动文化：帮助团队养成用数据说话的习惯
