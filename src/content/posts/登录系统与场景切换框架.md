# 登录系统与场景切换框架

## 1. 系统概述

游戏客户端的登录与场景切换是玩家体验的第一道关卡。本项目采用**分层登录架构**：

```
MSDK（腾讯 GCloud）登录平台
    ↓ 获取 Token
ZoneConnect（TCP 连接游戏服务器）
    ↓ 三次握手
ZoneLogin（发送登录协议）
    ↓ 服务器验证
SyncLoginData（同步玩家数据）
    ↓
GotoLobbyCultivation（进入大厅或剧情）
```

**关键技术点：**
- `CoroutineLockComponent` 防止登录请求并发
- 平台差异化处理（Android / iOS / PC / Editor）
- 登录后数据增量同步（背包 / 角色 / 道具 / 剧情进度）
- 场景切换统一走 `SceneChangeHelper`，带 Loading 面板

## 2. LoginHelper 核心实现

### 2.1 建立 TCP 连接

```csharp
// 位置：Hotfix/Function/Biz/Login/LoginHelper.cs
public static class LoginHelper
{
    // 建立到游戏服（Zone Server）的 TCP 连接
    public static async ETTask<(int errCode, string err)> ZoneConnect(
        Scene clientScene, ChannelType platform, string url)
    {
        // 使用 TaskCompletionSource 模式（ETTask.Create）等待异步回调
        var tcs = ETTask<(int, string)>.Create();
        
        // 自动补全协议前缀
        if (!url.StartsWith("tcp://"))
            url = ZString.Concat("tcp://", url);
        
        var conn = clientScene.GetComponent<ConnectorComponent>();
        conn.Url = url;
        
        conn.Connect(
            ConnectionID.GAME_CONNECTION,
            platform,
            onSuccess: () => tcs.SetResult((0, string.Empty)),         // 连接成功
            onError: (errCode, err) => tcs.SetResult((errCode, err)),  // 连接失败
            reconnect: false);  // 不自动重连（由业务层控制重连逻辑）
        
        return await tcs;  // 等待连接结果
    }
```

### 2.2 构建登录协议包

```csharp
    // 构建 ZoneLoginReq 协议包（包含版本信息、支付 Token、设备信息等）
    public static ZoneLoginReq CreateLoginRequest(
        ZoneSession zoneSession, uint logintype, 
        string name = "", EnumGenderType gender = EnumGenderType.EGenderTypePrivate)
    {
        var req = new ZoneLoginReq();
        
        // 平台识别（服务器用于差异化处理 Android/iOS/PC 的支付和审核逻辑）
#if UNITY_ANDROID && (!UNITY_EDITOR || EDITOR_DOWNLOAD)
        req.ClientType = (int)EnumZoneClientType.EClientTypeAnd;
#elif UNITY_IOS && (!UNITY_EDITOR || EDITOR_DOWNLOAD)
        req.ClientType = (int)EnumZoneClientType.EClientTypeIos;
#else
        req.ClientType = (int)EnumZoneClientType.EClientTypePc;  // Editor 和 PC 相同
#endif
        
        // 填入支付信息（MSDK 获取的 AccessToken / QQ 支付 Token）
        var payMsg = new ClientPayMsg();
        var info = NetworkComponent.AccountInfo;
        if (info != null)
        {
            var accessToken = SDKUtil.GetAccessToken();
            if (accessToken != null) payMsg.AccessInfo = accessToken;
            
            var qqPayToken = SDKUtil.GetQQPayToken(zoneSession.Player().QQPayToken);
            if (qqPayToken != null) payMsg.PayToken = qqPayToken;
            
            if (info.Pf != null) payMsg.PfStr = info.Pf;  // 平台标识（qq/wx/...)
        }
        req.PayMsg = payMsg;
        
        // 版本号（三段：App版本 / 资源版本 / APK版本）
        req.ClientVersion = new ClientVersionMsg
        {
            AppVersion = VersionUtil.toSimpleVersion(Application.version),
            ResVersion = VersionUtil.toSimpleVersion(VersionManager.Instance.ResVersion),
            ApkVersion = VersionUtil.toSimpleVersion(VersionPath.ApkVersion)
        };
        
        // 客户端状态（本局战斗的唯一标识，用于断线重连恢复）
        req.LoginUniqId = zoneSession.GetUniqID(logintype);
        
        // OSS 配置（用于热更新资源下载的 CDN 配置）
        req.OssMsg = EventSystem.Instance.Invoke<Evt_GI_GetOssInfo, OssInfo>(new Evt_GI_GetOssInfo());
        
        return req;
    }
```

### 2.3 发送登录协议（并发锁保护）

```csharp
    public static async ETTask<NetworkResult<ZoneLoginResp>> Login(
        ZoneLoginReq req, ZoneSession zoneSession,
        byte[] client_state = null,
        NetworkErrorHandle sendErrorHandle = null,
        NetworkReceiveErrorHandle receiveErrorHandle = null)
    {
        // ⚠️ 关键：协程锁防止并发登录（如网络抖动导致连续触发两次登录）
        using var coroutineLock = await CoroutineLockComponent.Instance
            .Wait(CoroutineLockType.ZoneLogin, 0);
        
        // 发送登录协议，await 等待服务器响应
        var nr = await zoneSession.SendAsync<ZoneLoginResp>(
            (uint)ZoneClientCmd.ZoneCsLogin, req,
            sendErrorHandle, receiveErrorHandle);
        
        return nr;
    }
```

```csharp
    // 登录成功后，批量同步服务器下发的初始数据
    private static void SyncLoginData(ZoneSession zoneSession, ZoneLoginResp resp)
    {
        // 角色（VTuber）数据同步
        var characterComp = zoneSession.Character();
        characterComp.SyncResidentCharacterFromLogin(resp.RoleInfo.VtuberInfo);
        characterComp.SyncCharacterListFromLogin(resp.RoleInfo.VtuberInfo);
        characterComp.SyncUnlockIPSequence(resp.RoleInfo.VtuberInfo);
        
        // 背包数据同步
        var backpackComp = zoneSession.Backpack();
        backpackComp.SyncItemList(
            resp.RoleInfo.ItemInfo.VirtualItem,         // 虚拟道具（货币/钻石等）
            resp.RoleInfo.ItemInfo.MainBagItemList);    // 主背包道具列表
    }
```

```csharp
    // 根据场景类型枚举查找对应场景 ID（从配置表 TbGameScene 读取）
    public static int GetSceneIdByType(ESceneType sceneType)
    {
        foreach (var scene in CfgManager.tables.TbGameScene.DataList)
        {
            if (scene.SceneType == sceneType)
                return scene.Id;
        }
        Log.Error(ZString.Format("GetSceneIdByType error! cannot find {0} scene", sceneType));
        return -1;
    }
```

## 4. 场景切换事件链

场景切换通过事件系统广播，各系统按顺序响应：

```
1. SceneChangeLoadPrepareEvent   → 准备阶段（清理上个场景资源）
2. SceneChangeLoadEvent          → 加载阶段（Addressables 加载场景资源）
3. SceneChangeLoadBattleEvent    → 战斗场景专属（加载角色、地图等战斗资源）
4. SceneChangeFinishEvent        → 加载完成（隐藏 Loading、初始化 UI）
```

```csharp
// 场景切换过程中的事件（订阅者按顺序处理）
[Event(SceneType.Client)]
public class SceneChangeLoadEventView : AEvent<Scene, SceneChangeLoadEvent>
{
    protected override async ETTask Run(Scene scene, SceneChangeLoadEvent args)
    {

        await AssetManager.Instance.LoadSceneAsync(args.SceneName, LoadSceneMode.Single);
    }
}
```

## 5. 错误处理

```csharp
    // 连接错误统一处理（断网、授权失败等）
    public static void OnConnectError(int errCode, string err)
    {
#if UNITY_EDITOR
        // Editor 专属：提供自助诊断提示（连接超时检查代理、授权失败检查账号）
        MsdkLogin.ClearLocalCacheToken();
        UIHelper.ShowMessageBox("登录问题自助指南(EditorOnly):\n..."
            + "\n1.连接超时/网络异常(201): 请检查本地网络是否正常，代理是否关闭"
            + "\n2.鉴权失败，请检查是否新号首次登录，需要在手机上登录一次注册");
#endif
        Log.Error($"网络异常 onConnectError: {err}, code: {errCode}");
        
        // 非用户主动取消的错误，退出登录让玩家重新登录
        if (errCode != (int)ConnectorErrorCode.UserCancel
            && errCode != (int)ConnectorErrorCode.UserDenied
            && errCode != (int)GCloud.MSDK.MSDKError.CANCEL
            && errCode != 26)  // 26: 微信授权错误
        {
            UIHelper.ShowTips(err);
            SDKUtil.LogoutGame(true);  // 清空 Token，强制重新登录
        }
    }
```

## 6. 常见问题与最佳实践

**Q: 登录流程卡住（Progress 一直 loading）？**  
A: 检查 `CoroutineLockComponent` 是否有死锁（旧的协程锁未释放）；检查 TCP 连接是否成功（`ZoneConnect` 是否返回）；检查服务器协议响应超时（默认 30 秒超时后触发 `receiveErrorHandle`）。

**Q: Android 和 iOS 登录行为有什么差异？**  
A: `ClientType` 字段告知服务器客户端类型，服务器用于：①App Store/Google Play 内购验证；②平台专属更新包下发；③iOS 审核期间关闭部分运营活动（遵守 Apple 审核规则）。

**Q: 场景切换时如何防止资源未卸载导致的内存泄漏？**  
A: `SceneChangeLoadPrepareEvent` 触发时，各系统（Audio、Battle、UI 等）监听此事件，主动调用 `OnSceneDestroy()` 释放持有的资源引用，再由 `AssetManager` 统一调用 `UnloadUnusedAssets`。
