# 游戏运行时类型信息替代方案与高性能多态分发系统完全指南

## 一、问题背景：IL2CPP 下的反射性能陷阱

Unity 发布 iOS、Android 高性能版本时会开启 IL2CPP 后端，将 C# IL 代码转换为 C++ 代码再编译为原生机器码。虽然运行时性能提升明显，但反射（Reflection）在 IL2CPP 下的代价远高于 Mono 运行时：

```csharp
// ❌ 危险：在热路径中调用反射
void ProcessMessage(object msg)
{
    var type = msg.GetType(); // IL2CPP 下约 200-500ns
    var method = type.GetMethod("Handle"); // 约 1-5μs，并且无法内联
    method.Invoke(null, new object[] { msg }); // 装箱 + 堆分配
}
```

### IL2CPP 反射性能测试数据

| 操作 | Mono JIT | IL2CPP | 差距 |
|------|---------|--------|------|
| `typeof(T)` | ~2ns | ~2ns | 1x |
| `obj.GetType()` | ~5ns | ~15ns | 3x |
| `GetMethod()` | ~500ns | ~3000ns | 6x |
| `MethodInfo.Invoke()` | ~800ns | ~5000ns | 6x |
| `Activator.CreateInstance()` | ~2μs | ~15μs | 7x |

在每帧处理数万消息、技能事件、碰撞回调的游戏场景中，这些开销会造成明显卡顿。

## 二、方案一：自定义类型 ID 注册表

### 2.1 设计原理

放弃 `Type` 对象，改用整数 ID 标识类型，所有分发逻辑基于整数比较（O(1) 字典查找或数组索引）。

```csharp
// TypeRegistry.cs - 编译时/启动时注册所有游戏类型
public static class TypeRegistry
{
    private static int _nextId = 1;
    private static readonly Dictionary<Type, int> _typeToId = new(256);
    private static readonly Dictionary<int, Type> _idToType = new(256);

    public static int Register<T>()
    {
        var type = typeof(T);
        if (_typeToId.TryGetValue(type, out var existingId))
            return existingId;

        var id = _nextId++;
        _typeToId[type] = id;
        _idToType[id] = type;
        return id;
    }

    public static int GetId<T>() => TypeId<T>.Value;
    public static int GetId(Type type) => _typeToId.TryGetValue(type, out var id) ? id : 0;
    public static Type GetType(int id) => _idToType.TryGetValue(id, out var type) ? type : null;
}

// 泛型静态字段技巧：每个 T 只初始化一次，之后是纯字段读取
public static class TypeId<T>
{
    public static readonly int Value = TypeRegistry.Register<T>();
}
```

### 2.2 在消息系统中应用

```csharp
// 消息基类携带类型 ID
public abstract class GameMessage
{
    // 子类通过 static readonly 字段缓存，不走虚方法
    public abstract int TypeId { get; }
}

// 具体消息
public class DamageMessage : GameMessage
{
    public static readonly int StaticTypeId = TypeId<DamageMessage>.Value;
    public override int TypeId => StaticTypeId;

    public int SourceEntityId;
    public int TargetEntityId;
    public float Amount;
    public DamageType Type;
}

public class HealMessage : GameMessage
{
    public static readonly int StaticTypeId = TypeId<HealMessage>.Value;
    public override int TypeId => StaticTypeId;

    public int TargetEntityId;
    public float Amount;
}

// 分发器：基于 int 的字典路由，零反射
public class MessageDispatcher
{
    // 使用 int 键的字典，比 Type 键快约 3-5 倍
    private readonly Dictionary<int, Action<GameMessage>> _handlers = new(64);

    public void Register<T>(Action<T> handler) where T : GameMessage
    {
        int typeId = TypeId<T>.Value;
        _handlers[typeId] = msg => handler((T)msg); // 直接类型转换，非反射
    }

    public void Dispatch(GameMessage message)
    {
        if (_handlers.TryGetValue(message.TypeId, out var handler))
            handler(message);
    }
    
    // 批量分发版本，减少字典查找开销
    public void DispatchBatch(ReadOnlySpan<GameMessage> messages)
    {
        foreach (var msg in messages)
        {
            if (_handlers.TryGetValue(msg.TypeId, out var handler))
                handler(msg);
        }
    }
}
```

## 三、方案二：手工虚表（Virtual Dispatch Table）

### 3.1 为什么要绕过 C# 虚方法？

C# 虚方法本身很快，但在以下场景有额外开销：
- 虚方法无法内联（JIT 有限，IL2CPP 更保守）
- 接口调用需要查询接口虚表（ITable Pointer Indirection）
- 泛型约束虚调用存在额外间接层

通过手工虚表，我们可以在确定类型后**完全消除间接调用**。

### 3.2 函数指针方案（C# 9+ unsafe）

```csharp
// 使用函数指针代替委托，零堆分配，可内联
public unsafe struct ComponentVTable
{
    public delegate*<Entity, void> OnUpdate;
    public delegate*<Entity, void> OnDestroy;
    public delegate*<Entity, BinaryWriter, void> Serialize;
    public delegate*<Entity, BinaryReader, void> Deserialize;
}

// 注册组件虚表
public static class ComponentRegistry
{
    private static ComponentVTable[] _vtables = new ComponentVTable[256];
    private static int _count = 0;

    public static int Register<T>() where T : IGameComponent
    {
        int id = _count++;
        _vtables[id] = new ComponentVTable
        {
            OnUpdate = &T.StaticOnUpdate,
            OnDestroy = &T.StaticOnDestroy,
            Serialize = &T.StaticSerialize,
            Deserialize = &T.StaticDeserialize,
        };
        return id;
    }

    public static ref ComponentVTable GetVTable(int typeId) => ref _vtables[typeId];
}

// 使用场景：比 C# 虚方法快约 2-4x，可被编译器内联
unsafe void UpdateComponent(Entity entity, int componentTypeId)
{
    ref var vtable = ref ComponentRegistry.GetVTable(componentTypeId);
    vtable.OnUpdate(entity); // 直接函数指针调用
}
```

### 4.2 Source Generator 实现

```csharp
[Generator]
public class MessageDispatcherGenerator : ISourceGenerator
{
    public void Initialize(GeneratorInitializationContext context)
    {
        context.RegisterForSyntaxNotifications(() => new MessageSyntaxReceiver());
    }

    public void Execute(GeneratorExecutionContext context)
    {
        if (context.SyntaxReceiver is not MessageSyntaxReceiver receiver)
            return;

        var messageTypes = CollectMessageTypes(context, receiver);
        
        // 生成分发器
        var sb = new StringBuilder();
        sb.AppendLine("// <auto-generated />");
        sb.AppendLine("using System;");
        sb.AppendLine("namespace Game.Network {");
        sb.AppendLine("public static partial class MessageDispatcher {");
        sb.AppendLine("  public static void Dispatch(int opcode, ReadOnlySpan<byte> data, IMessageSession session) {");
        sb.AppendLine("    switch (opcode) {");
        
        foreach (var (opcode, typeName) in messageTypes)
        {
            sb.AppendLine($"      case {opcode}:");
            sb.AppendLine($"        var msg{opcode} = MessagePool<{typeName}>.Rent();");
            sb.AppendLine($"        msg{opcode}.Deserialize(data);");
            sb.AppendLine($"        {typeName}Handler.Handle(msg{opcode}, session);");
            sb.AppendLine($"        MessagePool<{typeName}>.Return(msg{opcode});");
            sb.AppendLine($"        break;");
        }
        
        sb.AppendLine("      default:");
        sb.AppendLine("        LogUnknownOpcode(opcode);");
        sb.AppendLine("        break;");
        sb.AppendLine("    }");
        sb.AppendLine("  }");
        sb.AppendLine("}");
        sb.AppendLine("}");
        
        context.AddSource("MessageDispatcher.generated.cs", sb.ToString());
    }
    
    private IEnumerable<(int opcode, string typeName)> CollectMessageTypes(
        GeneratorExecutionContext context, 
        MessageSyntaxReceiver receiver)
    {
        foreach (var classDecl in receiver.Classes)
        {
            var model = context.Compilation.GetSemanticModel(classDecl.SyntaxTree);
            var symbol = model.GetDeclaredSymbol(classDecl);
            
            var attr = symbol?.GetAttributes()
                .FirstOrDefault(a => a.AttributeClass?.Name == "GameMessageAttribute");
            
            if (attr?.ConstructorArguments[0].Value is int opcode)
                yield return (opcode, symbol.ToDisplayString());
        }
    }
}
```

## 五、方案四：开放寻址哈希表的类型分发

### 5.1 自定义 int → Action 开放寻址哈希表

标准 `Dictionary<int, Action>` 在负载因子高时性能下降，且存在装箱风险。自定义开放寻址实现：

```csharp
/// <summary>
/// 针对 int 键优化的开放寻址哈希分发表
/// 比 Dictionary 快约 30-50%，零装箱
/// </summary>
public struct FixedDispatchTable<T>
{
    private struct Entry
    {
        public int Key;
        public Action<T> Handler;
        public bool IsOccupied;
    }

    private Entry[] _entries;
    private int _mask;

    public FixedDispatchTable(int capacity)
    {
        // 确保容量为2的幂
        var size = NextPowerOfTwo(capacity * 2); // 负载因子 0.5
        _entries = new Entry[size];
        _mask = size - 1;
    }

    public void Register(int key, Action<T> handler)
    {
        var idx = key & _mask;
        while (_entries[idx].IsOccupied && _entries[idx].Key != key)
            idx = (idx + 1) & _mask;
        
        _entries[idx] = new Entry { Key = key, Handler = handler, IsOccupied = true };
    }

    public bool TryDispatch(int key, T value)
    {
        var idx = key & _mask;
        while (_entries[idx].IsOccupied)
        {
            if (_entries[idx].Key == key)
            {
                _entries[idx].Handler(value);
                return true;
            }
            idx = (idx + 1) & _mask;
        }
        return false;
    }

    private static int NextPowerOfTwo(int n)
    {
        n--;
        n |= n >> 1; n |= n >> 2; n |= n >> 4; n |= n >> 8; n |= n >> 16;
        return n + 1;
    }
}

// 全局技能触发分发表（容量 512，满足大多数游戏需求）
public static class SkillDispatchTable
{
    private static FixedDispatchTable<SkillContext> _table = new(512);

    public static void Register(int skillId, Action<SkillContext> executor)
        => _table.Register(skillId, executor);

    public static bool Execute(int skillId, SkillContext context)
        => _table.TryDispatch(skillId, context);
}
```

## 六、完整工程示例：零反射的 ECS 组件系统

将以上方案组合，构建一个完整的 ECS 组件注册与调度系统：

```csharp
// ============================================================
// 组件接口（只定义静态入口，不使用虚方法）
// ============================================================
public interface IComponentStatic<TSelf> where TSelf : struct, IComponentStatic<TSelf>
{
    static abstract void OnAdd(int entityId, ref TSelf component);
    static abstract void OnRemove(int entityId, ref TSelf component);
    static abstract void OnUpdate(int entityId, ref TSelf component, float deltaTime);
}

// ============================================================
// 组件注册表（启动时一次性初始化）
// ============================================================
public static class ComponentSystem
{
    private struct ComponentMeta
    {
        public int TypeId;
        public string Name;
        public Action<int, float> UpdateAll; // 更新该类型所有实例
    }

    private static readonly ComponentMeta[] _metas = new ComponentMeta[256];
    private static int _count;

    public static int Register<T>() where T : struct, IComponentStatic<T>
    {
        var id = TypeId<T>.Value;
        _metas[_count++] = new ComponentMeta
        {
            TypeId = id,
            Name = typeof(T).Name,
            UpdateAll = (entityId, dt) =>
            {
                ref var comp = ref ComponentStorage<T>.Get(entityId);
                T.OnUpdate(entityId, ref comp, dt); // 静态接口调用，无虚派发
            }
        };
        return id;
    }

    public static void UpdateAllSystems(float deltaTime)
    {
        // 按注册顺序执行所有系统
        for (int i = 0; i < _count; i++)
        {
            _metas[i].UpdateAll(0, deltaTime);
        }
    }
}

// ============================================================
// 组件存储（SoA 结构化存储，Cache 友好）
// ============================================================
public static class ComponentStorage<T> where T : struct
{
    // 稀疏集合：entityId -> index 映射
    private static readonly int[] _sparse = new int[65536]; // entityId -> denseIndex
    private static T[] _dense = new T[1024];               // 紧密存储
    private static int[] _entityIds = new int[1024];        // 反向映射
    private static int _count;

    static ComponentStorage()
    {
        Array.Fill(_sparse, -1);
    }

    public static ref T Add(int entityId)
    {
        if (_count >= _dense.Length)
            Array.Resize(ref _dense, _dense.Length * 2);

        _sparse[entityId] = _count;
        _entityIds[_count] = entityId;
        _dense[_count] = default;
        return ref _dense[_count++];
    }

    public static ref T Get(int entityId)
    {
        var idx = _sparse[entityId];
        return ref _dense[idx];
    }

    public static bool Has(int entityId) => _sparse[entityId] >= 0;

    public static void Remove(int entityId)
    {
        var idx = _sparse[entityId];
        var last = _count - 1;
        
        // Swap with last (O(1) removal)
        _dense[idx] = _dense[last];
        _entityIds[idx] = _entityIds[last];
        _sparse[_entityIds[last]] = idx;
        _sparse[entityId] = -1;
        _count--;
    }

    // 批量迭代，Cache 友好
    public static Span<T> GetAll() => _dense.AsSpan(0, _count);
}

// ============================================================
// 使用示例
// ============================================================
public struct HealthComponent : IComponentStatic<HealthComponent>
{
    public float Current;
    public float Max;
    public float RegenPerSecond;

    public static void OnAdd(int entityId, ref HealthComponent component)
    {
        component.Max = 100f;
        component.Current = 100f;
        component.RegenPerSecond = 5f;
    }

    public static void OnRemove(int entityId, ref HealthComponent component) { }

    public static void OnUpdate(int entityId, ref HealthComponent component, float deltaTime)
    {
        if (component.Current < component.Max)
            component.Current = Math.Min(component.Max, component.Current + component.RegenPerSecond * deltaTime);
    }
}

// 启动时注册
public class GameBootstrap : MonoBehaviour
{
    void Awake()
    {
        // 一次性注册，之后零反射
        ComponentSystem.Register<HealthComponent>();
        ComponentSystem.Register<ManaComponent>();
        ComponentSystem.Register<MovementComponent>();
        ComponentSystem.Register<CombatComponent>();
    }

    void Update()
    {
        // 零反射更新所有 ECS 系统
        ComponentSystem.UpdateAllSystems(Time.deltaTime);
    }
}
```

## 八、最佳实践总结

### ✅ 推荐做法

1. **热路径禁用反射**：`GetType()`、`GetMethod()`、`Invoke()` 一律移出 Update/Fixed Update
2. **类型 ID 缓存**：使用 `TypeId<T>.Value` 泛型静态字段，访问开销接近零
3. **switch-case 优于字典**：消息 opcode 数量 < 1000 时，生成代码的 switch 比字典快 2x
4. **SoA 内存布局**：ECS 组件使用稀疏集合 + 紧密数组，最大化 Cache 命中率
5. **函数指针替代委托**：在 C# 9+ 且 unsafe 允许的场景，`delegate*` 可被内联

### ⚠️ 注意事项

1. 函数指针（`delegate*`）需要 `unsafe` 上下文，须在 Project Settings 中开启 "Allow unsafe code"
2. 静态接口（`static abstract`）需要 C# 11 / .NET 7+，Unity 2022.2+ 支持
3. 开放寻址哈希表容量固定，需提前估算最大消息类型数
4. Source Generator 调试需要 IDE 插件支持（VS2022 / Rider 2023+）

### 🔧 工具链建议

- **性能分析**：使用 Unity Profiler Deep Profile 定位热点，配合 `ProfilerMarker` 精确计时
- **验证零分配**：Frame Debugger + Memory Profiler 确认 GC 分配为 0
- **压测**：每帧注入 5 万+ 消息验证方案在高负载下的稳定性

## 总结

IL2CPP 下的反射性能陷阱是游戏客户端优化的重要议题。通过**类型 ID 注册表 + 手工虚表 + Source Generator + 自定义哈希表**四种方案的组合运用，可以在保持代码可维护性的前提下，将消息分发性能提升 **20-200 倍**，彻底消除 GC 分配，为高帧率游戏提供坚实的底层支撑。
