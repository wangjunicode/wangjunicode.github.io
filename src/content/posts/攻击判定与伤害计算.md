# 10 攻击判定与伤害计算

> 本文详细介绍从攻击框命中到最终伤害数值输出的完整流程，包括攻击判定结果类型、伤害计算公式、数值系统架构，以及暴击、格挡、背击等特殊情况的处理。

---

## 1. 系统概述

攻击判定与伤害计算是将"碰撞检测结果"转化为"游戏数值变化"的关键环节。当攻击框命中受击框后，系统需要：
1. 确定命中结果类型（普通/格挡/闪避/破防）
2. 计算基础伤害（攻击力 × 倍率 - 防御力）
3. 应用各种修正（暴击/背击/Buff加成/抗性）
4. 扣减目标血量/护盾
5. 触发战斗点事件（用于 Buff、被动技能响应）

### 1.1 攻击结果类型

```csharp
// 攻击/命中结果枚举
public enum EAttackResult
{

    Normal,          // 普通命中
    Critical,        // 暴击

// 受击结果（从受击方角度）
public enum EHurtResult
{
    Normal,         // 普通受击
    Parry,          // 格挡成功
    Doge,           // 闪避成功
    BreakDefense,   // 被破防
    PerformHurt,    // 演出受击（破防后的追打）
    SwordPlay,      // 拼刀结果
}
```

### 1.2 数值系统概览

伤害计算依赖 `NumericComponent` 中定义的各种属性：

| 数值 ID（ENumericId） | 含义 | 说明 |
|----------------------|------|------|
| `HP` | 当前生命值 | 受到伤害后扣除 |
| `HP_Max` | 最大生命值 | 血量上限 |
| `Guard` | 护盾值 | 优先于 HP 抵消伤害 |
| `Attack` | 攻击力（基础） | 伤害计算基础 |
| `Defense` | 防御力 | 减少受到的伤害 |
| `CriticalRate` | 暴击率（%） | 触发暴击概率 |
| `CriticalDmgRate` | 暴击伤害倍率（%） | 暴击时额外乘数 |
| `DmgRate` | 伤害加成率（%） | 攻击方额外加成 |
| `DmgReduceRate` | 伤害减免率（%） | 防御方减免 |
| `ElementDmgRate` | 元素伤害加成 | 火/冰/雷等元素加成 |

## 2. 架构设计

### 2.1 伤害计算的数据流

```
碰撞命中（ColliderComponent 检测到重叠）
    │
    ▼
生成 AttackInfo（攻击信息包）
    ├── Attacker（攻击者 Unit）
    ├── SkillId / SubSkillIndex
    ├── 攻击方向 / 受击点
    └── SubSkillConfigClone（技能数值配置）
    │
    ▼
触发战斗点事件 BPDef.PreDamage
    └→ Buff/被动技能响应（修改 AttackInfo 参数，如注入暴击）
    │
    ▼
AT_AttackDamage（伤害计算节点）
    │
    ├─ 读取攻击方数值（Attack/CriticalRate/DmgRate 等）
    ├─ 读取防御方数值（Defense/DmgReduceRate/Guard 等）
    ├─ 读取技能配置倍率（SubSkillConfigClone.DamageRate）
    ├─ 计算基础伤害
    ├─ 判断暴击（随机数 vs CriticalRate）
    ├─ 判断背击（IsBackHit → 额外加成）
    ├─ 扣减护盾（Guard 先于 HP 抵消）
    └─ 扣减 HP（NumericComponent 修改）
    │
    ▼
触发战斗点事件 BPDef.PostDamage
    └→ Buff/被动技能响应（如"受到伤害后回血"）
    │
    ▼
检查死亡（HP ≤ 0?）→ 触发 BPDef.Dead 事件 → FSM 进入 Dead 状态
```

### 2.2 NumericComponent —— 数值组件核心

数值系统采用**分层修改器（Modifier Layer）**设计，每个数值分为 8 层：

```
ENumericPart 枚举（数值分量）：
├── Base       ← 基础值（从配置表读取的初始值）
├── Min        ← 最小值限制
├── Max        ← 最大值限制
├── Add        ← 绝对值加成（Buff 的 +100 攻击力）
├── Pct        ← 百分比加成（Buff 的 +10% 攻击力）
├── FinalAdd   ← 最终绝对值加成（最后叠加）
├── FinalPct   ← 最终百分比加成（最后叠加）
└── （预留）
```

计算公式：
```
最终值 = Clamp(
    ((Base + Add) × (100 + Pct) / 100 + FinalAdd) × (100 + FinalPct) / 100,
    Min, Max
)
```

### 2.3 ModifierHandle —— 修改器句柄

每次添加数值修改都返回一个句柄，用于精确移除：

```csharp
public struct ModifierHandle
{
    public ushort NumericId;   // 数值 ID（如 HP/Attack）
    public byte Part;          // 数值分量（Base/Add/Pct等）
    public int Id;             // 修改器唯一 ID
    public int Version;        // 版本号（防止旧句柄误操作）

    // 有效性检查
    public bool IsValid => Id != 0 && NumericId != 0;
}
```

**句柄的重要性**：

```csharp
// ✅ 正确：使用句柄精确移除
var handle = numericComp.AddModifier(ENumericId.Attack, ENumericPart.Add, 100, myBuff);
// ... 一段时间后 ...
numericComp.RemoveModifier(handle);  // 精确移除这个 +100 攻击，不影响其他加成

// ❌ 错误：直接设值（会覆盖其他 Buff 的加成）
numericComp.SetAdd(ENumericId.Attack, 0);  // 把所有 Add 层都清零了！
```

### 3.1 NumericComponent.cs —— 数值组件

    // 技能段落的随机次数记录
    public DeterministicDictionary<Vector2_Int, DeterministicDictionary<int, int>> RandSkillFTimes = new();
    
    // 锁定状态（批量修改时锁定，防止中间状态触发事件）
    public bool locked = false;
    public bool Locked => locked;
    
    // 清理标志（已被销毁）
    public bool bCleared { get; set; } = false;
    
    // 静态初始化：计算最大数值 ID（用于数组分配）
    static NumericComponent()
    {
        foreach (int value in Enum.GetValues(typeof(ENumericId)))
        {
            if (value == (int)ENumericId.Max) continue;
            if (value > NumericIdMax) NumericIdMax = value;
        }
        NumericIdMax += 1;
    }
}

### 3.2 NumericInfo.cs —— 单个数值的计算逻辑

```csharp
public class NumericInfo
{
    // 各分量当前值
    private FP value = FP.Zero;    // Base
    private FP add = FP.Zero;      // Add 层总和
    private FP pct = FP.Zero;      // Pct 层总和
    private FP finalAdd = FP.Zero; // FinalAdd 层总和
    private FP finalPct = FP.Zero; // FinalPct 层总和
    private FP min = FP.MinValue;  // 最小值限制
    private FP max = FP.MaxValue;  // 最大值限制
    private FP final = FP.Zero;    // 缓存的最终值（避免重复计算）
    
    // 修改器列表（每层各有一个列表）
    private ModifierList[] modifierLists;
    
    // 核心公式：重新计算最终值
    private void RecalculateFinalReadOnly(out FP finalNew, out FP pctNew)
    {
        // final = ((Base + Add) × (100 + Pct) / 100 + FinalAdd) × (100 + FinalPct) / 100
        FP result = (((Value + add) * ((100 + pct) / 100)) + finalAdd) * ((100 + finalPct) / 100);
        result = TSMath.Clamp(result, this.min, this.max);
        finalNew = result;
        pctNew = result / max * 100;  // 百分比（如血量百分比）
    }
    
    // 重新计算并触发变化事件（如血量变化通知血条 UI）
    public bool RecalculateFinal()
    {
        var fnOld = final;
        RecalculateFinalReadOnly(out var finalNew, out var pctNew);
        var change = fnOld != finalNew;
        if (change)
        {
            final = finalNew;
            // 触发数值变化事件（血条/属性面板 UI 监听此事件更新）
            NumericWatcherComponent.Instance.Run(entityTemp,
                new NumericChange() { Entity = entityTemp, Old = fnOld, New = finalNew, NumericType = nid });
        }
        return change;
    }
    
    // 添加修改器（Buff 添加 +100 攻击力）
    public ModifierHandle AddModifier(ENumericPart part, FP value, object owner)
    {
        var list = GetList((int)part);
        var handle = list.Add(nid, (int)part, value, owner);
        ApplyPart((int)part, list.Total);  // 更新该层总值，触发重算
        return handle;
    }
    
    // 移除修改器（Buff 消失时移除加成）
    public void RemoveModifier(ModifierHandle handle)
    {
        var part = handle.Part;
        var list = modifierLists[part];
        if (list == null) return;
        list.RemoveByHandle(handle);
        ApplyPart(part, list.Total);  // 更新总值，触发重算
    }
}
```

### 3.5 AT_AttackDamage —— 伤害计算节点

```csharp
[Name("受到攻击伤害")]
[MemoryPackable]
public partial class AT_AttackDamage : ActionTask
{
    // 伤害类型（物理/魔法/真实等，对应不同的防御类型）
    [fsSerializeAs("at")]
    [MemoryPackOrder(0)]
    public EActionType dmgType;
    
    // Handler 中的伤害计算逻辑（伪代码）：
    /*
    1. 获取 AttackInfo（从上下文/黑板读取）
    2. 获取 attacker.NumericComponent[Attack].Final 攻击力
    3. 获取 SubSkillConfigClone.DamageRate 技能倍率
    4. 基础伤害 = 攻击力 × 技能倍率
    5. 触发 BPDef.PreDamage 事件（Buff 可在此修改基础伤害）
    6. 判断暴击：Random() < attacker[CriticalRate] → 暴击
       暴击伤害 = 基础伤害 × (1 + CriticalDmgRate / 100)
    7. 判断背击：AttackInfo.IsBackHit → 背击加成
    8. 计算减伤：减伤后伤害 = 伤害 × (1 - defender[DmgReduceRate] / 100)
    9. 护盾抵消：Guard -= 伤害，剩余伤害 → HP
    10. HP -= 剩余伤害（通过 RemoveModifier 或 SetValue）
    11. 触发 BPDef.PostDamage 事件
    12. 检查死亡：HP <= 0 → 触发 FSM Dead 状态
    */
}
```

### 3.6 ModifierList.cs —— 修改器列表

```csharp
public class ModifierList
{
    private NumericOp[] items;   // 修改器数组（固定数组，避免 List 的装箱）
    private int nextId = 1;      // 下一个修改器 ID（单调递增）
    private int version = 1;     // 版本号（Clear 时递增，使旧句柄失效）
    
    private int count;
    private FP total;            // 缓存该层的总值（O(1) 读取）

    // 添加修改器，返回句柄
    public ModifierHandle Add(int nid, int part, FP value, object owner)
    {
        if (count >= items.Length) Resize();  // 动态扩容
        var modifier = new NumericOp(value, nextId++, owner);
        items[count] = modifier;
        total += modifier.Value;              // 维护总值缓存
        count++;
        return new ModifierHandle(nid, part, modifier.Id, version);
    }

    // 通过句柄移除修改器（O(n)，通常 n 很小）
    public bool RemoveByHandle(ModifierHandle handle)
    {
        if (handle.Version != version) return false;  // 版本过期，安全跳过
        
        for (int i = 0; i < count; i++)
        {
            if (items[i].Id != handle.Id) continue;
            total -= items[i].Value;
            // 最后一个元素填补空位（O(1) 删除，不维护顺序）
            items[i] = items[--count];
            items[count] = default;
            return true;
        }
        return false;
    }
    
    // 批量清空（版本号递增，所有旧句柄自动失效）
    public void Clear()
    {
        count = 0;
        total = FP.Zero;
        nextId = 1;
        version++;  // 版本号递增，之前所有句柄都失效
    }
}
```

## 4. 特殊命中情况详解

### 4.1 暴击（Critical）

```
触发条件：Random(0,100) < CriticalRate（来自 NumericComponent）
伤害倍率：基础伤害 × (1 + CriticalDmgRate / 100)
触发事件：BPDef.OnCritical（暴击专属 Buff 可在此响应）
视觉反馈：跳字显示特殊暴击字体+颜色
```

### 4.2 背击（Back Hit）

```
触发条件：
    V = 攻击者位置 - 目标位置（归一化）
    F = 目标朝向（Forward）
    Dot(V, F) > 0.7 → 背击（夹角 < 约 45°）

伤害加成：通常由技能配置中的 BackHitRate 字段控制
额外效果：某些 Buff 会在背击时触发（如"背刺"被动）
```

### 4.3 格挡（Parry）

```
触发条件：
    目标处于格挡状态（FSM = ParryState）
    攻击框与格挡框相交（EColliderCheckType.Parry）

效果：
    ParryHitTimes++（记录格挡中被命中次数）
    伤害按格挡减伤率计算（如减少 50%）
    触发格挡特效和音效
    
破防条件：ParryHitTimes ≥ 配置的破防阈值
    → FSM 切换到 BreakDefense 状态
    → 攻击方可以发动追打
```

### 4.4 护盾（Guard）抵消

```
判定流程：
    if (Guard > 0) {
        实际伤害到 Guard：Guard -= min(伤害, Guard)
        溢出伤害 = max(伤害 - Guard_before, 0)
        HP -= 溢出伤害
    } else {
        HP -= 伤害
    }

护盾归零时：Guard Buff 自动移除（触发 BPDef.PostDamage）
```

## 5. 设计亮点

### 5.1 数值分层修改器——精确的叠加控制
数值系统的 8 层分量设计，确保不同来源的数值修改（装备、Buff、技能、被动）相互独立，移除任意一个来源时只影响该来源的贡献，不会影响其他来源。

### 5.2 ModifierHandle 版本号防御
`ModifierList.Clear()` 时版本号递增，使所有旧句柄自动失效。这防止了"持有过期句柄并误操作"的 Bug——如 Buff 被强制清除后，其 Handler 仍然尝试用旧句柄移除修改器，版本不匹配会安全返回 false 而不是误操作其他修改器。

### 5.3 战斗点事件系统的可扩展性
伤害流水线在关键节点（PreDamage、PostDamage、OnCritical）发布战斗点事件，任何 Buff 和被动技能都可以监听这些事件。这意味着可以用纯配置实现"受到伤害时回血 10%"、"暴击时额外获得 AP"等复杂效果，无需修改伤害计算代码。

### 5.4 护盾优先于血量的明确分层
护盾（Guard）和血量（HP）是独立的数值，伤害先消耗护盾再消耗血量，逻辑清晰。护盾层的变化通过 `BPDef.PreHealGuard`/`PostHealGuard` 事件通知，与血量事件完全解耦。

## 6. 常见问题与最佳实践

### Q1：如何给技能增加额外伤害？
**A**：通过以下方式之一：
1. **修改技能配置**：调整 `CSubSkill.DamageRate`（技能伤害倍率）
2. **Buff 在 PreDamage 时增益**：监听 `BPDef.PreDamage`，通过修改 AttackInfo 中的伤害参数
3. **临时修改 NumericComponent**：通过 `AddModifier(ENumericId.DmgRate, ENumericPart.Add, 50, buff)` 添加 +50% 伤害加成

### Q2：为什么要在 NumericInfo 中缓存 final 值，不每次都重算？
**A**：数值读取极其频繁（每帧、每次 AI 决策、每次 UI 刷新都需要），但修改相对稀少。缓存 final 值使读取 O(1)，只有在修改时重算，大幅提升性能。`RecalculateFinal()` 使用 `fnOld != finalNew` 防止无效事件触发（数值没变时不发布事件）。

### Q3：确定性随机数（RandExpFTimes）具体怎么用于暴击判断？
**A**：战斗初始化时用双方共享的随机种子初始化确定性随机数生成器。每次需要随机（如暴击判断）时，从该生成器取下一个随机值，同时 `RandExpFTimes[CriticalExpression]++` 记录调用次数。帧同步哈希校验时，比对各端的调用次数，若不一致说明存在逻辑分歧。

### Q4：如何实现"真实伤害"（穿透所有防御）？
**A**：通过 `EActionType.TrueDamage`（真实伤害类型）标识，在 `AT_AttackDamage` 的 Handler 中检查此类型，跳过防御减伤计算，直接扣减 HP。Buff 的 `PreDamage` 事件同样可以通过检查 ActionType 决定是否修改伤害。

### Q5：数值的 Max 层（上限）是否会影响 Modifier 的 Add 层？
**A**：是的，Max 层限制的是最终值（Final），不是 Add 层的累加。即使 Add 层加了很多，Final 也不会超过 Max。这用于实现"最大生命值上限"、"最大护盾量限制"等设计。
