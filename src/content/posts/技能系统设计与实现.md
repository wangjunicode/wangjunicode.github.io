# 04 技能系统设计与实现

> 本文详细介绍技能系统的完整流程，从技能的数据结构、释放流程、子技能（SubSkill）机制，到技能与 Timeline、FSM 的协作关系。

---

## 1. 系统概述

技能系统是战斗引擎中最复杂的子系统之一，它负责：
- 管理角色拥有的技能列表
- 判断技能是否满足释放条件（CD、AP、状态等）
- 驱动技能的执行流程（前摇→攻击帧→后摇）
- 配合 Timeline 精确控制每一帧的攻击判定、动作特效、Root Motion 等

### 1.1 技能系统核心概念

| 概念 | 说明 |
|------|------|
| `SkillGraph` | 技能逻辑图（UniScript FlowCanvas），技能的完整逻辑描述 |
| `SkillComponent` | Unit 上的技能状态组件，持有当前技能状态 |
| `SkillMgrComponent` | Unit 上的技能管理组件，维护已学技能列表 |
| `CSubSkill` | 子技能配置（来自 Luban），描述技能的数值参数 |
| `SubSkillIndex` | 当前执行到第几段子技能（连击索引） |
| `EPoseDirection` | 技能动作的朝向（影响后续可接的技能） |
| `EMoveSkillType` | 技能的移动类型（站立/前进/后退等） |

### 1.2 技能释放的完整流程

```
玩家输入（EInputKey.Skill_X）
    └→ GS_TryUseSkill 节点执行
         ├→ canUse 检查（CD/AP/状态判断）
         │    ├→ 条件满足 → onUse 分支（技能执行）
         │    └→ 条件不满足 → failed 分支（失败）
         └→ 执行 onUse：
              ├→ 扣除消耗（AP/Token等）
              ├→ 播放动画（Timeline 驱动）
              ├→ 攻击帧：激活 ColliderComponent 的攻击框
              ├→ 命中判定：ColliderComponent 与敌方碰撞
              ├→ 触发受击：目标 Unit 进入受击 FSM 状态
              └→ 技能结束：FSM 返回 Idle
```

## 2. 架构设计

### 2.1 SkillComponent 结构

```csharp
public class SkillComponent : Entity, IAwake, ILateUpdate, IFixedUpdate, IDestroy, IReset
{
    // 当前拥有的技能图列表
    public List<SkillGraph> SkillList { get; } = new();
    
    // 当前正在执行的技能
    public SkillGraph CurrentSkill { get; set; }
    
    // 当前执行到第几段连击
    public int CurrentSubSkillIndex { get; set; }

    // 当前技能的姿势方向（影响连击衔接）
    public EPoseDirection PoseDirection { get; set; }
    
    // 移动技能类型
    public EMoveSkillType CurMoveSkillType { get; set; } = EMoveSkillType.Stay;
    
    // 闪避状态
    public bool bInDodge { get; set; } = false;
    
    // 前摇状态（Pre-Cast Delay）
    public bool bInPreCast = false;
    public FP RestPreCastDelay;
    
    // 被中断的输入记录（防止打断后误触发）
    public HashSet<EInputKey> AbortInput { get; } = new();
}
```

### 2.3 子技能（SubSkill）机制

一个"技能"在配置上可以包含多段"子技能"（`CSubSkill`），每段子技能对应一段动作。例如：

```
普通攻击技能（SkillId = 1001）
├── SubSkill[0]: 第一段攻击（左挥）
│     ├── 动画：atk_01
│     ├── 攻击框：前方矩形，1帧激活
│     └── 伤害倍率：1.0x
├── SubSkill[1]: 第二段攻击（右挥）
│     ├── 动画：atk_02
│     ├── 攻击框：前方矩形+侧面，2帧激活
│     └── 伤害倍率：1.2x
└── SubSkill[2]: 第三段攻击（重击）
      ├── 动画：atk_03
      ├── 攻击框：大范围圆形
      └── 伤害倍率：2.0x
```

`SkillComponent.CurrentSubSkillIndex` 指向当前执行到哪段。每段结束时，根据玩家输入决定继续下一段（连击）还是回到 Idle。

### 2.4 技能与 Timeline 的关系

技能执行时，通过 `GS_PlayTimeline` 节点播放对应的 Timeline（UniScript 时间轴系统）。Timeline 上的 Clip（片段）精确控制每一帧的行为：

```
技能 Timeline（以普通攻击为例）
│
├─── [0~10帧] CLIP_RootMotion    ← 前摇位移（Root Motion 驱动）
├─── [3~12帧] CLIP_HurtCollider  ← 开启受击框（角色可被攻击）
├─── [5~8帧]  CLIP_SetAttackCollider ← 开启攻击框（判定命中）
├─── [8帧]    CLIP_AttackEnd     ← 攻击结束事件
├─── [5~10帧] CLIP_UniHurt       ← 受击配置（被打时的响应类型）
└─── [12帧]   结束，FSM 返回 Idle
```

### 2.5 GS_TryUseSkill 节点设计

这是触发技能执行的核心节点：

```
GS_TryUseSkill
├── Input: SkillId（要释放的技能 ID）
├── Input: 技能所需变量（如目标 Unit、施法位置等）
└── Output:
    ├── _successEnter  ← 技能开始时触发（即时）
    ├── _successComplete ← 技能完整执行完毕时触发
    └── _failed        ← 技能无法释放时触发（CD/条件不满足）
```

### 3.3 CLIP_RootMotion —— 动作位移控制

```csharp
[Attachable(typeof(ActorActionTrack))]
[Category("Root Motion")]
[Name("Root Motion")]
[MemoryPackable]
public partial class CLIP_RootMotion : ActionClip, IResourceMapping
{
    public bool motionWarping;        // 是否启用 Motion Warping（追踪目标位置）
    public MotionWarpingData motionWarppingData;  // Warp 配置
    
    public FP humanScale = FP.One;    // 人物缩放比例
    public FP rootScale = FP.One;     // Root Motion 位移缩放

    // 采样指定时间点的位移和旋转数据（从预烘焙的 RootMotionData 中读取）
    public void GetSampleData(FP time, ref TSVector displacement, ref TSQuaternion rotation)
    {
        if (_enable && rootMotionData != null)
        {
            rootMotionData.GetSampleData(time, ref displacement, ref rotation);
            displacement *= EngineDefine.UseAnimStream ? FP.One : humanScale;
        }            
    }
}
```

## 4. 技能执行时序详解

### 4.1 前摇阶段（Pre-Cast）

部分技能在真正释放前有一个"蓄力"阶段。玩家按下技能键后：
1. `bInPreCast = true`，`RestPreCastDelay` 赋值为前摇时间
2. 每个 FixedUpdate 减少 `RestPreCastDelay`
3. 倒计时结束 → 真正开始技能执行
4. 若等待期间玩家取消或被打断，则 `bInPreCast = false`，技能取消

### 4.2 技能执行阶段

技能通过 `GS_PlayTimeline` 节点推进 Timeline：
1. Timeline 启动，从第 0 帧开始
2. 每个 `FixedUpdate`，Timeline 向前推进一帧
3. 各 Clip 在激活范围内执行对应逻辑
4. 攻击框 Clip 激活时（`CLIP_SetAttackCollider`），`ColliderComponent` 开始检测碰撞
5. `CLIP_AttackEnd` 触发时，清理攻击状态

### 4.3 Motion Warping（位移吸附）

当技能需要让角色"自动追踪"到目标位置时，使用 Motion Warping：

```
CLIP_RootMotion.motionWarping = true
│
└→ 在 Warp 时间窗口内（startTime ~ endTime）：
     ├→ 读取目标当前位置（CurPredictPos）
     └→ 将 Root Motion 位移重新映射到指向目标方向
          （原始动画帧位移 × 缩放 → 重新方向化）
```

## 5. 设计亮点

### 5.1 技能配置与代码完全分离
技能逻辑（何时能用、如何执行）全部在 UniScript 图中配置，程序只提供底层 ActionTask 实现。策划可以独立迭代技能逻辑，甚至在不重新编译代码的情况下调整技能流程。

### 5.2 SubSkillConfigClone 动态修改技能参数
`SkillComponent` 维护了一份子技能配置的克隆（`SubSkillConfigClone`），允许 Buff 在运行时临时修改技能参数（如增加攻击框大小、提高伤害倍率），而不影响原始配置数据，技能结束后克隆自动重置。

### 5.3 AbortInput 防止误触发
当技能被打断时，记录被中断时的输入状态。技能恢复后会先消耗掉这些"残留输入"，防止因为打断瞬间的按键残留导致立即触发下一个行为，提升操作体验的流畅感。

### 5.4 技能图缓存池
`UnitComponent.SkillCache` 是一个技能图的对象池。技能图（JSON 反序列化后的 `SkillGraph` 对象）创建开销较大，使用完毕后归还池中，下次需要同种技能时直接取出复用，大幅减少 GC 压力。

### 5.5 姿势方向（PoseDirection）影响连击判定
`PoseDirection` 记录当前技能结束时角色的"姿势朝向"（前、后、左、右等）。下一段连击技能通过检查 PoseDirection 来决定播放哪个衔接动画，确保连击过渡的动画流畅性。

## 6. 常见问题与最佳实践

### Q1：如何给一个角色新增一个技能？
**A**：
1. 在 UniScript 编辑器中创建新的技能图（SkillGraph），配置 `canUse`（释放条件）和 `onUse`（执行逻辑）两个入口事件
2. 在 Luban 配置表中配置技能的数值参数（`CSubSkill`）
3. 在角色配置表中关联新技能 ID
4. 代码无需改动，框架自动加载

### Q2：技能 CD 是怎么实现的？
**A**：CD 通过 `GS_StartCD` 节点启动，本质是在 Unit 的黑板或数值组件中记录一个冷却时间戳。`CT_CheckCD` 条件节点检查 CD 是否已冷却，作为 `canUse` 的判断条件之一。

### Q3：技能被打断后怎么处理？
**A**：技能被打断时，`CurrentSkill` 对应的 SkillGraph 会被 Stop，FSM 切换到受击状态。SkillComponent 的 `AbortInput` 会记录被打断时的输入状态。技能图的 Handler 负责在 Stop 时做清理工作（清除攻击框、还原临时修改的数值等）。

### Q4：SkillGraph 和 UniFSM 有什么区别？
**A**：`SkillGraph` 是 FlowCanvas 图（流程图，线性执行），用于描述"一个技能的执行流程"；`UniFSM` 是状态机图，用于描述"角色在不同状态之间的转移逻辑"。技能执行是有明确开始和结束的，所以用 FlowCanvas；角色状态是持续性的、响应式的，所以用 FSM。

### Q5：SubSkillIndex 是如何递增的？
**A**：每段子技能的 Timeline 执行到 `CLIP_AttackEnd` 时，系统根据当前的玩家输入（是否按下了连击键）决定是否 `CurrentSubSkillIndex++`。若玩家没有在时间窗口内输入，则回到 `SubSkillIndex = 0`（第一段）。

### Q6：CLIP 和 ActionTask 有什么区别？
**A**：`CLIP_*` 是 Timeline 上的时间片段（有持续时长，如"在第 3~8 帧激活攻击框"）；`ActionTask` 是 FSM/FlowCanvas 上的即时动作节点（瞬间执行，如"添加一个 Buff"）。两者都通过 Handler 模式实现具体逻辑。
