# 伤害计算处理器的完整实现与分层架构

前面的文章分析了伤害公式的数学推导，本文聚焦于伤害计算在代码中的完整执行流程——从公式计算、Pre/Post 钩子、护盾与 HP 的分流计算，到吸血效果的处理。

`AT_AttackDamageHandler` 是整个战斗系统中最核心的"一行代码"——虽然只是一个处理器节点，但它串联了所有与伤害相关的系统。

---

## AT_AttackDamageHandler 的完整实现

```csharp
public class AT_AttackDamageHandler : AActionTaskHandler<AT_AttackDamage>
{
    protected override void OnExecute(AT_AttackDamage task)
    {
        // === 第一步：确定攻防双方 ===
        var attacker = task.Agent<Entity>()
            .GetComponent<HurtComponent>().AttackInfo.Attacker.RootUnit;
        var atkResult = attacker.GetAttackResult();  // 会心/破招/普通
        var enemy = task.Agent<Unit>();
        var enemyTeam = enemy.获取队伍();
        var attackerNumeric = attacker.GetComponent<NumericComponent>();
        var enemyNumeric = enemy.GetComponent<NumericComponent>();
        
        // === 第二步：计算基础伤害 ===
        EExpression.AttackDamage.TryGetFormulaBindValue(attacker, out var damage);
        
        // HP 直接伤害 = 穿血加成 × 基础伤害 + （破招时的额外增伤）
        FP hpDamage = (attackerNumeric.穿血伤害加成() + 
            (atkResult == EAttackResult.Critical ? attackerNumeric.当前破招增伤() : 0)) * damage;
        
        // === 第三步：护盾与 HP 伤害分流 ===
        var guardMulti = 1 + attackerNumeric.GetFinalValue(ENumericId.AtkGuardMulti) / 100;
        var hpMulti = 1 + attackerNumeric.GetFinalValue(ENumericId.AtkHpMulti) / 100;
        
        if (damage * guardMulti > enemyNumeric.当前抗压值())
        {
            // 伤害超过护盾上限：护盾归零 + 溢出部分转为 HP 伤害
            hpDamage += damage - enemyNumeric.当前抗压值();
            damage = enemyNumeric.当前抗压值();  // 护盾只扣到0
        }
        else
        {
            damage *= guardMulti;  // 护盾倍率加成（某些攻击对护盾伤害更高）
        }
        
        hpDamage *= hpMulti;  // HP 伤害倍率
        
        // === 第四步：Pre 事件（允许外部修改伤害）===
        attackerNumeric.SetPart(ENumericId.LastGuardDamage, ENumericPart.Base, damage);
        attackerNumeric.SetPart(ENumericId.LastHpDamage, ENumericPart.Base, hpDamage);
        
        // 触发 PreDamage 事件（Buff 可以在这里修改 LastGuardDamage / LastHpDamage）
        attacker.ClientScene().GetComponent<EventDispatcherComponent>()
            .FireEvent(new Evt_PointEvent() { Unit = enemy, Team = enemyTeam, EventID = BPDef.PreDamage });
        EventSystem.Instance.Publish(attacker.DomainScene(), 
            new Evt_PointEvent() { Unit = enemy, Team = enemyTeam, EventID = BPDef.PreDamage });
        
        // 读取 Pre 事件后可能被修改的值
        damage = attackerNumeric.GetFinalValue(ENumericId.LastGuardDamage);
        hpDamage = attackerNumeric.GetFinalValue(ENumericId.LastHpDamage);
        
        // === 第五步：锁血/锁护盾保护机制 ===
        FP lifeSteal = attackerNumeric.GetFinalValue(ENumericId.LifeSteal) / 100;
        
        // 重新检查护盾溢出
        if (damage > enemyNumeric.当前抗压值())
        {
            hpDamage += damage - enemyNumeric.当前抗压值();
            damage = enemyNumeric.当前抗压值();
        }
        
        // 锁护盾：某些状态下护盾不能降至0（保留1点）
        if (battleSpt.IsLockGraud(enemy))
        {
            var guard = enemyNumeric.GetFinalValue(ENumericId.RealGuard);
            damage = TSMath.Min(guard - 1, damage);
        }
        
        // 锁血：某些状态下HP不能降至0（保留1点）
        if (battleSpt.IsLockTeamHp(enemy))
        {
            var hp = enemyNumeric.队伍数值().GetFinalValue(ENumericId.RealTeamHp);
            hpDamage = TSMath.Min(hp - 1, hpDamage);
        }
        
        // === 第六步：实际扣除数值 ===
        // 扣除 HP（队伍共享）
        enemyNumeric.队伍数值().ModifyPart(ENumericId.RealTeamHp, ENumericPart.Base, -1 * hpDamage);
        
        // 吸血（HP）
        if (hpDamage * lifeSteal != FP.Zero)
            attacker.HealHp(hpDamage * lifeSteal);
        
        // 扣除护盾（角色个人）
        enemyNumeric.ModifyPart(ENumericId.RealGuard, ENumericPart.Base, -1 * damage);
        
        // 吸盾（护盾）
        if (damage * lifeSteal != FP.Zero)
            attacker.HealGuard(attacker, damage * lifeSteal);
        
        // === 第七步：Post 事件（伤害已施加，通知各系统）===
        attacker.ClientScene().GetComponent<EventDispatcherComponent>()
            .FireEvent(new Evt_AppendDamage() { Atker = attacker, Defer = enemy, Damage = damage, HpDamage = hpDamage });
        attacker.ClientScene().GetComponent<EventDispatcherComponent>()
            .FireEvent(new Evt_PointEvent() { Unit = enemy, Team = enemyTeam, EventID = BPDef.PostDamage });
        EventSystem.Instance.Publish(attacker.DomainScene(), 
            new Evt_PointEvent() { Unit = enemy, Team = enemyTeam, EventID = BPDef.PostDamage });
    }
}
```

## 护盾（Guard）与 HP 的双层生命系统

这套战斗设计引入了双层生命：

```
护盾（RealGuard）  →  个人属性，仅该角色承受
HP（RealTeamHp）  →  队伍属性，队伍成员共享
```

**护盾的角色**：护盾是第一道防线，优先承受伤害。护盾归零才会影响 HP。

**HP 的队伍性**：HP 是整个队伍共享的，无论哪个角色受到穿透到 HP 层的伤害，都从队伍 HP 池中扣除。这意味着——角色 A 的护盾被打穿，HP 减少，会影响整个队伍的存活状态。

### 伤害分流逻辑

```
基础伤害 = formula_result（数值公式计算）

护盾伤害 = min(基础伤害 × guardMulti, 当前护盾)
HP 伤害 = hpDamage + max(0, 基础伤害 - 当前护盾)
       + 穿血额外伤害
```

**穿血伤害（HP Direct Damage）**：部分伤害直接穿透护盾，不经过护盾减免，直接作用于 HP。这通过 `攻击方.穿血伤害加成()` 实现，让高穿透类技能更有威胁感。

**守护多线**：`AtkGuardMulti` 和 `AtkHpMulti` 分别控制对护盾和对 HP 的伤害倍率，允许设计"专门克制高护盾"或"专门克制低血量"的技能。

## Pre 钩子的设计精妙之处

```csharp
// 先写入临时变量
attackerNumeric.SetPart(ENumericId.LastGuardDamage, ENumericPart.Base, damage);
attackerNumeric.SetPart(ENumericId.LastHpDamage, ENumericPart.Base, hpDamage);

// 触发 Pre 事件（Buff 修改这两个临时变量）
EventSystem.Publish(new Evt_PointEvent { EventID = BPDef.PreDamage });

// 读取可能被修改后的值
damage = attackerNumeric.GetFinalValue(ENumericId.LastGuardDamage);
hpDamage = attackerNumeric.GetFinalValue(ENumericId.LastHpDamage);
```

**为什么要通过数值存储临时伤害值？**

如果 Pre 事件处理器需要修改伤害值，它需要一个"传递参数的通道"。直接通过事件参数修改有局限性（参数是只读的）。

把伤害值写入 `NumericComponent` 的特定 ID（`LastGuardDamage`/`LastHpDamage`），让 Pre 事件处理器可以用 `ModifyPart` 任意修改，甚至可以叠加多个 Buff 的修改效果（修改器系统支持多来源叠加）。

这是一个非常巧妙的设计，将"战斗中的临时计算中间值"也纳入了数值系统的管理。

## 锁血/锁护盾机制

```csharp
if (battleSpt.IsLockGraud(enemy))
{
    var guard = enemyNumeric.GetFinalValue(ENumericId.RealGuard);
    damage = TSMath.Min(guard - 1, damage);  // 护盾保留至少1点
}

if (battleSpt.IsLockTeamHp(enemy))
{
    var hp = enemyNumeric.队伍数值().GetFinalValue(ENumericId.RealTeamHp);
    hpDamage = TSMath.Min(hp - 1, hpDamage);  // HP保留至少1点
}
```

这是一种"保护机制"，在某些特殊情况下防止伤害计算将目标数值降至0：

- **剧情保护**：BOSS 的剧情必须在特定血量下发生，血量提前降至0会破坏剧情
- **技能保护**：某个 Buff 效果是"当护盾降至1时触发特效"，需要锁住护盾防止直接归零跳过特效
- **轮次保护**：某些轮次设计要求玩家无法一击必杀

`battleSpt.IsLockGraud/IsLockTeamHp` 从战斗脚本组件查询当前是否有锁定保护。

## 双目标广播

```csharp
// 本地广播（只在自己的场景中）
attacker.ClientScene().GetComponent<EventDispatcherComponent>()
    .FireEvent(new Evt_PointEvent { EventID = BPDef.PreDamage });

// 全局广播（通过 EventSystem）
EventSystem.Instance.Publish(attacker.DomainScene(), 
    new Evt_PointEvent { EventID = BPDef.PreDamage });
```

Pre/Post 事件通过两个渠道广播：

1. **LocalDispatcher（`ClientScene()`）**：触发本地场景中注册的监听器（如玩家角色的被动技能）
2. **GlobalEventSystem（`EventSystem.Publish`）**：触发全局注册的监听器（如统计系统、成就系统）

这种双重广播确保了不同类型的系统都能接收到伤害事件。

## 吸血效果

```csharp
FP lifeSteal = attackerNumeric.GetFinalValue(ENumericId.LifeSteal) / 100;

// 按 HP 伤害吸血（回复自身 HP）
if (hpDamage * lifeSteal != FP.Zero)
    attacker.HealHp(hpDamage * lifeSteal);

// 按护盾伤害吸盾（回复自身护盾）
if (damage * lifeSteal != FP.Zero)
    attacker.HealGuard(attacker, damage * lifeSteal);
```

吸血（`LifeSteal`）是造成伤害的同时回复自身生命的效果。这里区分了两种吸取：
- HP 吸血：回复自身队伍 HP
- 护盾吸取（吸盾）：回复自身护盾值

不等于零的检查避免了吸血量为 0 时的无效函数调用。

## 设计总结

`AT_AttackDamageHandler` 的七步流程展示了完整的伤害计算链路：

| 步骤 | 操作 | 目的 |
|------|------|------|
| 1 | 确定攻防双方 | 上下文初始化 |
| 2 | 公式计算基础伤害 | 数值系统核心 |
| 3 | 护盾/HP 分流 | 双层生命系统 |
| 4 | Pre 事件 + 临时值写入 | 允许外部修改伤害 |
| 5 | 锁血/锁护盾 | 特殊保护机制 |
| 6 | 实际扣除 + 吸血 | 最终数值结算 |
| 7 | Post 事件 | 通知各系统 |

关键设计哲学：
- **可扩展**：Pre/Post 钩子让任何系统都可以介入伤害计算
- **双层生命**：护盾（个人）+ HP（队伍）的设计增加策略深度  
- **临时变量**：借用 NumericComponent 传递 Pre 事件的修改参数
- **锁定保护**：战斗脚本可以干预伤害，支持剧情和特殊机制

这段代码虽然只有几十行，却体现了整个战斗系统设计哲学的精髓：每一行都有其理由，每一个扩展点都经过深思熟虑。
