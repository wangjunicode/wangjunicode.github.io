## 虚函数是什么

虚函数（virtual function）是 C++ 实现**运行时多态**的核心机制。当通过基类指针（或引用）调用虚函数时，会根据对象的实际类型来决定调用哪个版本的函数。

```cpp
class Animal {
public:
    virtual void Sound() { cout << "..."; }  // 虚函数
};

class Dog : public Animal {
public:
    void Sound() override { cout << "汪汪"; }  // 重写
};

Animal* animal = new Dog();
animal->Sound();  // 输出 "汪汪"（运行时决议，而非编译期）
```

## 实现机制：虚函数表（vtable）

### 虚函数表结构

虚函数通过**虚函数表（vtable）** 实现。每个含有虚函数的类都有一张虚函数表：

```
Animal 类的 vtable：
┌──────────────────────────────┐
│ Animal::Sound  → &Animal::Sound  │
│ Animal::Eat    → &Animal::Eat    │
└──────────────────────────────┘

Dog 类的 vtable（重写了 Sound）：
┌──────────────────────────────┐
│ Dog::Sound     → &Dog::Sound     │  ← 改写为 Dog 的版本
│ Animal::Eat    → &Animal::Eat    │  ← 继承自 Animal
└──────────────────────────────┘
```

### 虚表指针（vptr）

每个含虚函数的类的**对象**，其内存头部都有一个**虚表指针（vptr）**，指向该类的虚函数表：

```
Dog 对象内存布局：
┌──────────────┐
│  vptr        │ → 指向 Dog 的 vtable
├──────────────┤
│  成员变量 1   │
├──────────────┤
│  成员变量 2   │
└──────────────┘
```

## 编译期 vs 运行时

| 调用方式 | 决议时机 | 示例 |
|---------|--------|------|
| 通过指针/引用调用 | **运行时**（查 vtable）| `animal->Sound()` |
| 通过值调用 | **编译期**（静态绑定）| `animal.Sound()` |
| 非虚函数 | **编译期** | 任何非虚函数 |

```cpp
Animal a;
Dog d;

Animal& ref = d;
ref.Sound();      // 运行时决议 → 调用 Dog::Sound

Animal val = d;   // 对象切片！
val.Sound();      // 编译期决议 → 调用 Animal::Sound
```

## 虚函数表的共享

**虚函数表是类级别的，不是对象级别的**，同一个类的所有对象共享同一张虚函数表，但每个对象有自己的虚表指针（vptr）：

```
Dog 对象 d1:  vptr → Dog::vtable
Dog 对象 d2:  vptr → Dog::vtable（同一张表）
Dog 对象 d3:  vptr → Dog::vtable（同一张表）
```

## 纯虚函数与抽象类

```cpp
class Shape {
public:
    virtual float Area() = 0;  // 纯虚函数，= 0 表示没有实现
    virtual ~Shape() {}        // 虚析构函数（必须！）
};

class Circle : public Shape {
    float radius;
public:
    float Area() override { return 3.14f * radius * radius; }
};
```

- 包含纯虚函数的类是**抽象类**，不能直接实例化
- 派生类必须实现所有纯虚函数才能实例化

## 虚析构函数的必要性

如果基类指针指向派生类对象，且通过基类指针删除对象时，**必须将析构函数声明为虚函数**：

```cpp
class Base {
public:
    virtual ~Base() { cout << "Base 析构"; }
};

class Derived : public Base {
public:
    ~Derived() override { cout << "Derived 析构"; }
};

Base* p = new Derived();
delete p;
// 有虚析构：先调用 Derived 析构，再调用 Base 析构
// 无虚析构：只调用 Base 析构，Derived 的资源泄漏！
```

## 性能影响

虚函数相比普通函数有一点额外开销：
1. 每次调用需要**间接寻址**（通过 vptr 找 vtable，再找函数地址）
2. 每个对象多了一个 **vptr 指针**（通常 8 字节）
3. 编译器无法对虚函数进行**内联优化**

在游戏开发的热路径中（如每帧大量调用），需要注意虚函数的开销，考虑用 ECS 或组合模式替代。
