# 游戏热重载与运行时代码更新系统完全指南

## 一、热重载的本质与技术挑战

热重载（Hot Reload）是指在不重启应用程序的前提下，动态替换运行中的代码逻辑。在游戏客户端开发中，热重载技术贯穿了两个核心场景：

1. **开发期热重载**：加速迭代，修改逻辑后秒级生效，无需重启游戏
2. **运营期热更新**：线上版本不停服更新，推送补丁修复 Bug 或更新玩法逻辑

两者虽目标一致，但约束条件截然不同：
- 开发期可接受一定程度的状态丢失，关键是速度快
- 运营期必须保证状态连续性、版本回滚安全性，以及多平台审核合规

```
热重载技术栈全景
├── 开发期热重载
│   ├── Unity Domain Reload 优化（Skip Domain Reload）
│   ├── Roslyn 增量编译 + C# 热重载（Visual Studio / Rider）
│   ├── Assembly Hot Swap（Mono Soft Debugger 协议）
│   └── 脚本语言热重载（Lua reload / Python reload）
└── 运营期热更新
    ├── Lua 热更新（xlua / tolua / ulua）
    ├── C# 热更新（ILRuntime / HybridCLR）
    ├── 资源热更新（Addressables / AssetBundle）
    └── 逻辑补丁（IL Patch / Method Swap）
```

## 二、Unity Domain Reload 深度优化

### 2.1 Domain Reload 的性能瓶颈

Unity 每次进入 Play Mode，默认会执行完整的 Domain Reload：

```
Domain Reload 流程（耗时分布）
1. 销毁所有静态变量              ~200ms
2. 卸载当前 AppDomain            ~100ms  
3. 重新加载所有程序集             ~800ms
4. 重新初始化 Unity Native 状态   ~300ms
5. 执行所有 [InitializeOnLoad]    ~依项目规模
─────────────────────────────────
总计：大型项目可达 5~15 秒
```

### 2.2 跳过 Domain Reload 的配置

### 2.3 静态变量生命周期管理

```csharp
/// <summary>
/// 可热重载的静态状态容器
/// 配合 Skip Domain Reload 使用，确保进入 PlayMode 时状态正确初始化
/// </summary>
public static class HotReloadableState
{
    // 记录所有需要重置的静态字段
    private static readonly List<Action> _resetActions = new List<Action>();
    
    /// <summary>
    /// 注册需要在 PlayMode 开始时重置的静态变量
    /// </summary>
    public static void RegisterReset(Action resetAction)
    {
        _resetActions.Add(resetAction);
    }

    [RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.SubsystemRegistration)]
    private static void OnSubsystemRegistration()
    {
        // SubsystemRegistration 在每次进入 PlayMode 时都会触发（即使 Skip Domain Reload）
        foreach (var resetAction in _resetActions)
        {
            resetAction?.Invoke();
        }
        Debug.Log($"[HotReloadableState] 已重置 {_resetActions.Count} 个静态状态");
    }
}

/// <summary>
/// 使用示例：需要在 PlayMode 重置的管理类
/// </summary>
public static class GameManager
{
    private static int _score;
    private static bool _isGameRunning;
    private static Dictionary<string, object> _globalData;

    static GameManager()
    {
        // 注册重置逻辑
        HotReloadableState.RegisterReset(Reset);
    }

    private static void Reset()
    {
        _score = 0;
        _isGameRunning = false;
        _globalData = new Dictionary<string, object>();
    }

    public static int Score => _score;
}
```

## 三、HybridCLR：C# 原生热更新完全解析

### 3.1 HybridCLR 核心原理

HybridCLR（原名 Huatuo）是基于 IL2CPP 扩展的 C# 热更新方案，其核心思想是：

```
传统 IL2CPP 流程：
C# 代码 → IL → AOT 编译 → Native 机器码 → 无法热更

HybridCLR 流程：
热更新 C# 代码 → IL → 动态加载 → 混合解释执行
                              ↑
                    AOT 程序集提前编译的部分无需解释
                    热更新程序集在运行时解释执行
```

### 3.2 HybridCLR 工程集成

```csharp
/// <summary>
/// 热更新启动器：负责下载、加载热更新 DLL 并初始化
/// </summary>
public class HotUpdateLauncher : MonoBehaviour
{
    [Header("热更新配置")]
    [SerializeField] private string _remoteBaseUrl = "https://cdn.example.com/hotupdate/";
    [SerializeField] private string[] _hotUpdateDlls = { "HotUpdate.dll", "Game.Logic.dll" };
    [SerializeField] private string[] _aotMetaDlls = { "mscorlib.dll", "System.dll" };

    private async void Start()
    {
        await LoadHotUpdateAssemblies();
    }

    private async Task LoadHotUpdateAssemblies()
    {
        // Step 1：为 AOT 程序集补充元数据（解决泛型问题）
        await LoadAOTMetadata();

        // Step 2：下载并加载热更新 DLL
        foreach (var dllName in _hotUpdateDlls)
        {
            byte[] dllBytes = await DownloadDll(dllName);
            if (dllBytes != null)
            {
                Assembly hotUpdateAssembly = Assembly.Load(dllBytes);
                Debug.Log($"[HotUpdate] 加载成功: {hotUpdateAssembly.FullName}");
                
                // 触发热更新程序集的入口
                InvokeHotUpdateEntry(hotUpdateAssembly);
            }
        }
    }

    private async Task LoadAOTMetadata()
    {
        foreach (var dllName in _aotMetaDlls)
        {
            byte[] dllBytes = await DownloadDll(dllName);
            if (dllBytes != null)
            {
                // HybridCLR API：为 AOT 程序集补充元数据
                var err = HybridCLR.RuntimeApi.LoadMetadataForAOTAssembly(
                    dllBytes, 
                    HybridCLR.HomologousImageMode.SuperSet
                );
                Debug.Log($"[HotUpdate] AOT 元数据补充: {dllName} -> {err}");
            }
        }
    }

    private async Task<byte[]> DownloadDll(string dllName)
    {
        string url = $"{_remoteBaseUrl}{dllName}";
        
        // 先检查本地缓存
        string localPath = Path.Combine(Application.persistentDataPath, "hotupdate", dllName);
        if (File.Exists(localPath))
        {
            // 检查版本号（通过 manifest 文件）
            if (IsLocalVersionLatest(dllName))
            {
                return File.ReadAllBytes(localPath);
            }
        }
        
        // 下载远端最新版本
        using var request = UnityWebRequest.Get(url);
        await request.SendWebRequest();
        
        if (request.result == UnityWebRequest.Result.Success)
        {
            byte[] data = request.downloadHandler.data;
            
            // 缓存到本地
            Directory.CreateDirectory(Path.GetDirectoryName(localPath)!);
            File.WriteAllBytes(localPath, data);
            
            return data;
        }
        
        Debug.LogError($"[HotUpdate] 下载失败: {url}, {request.error}");
        return null;
    }

    private void InvokeHotUpdateEntry(Assembly assembly)
    {
        // 约定热更新程序集有统一的入口类
        var entryType = assembly.GetType("HotUpdateEntry");
        if (entryType != null)
        {
            var entryMethod = entryType.GetMethod("Run", 
                BindingFlags.Static | BindingFlags.Public);
            entryMethod?.Invoke(null, null);
        }
    }

    private bool IsLocalVersionLatest(string dllName)
    {
        // 读取本地 manifest 与远端 manifest 对比 MD5/版本号
        // 实际项目中通过版本管理服务器实现
        return false; // 简化示例，始终重新下载
    }
}
```

```csharp
        }
```

## 四、Lua 热更新：xlua 深度实践

### 4.1 Lua 热更新的核心机制

Lua 热更新基于一个关键特性：**Lua 的 `require` 机制与 `package.loaded` 缓存表**。

```lua
-- Lua require 原理
-- 第一次 require("module")：执行并缓存到 package.loaded
-- 后续 require("module")：直接返回缓存，不重新执行

-- 热重载的本质：清除缓存，强制重新加载
function hot_reload(module_name)
    package.loaded[module_name] = nil   -- 清除缓存
    return require(module_name)          -- 重新加载
end
```

### 4.2 xlua 集成与 C# 交互

```csharp
/// <summary>
/// Lua 虚拟机管理器：管理 xlua LuaEnv 生命周期与热重载
/// </summary>
public class LuaManager : SingletonMono<LuaManager>
{
    private LuaEnv _luaEnv;
    private float _gcInterval = 1.0f;
    private float _lastGCTime;

    protected override void Awake()
    {
        base.Awake();
        InitLuaEnv();
    }

    private void InitLuaEnv()
    {
        _luaEnv = new LuaEnv();
        
        // 注册自定义 Loader：从 Addressables 加载 Lua 脚本
        _luaEnv.AddLoader(CustomLuaLoader);
        
        // 注入全局工具函数
        _luaEnv.DoString(@"
            -- 热重载工具函数
            function hot_reload(module_name)
                package.loaded[module_name] = nil
                local ok, err = pcall(require, module_name)
                if not ok then
                    error('[HotReload] 重载失败: ' .. tostring(err))
                end
                print('[HotReload] 重载成功: ' .. module_name)
            end
            
            -- 批量热重载
            function hot_reload_all(modules)
                for _, name in ipairs(modules) do
                    hot_reload(name)
                end
            end
        ");
        
        // 执行入口脚本
        _luaEnv.DoString("require('Main')");
    }

    /// <summary>
    /// 自定义 Lua 文件加载器：优先从热更新目录加载，回退到 Resources
    /// </summary>
    private byte[] CustomLuaLoader(ref string filePath)
    {
        string hotUpdatePath = Path.Combine(
            Application.persistentDataPath, 
            "lua", 
            filePath.Replace('.', '/') + ".lua"
        );
        
        // 热更新目录优先
        if (File.Exists(hotUpdatePath))
        {
            return File.ReadAllBytes(hotUpdatePath);
        }
        
        // 回退到内置资源
        string resourcePath = "Lua/" + filePath.Replace('.', '/');
        TextAsset textAsset = Resources.Load<TextAsset>(resourcePath);
        if (textAsset != null)
        {
            return textAsset.bytes;
        }
        
        Debug.LogError($"[LuaManager] 找不到 Lua 文件: {filePath}");
        return null;
    }

    /// <summary>
    /// 执行 Lua 热重载（开发期使用，或运营期 Lua 层补丁）
    /// </summary>
    public void HotReloadLuaModule(string moduleName)
    {
        _luaEnv.DoString($"hot_reload('{moduleName}')");
    }

    /// <summary>
    /// 安全调用 Lua 函数
    /// </summary>
    public T CallLuaFunction<T>(string funcPath, params object[] args)
    {
        try
        {
            LuaTable luaTable = _luaEnv.Global;
            string[] parts = funcPath.Split('.');
            
            for (int i = 0; i < parts.Length - 1; i++)
            {
                luaTable = luaTable.Get<LuaTable>(parts[i]);
                if (luaTable == null)
                {
                    Debug.LogError($"[LuaManager] 路径不存在: {parts[i]}");
                    return default;
                }
            }
            
            var func = luaTable.Get<LuaFunction>(parts[^1]);
            if (func == null) return default;
            
            var results = func.Call(args);
            return results != null && results.Length > 0 ? (T)results[0] : default;
        }
        catch (Exception e)
        {
            Debug.LogError($"[LuaManager] 调用失败 {funcPath}: {e.Message}");
            return default;
        }
    }

    private void Update()
    {
        // 定期 GC，防止 Lua 内存泄漏
        if (Time.time - _lastGCTime > _gcInterval)
        {
            _luaEnv.Tick();
            _lastGCTime = Time.time;
        }
    }

    private void OnDestroy()
    {
        _luaEnv.Dispose();
    }
}
```

### 4.3 Lua 侧的热重载安全模式

```lua
-- SafeHotReload.lua
-- 带状态保留的安全热重载模块

local M = {}

-- 热重载前保存的状态
local _saved_state = {}

--- 保存当前模块状态（热重载前调用）
function M.save_state(module_name, state_getter)
    _saved_state[module_name] = state_getter()
end

--- 恢复模块状态（热重载后调用）
function M.restore_state(module_name, state_setter)
    local state = _saved_state[module_name]
    if state then
        state_setter(state)
        _saved_state[module_name] = nil
    end
end

--- 安全热重载：保留状态
--- @param module_name string 模块路径
--- @param state_getter function 获取当前状态的函数
--- @param state_setter function 恢复状态的函数
function M.safe_reload(module_name, state_getter, state_setter)
    -- 1. 保存当前状态
    if state_getter then
        M.save_state(module_name, state_getter)
    end
    
    -- 2. 执行热重载
    package.loaded[module_name] = nil
    local ok, err = pcall(require, module_name)
    
    if not ok then
        print('[SafeHotReload] 重载失败: ' .. tostring(err))
        return false
    end
    
    -- 3. 恢复状态
    if state_setter then
        M.restore_state(module_name, state_setter)
    end
    
    print('[SafeHotReload] 重载成功: ' .. module_name)
    return true
end

return M
```

## 五、运行时方法替换（Method Swap / IL Patch）

### 5.1 原理：通过反射修改方法指针

```csharp
/// <summary>
/// IL 级别的方法替换工具（仅限 Mono 环境，开发期调试使用）
/// 警告：此技术依赖 Mono 内部结构，生产环境请使用 HybridCLR
/// </summary>
public static class MethodPatcher
{
    // Mono 内部方法指针偏移（x64）
    private const int MethodInfoOffset = 0x60;

    /// <summary>
    /// 将 originalMethod 的方法体替换为 patchMethod
    /// </summary>
    public static void Patch(MethodInfo originalMethod, MethodInfo patchMethod)
    {
        if (originalMethod == null || patchMethod == null)
            throw new ArgumentNullException();

        // 验证签名兼容性
        ValidateSignature(originalMethod, patchMethod);

        unsafe
        {
            // 获取方法的原生函数指针
            IntPtr originalPtr = originalMethod.MethodHandle.GetFunctionPointer();
            IntPtr patchPtr = patchMethod.MethodHandle.GetFunctionPointer();

#if UNITY_EDITOR_WIN || UNITY_STANDALONE_WIN
            // Windows: 修改内存保护属性后写入
            VirtualProtect(originalPtr, new UIntPtr(1), 0x40, out uint oldProtect);
            
            byte* orig = (byte*)originalPtr.ToPointer();
            byte* patch = (byte*)patchPtr.ToPointer();
            
            // 写入 JMP 指令：E9 [相对偏移]
            long relativeOffset = (long)patchPtr - (long)originalPtr - 5;
            orig[0] = 0xE9;
            *((int*)(orig + 1)) = (int)relativeOffset;
            
            VirtualProtect(originalPtr, new UIntPtr(1), oldProtect, out _);
#endif
        }
    }

    private static void ValidateSignature(MethodInfo original, MethodInfo patch)
    {
        var origParams = original.GetParameters();
        var patchParams = patch.GetParameters();
        
        if (origParams.Length != patchParams.Length)
            throw new InvalidOperationException("方法参数数量不匹配");
        
        if (original.ReturnType != patch.ReturnType)
            throw new InvalidOperationException("方法返回类型不匹配");
    }

#if UNITY_EDITOR_WIN || UNITY_STANDALONE_WIN
    [DllImport("kernel32.dll")]
    private static extern bool VirtualProtect(
        IntPtr lpAddress, UIntPtr dwSize, uint flNewProtect, out uint lpflOldProtect);
#endif
}

/// <summary>
/// 方法替换的高级封装：支持通过特性标记补丁方法
/// </summary>
[AttributeUsage(AttributeTargets.Method)]
public class PatchMethodAttribute : Attribute
{
    public Type TargetType { get; }
    public string TargetMethodName { get; }

    public PatchMethodAttribute(Type targetType, string methodName)
    {
        TargetType = targetType;
        TargetMethodName = methodName;
    }
}

/// <summary>
/// 自动扫描并应用所有 [PatchMethod] 标记的方法替换
/// </summary>
public static class PatchManager
{
    public static void ApplyAllPatches(Assembly patchAssembly)
    {
        foreach (var type in patchAssembly.GetTypes())
        {
            foreach (var method in type.GetMethods(
                BindingFlags.Static | BindingFlags.Public | BindingFlags.NonPublic))
            {
                var attr = method.GetCustomAttribute<PatchMethodAttribute>();
                if (attr == null) continue;

                var targetMethod = attr.TargetType.GetMethod(
                    attr.TargetMethodName,
                    BindingFlags.Instance | BindingFlags.Static | 
                    BindingFlags.Public | BindingFlags.NonPublic);

                if (targetMethod != null)
                {
                    MethodPatcher.Patch(targetMethod, method);
                    Debug.Log($"[PatchManager] 已应用补丁: {attr.TargetType.Name}.{attr.TargetMethodName}");
                }
            }
        }
    }
}
```

```csharp
        // 1. 拉取远端清单
        HotUpdateManifest remoteManifest = await FetchRemoteManifest();
        if (remoteManifest == null) return false;

        // 2. 检查基准版本兼容性
        if (remoteManifest.AppVersion != Application.version)
        {
            Debug.Log($"[VersionMgr] 版本不兼容，需要更新安装包: {remoteManifest.AppVersion} != {Application.version}");
            // 引导用户去应用商店更新
            ShowForceUpdateDialog();
            return false;
        }

        // 3. 检查是否需要热更
        int localVersion = GetLocalHotUpdateVersion();
        if (remoteManifest.HotUpdateVersion <= localVersion)
        {
            Debug.Log($"[VersionMgr] 已是最新版本: v{localVersion}");
            return true;
        }

        // 4. 保存当前版本用于回滚
        _previousManifest = _currentManifest;

        // 5. 下载更新文件
        bool success = await DownloadUpdates(remoteManifest);
        if (!success)
        {
            Debug.LogError("[VersionMgr] 下载失败，保持当前版本");
            return false;
        }

        // 6. 验证文件完整性
        if (!VerifyDownloadedFiles(remoteManifest))
        {
            Debug.LogError("[VersionMgr] 文件验证失败，执行回滚");
            await RollbackToPrevious();
            return false;
        }

        // 7. 保存清单并更新版本号
        SaveManifest(remoteManifest);
        _currentManifest = remoteManifest;
        
        Debug.Log($"[VersionMgr] 更新成功: v{localVersion} → v{remoteManifest.HotUpdateVersion}");
        return true;
    }

    private async Task<bool> DownloadUpdates(HotUpdateManifest manifest)
    {
        var tasks = new List<Task<bool>>();
        int maxConcurrent = 3; // 最大并发下载数
        var semaphore = new SemaphoreSlim(maxConcurrent);

        foreach (var entry in manifest.Entries)
        {
            var capturedEntry = entry;
            tasks.Add(Task.Run(async () =>
            {
                await semaphore.WaitAsync();
                try
                {
                    return await DownloadFile(capturedEntry);
                }
                finally
                {
                    semaphore.Release();
                }
            }));
        }

        var results = await Task.WhenAll(tasks);
        return results.All(r => r);
    }

    private async Task<bool> DownloadFile(HotUpdateManifest.HotUpdateEntry entry)
    {
        string url = $"https://cdn.example.com/hotupdate/{entry.FileName}";
        string localPath = GetLocalFilePath(entry.FileName);

        using var request = UnityWebRequest.Get(url);
        await request.SendWebRequest();

        if (request.result != UnityWebRequest.Result.Success)
        {
            Debug.LogError($"[Download] 失败: {entry.FileName} - {request.error}");
            return false;
        }

        File.WriteAllBytes(localPath, request.downloadHandler.data);
        return true;
    }

    private bool VerifyDownloadedFiles(HotUpdateManifest manifest)
    {
        foreach (var entry in manifest.Entries)
        {
            string localPath = GetLocalFilePath(entry.FileName);
            if (!File.Exists(localPath)) return false;

            string localMd5 = ComputeMD5(localPath);
            if (localMd5 != entry.MD5)
            {
                Debug.LogError($"[Verify] MD5 不匹配: {entry.FileName}");
                return false;
            }
        }
        return true;
    }

    private async Task RollbackToPrevious()
    {
        if (_previousManifest == null) return;
        
        Debug.LogWarning("[VersionMgr] 执行热更新回滚...");
        
        // 恢复到上一个版本的文件
        foreach (var entry in _previousManifest.Entries)
        {
            string backupPath = GetBackupFilePath(entry.FileName);
            string currentPath = GetLocalFilePath(entry.FileName);
            
            if (File.Exists(backupPath))
            {
                File.Copy(backupPath, currentPath, overwrite: true);
            }
        }
        
        SaveManifest(_previousManifest);
        _currentManifest = _previousManifest;
        _previousManifest = null;
        
        Debug.Log($"[VersionMgr] 回滚成功: v{_currentManifest.HotUpdateVersion}");
    }

    private string GetLocalFilePath(string fileName) =>
        Path.Combine(Application.persistentDataPath, "hotupdate", fileName);

    private string GetBackupFilePath(string fileName) =>
        Path.Combine(Application.persistentDataPath, "hotupdate_backup", fileName);

    private string ComputeMD5(string filePath)
    {
        using var md5 = System.Security.Cryptography.MD5.Create();
        using var stream = File.OpenRead(filePath);
        byte[] hash = md5.ComputeHash(stream);
        return BitConverter.ToString(hash).Replace("-", "").ToLower();
    }

    private int GetLocalHotUpdateVersion()
    {
        if (_currentManifest != null) return _currentManifest.HotUpdateVersion;
        return PlayerPrefs.GetInt("HotUpdateVersion", 0);
    }

    private void SaveManifest(HotUpdateManifest manifest)
    {
        string json = JsonUtility.ToJson(manifest);
        string path = Path.Combine(Application.persistentDataPath, ManifestFileName);
        File.WriteAllText(path, json);
        PlayerPrefs.SetInt("HotUpdateVersion", manifest.HotUpdateVersion);
    }

    private async Task<HotUpdateManifest> FetchRemoteManifest()
    {
        string url = "https://cdn.example.com/hotupdate/manifest.json";
        using var request = UnityWebRequest.Get(url);
        await request.SendWebRequest();
        
        if (request.result == UnityWebRequest.Result.Success)
        {
            return JsonUtility.FromJson<HotUpdateManifest>(request.downloadHandler.text);
        }
        return null;
    }

    private void ShowForceUpdateDialog()
    {
        // UI 显示强制更新弹窗
        UIManager.Instance.ShowDialog("版本过旧", "请前往应用商店更新游戏", () =>
        {
            Application.OpenURL("https://play.google.com/store/apps/details?id=com.example.game");
        });
    }
}
```

## 七、开发期热重载工具链

### 7.1 Editor 实时监听文件变化自动热重载

```csharp
#if UNITY_EDITOR
using UnityEditor;

/// <summary>
/// 开发期 Lua 文件监听热重载工具
/// 修改 Lua 文件后自动通知游戏内重载对应模块
/// </summary>
[InitializeOnLoad]
public static class LuaFileWatcher
{
    private static FileSystemWatcher _watcher;
    private static readonly Queue<string> _pendingReloads = new Queue<string>();
    private static double _lastReloadTime;
    private const double ReloadDebounceMs = 500; // 防抖间隔

    static LuaFileWatcher()
    {
        string luaSourcePath = Path.Combine(Application.dataPath, "StreamingAssets", "Lua");
        if (!Directory.Exists(luaSourcePath)) return;

        _watcher = new FileSystemWatcher(luaSourcePath, "*.lua")
        {
            IncludeSubdirectories = true,
            NotifyFilter = NotifyFilters.LastWrite | NotifyFilters.FileName,
            EnableRaisingEvents = true
        };

        _watcher.Changed += OnLuaFileChanged;
        _watcher.Created += OnLuaFileChanged;

        EditorApplication.update += OnEditorUpdate;
        Debug.Log($"[LuaFileWatcher] 开始监听: {luaSourcePath}");
    }

    private static void OnLuaFileChanged(object sender, FileSystemEventArgs e)
    {
        string moduleName = ConvertPathToModuleName(e.FullPath);
        lock (_pendingReloads)
        {
            if (!_pendingReloads.Contains(moduleName))
                _pendingReloads.Enqueue(moduleName);
        }
    }

    private static void OnEditorUpdate()
    {
        if (!EditorApplication.isPlaying) return;
        if (EditorApplication.timeSinceStartup - _lastReloadTime < ReloadDebounceMs / 1000.0) return;
        if (_pendingReloads.Count == 0) return;

        _lastReloadTime = EditorApplication.timeSinceStartup;

        string moduleName;
        lock (_pendingReloads)
        {
            moduleName = _pendingReloads.Dequeue();
        }

        // 在主线程执行热重载
        LuaManager.Instance?.HotReloadLuaModule(moduleName);
        Debug.Log($"[LuaFileWatcher] 已热重载: {moduleName}");
    }

    private static string ConvertPathToModuleName(string filePath)
    {
        string luaSourcePath = Path.Combine(Application.dataPath, "StreamingAssets", "Lua") + "/";
        string relativePath = filePath.Replace(luaSourcePath, "").Replace("\\", "/");
        return relativePath.Replace(".lua", "").Replace("/", ".");
    }
}
#endif
```

## 八、最佳实践与方案选型

### 8.1 方案选型决策树

```
需要热重载吗？
├── 仅开发期加速迭代
│   ├── C# 逻辑 → Skip Domain Reload + [RuntimeInitializeOnLoadMethod]
│   └── Lua 脚本 → FileSystemWatcher + hot_reload()
│
└── 运营期线上热更
    ├── 需要支持 iOS（App Store 审核）
    │   ├── 逻辑热更 → HybridCLR（C#）或 xlua（Lua）
    │   └── 资源热更 → Addressables（合规方案）
    │
    └── 仅 Android / PC
        ├── 全量替换 → HybridCLR + 完整 DLL 下载
        └── 增量补丁 → IL Patch（风险高，谨慎使用）
```

### 8.2 热更新安全规范

| 规范项 | 说明 |
|--------|------|
| DLL 签名验证 | 下载后必须验证 MD5 和数字签名，防止篡改 |
| HTTPS 传输 | 所有热更资源必须走 HTTPS，防止中间人攻击 |
| 代码加密 | 热更 DLL 存储时进行 AES 加密，加载时解密 |
| 沙箱隔离 | 热更代码不能访问文件系统根目录和系统 API |
| 回滚机制 | 必须保留上一版本备份，验证失败时自动回滚 |
| 灰度发布 | 新版本先推送 1%~5% 用户验证稳定性 |

### 8.3 性能对比

| 技术方案 | 加载性能 | 运行性能 | 开发复杂度 | iOS 合规 |
|---------|---------|---------|-----------|---------|
| Skip Domain Reload | ★★★★★ | ★★★★★ | ★★★★★ | N/A |
| HybridCLR | ★★★★ | ★★★★ | ★★★ | ✅ |
| xlua/Lua | ★★★ | ★★★ | ★★★★ | ✅ |
| IL Patch (Mono) | ★★★★★ | ★★★★★ | ★★ | ❌ |

## 九、总结

游戏热重载体系是提升开发效率与运营灵活性的核心基础设施：

1. **开发期首选 Skip Domain Reload**：配合 `[RuntimeInitializeOnLoadMethod]` 管理静态状态，将进入 PlayMode 时间从秒级降至毫秒级
2. **运营期 C# 热更首选 HybridCLR**：原生 C# 语法，性能接近 AOT，iOS 合规，是当前最佳选择
3. **Lua 热更适合逻辑频繁变动的项目**：灵活性极高，但需要承担 C#/Lua 双语言维护成本
4. **版本管理与回滚是热更稳定性的底线**：MD5 验证、备份机制、灰度发布缺一不可
5. **安全性不可忽视**：热更代码需要签名验证和加密存储，防止恶意注入

合理运用热重载技术，能够让游戏团队在不停服的前提下快速响应线上问题，同时大幅提升开发迭代效率。
