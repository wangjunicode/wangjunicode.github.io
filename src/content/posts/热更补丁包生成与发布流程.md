# 05_热更补丁包生成与发布流程

> 面向刚入行的毕业生 · 建议搭配 BuildAssetsCommand.cs 和 HotUpdatePushWindow.cs 阅读

---

## 1. 系统概述

### 1.1 什么是热更补丁包？

热更补丁包是指游戏运行时可以动态替换的文件集合，主要包含：

| 类型 | 文件形式 | 说明 |
|------|---------|------|
| 热更代码 | `Game.Model.dll.bytes` 等 | C# 程序集（HybridCLR 格式） |
| AOT 元数据 | `mscorlib.dll.bytes` 等 | 补充 AOT 泛型元数据 |
| 资源 Bundle | `*.bundle` | Addressables 打包的资源 |
| Catalog | `catalog.json` | Addressables 资源映射表 |
| 配置数据 | `*.bytes` | 策划配置表（GameCfg/Data） |

### 1.2 补丁包生成工具链

本项目的补丁包生成工具链由以下部分组成：

```
开发机（Unity Editor）
├── BuildAssetsCommand.cs       ← 编译+打包 HybridCLR DLL 的 Editor 工具
├── HotUpdatePushWindow.cs      ← 一站式推送控制台（编译→打包→推送到手机）
└── CompileDllCommand           ← HybridCLR 内置的编译命令

GCloud 控制台
└── 上传补丁包 → 配置版本 → 发布（客户端通过 Dolphin 下载）
```

## 2. 架构设计

### 2.1 补丁包生成完整流程

```
[开发者触发]
    │
    ▼ HybridCLR/VGame/打包热更新程序集
    │
    ▼ CompileDllCommand.CompileDll(target)
    │   ├─ IL2CPP 编译热更程序集（生成裁剪后的 DLL）
    │   └─ 输出到 HybridCLRData/HotUpdateDlls/<target>/
    │
    ▼ CopyABAOTHotUpdateDlls(streamDstPath)
    │   ├─ CopyAOTAssembliesToDest()
    │   │   └─ 拷贝 AOT 元数据 DLL → StreamingAssets/HotUpdateDll/*.dll.bytes
    │   └─ CopyHotUpdateAssembliesToDest()
    │       └─ 拷贝热更 DLL → StreamingAssets/HotUpdateDll/*.dll.bytes
    │
    ▼ Addressables Build（Unity 编辑器手动触发）
    │   └─ 生成 catalog.json + *.bundle 文件
    │
    ▼ 打包 → 上传到 GCloud CDN
    │
    ▼ 在 GCloud 控制台配置版本 + 发布
    │
    ▼ 客户端下次启动时，Dolphin 检测并下载补丁包
```

### 2.2 文件存放路径

```
StreamingAssets/HotUpdateDll/           ← DLL 字节文件（随安装包发布）
├── Game.Model.dll.bytes                ← 热更程序集
├── Game.Function.dll.bytes
├── Game.FunctionView.dll.bytes
├── Game.Battle.Model.dll.bytes
├── Game.Battle.Function.dll.bytes
├── Game.UI.Model.dll.bytes
├── Game.UI.Function.dll.bytes
├── HotUpdateDllConfig.bytes            ← 热更 DLL 列表配置
├── AOTConfig.bytes                     ← AOT 元数据程序集列表
├── MacroConfig.bytes                   ← 游戏宏配置
├── mscorlib.dll.bytes                  ← AOT 元数据（标准库）
├── System.dll.bytes
└── ...

热更目录（Dolphin 下发后的目标目录）：
ResUpdate/<platform>/HotUpdateDll/      ← DLL 字节文件（覆盖安装包）
ResUpdate/catalog.json                  ← 资源映射表
ResUpdate/<platform>/                   ← Bundle 文件
```

### 2.3 开发阶段快速推送流程（HotUpdatePushWindow）

```
开发者打开 HybridCLR/打开推送控制台
    │
    ▼
配置中转服务器 IP:Port（21.91.218.52:8084）
    │
    ▼
连接中转服务器（WebSocket）
    │
    ▼
刷新设备列表 → 选择目标手机 → 配对
    │
    ▼ 点击"编译并推送"
    │

    └─ PushDllFiles()：通过 WebSocket 推送到手机的 ResUpdate 目录
```

## 3. 核心代码展示

### 3.1 DLL 编译入口（BuildAssetsCommand）

```csharp
// BuildAssetsCommand.cs
[MenuItem("HybridCLR/VGame/打包热更新程序集")]
public static void BuildAndCopyABAOTHotUpdateDlls()
{
    BuildTarget target = EditorUserBuildSettings.activeBuildTarget;
    
    // 1. 使用 HybridCLR 的编译命令编译热更 DLL
    CompileDllCommand.CompileDll(target);
    
    // 2. 将编译产物拷贝到 StreamingAssets
    CopyABAOTHotUpdateDlls(streamDstPath);
    
    // 3. 刷新 AssetDatabase
    AssetDatabase.Refresh();
}

// 批量拷贝：AOT + 热更 DLL
public static void CopyABAOTHotUpdateDlls(string destDir)
{
    if (!Directory.Exists(destDir))
        Directory.CreateDirectory(destDir);
    CopyAOTAssembliesToDest(destDir);    // 拷贝 AOT 元数据 DLL
    CopyHotUpdateAssembliesToDest(destDir); // 拷贝热更 DLL
}
```

### 3.2 AOT 元数据 DLL 拷贝

```csharp
// BuildAssetsCommand.cs
public static void CopyAOTAssembliesToDest(string destDir)
{
    var target = EditorUserBuildSettings.activeBuildTarget;
    
    // AOT DLL 来源：Il2CppStrip 后的目录（IL2CPP 构建时生成）
    string aotAssembliesSrcDir = SettingsUtil.GetAssembliesPostIl2CppStripDir(target);

    // 从 AOTConfig.bytes 文件读取需要补充元数据的程序集列表
    var path = Path.Combine(Application.streamingAssetsPath, "HotUpdateDll", "AOTConfig.bytes");
    if (File.Exists(path))
    {
        var aotAssembliesStr = File.ReadAllText(path);
        names.AddRange(aotAssembliesStr.Split(",", StringSplitOptions.RemoveEmptyEntries)
            .Select(s => s.Trim())
            .Where(s => !string.IsNullOrEmpty(s)));
    }

    foreach (var dll in names)
    {
        string srcDllPath = $"{aotAssembliesSrcDir}/{dll}.dll";
        if (!File.Exists(srcDllPath))
        {
            Debug.LogError($"AOT dll 不存在: {srcDllPath}，需要先完整构建一次主包！");
            continue;
        }
        string dllBytesPath = $"{aotAssembliesDstDir}/{dll}.dll.bytes";
        File.Copy(srcDllPath, dllBytesPath, true);
    }
}
```

**重要提示**：AOT DLL 必须来自**已完整构建过的主包**的产物目录。如果从未构建过主包，`GetAssembliesPostIl2CppStripDir` 目录不存在，AOT 元数据无法获取。这是新手最常见的坑之一。

### 3.3 热更 DLL 拷贝

```csharp
// BuildAssetsCommand.cs
public static void CopyHotUpdateAssembliesToDest(string destDir)
{
    var target = EditorUserBuildSettings.activeBuildTarget;
    
    // 热更 DLL 来源：HybridCLR 编译输出目录
    string hotfixDllSrcDir = SettingsUtil.GetHotUpdateDllsOutputDirByTarget(target);
    
    // SettingsUtil.HotUpdateAssemblyFilesExcludePreserved 来自 HybridCLR 设置
    // 列表中每个文件名如 "Game.Model.dll"
    foreach (var dll in SettingsUtil.HotUpdateAssemblyFilesExcludePreserved)
    {
        string dllPath = $"{hotfixDllSrcDir}/{dll}";
        string dllBytesPath = $"{destDir}/{dll}.bytes"; // 重命名为 .bytes 避免被 IL2CPP 误处理
        File.Copy(dllPath, dllBytesPath, true);
        Debug.Log($"拷贝热更 DLL: {dllPath} → {dllBytesPath}");
    }
}
```

### 3.4 AOT Config 生成（自动分析热更引用）

```csharp
// BuildAssetsCommand.cs
[MenuItem("HybridCLR/VGame/生成AOTConfig")]
public static void GenerateAOTConfig()
{
    var configName = "AOTConfig.bytes";
    var outputPath = Path.Combine(Application.streamingAssetsPath, "HotUpdateDll", configName);

    List<string> hotfixAssemblies = SettingsUtil.HotUpdateAssemblyNamesExcludePreserved;

    // 使用 HybridCLR 内置分析器，自动分析热更程序集引用了哪些 AOT 程序集
    var analyzer = new Link.Analyzer(
        MetaUtil.CreateHotUpdateAndAOTAssemblyResolver(
            EditorUserBuildSettings.activeBuildTarget, hotfixAssemblies));
    var refTypes = analyzer.CollectRefs(hotfixAssemblies);

    // 输出需要补充元数据的 AOT 程序集列表（以逗号分隔）
    var writer = new AotAssemliesConfigWriter();
    writer.Write(outputPath, refTypes);
    
    AssetDatabase.Refresh();
}
```

**这个工具的价值**：手动维护需要补充元数据的 AOT 程序集列表很容易遗漏。使用自动分析器，HybridCLR 会扫描所有热更代码，找出它们引用了哪些 AOT 泛型，生成精确的 AOTConfig，避免运行时泛型调用出错。

### 3.5 DLL 加密（可选，当前已注释）

```csharp
// BuildAssetsCommand.cs - DLL 加密算法（基于 XOR 和随机密钥）
private static void EncryptionBytesToPath(string path, string destPath)
{
    byte[] datas = File.ReadAllBytes(path);
    int length = datas.Length;
    
    // 生成随机 64 位主密钥
    System.Random random = new System.Random();
    UInt64 randomA = (UInt64)random.Next();
    UInt64 randomB = (UInt64)random.Next();
    UInt64 mainKey = (randomA << 32) | randomB;
    
    // 生成 128 字节的加密密钥表
    byte[] keys = new byte[128];
    // ... 密钥混淆算法 ...
    
    // XOR 加密
    for (int i = 0; i < length; ++i)
        datas[i] ^= keys[i % 128];
    
    // 将主密钥追加到文件末尾（客户端解密时读取）
    writer.Write(datas);
    writer.Write(mainKey); // 8 bytes
}
```

### 3.6 开发期推送窗口（HotUpdatePushWindow）

```csharp
// HotUpdatePushWindow.cs - 一键编译+推送
if (GUILayout.Button("编译并推送", GUILayout.Height(35)))
{
    if (ValidateInput())
    {
        CompileDllFiles();  // 1. 编译 DLL
        CopyDesignData();   // 2. 拷贝策划配置表
        PushDllFiles();     // 3. 通过 WebSocket 推送到手机
    }
}

// 推送策划配置数据
public static void CopyDesignData()
{
    string targetDir = Path.Combine(Application.streamingAssetsPath, "HotUpdateDll");
    string sourceDir = Path.Combine(Application.dataPath, "../GameCfg/Data");
    
    var files = Directory.GetFiles(sourceDir, "*.bytes", SearchOption.AllDirectories);
    foreach (var file in files)
    {
        string relativePath = Path.GetRelativePath(sourceDir, file);
        string destPath = Path.Combine(targetDir, relativePath);
        Directory.CreateDirectory(Path.GetDirectoryName(destPath)!);
        File.Copy(file, destPath, true);
    }
    Debug.Log($"拷贝了 {files.Length} 个配置文件到 {targetDir}");
}
```

## 4. 设计亮点

### 4.1 DLL 文件名加 `.bytes` 后缀

所有热更 DLL 在 StreamingAssets 中都以 `.dll.bytes` 形式存储（如 `Game.Model.dll.bytes`），而不是直接存储 `.dll`。这是因为：
- Unity 会对 `.dll` 文件进行特殊处理（尝试作为程序集加载）
- `.bytes` 后缀让 Unity 将其视为普通二进制资源，不做额外处理

### 4.2 平台缓存机制（CopyPlatformCache）

```csharp
// HotUpdatePushWindow.cs
private void CopyPlatformCache()
{
    // 将之前构建的 HybridCLR 缓存目录（WinData/MacData）
    // 恢复到当前工作目录，避免重复构建
    string cacheRoot = Path.Combine(projectRootPath, "HybridCLRData");
#if UNITY_EDITOR_WIN
    cacheRoot = developmentBuild 
        ? Path.Combine(cacheRoot, "WinData_Debug")
        : Path.Combine(cacheRoot, "WinData");
#endif
    // 将缓存恢复到 HybridCLRData 工作目录
    foreach (var cachePath in cachePaths)
    {
        CopyDirectoryRecursive(srcPath, dstPath);
    }
}
```

在 CI/CD 环境中，可以保存每次构建后的 HybridCLR 缓存，下次热更时直接恢复，跳过耗时的 IL2CPP 全量编译。

### 4.3 WebSocket 实时推送（开发利器）

`HotUpdatePushWindow` 通过 WebSocket 连接中转服务器，可以将编译好的 DLL 直接推送到连接到同一中转服务器的手机设备，实现"编辑器改代码 → 一键推送到手机测试"，整个流程不需要重新打包安装包，极大提升了开发迭代效率。

## 5. 常见问题与最佳实践

### Q1：提示"AOT dll 不存在"？
**A**：AOT DLL（Strip 后的程序集）只有在完整 IL2CPP 构建（Build Player）后才会生成。首次配置或更换构建机器后，需要先完整构建一次正式包，再进行热更打包。

### Q2：热更 DLL 加载后游戏崩溃？
**A**：最常见的原因是 AOT 泛型缺少元数据。症状：`ExecutionEngineException: Attempting to call method ... for which no ahead of time (AOT) code was generated`。
解决方案：
1. 运行 `HybridCLR/VGame/生成AOTConfig` 重新分析 AOT 引用
2. 运行 `HybridCLR/VGame/打包热更新程序集` 重新打包（会包含新的 AOTConfig）

### Q3：配置表更新了但游戏里不生效？
**A**：确认 `CopyDesignData()` 已将最新的 `.bytes` 文件拷贝到 `StreamingAssets/HotUpdateDll/`，并通过推送工具发送到手机。注意：配置表的热更路径是 `ResUpdate/<platform>/HotUpdateDll/`，确认推送目标路径正确。

### Q4：多人协作时如何管理 HotUpdateDll 目录？
**A**：`StreamingAssets/HotUpdateDll/` 目录应加入 `.gitignore`（因为不同开发者的本地编译产物不同）。在 CI/CD 流水线中统一生成，不在代码库中维护。

### Q5：发布热更的推荐流程是什么？
```
1. 代码提交到主干（Merge Request 通过 Code Review）
2. CI 自动触发：编译 DLL → 打 Addressables Bundle
3. QA 在测试包验证热更包内容
4. 上传到 GCloud 控制台（预发布环境）
5. 内审测试通过
6. 在 GCloud 控制台切换到正式服，按灰度比例发布
7. 监控崩溃率、更新成功率，确认稳定后全量发布
```
