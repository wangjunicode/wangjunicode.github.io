# 游戏内置购买与支付系统设计完全指南

## 前言

内置购买（IAP）是手游收入的核心来源，也是技术风险最高的系统之一。一个疏漏可能导致大规模刷单，损失数百万收入；一个不严谨的充值流程可能导致玩家充值失败但未到账，引发大量投诉。本文从架构设计到安全防护，系统性地介绍构建一套**健壮、安全、可运营**的游戏支付系统所需的全部技术知识。

---

## 一、支付系统整体架构

### 1.1 核心流程

```
玩家点击购买
     ↓
客户端创建订单（本地UUID）
     ↓
请求服务端创建预支付订单（防重放）
     ↓
调起平台支付（AppStore/GooglePlay/微信/支付宝）
     ↓
平台返回支付结果（收据/Token）
     ↓
客户端上报收据至服务端
     ↓
服务端向平台验证收据（服务端验证）
     ↓
验证通过 → 发放商品 → 更新订单状态
     ↓
客户端轮询/推送确认收货
```

### 1.2 关键安全原则

- **永远不要在客户端验证收据**：客户端验证100%可被绕过
- **服务端幂等性**：同一订单ID多次请求只发货一次
- **先验证后发货**：必须平台验证通过才能发货
- **补单机制**：网络异常时确保用户最终能收到商品

## 二、Unity IAP 多平台接入

### 2.1 初始化配置

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.Purchasing;
using UnityEngine.Purchasing.Extension;

/// <summary>
/// IAP管理器 - 统一封装多平台内购
/// </summary>
public class IAPManager : MonoBehaviour, IDetailedStoreListener
{
    public static IAPManager Instance { get; private set; }

    private IStoreController _storeController;
    private IExtensionProvider _extensions;

    // 商品配置
    [Serializable]
    public class ProductConfig
    {
        public string ProductId;
        public ProductType Type;          // Consumable/NonConsumable/Subscription
        public string IOSProductId;       // iOS AppStore商品ID
        public string AndroidProductId;   // GooglePlay商品ID
    }

    [SerializeField] private List<ProductConfig> _products;

    private bool _isInitialized = false;

    // 回调委托
    public event Action<string> OnPurchaseSuccess;  // productId
    public event Action<string, PurchaseFailureReason> OnPurchaseFailed;
    public event Action OnInitialized;
    public event Action<InitializationFailureReason> OnInitializeFailed;

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);
    }

    private void Start()
    {
        InitializePurchasing();
    }

    private void InitializePurchasing()
    {
        if (_isInitialized) return;

        var builder = ConfigurationBuilder.Instance(StandardPurchasingModule.Instance());

        foreach (var product in _products)
        {
            var ids = new IDs();
            ids.Add(product.IOSProductId, AppleAppStore.Name);
            ids.Add(product.AndroidProductId, GooglePlay.Name);
            builder.AddProduct(product.ProductId, product.Type, ids);
        }

        UnityPurchasing.Initialize(this, builder);
    }

    #region IDetailedStoreListener 实现

    public void OnInitialized(IStoreController controller, IExtensionProvider extensions)
    {
        _storeController = controller;
        _extensions = extensions;
        _isInitialized = true;
        
        Debug.Log("[IAP] 初始化成功");
        OnInitialized?.Invoke();

        // 处理未完成的购买（应用重启后恢复）
        RestorePendingPurchases();
    }

    public void OnInitializeFailed(InitializationFailureReason error)
    {
        Debug.LogError($"[IAP] 初始化失败: {error}");
        OnInitializeFailed?.Invoke(error);
    }

    public void OnInitializeFailed(InitializationFailureReason error, string message)
    {
        Debug.LogError($"[IAP] 初始化失败: {error} - {message}");
        OnInitializeFailed?.Invoke(error);
    }

    public PurchaseProcessingResult ProcessPurchase(PurchaseEventArgs args)
    {
        var product = args.purchasedProduct;
        Debug.Log($"[IAP] 购买成功: {product.definition.id}");

        // 将收据上报至服务端验证（异步）
        ValidateReceiptOnServer(product, args.purchasedProduct.receipt);

        // 返回Pending，等服务端验证后再调用ConfirmPendingPurchase
        return PurchaseProcessingResult.Pending;
    }

    public void OnPurchaseFailed(Product product, PurchaseFailureDescription failureDescription)
    {
        Debug.LogError($"[IAP] 购买失败: {product.definition.id} - {failureDescription.reason}");
        OnPurchaseFailed?.Invoke(product.definition.id, failureDescription.reason);
    }

    #endregion

    /// <summary>
    /// 发起购买
    /// </summary>
    public void BuyProduct(string productId)
    {
        if (!_isInitialized)
        {
            Debug.LogError("[IAP] 未初始化，无法购买");
            return;
        }

        var product = _storeController.products.WithID(productId);
        if (product != null && product.availableToPurchase)
        {
            _storeController.InitiatePurchase(product);
        }
        else
        {
            Debug.LogError($"[IAP] 商品不可购买: {productId}");
        }
    }

    /// <summary>
    /// 获取商品价格（本地化显示）
    /// </summary>
    public string GetLocalizedPrice(string productId)
    {
        if (!_isInitialized) return "---";
        var product = _storeController.products.WithID(productId);
        return product?.metadata.localizedPriceString ?? "---";
    }

    private async void ValidateReceiptOnServer(Product product, string receipt)
    {
        try
        {
            bool success = await PaymentServerAPI.ValidateReceipt(
                productId: product.definition.id,
                receipt: receipt,
                transactionId: product.transactionID
            );

            if (success)
            {
                // 验证通过，确认购买完成
                _storeController.ConfirmPendingPurchase(product);
                OnPurchaseSuccess?.Invoke(product.definition.id);
            }
            else
            {
                Debug.LogError($"[IAP] 服务端验证失败: {product.definition.id}");
                // 不调用Confirm，下次启动时会重试
            }
        }
        catch (Exception e)
        {
            Debug.LogError($"[IAP] 服务端验证异常: {e.Message}");
            // 网络异常时不调用Confirm，保留Pending状态
        }
    }

    private void RestorePendingPurchases()
    {
        // iOS需要调用RestoreTransactions
#if UNITY_IOS
        _extensions.GetExtension<IAppleExtensions>()
            .RestoreTransactions(result =>
            {
                Debug.Log($"[IAP] 恢复购买结果: {result}");
            });
#endif
    }
}
```

### 2.2 服务端收据验证 API

```csharp
using System;
using System.Net.Http;
using System.Text;
using System.Threading.Tasks;
using Newtonsoft.Json;
using UnityEngine;

/// <summary>
/// 支付服务端API客户端
/// </summary>
public static class PaymentServerAPI
{
    private static readonly HttpClient _httpClient = new HttpClient
    {
        Timeout = TimeSpan.FromSeconds(30)
    };

    private const string ServerBaseUrl = "https://api.yourgame.com/payment";

    [Serializable]
    public class ValidateRequest
    {
        public string UserId;
        public string ProductId;
        public string Receipt;
        public string TransactionId;
        public string OrderId;       // 客户端本地订单ID
        public string Platform;      // "iOS" / "Android"
        public long Timestamp;
        public string Sign;          // HMAC-SHA256签名
    }

    [Serializable]
    public class ValidateResponse
    {
        public int Code;
        public string Message;
        public bool Success;
        public string ServerId;      // 服务端订单ID
    }

    public static async Task<bool> ValidateReceipt(
        string productId, 
        string receipt, 
        string transactionId)
    {
        var userId = GameContext.CurrentUser.UserId;
        var orderId = PlayerPrefs.GetString($"order_{transactionId}", Guid.NewGuid().ToString());

        var request = new ValidateRequest
        {
            UserId = userId,
            ProductId = productId,
            Receipt = receipt,
            TransactionId = transactionId,
            OrderId = orderId,
            Platform = Application.platform == RuntimePlatform.IPhonePlayer ? "iOS" : "Android",
            Timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds(),
        };
        request.Sign = SignRequest(request);

        string json = JsonConvert.SerializeObject(request);
        var content = new StringContent(json, Encoding.UTF8, "application/json");

        try
        {
            var response = await _httpClient.PostAsync($"{ServerBaseUrl}/validate", content);
            string responseBody = await response.Content.ReadAsStringAsync();
            var result = JsonConvert.DeserializeObject<ValidateResponse>(responseBody);

            return result?.Success == true;
        }
        catch (TaskCanceledException)
        {
            throw new Exception("网络超时");
        }
    }

    private static string SignRequest(ValidateRequest req)
    {
        // HMAC-SHA256签名，密钥存于安全存储（不能硬编码）
        string rawStr = $"{req.UserId}|{req.ProductId}|{req.TransactionId}|{req.Timestamp}";
        using var hmac = new System.Security.Cryptography.HMACSHA256(
            Encoding.UTF8.GetBytes(SecureConfig.PaymentSignKey));
        byte[] hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(rawStr));
        return BitConverter.ToString(hash).Replace("-", "").ToLower();
    }
}
```

## 三、虚拟货币系统设计

### 3.1 双币种体系

大多数手游采用**双货币体系**：
- **钻石/元宝**（充值货币）：用真实货币购买，不可直接获得
- **金币/铜钱**（游戏货币）：游戏内产出，可通过活动获得

```csharp
using System;
using System.Collections.Generic;

/// <summary>
/// 虚拟货币管理器
/// </summary>
public class CurrencyManager
{
    public enum CurrencyType
    {
        Diamond = 1,    // 充值货币（高价值）
        Gold    = 2,    // 游戏货币
        Token   = 3,    // 活动代币（时效性）
    }

    private Dictionary<CurrencyType, long> _balances = new Dictionary<CurrencyType, long>();
    private readonly ICurrencyRepository _repository;
    private readonly ITransactionLog _transactionLog;

    public CurrencyManager(ICurrencyRepository repository, ITransactionLog log)
    {
        _repository = repository;
        _transactionLog = log;
    }

    /// <summary>
    /// 查询余额
    /// </summary>
    public long GetBalance(CurrencyType currency)
    {
        return _balances.TryGetValue(currency, out long val) ? val : 0;
    }

    /// <summary>
    /// 扣减货币（事务性）
    /// </summary>
    public bool Deduct(
        CurrencyType currency, 
        long amount, 
        string reason, 
        string relatedOrderId = null)
    {
        if (amount <= 0) throw new ArgumentException("扣减金额必须大于0");
        
        long currentBalance = GetBalance(currency);
        if (currentBalance < amount)
        {
            Debug.LogWarning($"[Currency] 余额不足: {currency} 需要{amount} 当前{currentBalance}");
            return false;
        }

        long newBalance = currentBalance - amount;
        _balances[currency] = newBalance;

        // 记录流水（服务端同步）
        var record = new TransactionRecord
        {
            Id = Guid.NewGuid().ToString(),
            UserId = GameContext.CurrentUser.UserId,
            Currency = currency,
            Delta = -amount,
            BalanceBefore = currentBalance,
            BalanceAfter = newBalance,
            Reason = reason,
            RelatedOrderId = relatedOrderId,
            Timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
        };

        _transactionLog.Write(record);
        _repository.SaveAsync(GameContext.CurrentUser.UserId, currency, newBalance);
        
        return true;
    }

    /// <summary>
    /// 充入货币
    /// </summary>
    public void Credit(
        CurrencyType currency, 
        long amount, 
        string reason, 
        string orderId = null)
    {
        if (amount <= 0) throw new ArgumentException("充入金额必须大于0");

        long currentBalance = GetBalance(currency);
        long newBalance = currentBalance + amount;
        _balances[currency] = newBalance;

        var record = new TransactionRecord
        {
            Id = Guid.NewGuid().ToString(),
            UserId = GameContext.CurrentUser.UserId,
            Currency = currency,
            Delta = amount,
            BalanceBefore = currentBalance,
            BalanceAfter = newBalance,
            Reason = reason,
            RelatedOrderId = orderId,
            Timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
        };

        _transactionLog.Write(record);
        _repository.SaveAsync(GameContext.CurrentUser.UserId, currency, newBalance);
    }
}

public class TransactionRecord
{
    public string Id;
    public string UserId;
    public CurrencyManager.CurrencyType Currency;
    public long Delta;
    public long BalanceBefore;
    public long BalanceAfter;
    public string Reason;
    public string RelatedOrderId;
    public long Timestamp;
}
```

## 四、补单机制

补单是支付系统中最重要的安全网，确保用户在网络异常、应用崩溃等情况下最终能收到商品。

### 4.1 客户端补单流程

```csharp
using System;
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 补单管理器 - 处理未完成的订单
/// </summary>
public class ReplenishmentManager : MonoBehaviour
{
    private const string PendingOrdersKey = "PendingOrders";
    private const float RetryInterval = 30f;  // 重试间隔（秒）
    private const int MaxRetries = 10;         // 最大重试次数

    [Serializable]
    private class PendingOrder
    {
        public string OrderId;
        public string ProductId;
        public string Receipt;
        public string TransactionId;
        public int RetryCount;
        public long CreateTime;
    }

    [Serializable]
    private class PendingOrderList
    {
        public List<PendingOrder> Orders = new List<PendingOrder>();
    }

    private PendingOrderList _pendingOrders;

    private void Start()
    {
        LoadPendingOrders();
        // 启动时尝试处理遗留订单
        StartCoroutine(ProcessPendingOrdersOnStart());
        // 周期性重试
        InvokeRepeating(nameof(RetryPendingOrders), RetryInterval, RetryInterval);
    }

    public void AddPendingOrder(string orderId, string productId, string receipt, string transactionId)
    {
        _pendingOrders.Orders.Add(new PendingOrder
        {
            OrderId = orderId,
            ProductId = productId,
            Receipt = receipt,
            TransactionId = transactionId,
            RetryCount = 0,
            CreateTime = DateTimeOffset.UtcNow.ToUnixTimeSeconds()
        });
        SavePendingOrders();
    }

    private IEnumerator ProcessPendingOrdersOnStart()
    {
        // 等待IAP初始化
        yield return new WaitUntil(() => IAPManager.Instance != null);
        yield return new WaitForSeconds(2f);
        
        RetryPendingOrders();
    }

    private async void RetryPendingOrders()
    {
        if (_pendingOrders.Orders.Count == 0) return;

        var toRemove = new List<PendingOrder>();

        foreach (var order in _pendingOrders.Orders)
        {
            if (order.RetryCount >= MaxRetries)
            {
                // 超过最大重试次数，转人工处理
                Debug.LogError($"[Replenishment] 订单{order.OrderId}重试超限，需人工处理");
                ReportUnresolvableOrder(order);
                toRemove.Add(order);
                continue;
            }

            try
            {
                bool success = await PaymentServerAPI.ValidateReceipt(
                    order.ProductId, order.Receipt, order.TransactionId);

                if (success)
                {
                    Debug.Log($"[Replenishment] 补单成功: {order.OrderId}");
                    toRemove.Add(order);
                    OnReplenishmentSuccess(order);
                }
                else
                {
                    order.RetryCount++;
                }
            }
            catch (Exception e)
            {
                Debug.LogWarning($"[Replenishment] 补单重试失败: {e.Message}");
                order.RetryCount++;
            }
        }

        foreach (var order in toRemove)
            _pendingOrders.Orders.Remove(order);

        if (toRemove.Count > 0)
            SavePendingOrders();
    }

    private void OnReplenishmentSuccess(PendingOrder order)
    {
        // 通知UI刷新货币显示
        GameEventBus.Publish(new CurrencyChangedEvent { ProductId = order.ProductId });
    }

    private void ReportUnresolvableOrder(PendingOrder order)
    {
        // 上报至客服系统，人工介入
        Debug.LogError($"[Replenishment] 需人工处理订单: {order.OrderId} | Product: {order.ProductId}");
    }

    private void LoadPendingOrders()
    {
        string json = PlayerPrefs.GetString(PendingOrdersKey, "{}");
        _pendingOrders = JsonUtility.FromJson<PendingOrderList>(json) ?? new PendingOrderList();
    }

    private void SavePendingOrders()
    {
        PlayerPrefs.SetString(PendingOrdersKey, JsonUtility.ToJson(_pendingOrders));
        PlayerPrefs.Save();
    }
}
```

## 五、防刷单安全体系

### 5.1 收据防重放

```csharp
// 服务端伪代码（Go）
// 订单幂等性检查
func ValidateAndFulfill(req ValidateRequest) (*ValidateResponse, error) {
    // 1. 检查TransactionId是否已处理（Redis缓存 + 数据库）
    if exists, _ := redis.Get(ctx, "txn:"+req.TransactionId); exists {
        // 已处理过，直接返回成功（幂等）
        return &ValidateResponse{Success: true, Message: "已处理"}, nil
    }
    
    // 2. 向平台验证收据
    platformResult, err := verifyWithPlatform(req.Platform, req.Receipt)
    if err != nil || !platformResult.Valid {
        return &ValidateResponse{Success: false, Message: "收据无效"}, nil
    }
    
    // 3. 检查ProductId是否匹配（防止小额收据购买大额商品）
    if platformResult.ProductId != req.ProductId {
        log.Error("ProductId不匹配！可能的攻击", req)
        return &ValidateResponse{Success: false, Message: "商品不匹配"}, nil
    }
    
    // 4. 事务性发货
    err = db.Transaction(func(tx *gorm.DB) error {
        // 创建订单记录
        order := &Order{...}
        if err := tx.Create(order).Error; err != nil {
            return err
        }
        // 发放商品
        if err := fulfillRewards(tx, req.UserId, req.ProductId); err != nil {
            return err
        }
        return nil
    })
    
    if err != nil {
        return nil, err
    }
    
    // 5. 标记已处理（Redis，防重放，设置过期时间略长于苹果收据有效期）
    redis.Set(ctx, "txn:"+req.TransactionId, "1", 90*24*time.Hour)
    
    return &ValidateResponse{Success: true}, nil
}
```

### 5.2 客户端安全加固

```csharp
/// <summary>
/// 支付安全检查器
/// </summary>
public static class PaymentSecurityChecker
{
    /// <summary>
    /// 校验支付环境安全性
    /// </summary>
    public static SecurityCheckResult CheckPaymentEnvironment()
    {
        var result = new SecurityCheckResult();

#if UNITY_IOS
        // 检测越狱
        result.IsJailbroken = CheckJailbreak();
        if (result.IsJailbroken)
            result.Risks.Add("设备已越狱，支付存在风险");
#endif

#if UNITY_ANDROID
        // 检测Root
        result.IsRooted = CheckRoot();
        if (result.IsRooted)
            result.Risks.Add("设备已Root，支付存在风险");
            
        // 检测非官方商店（国内特化）
        result.IsFromOfficialStore = CheckInstallSource();
#endif
        // 检测网络环境（不允许代理抓包）
        result.IsProxyDetected = CheckProxy();
        if (result.IsProxyDetected)
            result.Risks.Add("检测到代理，已禁用网络通信");

        return result;
    }

    private static bool CheckJailbreak()
    {
#if UNITY_IOS && !UNITY_EDITOR
        // 检测越狱常见路径
        string[] jailbreakPaths = {
            "/Applications/Cydia.app",
            "/Library/MobileSubstrate/MobileSubstrate.dylib",
            "/bin/bash",
            "/usr/sbin/sshd",
            "/etc/apt",
        };
        foreach (string path in jailbreakPaths)
        {
            if (System.IO.File.Exists(path)) return true;
        }
#endif
        return false;
    }

    private static bool CheckRoot()
    {
#if UNITY_ANDROID && !UNITY_EDITOR
        using AndroidJavaClass jc = new AndroidJavaClass("com.your.security.RootChecker");
        return jc.CallStatic<bool>("isRooted");
#endif
        return false;
    }
```

## 八、最佳实践总结

### 8.1 架构要点

1. **服务端验证铁律**：所有IAP收据必须通过服务端向平台API验证，客户端验证形同虚设
2. **幂等性设计**：以TransactionID为幂等键，防止网络重试导致重复发货
3. **Pending状态**：`ProcessPurchase`返回`Pending`，等服务端验证后再`Confirm`
4. **补单闭环**：App重启时处理所有未完成的Pending订单
5. **事务性发货**：数据库事务中同时写订单和发放商品，防止部分失败

### 8.2 安全要点

- 不信任客户端任何支付相关数据，一切以服务端验证为准
- 收据验证请求加签名，防止中间人篡改
- TransactionID在服务端去重，防止同一收据多次兑换
- 监控异常购买行为（高频购买、IP异常）

### 8.3 运营要点

- 商品配置服务端下发，支持不更新客户端的运营调整
- 所有价格变动留审计日志
- 充值流水保留至少5年（财务合规）
- 上线前在沙盒环境全流程回归测试

### 8.4 常见坑

| 坑 | 解法 |
|----|------|
| 客户端验证收据 | 改为服务端验证 |
| 忘记处理Pending购买 | 启动时遍历处理，IAPManager.OnInitialized回调中触发 |
| 发货不幂等 | 以TransactionId为唯一键 |
| 沙盒环境测试不充分 | 建立完整的沙盒测试矩阵（正常购买/网络断开/重启恢复） |
| 未考虑退款场景 | 平台退款后需撤销游戏内商品（订阅型关键） |

## 结语

支付系统是游戏变现的生命线，其设计需要在**用户体验**、**安全性**和**合规性**之间取得平衡。核心原则是：**服务端验证、幂等发货、补单保底、事务隔离**。任何对这四点的妥协，都可能导致严重的财务或合规风险。建议在游戏上线前，专门安排针对支付流程的安全测试和压力测试，把风险消灭在发布前。
