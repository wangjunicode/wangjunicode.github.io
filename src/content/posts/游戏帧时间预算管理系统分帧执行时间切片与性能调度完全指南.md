# 游戏帧时间预算管理系统：分帧执行、时间切片与性能调度完全指南

## 一、为什么需要帧时间预算管理

游戏运行在固定帧率下（60fps = 16.67ms/帧），每帧能使用的 CPU 时间是固定的"预算"。
常见的性能问题模式：

```
帧 N:   █████████████████░░  17.2ms（稍微超标，玩家感受到轻微卡顿）
帧 N+1: ████████████████████████  24ms（严重超标！丢帧）
帧 N+2: ████████████████  16ms（正常）
```

造成帧率不稳定的常见原因：
- **一次性大量计算**：加载数据、寻路计算、AI 评估
- **GC 触发**：瞬时内存分配引起 GC 停顿
- **资源加载同步阻塞**：纹理解压、Mesh 上传

**解决方案：帧时间预算管理**——将耗时操作切分为多帧执行，确保每帧耗时在预算内。

---

## 二、核心概念：时间切片（Time Slicing）

```
不使用时间切片：
帧 1: [初始化所有1000个敌人] → 50ms 严重卡顿

使用时间切片：
帧 1: [初始化 100 个]  → 5ms ✓
帧 2: [初始化 100 个]  → 5ms ✓
...
帧 10:[初始化最后100个] → 5ms ✓
总体流畅，代价是初始化延迟了 10 帧（约 167ms）
```

时间切片的核心是：**用延迟换流畅度**，对于玩家感知不强的后台操作是完美的权衡。

## 三、通用分帧任务调度器

### 3.1 核心调度器实现

```csharp
using System;
using System.Collections;
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 帧时间预算调度器
/// 每帧执行任务直到预算耗尽，未完成的任务延续到下一帧
/// </summary>
public class FrameBudgetScheduler : MonoBehaviour
{
    // ─── 单例 ─────────────────────────────────────
    public static FrameBudgetScheduler Instance { get; private set; }

    // ─── 配置 ─────────────────────────────────────
    [Header("Frame Budget Settings")]
    [Tooltip("每帧最大可用时间（毫秒），建议不超过帧时间的 40%")]
    [SerializeField] private float budgetMs = 4.0f;

    [Tooltip("是否根据当前帧率动态调整预算")]
    [SerializeField] private bool dynamicBudget = true;

    [Tooltip("目标帧率")]
    [SerializeField] private int targetFps = 60;

    // ─── 任务队列 ─────────────────────────────────
    public enum Priority { High = 0, Normal = 1, Low = 2 }

    private class ScheduledTask
    {
        public IEnumerator  Iterator;
        public Priority     Priority;
        public string       Name;
        public Action       OnComplete;
        public bool         IsDone;
    }

    // 三个优先级的队列
    private readonly Queue<ScheduledTask>[] _queues = new Queue<ScheduledTask>[3]
    {
        new Queue<ScheduledTask>(),
        new Queue<ScheduledTask>(),
        new Queue<ScheduledTask>(),
    };

    private int _totalPending;

    // ─── 统计 ─────────────────────────────────────
    public float LastFrameUsedMs   { get; private set; }
    public float CurrentBudgetMs   => _currentBudgetMs;
    public int   PendingTaskCount  => _totalPending;

    private float _currentBudgetMs;
    private System.Diagnostics.Stopwatch _sw = new();

    private void Awake()
    {
        if (Instance != null && Instance != this) { Destroy(gameObject); return; }
        Instance = this;
        _currentBudgetMs = budgetMs;
    }

    // ─── 提交任务 ─────────────────────────────────
    /// <summary>
    /// 提交一个分帧迭代器任务
    /// 迭代器中 yield return null 表示"本帧到此为止，下帧继续"
    /// </summary>
    public ScheduledTask Submit(IEnumerator iterator,
                                 Priority priority  = Priority.Normal,
                                 string   taskName  = "Task",
                                 Action   onComplete = null)
    {
        var task = new ScheduledTask
        {
            Iterator   = iterator,
            Priority   = priority,
            Name       = taskName,
            OnComplete = onComplete,
        };
        _queues[(int)priority].Enqueue(task);
        _totalPending++;
        return task;
    }

    // ─── 每帧调度 ─────────────────────────────────
    private void Update()
    {
        // 动态预算：当帧率低时减少预算，保护帧率
        if (dynamicBudget)
            AdjustBudget();

        _sw.Restart();
        float budgetTicks = _currentBudgetMs * System.Diagnostics.Stopwatch.Frequency / 1000f;

        // 按优先级顺序执行
        for (int pri = 0; pri < 3; pri++)
        {
            var queue = _queues[pri];
            while (queue.Count > 0)
            {
                // 检查时间预算
                if (_sw.ElapsedTicks >= budgetTicks) goto BudgetExhausted;

                var task = queue.Peek();

                // 执行迭代器的一步
                bool hasMore = task.Iterator.MoveNext();

                if (!hasMore)
                {
                    // 任务完成
                    queue.Dequeue();
                    _totalPending--;
                    task.IsDone = true;
                    task.OnComplete?.Invoke();
                }
                // 若还有更多步骤，任务留在队列头等待下一帧
                // （注意：只有 yield return null 时才让出，其他 yield 直接继续）
            }
        }

        BudgetExhausted:
        LastFrameUsedMs = (float)(_sw.ElapsedTicks * 1000.0 /
                                   System.Diagnostics.Stopwatch.Frequency);
    }

    private void AdjustBudget()
    {
        float targetFrameMs = 1000f / targetFps;
        float currentFps    = 1f / Time.unscaledDeltaTime;

        if (currentFps < targetFps * 0.9f)
        {
            // 帧率偏低，缩减预算
            _currentBudgetMs = Mathf.Max(1f, _currentBudgetMs - 0.2f);
        }
        else if (currentFps > targetFps * 0.98f)
        {
            // 帧率充裕，增加预算（但不超过帧时间的50%）
            _currentBudgetMs = Mathf.Min(budgetMs, _currentBudgetMs + 0.1f);
        }
    }

    // ─── 便捷 API ─────────────────────────────────
    /// <summary>清空所有待处理任务</summary>
    public void ClearAll()
    {
        foreach (var q in _queues) q.Clear();
        _totalPending = 0;
    }
}
```

### 3.2 使用示例：分帧初始化大量敌人

```csharp
public class EnemySpawner : MonoBehaviour
{
    [SerializeField] private GameObject enemyPrefab;
    [SerializeField] private int        enemyCount = 2000;

    private void Start()
    {
        // 分帧生成 2000 个敌人，避免一帧卡死
        FrameBudgetScheduler.Instance.Submit(
            SpawnEnemiesIterator(),
            FrameBudgetScheduler.Priority.Normal,
            "SpawnEnemies",
            () => Debug.Log("所有敌人生成完毕！")
        );
    }

    private IEnumerator SpawnEnemiesIterator()
    {
        for (int i = 0; i < enemyCount; i++)
        {
            var pos = new Vector3(
                UnityEngine.Random.Range(-50f, 50f),
                0,
                UnityEngine.Random.Range(-50f, 50f)
            );
            Instantiate(enemyPrefab, pos, Quaternion.identity);

            // 每生成 20 个让出一次（允许调度器检查时间预算）
            if (i % 20 == 0)
                yield return null;
        }
    }
}
```

## 四、更精细的时间切片：步进式迭代器

上面的调度器以"迭代器步骤"为切片粒度，有时我们需要在单个 Update 中也控制精确耗时：

```csharp
/// <summary>
/// 时间切片执行器：在指定时间内尽可能多地处理工作项
/// </summary>
public static class TimeSlice
{
    /// <summary>
    /// 在 budgetMs 时间内处理 items，返回下次应从哪个索引继续
    /// </summary>
    public static int Process<T>(
        IList<T>   items,
        int        startIndex,
        float      budgetMs,
        Action<T>  processAction)
    {
        var sw    = System.Diagnostics.Stopwatch.StartNew();
        long budget = (long)(budgetMs * System.Diagnostics.Stopwatch.Frequency / 1000.0);

        int i = startIndex;
        while (i < items.Count && sw.ElapsedTicks < budget)
        {
            processAction(items[i]);
            i++;
        }
        return i; // 返回下次起始索引
    }

    /// <summary>
    /// 将 Action<T> 包装成可用于 FrameBudgetScheduler 的迭代器
    /// </summary>
    public static IEnumerator ToIterator<T>(
        IList<T>   items,
        Action<T>  processAction,
        int        batchSize = 32)
    {
        for (int i = 0; i < items.Count; i++)
        {
            processAction(items[i]);
            if (i % batchSize == batchSize - 1)
                yield return null; // 每批次让出
        }
    }
}
```

### 使用示例：分帧 NavMesh 路径计算

```csharp
using UnityEngine.AI;

public class PathfindingManager : MonoBehaviour
{
    private struct PathRequest
    {
        public NavMeshAgent Agent;
        public Vector3      Destination;
    }

    private Queue<PathRequest> _pendingRequests = new();
    private int _processedThisFrame;

    public void RequestPath(NavMeshAgent agent, Vector3 dest)
    {
        _pendingRequests.Enqueue(new PathRequest { Agent = agent, Destination = dest });
    }

    private void Update()
    {
        // 每帧最多处理 10 个路径请求（每个 CalculatePath 约 0.1ms）
        _processedThisFrame = 0;
        float budget = 1.0f; // 1ms
        var sw = System.Diagnostics.Stopwatch.StartNew();

        while (_pendingRequests.Count > 0 &&
               sw.ElapsedMilliseconds < budget)
        {
            var req = _pendingRequests.Dequeue();
            if (req.Agent != null && req.Agent.isActiveAndEnabled)
            {
                req.Agent.SetDestination(req.Destination);
                _processedThisFrame++;
            }
        }
    }
}
```

## 五、与 UniTask 结合的异步帧预算系统

UniTask 提供了更现代的异步编程模型，与帧预算结合使用更优雅：

```csharp
#if UNITASK_AVAILABLE
using Cysharp.Threading.Tasks;

public static class FrameBudgetAsync
{
    /// <summary>
    /// 异步分帧处理大量数据
    /// </summary>
    public static async UniTask ProcessBatchedAsync<T>(
        IList<T>               items,
        Func<T, UniTask>       asyncProcessor,
        float                  budgetMsPerFrame = 4f,
        IProgress<float>       progress         = null,
        System.Threading.CancellationToken ct = default)
    {
        int total     = items.Count;
        var sw        = System.Diagnostics.Stopwatch.StartNew();
        long budget   = (long)(budgetMsPerFrame * System.Diagnostics.Stopwatch.Frequency / 1000.0);

        for (int i = 0; i < total; i++)
        {
            ct.ThrowIfCancellationRequested();

            await asyncProcessor(items[i]);

            progress?.Report((float)(i + 1) / total);

            // 检查是否超出预算，超出则等到下一帧
            if (sw.ElapsedTicks >= budget)
            {
                await UniTask.Yield(PlayerLoopTiming.Update);
                sw.Restart();
            }
        }
    }

    /// <summary>
    /// 在指定时间预算内执行同步工作，超出则 yield 到下一帧
    /// </summary>
    public static async UniTask RunWithBudgetAsync(
        Func<bool>  workUnit,       // 返回 true = 还有工作
        float       budgetMs = 4f,
        System.Threading.CancellationToken ct = default)
    {
        var  sw     = System.Diagnostics.Stopwatch.StartNew();
        long budget = (long)(budgetMs * System.Diagnostics.Stopwatch.Frequency / 1000.0);

        while (!ct.IsCancellationRequested)
        {
            bool hasMore = workUnit();
            if (!hasMore) break;

            if (sw.ElapsedTicks >= budget)
            {
                await UniTask.Yield(PlayerLoopTiming.Update);
                sw.Restart();
            }
        }
    }
}

// 使用示例
public class DataLoader : MonoBehaviour
{
    private async UniTaskVoid LoadLargeDataAsync()
    {
        var records = LoadAllRecordsFromDisk(); // 假设有 10000 条

        await FrameBudgetAsync.ProcessBatchedAsync(
            records,
            async record => {
                // 每条记录的处理逻辑（可以是同步或异步）
                ProcessRecord(record);
                // 不需要 await，ProcessBatchedAsync 会自动切帧
            },
            budgetMsPerFrame: 3f,
            progress: new Progress<float>(p => Debug.Log($"加载进度: {p:P0}"))
        );

        Debug.Log("数据加载完成");
    }
}
#endif
```

```csharp
/// <summary>
/// 运行时帧预算监控面板（仅开发模式）
/// </summary>
public class FrameBudgetMonitor : MonoBehaviour
{
    [SerializeField] private bool showHUD = true;

    private float _smoothedUsage;
    private readonly Queue<float> _history = new(60);

    private void Update()
    {
        if (FrameBudgetScheduler.Instance == null) return;

        float usage = FrameBudgetScheduler.Instance.LastFrameUsedMs;
        _smoothedUsage = Mathf.Lerp(_smoothedUsage, usage, 0.1f);

        if (_history.Count >= 60) _history.Dequeue();
        _history.Enqueue(usage);
    }

    private void OnGUI()
    {
        if (!showHUD || FrameBudgetScheduler.Instance == null) return;

        var scheduler = FrameBudgetScheduler.Instance;
        float budget  = scheduler.CurrentBudgetMs;
        float used    = _smoothedUsage;
        float ratio   = Mathf.Clamp01(used / budget);

        // 绘制进度条
        GUI.Box(new Rect(10, 10, 200, 80), "Frame Budget");

        // 颜色：绿→黄→红
        GUI.color = Color.Lerp(Color.green, Color.red, ratio);
        GUI.Box(new Rect(12, 45, 196 * ratio, 14), GUIContent.none);
        GUI.color = Color.white;

        GUI.Label(new Rect(12, 30, 200, 20),
            $"Used: {used:F2}ms / Budget: {budget:F1}ms  ({ratio:P0})");
        GUI.Label(new Rect(12, 62, 200, 20),
            $"Pending Tasks: {scheduler.PendingTaskCount}");
    }
}
```

```csharp
/// <summary>
/// 按优先级分配帧预算——高优先级任务获得更多时间
/// </summary>
public class PriorityBudgetAllocator
{
    private const float TotalBudgetMs = 8.0f;

    // 优先级权重分配（高:中:低 = 5:3:2）
    private static readonly float[] Weights = { 5f, 3f, 2f };
    private static readonly float   TotalWeight = 10f;

    public static float GetBudget(FrameBudgetScheduler.Priority priority)
    {
        return TotalBudgetMs * Weights[(int)priority] / TotalWeight;
    }
}
```

## 八、常见陷阱与解决方案

### 陷阱 1：过细的切片粒度

```csharp
// ❌ 错误：每个元素 yield 一次，调度开销比实际工作更大
for (int i = 0; i < items.Length; i++)
{
    Process(items[i]);
    yield return null; // 每次都让出
}

// ✅ 正确：批量处理后才让出
const int BatchSize = 64;
for (int i = 0; i < items.Length; i++)
{
    Process(items[i]);
    if (i % BatchSize == BatchSize - 1)
        yield return null;
}
```

### 陷阱 2：忽略任务饥饿

```csharp
// ❌ 错误：高优先级任务持续提交，低优先级永远得不到执行（饥饿）

// ✅ 解决：老化机制（Aging）——等待时间越长，优先级越高
private void AgeTasks()
{
    foreach (var (task, frameWaited) in _waitingTasks)
    {
        if (frameWaited > 300) // 等待超过 300 帧（约 5 秒）
            UpgradePriority(task);
    }
}
```

### 陷阱 3：分帧执行中途对象销毁

```csharp
// ✅ 使用 CancellationToken 或弱引用处理
private IEnumerator SafeIterator(WeakReference<Transform> weakRef)
{
    for (int i = 0; i < 1000; i++)

        // 每步检查对象是否仍然存在

            yield break; // 对象已销毁，终止

        if (i % 20 == 0)

    }
```

## 九、最佳实践总结

1. **预算设定**：建议每帧调度系统预算不超过帧时间的 30%（60fps 下约 5ms）
2. **批量大小**：每次 yield 前处理 16~64 个工作单元，平衡切片开销与响应性
3. **优先级分层**：玩家可感知的操作（生成特效、响应输入）用 High，后台计算用 Low
4. **动态预算**：根据当前实际帧率动态收缩/扩展预算，避免雪崩
5. **饥饿保护**：长时间等待的低优先级任务应自动提升优先级
6. **对象生命周期**：分帧迭代器中必须处理目标对象被销毁的情况
7. **监控可视化**：开发期间务必使用帧预算 HUD，数据驱动调优
8. **与 UniTask 结合**：异步链式场景使用 `UniTask.Yield()` + 预算检查，代码更清晰
9. **避免过度设计**：简单场景直接用 `StartCoroutine` 分帧，不需要引入完整调度系统

## 十、小结

帧时间预算管理是游戏客户端性能工程的重要一环。通过将耗时操作切分为多帧执行，配合优先级调度和动态预算调整，可以在不牺牲功能的前提下保持稳定的帧率。从简单的迭代器分帧到完整的调度系统，开发者应根据项目规模选择合适的粒度，以 Profiler 数据为依据持续优化，最终构建出流畅且可维护的高性能游戏体验。
